# Project Rules

## Git: синхронизация перед изменениями

**Перед любыми изменениями в репозитории — ВСЕГДА сначала выполни `git pull`.** Это гарантирует, что ты работаешь с актуальной версией и не создашь конфликтов с изменениями других участников.

**После каждого изменения — спроси пользователя, хочет ли он запушить.** Не пуши автоматически, всегда жди подтверждения.

## Навигация по репозиторию

**Перед поиском информации в этом репозитории — ВСЕГДА сначала прочитай `index.md` в корне проекта.** Этот файл содержит полную карту всех файлов и папок. Используй его чтобы понять структуру и найти нужные файлы, прежде чем искать вслепую.

## UTM-метки: обязательная проверка

**Если ты генерируешь или редактируешь контент, в котором есть ссылки (посты, рассылки, карточки, лендинги, описания видео) — ВСЕГДА проверь, что ссылки содержат UTM-метки.**

- Справочник UTM-меток: `3-Marketing/utm-system.md`
- Если в контенте есть ссылка без UTM — предложи добавить метку по справочнику, даже если пользователь не просил
- Если это вебинар — используй раздел «Воронка вебинара» из справочника, чтобы подготовить полный набор меток на все этапы
