# Скилл: Создание тезисов главы (Double Diamond — РАСХОЖДЕНИЕ)

Этот скилл превращает поток мыслей в РАСШИРЕННЫЕ тезисы через 3 исследовательских трека: `knowledge-researcher`, `expert-hunter`, `contrarian`. Треки можно выполнять параллельно (если среда поддерживает подзадачи) или последовательно одним агентом, но с тем же протоколом качества.

Приоритет: качество тезисов > количество, точность > полнота покрытия.

## Рабочие пути и инициализация

Перед стартом вычисли корень репозитория и рабочие каталоги:

```bash
ROOT="$(git rev-parse --show-toplevel 2>/dev/null || pwd)"
THOUGHTS_DIR_MAIN="$ROOT/3-Marketing/2-Writing-Book/1-Raw-Thoughts"
THOUGHTS_DIR_FALLBACK="$ROOT/3-Marketing/2-PR-Articles/1-Thoughts"

# Canonical output dir for PR-articles (always write results here)
THESIS_DIR="$ROOT/3-Marketing/2-PR-Articles/2-Theses"

# Optional legacy dir (read-only, if you need to look up older files)
THESIS_DIR_LEGACY="$ROOT/3-Marketing/2-Writing-Book/2-Theses"

if [ -d "$THOUGHTS_DIR_FALLBACK" ]; then
  THOUGHTS_DIR="$THOUGHTS_DIR_FALLBACK"
else
  THOUGHTS_DIR="$THOUGHTS_DIR_MAIN"
fi

mkdir -p "$THESIS_DIR"
```

Собери slug для temp-папки и создай рабочую директорию:

```bash
SLUG="$(printf '%s' "{название-главы}" \
  | tr '[:upper:]' '[:lower:]' \
  | sed -E 's/[^a-z0-9]+/-/g; s/^-+|-+$//g; s/-+/-/g')"
TEMP_DIR="$ROOT/tmp/thesis-${SLUG}"
mkdir -p "$TEMP_DIR"
```

В этом документе обозначение `temp-папка` всегда означает путь из переменной `$TEMP_DIR`.

## Входные данные

Пользователь даёт:
1. **Название главы** (или тему)
2. **Поток мыслей** — неструктурированные идеи, голосовые заметки, тезисы

ИЛИ:

1. **Путь к файлу мысли/набросков** из `3-Marketing/2-PR-Articles/1-Thoughts/` (приоритет) или `3-Marketing/2-Writing-Book/1-Raw-Thoughts/`
2. **(Опционально) пояснение** — что важно вытащить в тезисы

ИЛИ:

1. **Ссылку на существующий файл тезисов** (обычно из `3-Marketing/2-PR-Articles/2-Theses/`, legacy: `3-Marketing/2-Writing-Book/2-Theses/`)
2. **Фидбэк** — что доработать, углубить, изменить

## ЭТАП 0: Определение режима работы (CREATION vs ITERATION)

**Выполни ДО ВСЕГО остального.** Определи, какой режим нужен:

### Как определить режим

| Сигнал | Режим |
|--------|-------|
| Пользователь дал Raw Thoughts / поток мыслей / новую тему | **CREATION** → иди к ЭТАП 1 |
| Пользователь дал путь к файлу из `3-Marketing/2-PR-Articles/1-Thoughts/` | **CREATION** → иди к ЭТАП 1 |
| Пользователь дал ссылку на существующий файл `Theses-*.md` | **ITERATION** → иди к ЭТАП И1 |
| В файле тезисов заполнено поле `## 🔄 Фидбэк автора для итерации` | **ITERATION** → иди к ЭТАП И1 |
| Пользователь написал "доработай тезисы" / "углуби" / "поправь" | **ITERATION** → иди к ЭТАП И1 |

### Автодетект

1. Если пользователь дал путь к файлу — прочитай его
2. Если путь указывает на `.../1-Thoughts/...` или `.../1-Raw-Thoughts/...` → сразу **CREATION mode**
3. Проверь: есть ли секция `## 🔄 Фидбэк автора для итерации` с заполненным содержимым (НЕ placeholder)?
4. Проверь: есть ли метаданные `Версия:` в шапке?
5. Если да → **ITERATION mode**. Если нет → **CREATION mode**

### Маршрутизация

- **CREATION mode:** Этапы 1 → 2 → 3 → 4 → 5 (полный цикл с 3 исследовательскими треками)
- **ITERATION mode:** Этапы И1 → И2 → И3 → И4 (точечная доработка без полного ресерча)

|-------------|------|------------|

| Отчёт каждого агента | `agent-{имя}.md` | Полный отчёт агента (агенты сохраняют сами) |
| ЭТАП 2.5 | `02-first-principles.md` | Разбор тезисов на первые принципы и скрытые допущения |
| ЭТАП 2.5 | `03-order-effects.md` | Карта последствий 2-го и 3-го порядка |

| ЭТАП 2.5 | `03-thesis-bank.md` | Широкий пул кандидатных тезисов для отбора |
| Каждый раунд QC | `coverage-round-{N}.md` | Coverage Score + слабые места + задания тиммейтам |
| ЭТАП 4 | `04-synthesis.md` | Финальный синтез перед записью |

### Правила

1. **После каждого этапа:** Сохрани результат в файл + обнови `_spring-context.md`
2. **Перед каждым этапом:** Перечитай `_spring-context.md` и все указанные в нём файлы
3. **НЕ удаляй** temp-файлы по завершении — они нужны для следующих скиллов (draft, edit)
4. **Агенты-тиммейты** тоже сохраняют свои отчёты в эту папку (см. промпты ниже)

### ОБЯЗАТЕЛЬНЫЕ ССЫЛКИ НА ИСТОЧНИКИ

⚠️ **Железное правило:** Каждый упомянутый источник ОБЯЗАН иметь ссылку.

| Тип источника | Что указывать |
|---------------|---------------|
| **Веб-статья / блог** | Реальный URL (https://...) — найденный через `web.search_query` + `web.open` |
| **Твит / тред в X** | URL на пост (https://x.com/...) |
| **Книга** | Название, автор, год, ISBN если возможно |
| **Knowledge Base (AURA/AJTBD)** | Полный путь к файлу (`Knowledge/AJTBD/fundamentals.md`) |
| **Подкаст / видео** | URL или название + платформа + дата |

**НЕ ДОПУСКАЕТСЯ:**
- Упоминать эксперта без ссылки на конкретную статью/пост
- Писать "по мнению X" без URL на первоисточник
- Ссылаться на "исследование показало" без конкретного URL или названия
- Придумывать/генерировать URL — только реальные ссылки, найденные через `web.search_query`

**Агенты-тиммейты** обязаны соблюдать это правило в своих отчётах.

### ПОЛИТИКА СВЕЖЕСТИ ИСТОЧНИКОВ (ОБЯЗАТЕЛЬНО)

Жёсткое правило по дате для доказательной базы:

- Внешние источники (статьи, треды, исследования, посты): **только с датой публикации не раньше `2025-06-01`**.
- Источник без даты публикации: **не использовать** как доказательство.
- Knowledge Base можно использовать как методологическую опору, но любые фактологические утверждения по рынку/трендам подтверждай внешними источниками `>= 2025-06-01`.

Приоритет свежести:

- **Приоритет 1 (максимальный):** источники за последние 3 месяца от даты запуска.
- **Приоритет 2:** источники в диапазоне `2025-06-01` → начало 3-месячного окна.
- На `2026-03-02` окно последних 3 месяцев начинается с `2025-12-02`.

Минимальные квоты:

- `>= 80%` внешних источников — из последних 3 месяцев.
- `100%` внешних источников — не раньше `2025-06-01`.

Нарушение хотя бы одного правила выше => этап ресерча считается незавершённым.

## ЭТАП 2: Оркестрация 3 исследовательских треков

**ТЫ (лид) = Quality Controller.** Твоя роль — координировать треки, проверять качество, отправлять на доработку и решать, когда ресерч завершён.

### Шаг 2.1: Подготовка

Определи ключевые термины и темы из потока мыслей автора. Сформулируй их в виде списка — это входные данные для всех треков.

Зафиксируй `Topic Lock` в `_spring-context.md`:

- Главный вопрос статьи (1 фраза)
- Что точно in-scope (3-7 пунктов)
- Что out-of-scope (3-7 пунктов)
- Какой тип ценности ищем: стратегия / тактика / контринтуиция / фреймворк / кейсы

### Шаг 2.2: Выбери режим исполнения

`Mode A — Parallel` (если среда поддерживает параллельные подзадачи):
- Запусти треки `knowledge-researcher`, `expert-hunter`, `contrarian` параллельно.
- Для каждого трека используй промпт ниже и отдельный файл отчёта в `$TEMP_DIR`.

`Mode B — Sequential` (fallback по умолчанию):
- Выполни те же 3 трека последовательно в одном агенте.
- После завершения каждого трека зафиксируй результат в соответствующий `agent-*.md`.
- Перед стартом следующего трека перечитай уже сохранённые отчёты, чтобы имитировать межтрековую синхронизацию.

### Шаг 2.3: Подготовь рабочие файлы координации

Создай/обнови:
- `$TEMP_DIR/agent-requests.md` — запросы лида в треки (что доисследовать, что проверить)
- `$TEMP_DIR/agent-log.md` — журнал статусов треков, дедлайны, выполненные итерации

- Объяви команде: "Ресерч завершён. Coverage Score: {X}%. Переходим к синтезу."
- Начинай ЭТАП 4

Дополнительные hard-gates (обязательные независимо от среднего Coverage Score):

- `Freshness Score >= 90%`
- `Expansion Quality >= 85%`
- `Depth Score >= 85%`

Если любой hard-gate не выполнен, цикл ресерча продолжается.

**Максимум 2 раунда итерации.** Если после 2 раундов Coverage Score < 85% — переходи к ЭТАП 4 с тем, что есть. Не затягивай.

## Принцип работы

**РАСХОЖДЕНИЕ по Double Diamond через Track Orchestration:**
- 3 трека работают параллельно (или последовательно с file-handoff)
- expert-hunter ищет ТОЧНЫЕ новые тезисы (межотраслевые аналогии, провокации, свежие данные)
- knowledge-researcher УГЛУБЛЯЕТ авторские тезисы данными из KB (НЕ создаёт тезисы из других глав)
- contrarian превращает контраргументы в самостоятельные тезисы (провокации, синтез, футуризм)
- обязательный Expansion Lab: first-principles + последствия 2/3 порядка + комбинаторика
- **Качество > количество** — каждый новый тезис проходит тест релевантности
- широкий банк 30-60 кандидатов + shortlist 15-25 новых тезисов
- Лид = Quality Controller, Coverage Score ≥ 85% = выход
- 1 раунд агентов + 1 раунд доработки (максимум 2 раунда итерации)
- Приоритет СВЕЖЕСТИ: Last 3 months >> >=2025-06-01; всё, что раньше, исключается
- НИЧЕГО не удаляем, спорные моменты ПОДСВЕЧИВАЕМ
- **Выходной файл = меню для автора**: авторские + новые, автор выбирает

## Handoff в Draft

После финализации тезисов явно предложить следующий шаг:

- Запустить `$new-article-draft` для сборки черновика статьи из тезисов.
- Передать путь к текущему `Theses-*.md` и целевое издание.
- Уточнить, что draft-этап должен провести reverse-engineering most-read материалов издания и применить их tone-of-voice к секциям.
