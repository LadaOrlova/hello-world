# Анализ конкурентов: AI-гайд по продуктовому мышлению + обучение вайбкодингу

Дата анализа: 2026-03-01

## Контекст продукта

Гибрид курса и инструмента. Продукт, в который ты заходишь, рассказываешь, что хочешь сделать. Он:
1. Задает вопросы -- помогает написать продуктовые требования, выделить риски
2. Обучает в процессе -- объясняет "почему мы это делаем", "как это работает"
3. Генерирует документы -- PRD, аналитический план, лендинг-копи для передачи в Lovable, Replit, Cursor или Claude Code
4. Учит вайбкодингу -- практические микро-гайды для Cursor, Claude Code
5. НЕ генерирует код / НЕ деплоит продукт
6. Output = документы + знания

Целевые сегменты:
1. Non-Technical Founders (0 -> 1): понять что строить + научиться строить
2. Solopreneurs / Side-Hustlers: быстро валидировать идею + получить план
3. AI-Curious Professionals: получить practical skills по product thinking + vibe-coding

## Карта конкурентного ландшафта

Конкуренты разбиты на 5 категорий:

1. **AI PRD-генераторы и продуктовые AI-инструменты** -- инструменты создания продуктовых документов
2. **AI-валидаторы идей и стартап-коучи** -- AI-помощники для валидации и планирования бизнеса
3. **Курсы/буткемпы вайб-кодинга** -- обучение созданию продуктов с AI
4. **AI-coding платформы (инструменты)** -- инструменты для непосредственного создания продуктов
5. **Образовательные платформы по продуктовому менеджменту** -- курсы product thinking

## Ключевые выводы

### 1. Главный гэп: никто не объединяет product thinking + vibe-coding обучение + AI-документогенерацию

Рынок четко разделен:
- **AI PRD-инструменты** (ChatPRD, PRDGPT) генерируют документы, но не учат думать и не учат реализовывать
- **AI-валидаторы** (IdeaProof, DimeADozen, ValidatorAI) помогают понять "стоит ли строить", но не помогают строить
- **Вайбкодинг-курсы** (Buildcamp, Scrimba, Product School) учат строить, но не учат думать о продукте
- **AI-coding платформы** (Lovable, Cursor) -- чистые инструменты без обучения

Ни один конкурент не решает full job: "Я хочу сделать продукт. Научи меня думать правильно, помоги написать документы, и покажи, как это реализовать через вайбкодинг."

### 2. Ближайший конкурент -- LeanPivot.ai

LeanPivot.ai наиболее близок к нашей концепции: AI-коуч + инструменты + стадийный подход + даже VIBE Coding Tool. Однако ключевые отличия:
- LeanPivot использует Lean Startup, не AJTBD -- другая методологическая основа
- LeanPivot перегружен инструментами (64 штуки) без глубокого обучения
- Нет интеграции с реальными coding-платформами (Lovable, Cursor, Claude Code)
- Обучение вайбкодингу поверхностное (один инструмент, а не микро-гайды)
- Много функций "Coming Soon"
