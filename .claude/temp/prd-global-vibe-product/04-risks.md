# Рискованные предположения v2 (Risk Assumption Test)

Формула: **Score = (P × I) / C**
- P (1-5) — вероятность наступления риска
- I (1-5) — стоимость последствий
- C (1-5) — стоимость проверки

---

### Риск 1: ChatGPT/Claude как прямой конкурент

**Предположение:** Люди готовы платить $29-49/мес за structured AI-guided experience, когда могут получить "80% результата" через бесплатный ChatGPT/Claude с правильным промптом.

**Риск:** Если нет значимой разницы между нашим продуктом и "промпт в ChatGPT", продукт мёртв. Нулевая конверсия free→paid.

**Категория:** Готовность сегмента купить (ценностная гипотеза)

**P (вероятность):** 4/5 — Очень высокая. Люди уже активно используют ChatGPT для написания PRD. Промпты типа "write me a PRD for..." дают 70-80% результата бесплатно.
**I (последствия):** 5/5 — Убьёт бизнес. Если нет WTP, продукта не существует.
**C (стоимость проверки):** 2/5 — Дешёво: smoke-тест лендинг + pre-order → можно проверить за $200-500 на рекламу + 1 неделю.

**Score = 4×5/2 = 10.0**

**Методы проверки:**
1. Smoke-тест: лендинг с waitlist + описание ценности → measure конверсия
2. Сравнительный тест: дать 20 людям ChatGPT-промпт vs наш гайд, сравнить результаты и NPS
3. Pre-order: предложить $29 early-bird → измерить CR
4. Интервью с 10 людьми, которые уже пробовали "PRD через ChatGPT": чего не хватило?

### Риск 2: Качество AI-guided experience

**Предположение:** AI-гайд способен давать quality of guidance сопоставимый с живым экспертом/ментором на уровне достаточном, чтобы пользователь дошёл до результата (PRD + первый прототип).

**Риск:** Если AI даёт generic/поверхностные рекомендации, пользователи бросают на середине. Retention = 0, repeat usage = 0.

**Категория:** Операционные / технологические

**P (вероятность):** 3/5 — Средне-высокая. LLM хороши в структурированных задачах, но персонализация + обучение "на лету" — сложнее. Зависит от качества промптов и системы.
**I (последствия):** 5/5 — Убьёт продукт. "AI-гайд" без качественного гайдинга = пустой маркетинг.
**C (стоимость проверки):** 3/5 — Средне: нужно построить MVP-прототип (2-3 недели) и протестировать на 10-20 людях. Или начать с "Wizard of Oz" (human-in-the-loop).

**Score = 3×5/3 = 5.0**

**Методы проверки:**
1. Wizard of Oz: вручную вести 5-10 людей через процесс (человек + AI), замерить completion rate
2. MVP-прототип: минимальная версия гайда (3-5 шагов), тест на 20 людях
3. A/B: AI-гайд vs просто чеклист с видео — кто доходит до результата чаще?
4. NPS после прохождения первого цикла

### Риск 3: WTP за подписку (Freemium → Paid конверсия)

**Предположение:** Конверсия free→paid составит ≥8-10% при ценнике $29-49/мес, что обеспечит sustainable unit economics.

**Риск:** Если free-версия даёт "достаточно", люди не конвертируются. Или ценник слишком высок для education product. CAC > LTV.

**Категория:** Юнит-экономика (маржинальность)

**P (вероятность):** 3/5 — Средне-высокая. Типичная конверсия freemium = 2-5%. Нужно 8-10% для сходимости.
**I (последствия):** 4/5 — Сломает экономику. Без конверсии нет revenue.
**C (стоимость проверки):** 2/5 — Дешёво: fake door test, pre-order, early access за деньги.

**Score = 3×4/2 = 6.0**

**Методы проверки:**
1. Fake door: на лендинге кнопка "Get Pro" → checkout page → measure intent
2. Pre-order campaign: $29 early-bird access → measure CR
3. Интервью с 10 целевыми: "сколько бы вы заплатили за X?"
4. Тест разных ценников ($19 vs $29 vs $49) на split landing pages

### Риск 4: Completion Rate (дохождение до результата)

**Предположение:** ≥40% начавших пользователей дойдут до конца (получат PRD + план вайбкодинга). Это критично для retention и word-of-mouth.

**Риск:** Типичный completion rate онлайн-курсов = 3-15%. Если наш guided experience не значительно лучше, 85%+ бросят на середине.

**Категория:** Масштабирование спроса

**P (вероятность):** 4/5 — Высокая. Completion rate — историческая проблема всех education products. Даже "learn by doing" не гарантирует.
**I (последствия):** 4/5 — Сильно снижает WoM, retention, LTV. Люди не рекомендуют то, что не дошли.
**C (стоимость проверки):** 3/5 — Средне: нужен MVP + 50-100 пользователей для статистики.

**Score = 4×4/3 = 5.3**

**Методы проверки:**
1. MVP с 50+ пользователями: track funnel drop-off по шагам
2. Cohort analysis: где именно бросают (step 2? step 5?)
3. Интервью с "бросившими": почему остановились?
4. A/B: разные форматы (short vs long, с/без видео, с/без чата)

### Риск 5: Контент + методология не переносимы на self-service AI

**Предположение:** Методология AURA (отработанная на 11000 выпускников в формате живых курсов) может быть эффективно перенесена в self-service AI-формат без потери качества.

**Риск:** Методология может работать только с живым ведущим (контекстуальная обратная связь, мотивация, давление группы). AI-версия даёт "бледную копию".

**Категория:** Операционные / технологические

**P (вероятность):** 3/5 — Средняя. Часть методологии алгоритмизируема (вопросы, шаблоны). Но "инсайты на основе контекста" — сложнее.
**I (последствия):** 4/5 — Если методология не переносима, теряем ключевое преимущество.
**C (стоимость проверки):** 2/5 — Дешёво: создать AI-версию одного модуля + тест на 10 людях vs живой ведущий.

**Score = 3×4/2 = 6.0**

**Методы проверки:**
1. Пилот: один модуль (сегментация) в AI-формате vs тот же модуль с куратором
2. NPS/качество output сравнить: AI-generated segments vs curator-guided segments
3. Тест с выпускниками BOOST: "Попробуйте AI-версию, сравните с опытом интенсива"
4. Экспертная оценка output: пригласить 3-5 экспертов оценить качество AI-guided PRD

### Риск 6: Retention (возвращаемость)

**Предположение:** Пользователи будут возвращаться с новыми идеями/проектами, обеспечивая monthly retention ≥30% на M3.

**Риск:** Продукт — "одноразовый": получил PRD → ушёл. Нет причины возвращаться. Это убивает подписочную модель.

**Категория:** Юнит-экономика (маржинальность)

**P (вероятность):** 4/5 — Высокая. PRD = разовый output. Если нет recurring value, подписка не работает.
**I (последствия):** 4/5 — Убьёт unit economics. Без retention LTV = 1 месяц × $29 = $29. При CAC $20+ бизнес не сходится.
**C (стоимость проверки):** 3/5 — Средне: нужна когорта 50-100 людей на 2-3 месяца.

**Score = 4×4/3 = 5.3**

**Методы проверки:**
1. Cohort analysis после MVP: M1/M2/M3 retention
2. Интервью с users после первого проекта: "Зачем вернуться?"
3. Тест recurring value: добавить weekly insights, progress tracking, community → retention impact
4. Survey: "Что заставило бы вас продолжить подписку после получения PRD?"

### Риск 7: Каналы привлечения для англоязычного рынка

**Предположение:** Можем привлекать англоязычных пользователей по CAC ≤ $20 через SEO/GEO + контент + paid ads.

**Риск:** Англоязычный рынок education/AI крайне конкурентный по рекламе. CAC может быть $50-100+. При LTV $87 (3 мес × $29) бизнес не сходится.

**Категория:** Масштабирование спроса

**P (вероятность):** 3/5 — Средняя. SEO/GEO может работать для нишевых запросов. Paid ads в education = дорого.
**I (последствия):** 3/5 — Не убьёт продукт, но замедлит рост. Можно скорректировать каналы.
**C (стоимость проверки):** 2/5 — Дешёво: $500 тестовый бюджет на Google Ads / Meta → CPC, CTR, CAC.

**Score = 3×3/2 = 4.5**

**Методы проверки:**
1. Google Ads тест: $200-500 бюджет → CPC для ключевых запросов (vibe coding PRD, AI product requirements)
2. SEO research: конкурентность ключевых запросов (Ahrefs/SEMrush)
3. Organic content test: 5 статей/постов → track organic traffic
4. Product Hunt launch: бесплатный канал, замерить traffic + conversion

### Риск 8: Рынок "learn to build" слишком фрагментирован

**Предположение:** Есть достаточно большой consolidated market для "AI-guided product thinking + vibe-coding education". Люди ищут именно это сочетание.

**Риск:** Рынок разделён: люди ищут ИЛИ "product management course" ИЛИ "learn to code with AI" — но не оба одновременно. Наш гибрид не попадает ни в один сформировавшийся запрос.

**Категория:** Спрос на рынке

**P (вероятность):** 3/5 — Средняя. Тренд "build your own product" растёт, но пока не сформировался как категория.
**I (последствия):** 3/5 — Если нет единого спроса, придётся продавать два продукта, не один. Сложнее, но не смертельно.
**C (стоимость проверки):** 1/5 — Очень дешёво: keyword research + landing page test с разными позиционированиями.

**Score = 3×3/1 = 9.0**

**Методы проверки:**
1. Keyword research: есть ли запросы типа "AI product building guide", "vibe coding PRD"?
2. Landing page split-test: "Product Thinking AI" vs "Vibe Coding Guide" vs "Build Your Product" — какой конвертирует лучше?
3. Reddit/Twitter research: как люди формулируют эту потребность?
4. Competitor analysis: есть ли кто-то, кто уже объединяет product thinking + vibe-coding?

### Риск 9: Vibe-coding micro-guides быстро устаревают

**Предположение:** Micro-guides по Cursor, Claude Code, Lovable будут актуальны достаточно долго (>3 месяца), чтобы оправдать инвестицию в создание.

**Риск:** AI-инструменты обновляются каждые 2-4 недели. Гайд "как настроить Cursor" может устареть через месяц. Постоянное обновление = высокие операционные расходы.

**Категория:** Операционные

**P (вероятность):** 4/5 — Высокая. AI-инструменты обновляются стремительно.
**I (последствия):** 2/5 — Не убьёт продукт, но увеличит cost-to-serve. Может снизить NPS.
**C (стоимость проверки):** 1/5 — Очень дешёво: просто отследить частоту breaking changes в Cursor/Claude Code за последние 3 месяца.

**Score = 4×2/1 = 8.0**

**Методы проверки:**
1. Audit: проверить changelog Cursor, Claude Code, Lovable за последние 6 месяцев — сколько breaking changes?
2. Модульная архитектура: micro-guides как отдельные модули, обновляемые независимо
3. Community-driven updates: позволить пользователям отмечать устаревший контент
4. Оценить: можно ли генерировать micro-guides автоматически через AI?
