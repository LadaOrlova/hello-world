# PRD: LaunchPilot -- AI-платформа "от идеи до работающего продукта"

Дата: 2026-03-01
Версия: 1.0

---

## 1. Обзор продукта

### Название

**LaunchPilot** -- AI-powered platform that turns your idea into a working product.

### Цель продукта

Глобальная англоязычная AI-платформа, которая ведёт человека от идеи до работающего задеплоенного продукта. Стратегия "убить работу обучения" -- человек не учится делать продукт, а делает продукт, система учит по ходу. Платформа одновременно является инструментом и образовательной средой.

### Ценность (Value Proposition)

Единственный продукт на рынке, который объединяет:
1. **AI-guided experience** -- пошаговый AI-гайд от идеи до деплоя (чего нет у Bolt, Replit, Lovable)
2. **Продуктовое мышление** -- встроенная методология AJTBD: что строить, для кого, как валидировать (чего нет у курсов и инструментов)
3. **Self-service формат** -- без расписания, без когорт, без живого наставника (чего нет у Maven, FI Bootcamp)

## 2. Целевые пользователи

### Сегмент 1: Non-technical Founders "0->1" (приоритетный)

**Кто они:** 25-45 лет, бизнес/маркетинг-бэкграунд, без опыта программирования. Увидели в Twitter/X или YouTube, как кто-то собрал приложение за выходные с помощью AI. Чувствуют FOMO и возбуждение.

**Jobs:**
- Когда есть бизнес-идея и нет tech-навыка -- превратить идею в работающий прототип -- проверить, будут ли платить
- Когда нанять разработчиков дорого ($10-50K) -- собрать MVP самостоятельно -- не тратить деньги до валидации
- Когда не знаю с чего начать -- получить пошаговый план от идеи до запуска -- не терять месяцы

**Критерии успеха:**
- Работающий прототип за < 2 недели (не макет, а реально работающий продукт)
- Получить первых пользователей / первую обратную связь
- Понять, стоит ли продолжать инвестировать
- Не потратить > $500 на этапе валидации

### Сегмент 3: Solopreneurs / Side-hustlers (приоритетный)

**Кто они:** 25-45 лет, разработчики/маркетологи/дизайнеры/аналитики с основной работой. Часто уже пробовали что-то запустить и не довели до конца. Подписаны на Indie Hackers, r/SaaS, Pieter Levels, Marc Lou.

**Jobs:**
- Когда хочу дополнительный доход -- построить SaaS/digital product -- не зависеть от одного работодателя
- Когда ограничено время (основная работа, семья) -- получить максимально эффективную систему -- не терять месяцы
- Когда вижу успешных indie hackers -- повторить путь системно -- в перспективе уволиться

**Критерии успеха:**
- Построить и запустить первый продукт за < 1 месяц (работая по вечерам/выходным)
- Получить первых платящих клиентов (хотя бы $100 MRR)
- Не тратить > $100/мес на инструменты
- Async-режим, без привязки к расписанию

### Сегмент 2: Career Builders (вторичный)

**Кто они:** PM, продуктовые дизайнеры, бизнес-аналитики, 28-40 лет, в tech-компаниях США, UK, Европы. Карьерная тревога из-за AI-трансформации.

**Jobs:**
- Когда AI меняет индустрию -- освоить AI-инструменты на практике -- оставаться конкурентоспособным
- Когда коллега показал прототип, собранный за день -- получить структурированную систему обучения -- не тратить недели на YouTube

**Критерии успеха:**
- Собрать работающий прототип за < 1 неделю
- Получить навыки для резюме (AI product building, vibe-coding)
- Учиться в свободное время, без расписания

### Сегмент 4: Builders в организации (вторичный)

**Кто они:** Сотрудники компаний, видящие возможность автоматизировать процессы или создать внутренние инструменты.

**Jobs:**
- Когда компании нужны внутренние инструменты -- собрать их самостоятельно -- не зависеть от IT-отдела
- Когда хочу приносить больше ценности организации -- освоить создание продуктов -- стать драйвером инноваций

**Критерии успеха:**
- Работающий внутренний инструмент за < 2 недели
- Показать результат руководству

### Сегмент 5: AI Future-Proofers (вторичный)

**Кто они:** Профессионалы, обеспокоенные тем, что AI меняет их профессию. Хотят "быть на волне".

**Jobs:**
- Когда AI угрожает моей профессии -- научиться использовать AI как инструмент -- превратить угрозу в преимущество

**Критерии успеха:**
- Практические навыки работы с AI-инструментами
- Уверенность в своей конкурентоспособности

## 3. Функциональные требования

### 3.1. Core Job 1: "Сформулировать, что именно строить" (Product Thinking)

**Связь с сегментами:** Founders 0->1, Solopreneurs
**Незакрытый gap на рынке:** Ни один конкурент не помогает сформулировать идею, выбрать сегмент, определить jobs

#### Критическая последовательность работ (КПР):

1. **Описать идею продукта**
2. **Определить целевого пользователя и его работу (job)**
3. **Сформулировать ценность (value proposition)**
4. **Определить MVP-скоуп (что строить первым)**

#### Функциональные требования:

**FR-1.1. AI-интервью для формулировки идеи**

Пользователь может описать свою идею в свободной форме (текст или голос), и AI задаёт уточняющие вопросы, чтобы сформулировать структурированное описание продукта.

- Критерии приёмки:
  - AI задаёт не менее 5 уточняющих вопросов по ключевым аспектам: кто пользователь, какую проблему решает, чем отличается от существующих решений
  - На выходе формируется структурированный документ с: описанием идеи, целевым пользователем, ключевой проблемой, предлагаемым решением
  - Время диалога: 5-10 минут
  - Пользователь может ввести ответы текстом (голосовой ввод -- в следующих итерациях)
- Связь с Aha-моментом: шаг 1 из guided flow. Пользователь впервые видит свою сырую идею структурированной

**FR-1.2. Генерация PRD (Product Requirements Document)**

Пользователь может получить PRD своего продукта на основе результатов AI-интервью.

- Критерии приёмки:
  - PRD содержит: описание продукта, целевая аудитория, ключевые фичи MVP, user stories, критерии успеха
  - PRD генерируется автоматически после завершения AI-интервью
  - Пользователь может редактировать и итерировать PRD через диалог с AI
  - Формат: Markdown, доступен для копирования и скачивания
- Связь с Aha-моментом: шаг 2. "PRD моего продукта за 10 минут!"

**FR-1.3. Определение целевого сегмента и jobs (упрощённый AJTBD)**

Пользователь получает от AI рекомендации по целевому сегменту: кто его пользователи, какие у них работы (jobs), какие критерии успеха.

- Критерии приёмки:
  - AI генерирует 2-3 гипотезы сегментов с описанием jobs в формате "Когда [контекст], хочу [действие], чтобы [результат]"
  - Для каждого сегмента указываются: размер (оценка), острота проблемы, конкурентные альтернативы
  - Пользователь выбирает приоритетный сегмент для MVP
  - Рекомендация подкрепляется объяснением "почему этот сегмент" (обучение по ходу)
- Связь с Aha-моментом: обучающий элемент, встроенный в процесс -- пользователь учится продуктовому мышлению, пока делает свой продукт

**FR-1.4. MVP-скоупинг**

Пользователь может получить рекомендацию, что включить в MVP, а что отложить на потом.

- Критерии приёмки:
  - AI разделяет фичи из PRD на: "Must have для MVP" и "Отложить"
  - Обоснование каждого решения (почему эта фича важна/не важна для первой версии)
  - Оценка объёма MVP: "Это можно собрать за N часов с помощью AI-кодинга"
  - Пользователь может переместить фичи между группами

### 3.2. Core Job 2: "Построить работающий прототип" (AI-Coding)

**Связь с сегментами:** Все сегменты
**Конкурентный паритет с:** Bolt.new, Lovable, Replit

#### Критическая последовательность работ (КПР):

1. **Сгенерировать код лендинга**
2. **Сгенерировать код MVP-приложения**
3. **Предпросмотреть результат**
4. **Итерировать через диалог с AI**

#### Функциональные требования:

**FR-2.1. Генерация лендинга из PRD**

Пользователь может одним действием превратить PRD в работающий лендинг.

- Критерии приёмки:
  - Лендинг генерируется на основе PRD: заголовок, описание проблемы, решение, CTA, отзывы (placeholder)
  - Используется современный стек: HTML/CSS/JS или React/Next.js + Tailwind
  - Лендинг адаптивный (mobile-responsive)
  - Время генерации: < 2 минут
  - Результат доступен для предпросмотра прямо в платформе
- Связь с Aha-моментом: шаг 3 -- "Мой лендинг за 2 минуты!" -- это ключевой Aha-момент для сегментов 1, 2, 5

**FR-2.2. Генерация MVP-приложения из PRD**

Пользователь может превратить PRD в работающий прототип приложения.

- Критерии приёмки:
  - AI генерирует full-stack приложение на основе PRD и выбранного MVP-скоупа
  - Стек: Next.js + Tailwind CSS + PostgreSQL (или SQLite для простоты MVP)
  - Приложение включает: базовый UI, роутинг, работу с данными
  - Результат доступен для предпросмотра в браузере
  - Время генерации MVP: < 10 минут
- Связь с Aha-моментом: шаг 4 -- "Работающее приложение из моей идеи!"

**FR-2.3. Итеративная доработка через диалог**

Пользователь может дорабатывать сгенерированный код через текстовые инструкции на естественном языке.

- Критерии приёмки:
  - Пользователь пишет инструкцию на естественном языке: "Добавь форму регистрации", "Сделай кнопку синей", "Добавь страницу с ценами"
  - AI вносит изменения в код и показывает обновлённый предпросмотр
  - История изменений сохраняется (возможность откатить)
  - AI объясняет, что изменил и почему (обучение по ходу)

**FR-2.4. Предпросмотр в реальном времени**

Пользователь видит результат генерации в реальном времени без необходимости настройки среды разработки.

- Критерии приёмки:
  - Встроенный preview в браузере (iframe или аналог)
  - Обновление preview после каждого изменения: < 5 секунд
  - Поддержка мобильного предпросмотра (переключение viewport)
  - Работает без установки чего-либо на компьютер пользователя

### 3.3. Core Job 3: "Задеплоить и показать людям" (Launch)

**Связь с сегментами:** Founders 0->1, Solopreneurs
**Конкурентный паритет с:** Bolt.new (встроенный деплой), Lovable (one-click деплой)

#### Критическая последовательность работ (КПР):

1. **Задеплоить приложение на публичный URL**
2. **Подключить домен (опционально)**
3. **Поделиться ссылкой**

#### Функциональные требования:

**FR-3.1. One-click деплой**

Пользователь может задеплоить свой продукт на публичный URL одним действием.

- Критерии приёмки:
  - Кнопка "Deploy" / "Publish" в интерфейсе
  - Деплой на поддомен платформы (app-name.launchpilot.app) -- free tier
  - HTTPS включён по умолчанию
  - Время деплоя: < 3 минут
  - Пользователь получает публичную ссылку для шеринга
  - Приложение доступно 24/7

**FR-3.2. Подключение кастомного домена (платный тир)**

Пользователь может подключить свой домен к задеплоенному приложению.

- Критерии приёмки:
  - Инструкция по настройке DNS
  - Автоматическая генерация SSL-сертификата
  - Домен начинает работать в течение < 1 часа после настройки DNS

### 3.4. Core Job 4: "Валидировать идею и получить обратную связь" (Validation)

**Связь с сегментами:** Founders 0->1, Solopreneurs
**Незакрытый gap на рынке:** Только FI Bootcamp частично закрывает эту job

#### Критическая последовательность работ (КПР):

1. **Настроить аналитику**
2. **Получить первый трафик**
3. **Собрать обратную связь**
4. **Итерировать продукт**

#### Функциональные требования:

**FR-4.1. Встроенная аналитика (PostHog)**

Пользователь может видеть базовые метрики своего продукта без настройки внешних инструментов.

- Критерии приёмки:
  - PostHog интегрирован по умолчанию при деплое
  - Пользователь видит: количество посетителей, источники трафика, конверсия по ключевым действиям
  - Dashboard доступен в интерфейсе платформы
  - Не требует настройки от пользователя

**FR-4.2. AI-рекомендации по валидации**

Пользователь получает пошаговый план валидации своей идеи после деплоя.

- Критерии приёмки:
  - AI генерирует чек-лист: "Как получить первых 10 пользователей"
  - Рекомендации адаптированы к типу продукта (SaaS, marketplace, tool)
  - Включает: где найти пользователей, что спросить, как интерпретировать обратную связь
  - Встроенные шаблоны для сбора фидбека (простая форма или email)

**FR-4.3. AI-помощник по итерированию**

Пользователь может получить рекомендации по улучшению продукта на основе собранной обратной связи.

- Критерии приёмки:
  - Пользователь вводит фидбек от пользователей (текст)
  - AI анализирует фидбек и предлагает конкретные изменения в продукте
  - Рекомендации привязаны к jobs целевого сегмента (из PRD)
  - Пользователь может применить рекомендации одним кликом (AI вносит изменения в код)

### 3.5. Core Job 5: "Монетизировать продукт" (Monetization)

**Связь с сегментами:** Solopreneurs, Founders 0->1
**Конкурентный паритет с:** Maven VC (Stripe), Replit (Stripe integration)

#### Функциональные требования:

**FR-5.1. Подключение оплаты (Stripe)**

Пользователь может подключить Stripe к своему продукту для приёма платежей.

- Критерии приёмки:
  - Guided flow: пошаговая инструкция по подключению Stripe
  - AI генерирует код для интеграции (checkout page, pricing page)
  - Поддержка одноразовых платежей и подписок
  - Тестовый режим Stripe для проверки до go-live

### 3.6. Core Job 6: "Освоить AI-инструменты" (Learning)

**Связь с сегментами:** Career Builders, AI Future-Proofers, Builders в организации
**Стратегия:** "Убить работу обучения" -- обучение встроено в процесс создания

#### Функциональные требования:

**FR-6.1. Контекстное обучение (Learning-by-Doing)**

Пользователь получает образовательный контент именно в тот момент, когда он релевантен текущему шагу.

- Критерии приёмки:
  - На каждом шаге guided flow AI объясняет: "Что мы делаем", "Почему именно это", "Какой принцип за этим стоит"
  - Объяснения короткие (2-3 предложения), с возможностью "Узнать больше"
  - Покрываемые темы: продуктовое мышление (JTBD, сегментация, MVP), основы вайб-кодинга (промптинг, стеки, деплой)
  - Обучение не блокирует прогресс -- пользователь может пропустить объяснение

**FR-6.2. Библиотека знаний**

Пользователь может обращаться к структурированной базе знаний по продуктовому мышлению и вайб-кодингу.

- Критерии приёмки:
  - Разделы: "Как формулировать идею", "Как определить целевую аудиторию", "Как валидировать", "Основы вайб-кодинга", "Как деплоить", "Как монетизировать"
  - Формат: короткие статьи (3-5 минут чтения) с примерами
  - Контент доступен бесплатно (часть free tier)
  - Поиск по библиотеке

### 3.7. Guided Flow -- центральная механика продукта

**Связь с рисками:** Риск #1 (guided experience не даёт результата)

#### Функциональные требования:

**FR-7.1. Пошаговый guided flow "от идеи до деплоя"**

Пользователь проходит структурированный путь из 5-7 шагов, который приводит к конкретному результату -- задеплоенному продукту.

- Критерии приёмки:
  - Шаги guided flow:
    1. "Расскажи о своей идее" (AI-интервью, 5-10 мин)
    2. "Вот PRD твоего продукта" (генерация, 2 мин)
    3. "Вот лендинг твоего продукта" (генерация + предпросмотр, 2 мин) -- **Aha-момент**
    4. "Вот рабочий прототип" (генерация MVP, 5-10 мин)
    5. "Доработай под себя" (итерация через диалог, 10-20 мин)
    6. "Опубликуй" (one-click деплой, 2 мин)
    7. "Получи первых пользователей" (план валидации)
  - На каждом шаге: прогресс-бар, объяснение "что дальше", кнопка "Продолжить"
  - Пользователь может вернуться к любому шагу
  - Общее время прохождения: 30-60 минут до задеплоенного продукта
  - Flow не блокирующий: пользователь может пропустить шаг или вернуться позже

**FR-7.2. Механика "не дать застрять" (Anti-Stuck)**

Продукт активно помогает пользователю преодолеть точки, где он может бросить.

- Критерии приёмки:
  - AI обнаруживает, что пользователь "завис" (> 3 минут бездействия на одном шаге) и предлагает помощь
  - На шагах с высокой вероятностью "застревания" (формулировка идеи, итерация кода) AI предлагает: примеры, шаблоны, подсказки
  - Если AI-генерация выдаёт ошибку -- автоматическое исправление с объяснением
  - "Escape hatch" на каждом шаге: "Я застрял, помоги" -- AI диагностирует проблему и предлагает решение
  - Предзаполненные шаблоны для тех, у кого нет идеи: "Попробуй собрать [SaaS для фрилансеров / landing page builder / habit tracker]"

**FR-7.3. Progress tracking и Milestones**

Пользователь видит свой прогресс и получает positive reinforcement.

- Критерии приёмки:
  - Visual progress bar: "Шаг 3 из 7: Лендинг сгенерирован"
  - Milestones с celebration: "Поздравляем! Твой PRD готов", "Твой продукт в интернете!"
  - History: список всех проектов пользователя с их статусом
  - Статистика: "Ты создал 2 проекта, задеплоил 1"

### 3.8. Retention-механики

**Связь с рисками:** Риск #6 (низкий retention)

#### Функциональные требования:

**FR-8.1. Множественные проекты**

Пользователь может создавать несколько проектов (идей/продуктов) в одном аккаунте.

- Критерии приёмки:
  - Dashboard с карточками проектов
  - Каждый проект -- независимый guided flow
  - Quick start: "Создать новый проект" -- 1 клик
  - Free tier: до 3 проектов

**FR-8.2. Итерационные задачи (Next Steps)**

После первого деплоя AI предлагает следующие шаги для развития продукта.

- Критерии приёмки:
  - Чек-лист "Что делать дальше": добавить аналитику, собрать фидбек, подключить оплату, улучшить дизайн
  - Каждый элемент чек-листа -- кликабельный, ведёт в соответствующий guided flow
  - Персонализация: рекомендации зависят от типа продукта и выбранного сегмента
  - Email-напоминания: "Ты задеплоил продукт 3 дня назад. Получил первых пользователей?"

**FR-8.3. Email-nudges и accountability**

Продукт поддерживает вовлечённость через email-напоминания.

- Критерии приёмки:
  - Автоматические email:
    - D+1 после регистрации без завершения guided flow: "Ты начал -- продолжи!"
    - D+3 после деплоя: "Как дела с первыми пользователями?"
    - D+7: "Пришло время итерировать. Вот что можно улучшить"
    - D+14: "Готов подключить оплату?"
  - Пользователь может отписаться от nudges
  - Частота: не чаще 2 раз в неделю

## 7. Учёт рисков

### Риск #1 (Score 10.0): Guided AI experience не даёт результата

**Как PRD минимизирует этот риск:**

1. **Anti-stuck механики (FR-7.2):** AI обнаруживает бездействие пользователя и проактивно предлагает помощь. Предзаполненные шаблоны для тех, кто не знает с чего начать.

2. **Автоматическое исправление ошибок:** Если сгенерированный код не работает, AI автоматически диагностирует и исправляет ошибку, объясняя, что произошло.

3. **Максимально короткий guided flow (7 шагов):** Меньше шагов = меньше мест, где можно застрять. Каждый шаг -- конкретный, атомарный результат.

4. **"Escape hatch" на каждом шаге:** Кнопка "Я застрял" вызывает AI-помощника, который диагностирует проблему.

5. **Предвыбранный стек и инфраструктура:** Пользователь не выбирает технологии, не настраивает среду, не деплоит вручную -- всё автоматизировано.

6. **Scaffolding:** На первых шагах AI делает 80% работы, постепенно пользователь берёт на себя больше (аналогия с BOOST Intensive).

**Метрика валидации:** Completion Rate guided flow (цель: > 50% пользователей, начавших flow, доходят до деплоя).

### Риск #2 (Score 7.5): Не готовы платить за self-service подписку

**Как PRD минимизирует этот риск:**

1. **Aha-момент до paywall:** Пользователь получает работающий лендинг и прототип БЕСПЛАТНО, до любых запросов оплаты. Aha-момент сдвинут на 10-15 минуту первой сессии.

2. **Чёткая граница free/paid:**
   - Free tier: 1 полный guided flow (от идеи до деплоя), 3 проекта, деплой на поддомен платформы, базовая аналитика
   - Paid tier ($29-49/мес): неограниченные проекты, кастомный домен, продвинутая аналитика, Stripe-интеграция, приоритетная генерация

3. **Ценность paid видна из free:** В free tier AI показывает, что доступно в paid ("Хочешь подключить свой домен? Это доступно в Pro").

4. **Fallback на разовые покупки:** Если подписка не работает -- возможность добавить one-time покупки (например, "Launch Pack" за $99 -- деплой + домен + аналитика на 3 месяца).

**Метрика валидации:** Free-to-Paid conversion rate (цель: > 3%).

### Риск #6 (Score 5.3): Низкий retention

**Как PRD минимизирует этот риск:**

1. **Множественные проекты (FR-8.1):** Причина возвращаться -- "У меня ещё идея, давай соберу и её".

2. **Next Steps после деплоя (FR-8.2):** AI не бросает пользователя после первого результата, а предлагает: "Собери фидбек -> Итерируй -> Подключи оплату -> Масштабируй".

3. **Email-nudges (FR-8.3):** Автоматические напоминания возвращают пользователя через D+1, D+3, D+7, D+14.

4. **Итерационная петля:** Продукт спроектирован так, что каждый результат (деплой) создаёт следующую задачу (валидация), которая создаёт следующую задачу (итерация), и так далее.

5. **Progression system:** Пользователь видит, на каком этапе находится его продукт: "Idea -> PRD -> Prototype -> Deployed -> Validated -> Monetized". Каждый следующий этап -- причина вернуться.

**Метрика валидации:** D7 retention > 30%, D30 retention > 15%.

## 9. Что НЕ входит в scope (MVP)

### Явные исключения

| Что исключено | Почему |
|---------------|--------|
| Мобильное приложение | MVP работает как web-app. Мобильная версия -- после валидации |
| Community / форум / чат между пользователями | Команда 1-3 чел. не может модерировать community. Рассмотреть после 1000+ пользователей |
| Живые воркшопы / менторство | Стратегия -- self-service. Живые сессии масштабируются линейно, не подходят для 1-3 чел |
| Мобильные приложения (генерация) | MVP -- только web-приложения. React Native / Flutter -- следующий этап |
| Marketplace шаблонов | Требует комьюнити и модерации. MVP -- фиксированный набор шаблонов |
| Team collaboration | MVP -- single-user. Командные фичи -- после 1000+ пользователей |
| Интеграции (кроме Stripe и PostHog) | MVP -- минимум интеграций. Zapier, n8n -- следующий этап |
| White-label / Agency решение | Не core job для MVP-сегментов |
| AI-генерация бэкенд-логики (сложные API, ML) | MVP -- простые CRUD-приложения и лендинги. Сложная логика -- следующий этап |
| Голосовой ввод | Сложная фича, не критична для первой версии. Текстовый ввод достаточен |
| Мультиязычность платформы | MVP -- только английский язык |
| Экспорт кода (download project) | MVP -- код живёт на платформе. Экспорт -- в paid tier следующего этапа |
| Кастомизация AI-моделей | MVP -- одна модель (Claude). Выбор моделей -- следующий этап |
| A/B тестирование для пользовательских приложений | Продвинутая фича для масштабирования |
| CI/CD pipeline | MVP -- manual deploy. Автоматический деплой -- следующий этап |

### Ограничения MVP

- **Лимит на сложность генерируемых приложений:** MVP фокусируется на лендингах, простых SaaS (CRUD), портфолио-сайтах. Сложные приложения с ML, real-time, множественными API -- за пределами MVP.
- **Количество проектов:** Free tier: 3 проекта. Paid: 10 проектов.
- **AI-запросы:** Free tier: ограничение по количеству AI-запросов в день (аналогично Bolt.new).
- **Хостинг задеплоенных приложений:** Базовый план -- ограниченные ресурсы (достаточно для MVP/прототипа, не для production-нагрузки).
