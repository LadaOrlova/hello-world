# Bookconcept Notes — AI Changes PM Role (MtP)

## Паттерн подачи
КЕЙС → МАТЧАСТЬ → HOW TO

## Целевые сегменты (релевантные для MtP-статьи)
1. **Корпоративные PM** — Core Job: "иметь чёткую логику принятия решений", Big Job: "стать экспертом, влиять на решения"
2. **Product-lead стагнирующего продукта** — Core Job: "найти точки роста с AI"
3. **Фаундер 0->1** — Core Job: "быстро и дешёво валидировать идею"

## Принципы написания
- Язык понятный и продактам, и топ-менеджерам
- Кейсы из крупных компаний
- Фокус на аргументации, коммуникации, "продаже" решений внутри
- Встроенность в стратегию (не только UX/фичи)

## Язык и тон
- Книга на русском → перевод на английский
- Но статья для MtP = изначально на АНГЛИЙСКОМ
- Стиль MtP ПРИОРИТЕТНЕЕ стиля книги (по скиллу)

## Ключевые методологии из книги
- AURA Framework (каузальная цепочка рисков)
- AJTBD (Advanced Jobs to Be Done)
- RAT (Risk Assumption Test)
- ABCDX-сегментация
- Critical Chain
- Four Forces

## NOTE
Для MtP-статьи: НЕ злоупотреблять KB-терминами. Использовать идеи без жаргона. MtP-аудитория знает JTBD, но не знает AURA.
