# Скилл: Справочная мини-глава (Reference Chapter)

Этот скилл создаёт КОРОТКУЮ справочную главу. Никаких историй, никаких hook'ов — только практика.

## Входные данные

Пользователь даёт:
1. **Файл тезисов** — из папки `2. Theses/` (Theses-{название}.md)
2. **(Опционально) Инструкция** — поле "Инструкция от Вани Замесина"

---

## ПРОТОКОЛ СОХРАНЕНИЯ КОНТЕКСТА (Anti-Compaction)

⚠️ **КРИТИЧЕСКИ ВАЖНО:** При длинных сессиях Claude Code компактит контекст, теряя детали. Для защиты — **сохраняй промежуточные результаты на диск**.

### Папка временных файлов

В самом начале работы создай папку через Bash:
```bash
mkdir -p /Users/zamesinivan/Documents/WORK/Zamesin-Academy-Team/tmp/ref-{название-главы}/
```

### Файл пружинного контекста (_spring-context.md)

Обновляй после КАЖДОГО этапа:

```markdown
# Spring Context — Ref: {название главы}

## Текущий этап: {N}
## Сохранённые файлы
- `01-kb-notes.md` — выжимка из Knowledge Base
- `02-draft.md` — черновик справочной главы
```

### Расписание сохранения

| После этапа | Файл | Содержимое |
|-------------|------|------------|
| ЭТАП 1 | `01-kb-notes.md` | Определения, принципы, ошибки из Knowledge Base |
| ЭТАП 2 | `02-draft.md` | Черновик справочной главы |

### Правила

1. **После каждого этапа:** Сохрани результат + обнови `_spring-context.md`
2. **Перед каждым этапом:** Перечитай `_spring-context.md` и все указанные файлы

### ОБЯЗАТЕЛЬНЫЕ ССЫЛКИ НА ИСТОЧНИКИ

⚠️ **Железное правило:** Каждый упомянутый источник ОБЯЗАН иметь ссылку.

- **Knowledge Base** → полный путь к файлу (`Knowledge/AJTBD/fundamentals.md`)
- **Определения из методологии** → путь к файлу-источнику
- **"См. также"** → ссылки на конкретные главы или файлы

**НЕ ДОПУСКАЕТСЯ:** ссылаться на источник без указания конкретного файла или URL.

## Характеристики справочной главы

- **Длина:** ~1 страница A4 (300-500 слов)
- **Без историй и hook'ов**
- **Чисто практика:** что это + когда нужно + что делать
- **Минимум теории**
- **Максимум actionability**

## ЭТАП 2: Написать черновик справочной главы

**Структура (строго соблюдать):**

1. **Что это такое** — 2-4 предложения, определение (используй точную формулировку из Knowledge Base)
2. **Когда применяется** — список ситуаций (буллеты)
3. **Главная цель** — 1-2 предложения
4. **Алгоритм действий** — пронумерованные шаги
5. **Ключевые принципы** — 3-5 принципов одним предложением каждый
6. **Частые ошибки** — формат ❌ Ошибка → ✅ Решение
7. **См. также** — ссылки на связанные главы

## Стиль

### Что использовать
- Короткие предложения
- Буллеты и нумерованные списки
- Прямое обращение ("ты", "вы")
- Конкретика (цифры, названия)

### Чего избегать
- Длинных абзацев
- Историй и примеров (минимум)
- AI-паттернов
- Воды и теоретизирований
