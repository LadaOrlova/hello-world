# Скилл: Создание тезисов главы (Deep Research Edition)

Этот скилл превращает поток мыслей в РАСШИРЕННЫЕ тезисы через:
- **Deep Research (gpt-5.2-pro, reasoning: xhigh)** — заменяет expert-hunter + contrarian. Один мощный вызов вместо 16-20 отдельных WebSearch. First principles анализ + эксперты + контраргументы.
- **Knowledge-researcher (Claude agent)** — углубляет тезисы данными из локальной базы AURA/AJTBD.

Оба работают **ПАРАЛЛЕЛЬНО** → 30-40 минут вместо часа.

## Входные данные

Пользователь даёт:
1. **Название главы** (или тему)
2. **Поток мыслей** — неструктурированные идеи, голосовые заметки, тезисы

ИЛИ:

1. **Ссылку на существующий файл тезисов** (из `3-Writing-Book/2-Theses/`)
2. **Фидбэк** — что доработать, углубить, изменить

## ЭТАП 0: Определение режима работы (CREATION vs ITERATION)

**Выполни ДО ВСЕГО остального.** Определи, какой режим нужен:

### Как определить режим

| Сигнал | Режим |
|--------|-------|
| Пользователь дал Raw Thoughts / поток мыслей / новую тему | **CREATION** → иди к ЭТАП 1 |
| Пользователь дал ссылку на существующий файл `Theses-*.md` | **ITERATION** → иди к ЭТАП И1 |
| В файле тезисов заполнено поле `## 🔄 Фидбэк автора для итерации` | **ITERATION** → иди к ЭТАП И1 |
| Пользователь написал "доработай тезисы" / "углуби" / "поправь" | **ITERATION** → иди к ЭТАП И1 |

### Автодетект

1. Если пользователь дал путь к файлу — прочитай его
2. Проверь: есть ли секция `## 🔄 Фидбэк автора для итерации` с заполненным содержимым (НЕ placeholder)?
3. Проверь: есть ли метаданные `Версия:` в шапке?
4. Если да → **ITERATION mode**. Если нет → **CREATION mode**

### Маршрутизация

- **CREATION mode:** Этапы 1 → 2 → 3 → 4 → 5 (DR + Knowledge-researcher)
- **ITERATION mode:** Этапы И1 → И2 → И3 → И4 (точечная доработка)

### ЗАДАЧА 2: Контраргументы и проверка актуальности

Для КАЖДОГО авторского тезиса:
1. Найди СИЛЬНЫЕ контраргументы с URL — кто и почему не согласен

### ЗАДАЧА 3: First Principles Analysis

Проведи анализ из первых принципов (метод Elon Musk / Aristotle):

1. **Определи аксиомы** — что стало бесплатным или почти бесплатным в контексте темы главы:
   - Написание кода = ~$0
   - Исследования и анализ = ~$0
   - Когнитивный труд на задачу = ~$0
   - Координация = ~$0
   - Создание прототипа = ~$0
   - Другие аксиомы, СПЕЦИФИЧНЫЕ для темы главы

2. **Построй логические цепочки:**
   Если X стало бесплатным → что следует? → что следует из этого? → и ещё дальше?
   Иди до НЕОЧЕВИДНЫХ выводов — тех, которые не видны с первого взгляда.

3. **Выведи НОВЫЕ тезисы** — правила и принципы, которые раньше не работали или не существовали, а теперь стали фундаментально верными.

### ЗАДАЧА 4: Формулирование новых тезисов

На основе задач 1-3 сформулируй новые тезисы, которых НЕТ в авторских. Для каждого:

1. **Название тезиса** — чёткая формулировка
2. **Суть** — 3-10 предложений, раскрывающих тезис
3. **First Principles цепочка:**
   - Аксиома: что стало бесплатным/дешёвым
   - Цепочка: если X → то Y → то Z → НЕОЧЕВИДНЫЙ ВЫВОД
4. **Источник** — эксперт/исследование + URL
5. **Почему автор мог не упомянуть** — это новое? спорное? из смежной области?
6. **Связь с авторскими тезисами** — спорит / дополняет / расширяет / независимый

**Тест релевантности** (каждый новый тезис ОБЯЗАН пройти):
1. "Можно ли убрать из главы без потери ценности?" → Если да — НЕ включай

3. "Он уточняет, расширяет или переворачивает авторский тезис?" → Если нет — НЕ включай

### Проверка Coverage Score (один раз, без итераций)

Оцени по 6 критериям:

| Критерий | Вопрос | Оценка |
|----------|--------|--------|
| **Экспертное покрытие** | Найдены ли мнения ключевых экспертов? | 0-100% |
| **Свежесть источников** | >50% источников 2024-2026? | 0-100% |
| **Глубина тезисов** | Каждый тезис отвечает на "почему?" на 2+ уровня? | 0-100% |
| **Контраргументы** | Для ключевых тезисов есть контрпозиции? | 0-100% |
| **Knowledge Base** | Каждый тезис связан с механикой из Knowledge/? | 0-100% |
| **Actionability** | Читатель поймёт, что делать? | 0-100% |
| **Ссылки на источники** | У каждого источника есть URL/путь? | 0-100% |
| **🆕 Качество расширения** | Новые тезисы трансформируют понимание? | 0-100% |

**Средний Coverage Score** = среднее по 8 критериям.

### Действия по результатам

**Если Coverage Score ≥ 80%:** Переходи к ЭТАП 4. DR дал достаточно материала.

**Если Coverage Score < 80%:** Определи самые слабые критерии и ТОЧЕЧНО дополни:
- Если слабый KB — отправь knowledge-researcher на доработку (только конкретный вопрос)
- Если слабые контраргументы или свежесть — сделай 2-3 WebSearch самостоятельно
- НЕ перезапускай Deep Research — это дорого и долго
