# Скилл: Кейс → контент для каналов

Превращает готовый кейс выпускника в:
1. **Набор карточек** для Telegram-карусели (канал @zamesin, бот, «Поток»)
2. **Подпись к посту** (не дублирует карточки)
3. **HTML-письмо** для email-рассылки

---

## ЭТАП 0: Сбор входных данных

### 0.1. Найти кейс

Если пользователь не указал файл — спросить через AskUserQuestion:
"Какой кейс берём? Укажи файл или название выпускника."

Прочитать файл кейса. Если кейса нет в репо — попросить вставить текст.

### 0.2. Определить параметры (AskUserQuestion, 4 вопроса одновременно)

**Вопрос 1 — Цель:**
"Какая цель этого контента?"
- Доказать экспертизу (Recommended)
- Продать курс
- Привлечь на вебинар/мероприятие
- Широкий охват / первое касание

**Вопрос 2 — Целевой сегмент:**
"Для какого сегмента делаем?"
- Job 1: Запустить свой продукт (25%, фаундеры, собственники)
- Job 2: Получить систему вместо чуйки (30%, продакты)
- Job 3: Починить стагнирующий продукт (20%, PM + собственники)
- Job 5: Усилить маркетинг (10%, маркетологи, CMO)

**Вопрос 3 — Каналы (multiSelect: true):**
"Куда пойдёт контент?"
- Telegram @zamesin (карточки + подпись)
- Telegram-бот (карточки + подпись)
- Email-рассылка (HTML)
- Instagram (карточки + подпись)

**Вопрос 4 — Ссылка:**
"Куда ведём? Пришли ссылку."
- zamesin.ru/producthowto (на курс)
- Другая ссылка (напишу)

### 0.3. QR-код

После получения ответов — спросить через AskUserQuestion:
"Нужен QR-код на CTA-карточке?"
- Да
- Нет (Recommended)

## ЭТАП 2: Генерация карточек

### 2.1. Прочитать контекст

Обязательно прочитать:
1. Файл кейса (из ЭТАПА 0)
2. Файл сегмента из `1-Context/Segments-and-jobs/` — для выбранного Job
3. Правила кейс-постов: `4-Personal-files/Vladlena-Orlova/case-post-rules.md`
4. Структуры карточек: `4-Personal-files/Vladlena-Orlova/content-system/card-structures.md`
5. UTM-систему: `4-Personal-files/Vladlena-Orlova/content-system/utm-system.md`

### 2.2. Создать набор карточек

По выбранному подходу создать 7-9 карточек. Каждая карточка:

```
### КАРТОЧКА {N}: {название слайда}

{Текст карточки — крупный, короткий, читается с телефона}
```

**Правила текста на карточках:**
- Короткие фразы, крупный шрифт
- Одна мысль — одна карточка
- Цифры выделять
- Клиент — герой, не Замесин
- Последняя карточка — CTA со ссылкой (+ QR если запросили)

### 2.3. Создать подпись к посту

Подпись к посту для Telegram. **НЕ ДУБЛИРУЕТ** содержание карточек.

Подпись содержит:
- Контекст / мысль автора (2-3 предложения)
- Ссылка с UTM-меткой

### 2.4. Собрать UTM-метку

По правилам из `utm-system.md`:
```
{ссылка}?utm_source={source}&utm_medium=card&utm_campaign=case_{название}
```

Где `utm_source` определяется по выбранному каналу:
- @zamesin → `tg_zamesin`
- Бот → `tg_bot`
- Email → `email`
- Instagram → `ig`

**Если каналов несколько — для каждого своя UTM-ссылка.**

### 2.5. QR-код (если запросили)

Генерировать QR-код:
- **Стиль:** округлые линии (rounded)
- **Содержимое:** ссылка с UTM, где `utm_medium=qr`
- **Размещение:** на CTA-карточке (последней)

## ЭТАП 4: HTML-рассылка (если выбран канал Email)

Если email не выбран — пропустить этот этап.

### 4.1. Создать HTML-письмо

На основе тех же данных кейса создать email-рассылку. **Формат отличается от карточек:**

- **Заголовок:** результат + цифра (как в правилах кейс-постов)
- **Тело:** текстовое письмо, НЕ набор карточек. Структура:
  1. Результаты в цифрах (буллеты)
  2. Контекст (3-4 предложения)
  3. Ключевые действия (без нумерации шагов)
  4. Выводы (макс 3 пункта)
  5. Цитата клиента
  6. CTA-кнопка со ссылкой + UTM (`utm_source=email&utm_medium=email`)

- **HTML:** чистый, инлайновые стили, адаптивный. Без внешних CSS.
- **Ширина:** max-width 600px
- **Шрифты:** системные (Arial, Helvetica, sans-serif)
- **CTA-кнопка:** крупная, контрастная

### 4.2. Итерация рассылки

Показать HTML-превью автору. Спросить через AskUserQuestion:
"Что делаем с рассылкой?"
- Готово
- Дать фидбэк

## ЭТАП 5: Сохранение

### 5.1. Генерация slug

Короткий slug из названия кейса (3-4 слова, kebab-case, латиница).
Пример: `pleada-ajtbd-ai`, `lift-check-x3`.

### 5.2. Сохранить файлы

**Карточки для Telegram:**
`3-Marketing/Telegram-posts/{YYYY-MM-DD}-{slug}-carousel.md`

**Карточки для бота:**
`3-Marketing/Bot/{YYYY-MM-DD}-{slug}-cards.md`

**HTML-рассылка:**
`3-Marketing/Email/{YYYY-MM-DD}-{slug}-email.html`

Формат файла карточек:
```markdown
---
date: {YYYY-MM-DD}
case: {название кейса}
source: {ссылка на кейс на сайте, если есть}
target_segment: {Job N}
goal: {цель}
approach: {подход A/Б/В/Г/Д}
channels: [{список каналов}]
utm_links:
  tg_zamesin: {полная ссылка с UTM}
  tg_bot: {полная ссылка с UTM}
  email: {полная ссылка с UTM}
status: final
---

# {Заголовок}

## Подпись к посту

{подпись}

---

## Карточки

### КАРТОЧКА 1: {название}
{текст}

### КАРТОЧКА 2: {название}
{текст}

...
```

### 5.3. Вывод автору

После сохранения показать:
1. Финальные карточки одним блоком (готовы к использованию)
2. Подпись к посту
3. UTM-ссылки для каждого канала
4. Пути к сохранённым файлам
5. Если HTML — путь к файлу рассылки

## ЗНАНИЯ: Правила кейс-постов

Полные правила в `4-Personal-files/Vladlena-Orlova/case-post-rules.md`. Ключевое:

1. **Заголовок:** Результат + цифра. НЕ описание процесса.
2. **Структура:** Результаты → Контекст → Действия → Выводы (макс 3) → Цитата
3. **Клиент — герой.** Замесин = методология, клиент = действие и результат.
4. **Резать детали.** Только логика «проблема → действие → результат».

## ЗНАНИЯ: Структуры карточек

Полные структуры в `4-Personal-files/Vladlena-Orlova/content-system/card-structures.md`. Краткий выбор:

### Подход А: Контекст + поворотный момент + шаги
**Когда:** доказать экспертизу. **7-9 карточек.**
Hook → Контекст → Узнаёшь себя? → Поворотный момент → Шаг 1 → Шаг 2 → Результаты → Принцип → Цитата + CTA

### Подход Б: Точка Б + снижение барьеров
**Когда:** продать курс, снять страхи. **7 карточек.**
Hook (результат) → Ещё цифры → Было (Точка А) → Барьер 1 → Барьер 2 → Барьер 3 → Принцип + CTA

### Подход В: Активирующее знание + увольнение конкурентов
**Когда:** изменить убеждение. **7-8 карточек.**
Hook (провокация) → Старый способ → Почему не работает → Новый способ → Как выглядит → Результат → Принцип → CTA

### Подход Г: Через Core Jobs / Micro Jobs
**Когда:** объяснить методологию. **7-8 карточек.**
Hook → Контекст → Big Job 1 → Big Job 2 → Big Job 3 → Micro Jobs → Результат → Принцип + CTA

### Подход Д: Через Aha-момент
**Когда:** широкий охват, первое касание. **5-7 карточек.**
Hook (факт) → Контекст → Aha-момент → Было/стало → Результат → "А у вас?" → CTA

---

## ЗНАНИЯ: UTM-система

Полные правила в `4-Personal-files/Vladlena-Orlova/content-system/utm-system.md`.

**Формат:** `{url}?utm_source={source}&utm_medium={medium}&utm_campaign={campaign}`

**Главное правило:** ВСЕ ссылки с UTM. Исключение — deep links в боте (сломаются).

**При создании контента ВСЕГДА:**
1. Спросить "Куда ведёшь? Пришли ссылку"
2. Собрать UTM по правилам
3. Никогда не вставлять ссылку без UTM

---

## ЗНАНИЯ: QR-коды

- Стиль: округлые линии (rounded), не квадратные пиксели
- Содержимое: полная ссылка с UTM (`utm_medium=qr`)
- Размещение: на CTA-карточке, слайде, или в сторис
- Спрашивать "Нужен QR-код?" при создании любого контента
