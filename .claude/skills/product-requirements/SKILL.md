# Генерация продуктовых требований (PRD) v3

Скилл превращает идею продукта в полноценный продуктовый пакет через быстрый процесс:
1. Сбор идеи и настроек (1 раунд вопросов)
2. Сбор бизнес-контекста (2 раунда по 4 вопроса)
3. Автогенерация: AURA-рекомендации + сегменты (80% рынка) + риски
4. Подтверждение пользователем (1 раунд)
5. Запуск команды из 6 агентов с кросс-проверкой

**Принцип скорости:** вопросы задаются ПАЧКАМИ (до 4 за раз), а не по одному. Всего 4-5 обращений к пользователю до запуска агентов.

**На выходе — папка с 5 файлами:**
- `01-prd.md` — продуктовые требования + edge-кейсы
- `02-analytics-plan.md` — аналитическая система с кодом событий
- `03-landing-copy.md` — полный текст лендинга по AURA
- `04-marketing-hypotheses.md` — маркетинговые гипотезы + cross-sell / upsell / retention
- `05-unit-economics.md` — модель юнит-экономики

### Шаг 2.3: Генерация рисков (RAT)

На основе идеи, бизнес-контекста и сгенерированных сегментов — сгенерируй рискованные предположения.

**Категории риска:**
- Спрос на рынке
- Экономически привлекательный сегмент
- Готовность сегмента купить (ценностная гипотеза)
- Юнит-экономика (маржинальность)
- Масштабирование спроса
- Операционные / регуляторные / технологические

**Формула ранжирования:**
```
Score = (P × I) / C
```
- **P** (1-5) — вероятность наступления риска
- **I** (1-5) — стоимость последствий
- **C** (1-5) — стоимость проверки

**Формат карточки риска:**

```markdown
### Риск N: [Название]

**Предположение:** [конкретная проверяемая гипотеза с цифрами/условиями]
**Риск:** [почему это критично для бизнеса]
**Категория:** [одна из 6 категорий]
**P:** N/5 — [обоснование]
**I:** N/5 — [обоснование]
**C:** N/5 — [обоснование]
**Score = P×I/C = [значение]**
**Методы проверки:** [2-5 быстрых экспериментов]
```

Отсортируй по Score по убыванию. Выведи все риски.

## ЭТАП 3: Команда из 6 агентов

### Порядок выполнения

```
Агент 1 (Конкурентный аналитик)
    ↓
Агент 2 (Проектировщик PRD)
    ↓
Агент 3 (Edge-case аналитик)
    ↓
Агент 4 (Аналитический архитектор) ─┐ ПАРАЛЛЕЛЬНО
Агент 5 (Копирайтер и маркетолог)  ─┘
    ↓
Агент 6 (Кросс-верификатор) — ПОСЛЕ ВСЕХ
```

**Важно:** Агенты 4 и 5 запускаются **параллельно** (одновременно) после завершения Агента 3. Агент 6 запускается только после завершения всех пяти предыдущих агентов.

**Правило языка:** В промпт каждого агента добавляй финальную строку: `Весь output пиши на языке: {language из 00-product-idea.md}.`

**Правило пути:** Перед запуском агентов вычисли конкретный путь временной папки: `.claude/temp/prd-{короткое-имя-продукта}/`, где `{короткое-имя-продукта}` берётся из `00-product-idea.md`. Подставь этот конкретный путь во все промпты агентов вместо `{путь к temp}`. Агенты должны получить реальный путь, а не плейсхолдер.
