# Генерация продуктовых требований (PRD) v2

Скилл превращает идею продукта в полноценный продуктовый пакет через последовательный процесс:
1. Сбор идеи, языка, инструмента аналитики
2. Глубокий сбор бизнес-контекста (10-12 вопросов в 3 раундах) + предложения по улучшению бизнес-цели по AURA
3. Генерация гипотез сегментов и core jobs
4. Генерация и ранжирование рискованных предположений
5. Запуск команды из 6 агентов с кросс-проверкой

**На выходе — папка с 5 файлами:**
- `01-prd.md` — продуктовые требования + edge-кейсы
- `02-analytics-plan.md` — аналитическая система с кодом событий
- `03-landing-copy.md` — полный текст лендинга по AURA
- `04-marketing-hypotheses.md` — маркетинговые гипотезы + cross-sell / upsell / retention
- `05-unit-economics.md` — модель юнит-экономики

## ЭТАП 4: Генерация и ранжирование рискованных предположений

Сгенерируй рискованные предположения по методологии Risk Assumption Test.

### Входные данные для генерации
- Идея продукта (ЭТАП 0)
- Полный бизнес-контекст (ЭТАП 1, все 3 раунда)
- Выбранные сегменты и jobs (ЭТАП 3)

### Категории риска
- Спрос на рынке
- Экономически привлекательный сегмент
- Готовность сегмента купить (ценностная гипотеза)
- Юнит-экономика (маржинальность)
- Масштабирование спроса
- Операционные / регуляторные / технологические

### Формула ранжирования

```
Score = (P × I) / C
```

Где:
- **P** (1-5) — вероятность наступления риска (5 = чистая гипотеза, 1 = есть эмпирика)
- **I** (1-5) — стоимость последствий, если риск срабатывает (5 = убьёт бизнес, 1 = локальный сбой)
- **C** (1-5) — стоимость проверки (5 = очень дорого, 1 = проверить почти бесплатно)

Чем выше Score — тем приоритетнее проверка.

### Формат карточки риска

```markdown
### Риск N: [Название]

**Предположение:** [конкретная проверяемая гипотеза с цифрами/условиями]

## Методология AURA для коммуникации

Ключевые принципы:
- Всегда коммуницируй через Big Job (мотивация), не через Core Job (конкретные шаги)
- 3 базовых сообщения: (1) какие работы выполним, (2) какую ценность дадим с КОНКРЕТНЫМИ критериями успеха, (3) снимаем страхи
- Не используй абстракции ("быстро", "качественно", "надёжно") — только конкретику ("за 15 минут", "гарантия 5 лет", "200+ отзывов")
- Контекст и триггер: человек должен узнать себя в тексте
- Язык текста = язык клиента, а не язык фич

Структура коммуникации (цепочка):
Job → Big Job → Ценность → Micro-jobs → Барьеры → Снятие барьеров → Конкуренты → Увольнение конкурентов → Активирующие знания

## ЧАСТЬ 1: Полный текст лендинга

### Блок 1: Hero-секция
- **One-liner** по формуле: [Что это] + [Core Job] + [Ценность]
- **Tagline** (баннерный заголовок) — 3 варианта по формулам:
  1. Core Job для Big Job
  2. Core Job + ценность
  3. Триггер → Core Job
- **Подзаголовок:** 1-2 предложения, раскрывающие ценность
- **CTA-кнопка:** текст кнопки + что получит пользователь

### Блок 2: Core Job + Ценность
- Какую работу выполняет продукт
- За счёт каких фич (не абстрактно, а через micro-jobs)
- Конкретные критерии успеха (цифры, сроки, гарантии)

### Блок 3: Micro Jobs
- Список из 3-6 micro-jobs, которые выполняет продукт
- Для каждой: что делает + какую ценность даёт
- [CTA: выполнить самую легко реализуемую micro-job]

### Блок 4: Aha-момент
- Дай пользователю прожить вкус ценности прямо на лендинге
- Способы: демо, калькулятор, бесплатный мини-результат, интерактивный элемент
- 2-3 конкретных способа для этого продукта

### Блок 5: Ценность по jobs
- Для каждой Core Job / Micro Job:
  - Ценность N: [что получает клиент] + [конкретный критерий успеха]

### Блок 6: "Узнаёшь себя?" (сегменты и триггеры)
- Для каждого целевого сегмента:
  - Ты хочешь [Big Job]?
  - Триггер: [конкретная ситуация]
  - Эмоция в точке А: [что чувствуешь сейчас]
  - Проблема: [что мешает]

### Блок 7: Как это работает
- Для каждой Core Job:
  - Micro Job 1 + ценность при выполнении
  - Micro Job 2 + ценность
  - Micro Job 3 + ценность

### Блок 8: Точка Б (Big Job + эмоции)
- Эмоции в точке Б для каждого Big Job
- Образ точки Б (описательно, для визуализации)
- Конкретные результаты: "Ты сможешь [Big Job 1], [Big Job 2]"

### Блок 9: Снижение барьеров
- 4-6 главных барьеров (страхов) клиента
- Для каждого: барьер → как мы его снимаем (кейсы, гарантии, отзывы, FAQ)

### Блок 10: Увольнение конкурентов
- Для каждого конкурента/альтернативы из 06-competitors.md:
  - Почему текущее решение плохо выполняет job
  - Почему наше решение лучше (конкретные преимущества)

### Блок 11: Финальный CTA
- Призыв к действию с напоминанием Big Job
- Снятие последнего страха
- Активирующие знания: почему новый способ лучше

**Для каждого блока пиши ПОЛНЫЙ готовый текст, не заглушки и не [подставить].**

## ЧАСТЬ 3: Гипотезы стратегии воронки

На основе графа jobs (Previous Job → Core Job → Next Job → Big Job):

### Воронка привлечения
- **Previous Job → Lead Magnet:** Какую предыдущую работу клиента можно выполнить бесплатно, чтобы привлечь его? (чеклист, калькулятор, гайд, шаблон)
- **Активирующие знания → Контент:** Какие знания нужно загрузить клиенту, чтобы он понял ценность нового способа? (статьи, видео, вебинары)
- **Вирусные jobs:** Есть ли в продукте работы, которые выполняются ДЛЯ других или ВМЕСТЕ с другими? (если да — это бесплатный канал)

### Воронка конверсии
- **Aha-момент на лендинге:** Как дать прожить ценность ещё до покупки?
- **7 факторов конверсии** (по AURA): для каждого фактора — как продукт его обеспечивает:
  1. Правильный лид (из целевого сегмента)
  2. Превосходная ценность
  3. Ценность понятна
  4. Нет страхов к работе
  5. Нет страхов к продукту
  6. Низкие затраты на активацию
  7. Технически всё работает

## ЧАСТЬ 4: Cross-sell, Upsell, Retention-механики

### Cross-sell гипотезы (3-5 штук)

| # | Next Job клиента | Что предлагаем | Триггер (когда предлагаем) | Связь с текущим job | Ожидаемая конверсия |

### Upsell гипотезы (2-3 штуки)
На основе Big Job и усложнения Core Job:
| # | Триггер upsell | Что предлагаем | Value metric | Ожидаемый рост чека |

### Retention-механики (3-5 штук)
На основе алгоритмов AURA (привычка, активация ценности, next job, экосистема):
| # | Механика | Реализация в продукте | Ожидаемый эффект на retention |

Обязательные механики:
- Сдвиг Aha-момента влево (к первой сессии)
- Выход на следующую работу (Next Job) для удержания
- Частотность: как увеличить частоту использования
- Переиспользование существующих привычек (не прививать новые)
