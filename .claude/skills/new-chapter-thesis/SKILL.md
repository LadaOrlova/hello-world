# Скилл: Создание тезисов главы (Double Diamond — РАСХОЖДЕНИЕ)

Этот скилл превращает поток мыслей в РАСШИРЕННЫЕ тезисы через **команду агентов (Agent Team)**, которые исследуют тему параллельно, спорят друг с другом и проверяют гипотезы. Приоритет: качество тезисов > количество, точность > полнота покрытия.

## Входные данные

Пользователь даёт:
1. **Название главы** (или тему)
2. **Поток мыслей** — неструктурированные идеи, голосовые заметки, тезисы

ИЛИ:

1. **Ссылку на существующий файл тезисов** (из `2-Writing-Book/2-Theses/`)
2. **Фидбэк** — что доработать, углубить, изменить

---

## ЭТАП 0: Определение режима работы (CREATION vs ITERATION)

**Выполни ДО ВСЕГО остального.** Определи, какой режим нужен:

### Как определить режим

| Сигнал | Режим |
|--------|-------|
| Пользователь дал Raw Thoughts / поток мыслей / новую тему | **CREATION** → иди к ЭТАП 1 |
| Пользователь дал ссылку на существующий файл `Theses-*.md` | **ITERATION** → иди к ЭТАП И1 |
| В файле тезисов заполнено поле `## 🔄 Фидбэк автора для итерации` | **ITERATION** → иди к ЭТАП И1 |
| Пользователь написал "доработай тезисы" / "углуби" / "поправь" | **ITERATION** → иди к ЭТАП И1 |

### Автодетект

1. Если пользователь дал путь к файлу — прочитай его
2. Проверь: есть ли секция `## 🔄 Фидбэк автора для итерации` с заполненным содержимым (НЕ placeholder)?
3. Проверь: есть ли метаданные `Версия:` в шапке?
4. Если да → **ITERATION mode**. Если нет → **CREATION mode**

### Маршрутизация

- **CREATION mode:** Этапы 1 → 2 → 3 → 4 → 5 (полный цикл с Agent Team)
- **ITERATION mode:** Этапы И1 → И2 → И3 → И4 (точечная доработка без полного ресерча)

## ПРОТОКОЛ СОХРАНЕНИЯ КОНТЕКСТА (Anti-Compaction)

⚠️ **КРИТИЧЕСКИ ВАЖНО:** При длинных сессиях Claude Code компактит контекст, теряя детали и снижая качество. Для защиты — **сохраняй промежуточные результаты на диск и перечитывай их перед каждым этапом**.

### Папка временных файлов

В САМОМ НАЧАЛЕ работы создай папку через Bash:

### Файл пружинного контекста (_spring-context.md)

Это ГЛАВНЫЙ файл восстановления. Обновляй его через Write после КАЖДОГО этапа:

```markdown
# Spring Context — Thesis: {название главы}

## Текущий этап: {N} — {название этапа}
## Статус: {в процессе / завершён}

## Тема главы
{1-2 предложения}

## Ключевые решения
- {решение 1}
- {решение 2}

## Сохранённые файлы
- `01-raw-theses.md` — извлечённые тезисы из потока мыслей
- `agent-knowledge-researcher.md` — отчёт агента
- `agent-expert-hunter.md` — отчёт агента
- `agent-contrarian.md` — отчёт агента
- `coverage-round-{N}.md` — последний coverage score
- `04-synthesis.md` — финальный синтез

## Следующий шаг
{что делать дальше}
```

### Расписание сохранения

| После этапа | Файл | Содержимое |
|-------------|------|------------|
| ЭТАП 1 | `01-raw-theses.md` | ВСЕ извлечённые тезисы (полностью!) |
| Отчёт каждого агента | `agent-{имя}.md` | Полный отчёт агента (агенты сохраняют сами) |
| Каждый раунд QC | `coverage-round-{N}.md` | Coverage Score + слабые места + задания тиммейтам |
| ЭТАП 4 | `04-synthesis.md` | Финальный синтез перед записью |

### Правила

1. **После каждого этапа:** Сохрани результат в файл + обнови `_spring-context.md`
2. **Перед каждым этапом:** Перечитай `_spring-context.md` и все указанные в нём файлы
3. **НЕ удаляй** temp-файлы по завершении — они нужны для следующих скиллов (draft, edit)
4. **Агенты-тиммейты** тоже сохраняют свои отчёты в эту папку (см. промпты ниже)

### ОБЯЗАТЕЛЬНЫЕ ССЫЛКИ НА ИСТОЧНИКИ

⚠️ **Железное правило:** Каждый упомянутый источник ОБЯЗАН иметь ссылку.

| Тип источника | Что указывать |
|---------------|---------------|
| **Веб-статья / блог** | Реальный URL (https://...) — найденный через WebSearch/WebFetch |
| **Твит / тред в X** | URL на пост (https://x.com/...) |
| **Книга** | Название, автор, год, ISBN если возможно |
| **Knowledge Base (AURA/AJTBD)** | Полный путь к файлу (`Knowledge/AJTBD/fundamentals.md`) |
| **Подкаст / видео** | URL или название + платформа + дата |

**НЕ ДОПУСКАЕТСЯ:**
- Упоминать эксперта без ссылки на конкретную статью/пост
- Писать "по мнению X" без URL на первоисточник
- Ссылаться на "исследование показало" без конкретного URL или названия
- Придумывать/генерировать URL — только реальные ссылки, найденные через WebSearch

**Агенты-тиммейты** обязаны соблюдать это правило в своих отчётах.

### ЭТАП И1: Разбор фидбэка и классификация задач

1. **Прочитай файл тезисов** целиком
2. **Найди секцию** `## 🔄 Фидбэк автора для итерации`
3. **Классифицируй каждый пункт фидбэка** в один из типов:

| Тип действия | Пример | Сложность |
|-------------|--------|-----------|
| **УДАЛИТЬ** | "Удали тезис 5" | Лёгкая — без агентов |
| **ПЕРЕФОРМУЛИРОВАТЬ** | "Перефразируй тезис 2 проще" | Лёгкая — без агентов |
| **ОБЪЕДИНИТЬ** | "Объедини тезисы 2 и 4" | Лёгкая — без агентов |
| **РАЗДЕЛИТЬ** | "Раздели тезис 3 на два" | Лёгкая — без агентов |
| **ПЕРЕСТАВИТЬ** | "Тезис 6 должен идти первым" | Лёгкая — без агентов |
| **УГЛУБИТЬ** | "Углуби тезис 3 — мало конкретики" | Глубокая — нужен агент |
| **ДОБАВИТЬ** | "Добавь тезис про влияние AI" | Глубокая — нужен агент |
| **ПРОВЕРИТЬ** | "Проверь актуальность тезиса 1" | Глубокая — нужен агент |

4. **Составь план действий** — список конкретных задач с типами

### ЭТАП И2: Определение стратегии (Light vs Targeted-Deep)

На основе классификации из И1 выбери стратегию:

#### Стратегия A: Light (без агентов)

**Когда:** ВСЕ пункты фидбэка — лёгкие (удалить, переформулировать, объединить, разделить, переставить).

Действуй сам: правь файл напрямую. Переходи сразу к ЭТАП И4.

#### Стратегия B: Targeted-Deep (1-2 агента на конкретные задачи)

**Когда:** есть хотя бы один пункт типа УГЛУБИТЬ, ДОБАВИТЬ или ПРОВЕРИТЬ.

Запусти **ТОЛЬКО нужных** агентов (1-2, НЕ всю команду из 3):

| Задача из фидбэка | Какой агент | Что делает |
|-------------------|-------------|------------|
| "Углуби тезис X" | expert-hunter ИЛИ knowledge-researcher | Ищет доп. данные ТОЛЬКО по теме тезиса X |
| "Добавь тезис про Y" | expert-hunter + knowledge-researcher | Ресерч ТОЛЬКО по теме Y |
| "Проверь актуальность X" | expert-hunter | Ищет свежие источники 2025-2026 ТОЛЬКО по теме X |
| "Нужны контраргументы к X" | contrarian | Ищет контрпозиции ТОЛЬКО к тезису X |

⚠️ **КРИТИЧЕСКИ ВАЖНО:** Агент получает КОНКРЕТНУЮ узкую задачу из фидбэка. НЕ "исследуй всю тему заново". Пример:
- ✅ "Найди 3-5 свежих источника 2025-2026 про метрики PMF для AI-продуктов"
- ❌ "Исследуй тему PMF" ← это CREATION mode, не делай так в итерации

## Принцип работы

**РАСХОЖДЕНИЕ по Double Diamond через Agent Team:**
- 3 тиммейта работают ПАРАЛЛЕЛЬНО и ОБЩАЮТСЯ друг с другом
- expert-hunter ищет ТОЧНЫЕ новые тезисы (межотраслевые аналогии, провокации, свежие данные)
- knowledge-researcher УГЛУБЛЯЕТ авторские тезисы данными из KB (НЕ создаёт тезисы из других глав)
- contrarian превращает контраргументы в самостоятельные тезисы (провокации, синтез, футуризм)
- **Качество > количество** — каждый новый тезис проходит тест релевантности
- **Максимум 10-12 новых тезисов**, отфильтрованных по качеству
- Лид = Quality Controller, Coverage Score ≥ 85% = выход
- 1 раунд агентов + 1 раунд доработки (максимум 2 раунда итерации)
- Приоритет СВЕЖЕСТИ: 2025-2026 > 2024 > старые
- НИЧЕГО не удаляем, спорные моменты ПОДСВЕЧИВАЕМ
- **Выходной файл = меню для автора**: авторские + новые, автор выбирает
