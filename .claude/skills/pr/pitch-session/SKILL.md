# Скилл 2: Питч-сессия — тезисы и черновик

Совместная работа с Иваном: из согласованной темы → готовый питч с тезисами. Работает в двух режимах: создание нового питча и итерация по фидбэку.

---

## ЭТАП 0: Определение режима (CREATION vs ITERATION)

**Выполни первым делом.**

| Сигнал | Режим |
|--------|-------|
| Пользователь дал тему из Скилла 1 (новую) | **CREATION** → Этап 1 |
| Пользователь дал путь к существующему файлу питча | **ITERATION** → Этап И1 |
| В файле заполнено поле `## 🔄 Фидбэк для итерации` | **ITERATION** → Этап И1 |
| Пользователь написал «доработай», «углуби», «поправь» | **ITERATION** → Этап И1 |

---

## РЕЖИМ CREATION

### Этап 1: Принять входные данные

Получить от пользователя (из файла Скилла 1 или напрямую):
- СМИ / подкаст
- Хост / журналист
- Согласованная тема
- Угол подачи
- Формат (подкаст / статья / колонка)

Если чего-то нет — спросить у пользователя.

### Этап 2: Запрос контекста у Ивана

Перед генерацией тезисов задать Ивану **1-2 точечных вопроса**. Не больше — сессия быстрая.

Примеры вопросов (выбрать релевантные):
- «Какой кейс лучше всего иллюстрирует эту тему? Хочу поставить его в центр»
- «Есть ли что-то провокационное/контринтуитивное что ты хочешь сказать?»
- «Что ты говоришь обычно по этой теме, что людей удивляет больше всего?»
- «Есть цифры или данные, которые ещё нигде не звучали?»

Если Иван говорит «давай сам» — генерировать на основе базы знаний.

### Этап 3: Генерация тезисов

Создать 5-7 тезисов. Структура каждого:

```
**Тезис [N]:** [Утверждение одной строкой — конкретно, не абстрактно]
**Суть:** [2-4 предложения раскрытия]
**Доказательство:** [кейс / цифра / пример]
**Почему это важно для аудитории [издания]:** [1 предложение]
```

Принципы хороших тезисов:
- Конкретное утверждение, не «JTBD это важно»
- Есть реальный кейс или цифра
- Провокация или контринтуитивный поворот приветствуется
- Язык Ивана — живой, без корпоративщины

### Этап 4: Черновик питча

После тезисов — генерировать черновик письма/сообщения хосту.

```markdown
**Варианты темы письма:**
1. [Прямой]
2. [Интригующий / вопрос]

**Черновик:**

[Открытие — персонализированная ссылка на их недавний выпуск/статью]

[News hook — почему тема актуальна прямо сейчас]

[Что предлагаем — Иван как гость/автор + что даём эксклюзивно: кейс, данные, угол]

[Конкретный CTA — не «было бы интересно», а «готов записать 30-минутный эпизод в [месяц]»]

Best,
Ivan Zamesin
```

Длина: 100-150 слов максимум.

### Этап 5: Сохранение

Сохранить в файл:
```
pitches/pitch-[название-сми]-[дата].md
```

Шаблон файла:

```markdown
# Питч: [СМИ] — [Хост]
**Дата:** [YYYY-MM-DD]
**Версия:** 1
**Статус:** черновик

## Входные данные
- **Тема:** [тема из Скилла 1]
- **Угол:** [угол]
- **Формат:** [подкаст/статья/колонка]

## Тезисы

[тезисы 1-7]

## Черновик питча

**Subject line варианты:**
1.
2.

**Текст:**
[черновик]

## 🔄 Фидбэк для итерации

> *[Заполни это поле — скилл автоматически перейдёт в режим доработки]*
>
> Примеры:
> - «Тезис 3 слабый — нет конкретики, добавь кейс»
> - «Переделай открытие питча — слишком общее»
> - «Добавь тезис про AI-угол»
> - «Тезисы 2 и 4 про одно — объедини»
> - «Питч слишком длинный — сократи до 100 слов»
```

Сказать пользователю: «Готово. Питч сохранён. Хочешь что-то поправить — заполни поле фидбэка и скажи 'доработай'. Или переходим к следующему?»

---

## РЕЖИМ ITERATION

### Этап И1: Разбор фидбэка

1. Прочитать файл питча
2. Найти секцию `## 🔄 Фидбэк для итерации`
3. Классифицировать каждый пункт:

| Тип | Пример | Сложность |
|-----|--------|-----------|
| **УДАЛИТЬ** | «Удали тезис 5» | Лёгкая |
| **ПЕРЕФОРМУЛИРОВАТЬ** | «Тезис 2 — проще» | Лёгкая |
| **ОБЪЕДИНИТЬ** | «Тезисы 2 и 4 — одно» | Лёгкая |
| **СОКРАТИТЬ** | «Питч длиннее 150 слов» | Лёгкая |
| **УГЛУБИТЬ** | «Тезис 3 — мало конкретики» | Нужен доп. контекст от Ивана |
| **ДОБАВИТЬ** | «Добавь тезис про AI» | Нужен доп. контекст от Ивана |
| **ПЕРЕДЕЛАТЬ ПИТЧ** | «Открытие слишком общее» | Нужен доп. контекст |

### Этап И2: Выполнение

**Лёгкие правки** — делать сразу без вопросов.

**Для УГЛУБИТЬ / ДОБАВИТЬ / ПЕРЕДЕЛАТЬ** — задать Ивану точечный вопрос:
- «Для тезиса 3 про сегментацию — есть конкретный кейс с цифрами?»
- «Для нового тезиса про AI — что именно хочешь сказать?»

Не задавать больше 2 вопросов за раз.

### Этап И3: Обновление файла

⚠️ **Обновлять ТОТ ЖЕ файл, не создавать новый.**

1. Применить все правки
2. Обновить метаданные:
   - `**Версия:**` → инкрементировать (+1)
   - `**Дата:**` → текущая
   - `**Статус:**` → обновить если нужно
3. Очистить поле фидбэка до placeholder
4. Сообщить что изменилось: «Обновила версию 2: переформулированы тезисы 2 и 4, сокращён питч до 120 слов. Готово?»

## База знаний: Иван Замесин

### Экспертиза

**Advanced JTBD:** AJTBD = форк классического Jobs To Be Done, фокус на алгоритмах и применимости. Граф работ, 4 силы прогресса, ABCDX-сегментация, количественная валидация.

**AI + product:** «С нейронками будем проверять тысячи гипотез в год». SaaSocalypse — конец эпохи SaaS, начало эпохи агентов. Vibe coding + Claude Code для product managers.

**PMF:** Алгоритм поиска PMF, traction map исследований, как найти и выбрать сегмент до строчки кода.

**Psychology:** Прокрастинация = избегание тяжёлых эмоций. Psychological glass ceilings. Creative sabbatical для builders.

### Кейсы (доказательная база)

- **PLEADA Real Estate** (St. Pete): +30% client meetings, ROMI 800%→1200%, 20M+ RUB revenue — AJTBD segmentation
- **Auto parts market leader**: +5% revenue per B2B client = +90M RUB/month — AJTBD + new IT product; PM became CPO
- **9000+ graduates** of «How to Build a Product»

### Ключевая цитата

«People don't need our products, features, mockups, websites. They need to accomplish what matters to them and feel better.»

### Что НЕ нужно

- Корпоративный/академический тон
- Теория без кейсов
- «Ещё один JTBD евангелист» — у него своя методология, форк

## Чек-лист качества питча

Перед финализацией:

- [ ] Тезисы конкретные, не абстрактные
- [ ] В каждом тезисе есть кейс или цифра
- [ ] Есть провокационный/контринтуитивный угол
- [ ] Питч 100-150 слов
- [ ] Открытие персонализировано под конкретного хоста
- [ ] News hook — почему тема актуальна именно сейчас
- [ ] CTA конкретный (не «было бы интересно»)
- [ ] Язык живой, не корпоративный
- [ ] Файл сохранён с правильной версией и датой
