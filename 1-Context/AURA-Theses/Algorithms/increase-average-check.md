# Алгоритм повышения среднего чека

> Базовые принципы юнит-экономики (экономика внутри сегмента, B2B/B2C смещение, монетизация работ) описаны в [unit-economics.md](../UnitEconomics/unit-economics.md).

## Одно решение — много работ для роста среднего чека
- **Работы внутри одного человека**: Если работы внутри одного человека → это задача роста возвращаемости, роста LTV, роста среднего чека.
- **Следствие**: Из этого принципа вытекает большое количество механик решения бизнес-задач (рост возвращаемости, средний чек, удержание, масштабирование).
- **Пример Яндекс.Такси**: Утром эконом, днём комфорт, на длинной поездке бизнес-класс. Разные работы одного человека → разные чеки.

## Смещение Up-market / Down-market
- **Уйти в Premium**: Меньше клиентов, выше чек, выше требования.
- **Уйти в Mass Market**: Больше клиентов, ниже чек, проще продукт.

## Фокусировка на высокомаржинальных сегментах
- **Кейс Анастасии Гусенцовой (агентство личного бренда)**: Сфокусировалась на сегменте топ-менеджеров вместо экспертов. Результат: рост среднего чека в 3 раза, рост выручки 60% за 3 месяца.
- **Типичный паттерн**: Узнать сегменты → отбросить убыточные → сфокусироваться на самых маржинальных → рост чека и выручки.

## Cross-sell через Super App
- Если мы уже привлекли клиента (заплатили CAC) и удерживаем его, выгоднее продать ему решение *других* его джобов (Cross-sell).
- Пример: Яндекс Go (Такси → Еда → Лавка → Доставка). Экономия на привлечении, рост LTV.
- Кросс-продажи в экосистеме: Продавать другие продукты существующим пользователям дешевле, чем привлекать новых (пример: Amazon Prime).

## Бюджет на джобу (ARPU) как критерий выбора сегмента
- **Бюджет на джобу (ARPU)**: Один из критериев привлекательности сегмента.
- **При выборе фокусного сегмента**: Учитывать бюджет, который люди тратят на выполнение работы. Чем выше бюджет — тем выше потенциальный средний чек.
