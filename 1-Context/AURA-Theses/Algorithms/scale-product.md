# Алгоритм масштабирования продукта

## Масштабирование через сегменты
- **Добавить сегмент**: Нашли A/B, масштабировались, добавили новый сегмент C (который для нас станет новым A).
- **Смещение (Up-market / Down-market)**:
    - Уйти в Premium (меньше клиентов, выше чек, выше требования).
    - Уйти в Mass Market (больше клиентов, ниже чек, проще продукт).

## Географическое масштабирование
- Проверка: существуют ли такие же сегменты с такими же джобами в других странах?
- Культурный код меняет контекст, но часто базовые джобы (безопасность, доход) универсальны.

## Super App стратегия
- Если мы уже привлекли клиента (заплатили CAC) и удерживаем его, выгоднее продать ему решение *других* его джобов (Cross-sell).
- Пример: Яндекс Go (Такси → Еда → Лавка → Доставка). Экономия на привлечении, рост LTV.

## Экосистема
- Связывание продуктов так, что 1+1 > 2.
- Повышение Switching Costs (издержек переключения). Уйти из Apple сложно, потому что часы не работают с Android.

## Стратегии масштабирования (из Workshop 7)
- **Стратегия "Вертикальная интеграция"**: Взять под контроль предыдущие или следующие работы в value chain (пример: Tesla делает батареи сама).
- **Стратегия "Горизонтальная экспансия"**: Добавлять смежные сегменты с похожими работами (пример: Uber → Uber Eats).
- **Стратегия "Географическая экспансия"**: Запуск в новых странах/городах с теми же работами (пример: Airbnb во всех странах).
- **Стратегия "New Core Job"**: Полная смена фокуса на другую работу (пример: Nintendo — игральные карты → игровые консоли).
- **Pivot vs масштабирование**: Pivot — это смена сегмента или Core Job, когда текущий не работает. Масштабирование — это добавление при успешном продукте.

## Стратегии масштабирования и выхода из конкуренции (из Workshop 12)
- **Усложнить уход клиента (Dark Pattern)**: Сделать отписку/отмену сложной. Работает, но сжирает бренд. Пример: интернет-провайдер — подключение бесплатное, а расторгнуть контракт — привези роутер в офис. Отписка от плюса — целый квест.
- **Залочить уникальную ценность**: Эксклюзивный контент, который есть только у тебя (пример: Spotify + Joe Rogan, Disney+ и Marvel, God of War = покупай PlayStation). Чаще всего лочится контент (аудио, видео, игры).
- **Направить внимание клиента**: Вставить свои продукты/рекламу в критическую цепочку клиента. Человек пришёл за одним — останавливаем и говорим «смотри сюда». Пример: лаунчеры игр (запускаешь StarCraft — продают Diablo, Overwatch), Google (гуглишь одно — показывают рекламу другого), новостные сайты. Гиперценность: Google просто сделал корджобу «найти информацию» и на этом строит миллиарды через перенаправление внимания.
- **Выйти на предыдущую работу**: Сделать продукт/контент для работы, которая идёт ДО core job (пример: агрегаторы, гайды, калькуляторы). Пример: агрегатор перевозчиков делает визард «сколько у вас спален» → продаёт лидов перевозчикам. Весь контент-маркетинг построен по этой стратегии. Банки.ру, Сравни.ру — вышли из прямой конкуренции с банками, являются каналами лидогенерации.
- **Выйти на работу выше уровня (главная стратегия)**: Два подвида:
  - **С убийством работ**: Кардинально упростить, убить огромное количество работ ниже уровня (пример: Zoom vs командировки, Яндекс.Такси vs владение машиной). Кардинальное упрощение, ускорение, удешевление.
  - **Без убийства работ**: Просто закрыть все small джобы — они стали нашими core джобами — core джоба поднялась на уровень выше, но убийства не произошло.
- **Смещаться/добавлять сегменты**: Горизонтальная экспансия на соседние сегменты с вариациями работ. Можно добавить сегмент (текущий хорош + хотим ещё), можно сместиться (текущий неприкольный, есть кардинально лучше). Пример: ипотечные продукты Футера — разные люди, разные контексты (самозанятый, ИП, молодая семья, иностранец).
- **Up-market / Down-market**: Смещение по цене. Up-market — состоятельные люди/богатые бизнесы, готовые платить x2-x100 за вариацию корджобы. Down-market — огромный рынок в штуках, но средний чек сильно ниже. Пример: X5 — Чижик (дискаунтер) + Пятёрочка (средний) + Перекрёсток (премиум). Toyota — Daihatsu (дешёвый) + Toyota (средний) + Lexus (премиум).
- **B2B ↔ B2C**: Переход из B2C в B2B или наоборот на тех же технологиях. Кейс GetTaxi: проиграли рынок такси Uber/Яндекс.Такси → на тех же ресурсах сделали B2B (управление флотом, управление поездками сотрудников) → выжили → запустили Get → хорошо себя чувствуют на рынке чёрных кабов в Лондоне. Кейс Vimeo: проиграли видеохостинг YouTube → ушли в B2B.
- **Super App**: Выполнять другие работы клиента, не связанные с исходным Big Job. Сначала сегментировать людей по признаку «он у нас есть и может быть маржинален для других джобов», потом делать суперап. Пример: Яндекс.Go (возили по городу → продают продукты → готовая еда). Все торговые центры: привести тебя в торгач, чтобы и джинсы купил, и наушники, и кофе попил, и с ребёнком в кинотеатр сходил. Все аэропорты — по этой же стратегии.
- **Экосистема**: Связать несколько продуктов для создания уникальной ценности, которую невозможно получить без комбинации продуктов. Пример Apple: наушники + телефон + ноутбук → Zoom-созвон автоматически переключается с телефона на компьютер, наушники переключаются автоматически → убито 3-6 работ. Бизнес-ценность экосистемы: когда человек хочет уйти (купить Samsung Galaxy Buds) — теряет эту ценность. Мощная стратегия удержания.

### Общие джобы в разных графах
- **Принцип**: Можно находить джобу или набор джобов, которые встречаются в графах **разных** сегментов с разными бигджобами.
- **Ключевой критерий**: Вариативность бигджобов — джоба используется для очень разных целей очень разными людьми.
- **Примеры**: Ручка-бумага (записать что-то — в огромном количестве графов), деньги (заплатить — для разнообразного), Telegram (связаться, собрать людей — для огромного количества бигджобов), автомобиль (из А в Б, для развлечения, детей в школу, мебель привезти), поисковик, доставка, ноутбук, дороги.

### Ранжирование стратегий
- **Нет универсального ранжирования по мощности**: Но для каждой бизнес-задачи есть импакт.
- **Оценка по RICE**: Применяете гипотезы над графами и сегментами → для увеличения размера рынка задаёте импакт по 10-балльной шкале (10 = максимальное увеличение) относительно всех остальных гипотез.

## Экзистенциальный ответ: как делать продукт
- **На горизонте лет**: Продукты постоянно убивают работы и поднимаются на всё более высокий уровень работ.
- **Постоянное упрощение графа**: Каждое новое поколение продуктов делает граф работ проще — меньше работ для достижения той же Big Job.
- **Пример**: Доехать из точки А в точку Б. Раньше: купить машину, заправить, найти маршрут, припарковаться. Сейчас: открыть приложение, нажать кнопку.
- **Тренд бандлинга и анбандлинга**: Продукты то разделяются на узкие решения (анбандлинг), то собираются обратно в Super App (бандлинг).
- **Движущая сила**: Мозг стремится более эффективно решать задачи. Продукты, которые это делают — побеждают.

## Масштабирование через одно решение — много работ
- **Работы в разных людях**: Если работы есть в разных людях → это задача масштабирования продукта.
- **Пример Яндекс.Гоу**: Добавили такси, доставку еды, доставку товаров, аренду самокатов — открыли более широкую аудиторию и выросли в существующем сегменте.
- **Примеры других продуктов**: Авито (вещи, недвижимость, услуги), Тинькофф (банк + TP + город), Delivery (доставка из ресторанов и аптек).

## Стратегия базового масштабирования
- **Базовая стратегия**: Не знали сегменты → узнали → выбрали фокусный сегмент → создали ценность → создали воронку → купили клиентов → оптимизировали воронку → добавили новый соседний сегмент.
- **Стратегия добавления сегментов**: Яндекс.Такси начал с эконома → добавил комфорт → добавил ультиму → добавил Maybach. Разные работы, разные люди, разные критерии успеха.
