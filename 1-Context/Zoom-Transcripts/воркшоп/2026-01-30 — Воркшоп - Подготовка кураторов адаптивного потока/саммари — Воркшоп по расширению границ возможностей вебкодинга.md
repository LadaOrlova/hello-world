## Ключевые концепции

### 1. Парадигма веб-кодинга и «перепрошивка» мозга
- **Суть идеи:** Использование ИИ-инструментов для кодинга — это не просто новый навык, а изменение сознания. То, что кажется невозможным сегодня, становится рутиной через 1–3 месяца, когда мозг адаптируется к новым скоростям и возможностям инструментов.
- **Почему это важно:** Это снимает барьер перед «невозможным». Цель — научить не конкретным кнопкам, а методологии экспериментов: пробовать, ломать и чинить с помощью ИИ.
- **Ключевой пример:** Сервис «Аура», созданный одним человеком без навыков программирования.

### 2. Психологическое сопротивление интерфейсам
- **Суть идеи:** Сложные IDE (как Cursor) и терминалы пугают новичков («шаманство», избыток информации), что ведет к потере нити понимания. 
- **Почему это важно:** Важно упрощать среду обучения, чтобы когнитивная нагрузка уходила на созидание, а не на борьбу с интерфейсом.
- **Ключевой пример:** Переход с темной темы на светлую. Светлая тема помогает новичкам лучше видеть структуру (блоки, файлы, агенты) и меньше ассоциировать процесс с «хакерской» сложностью.

### 3. Культура DevOps и FinOps в облачной среде
- **Суть идеи:** Переход в облако (SaaS) — это делегирование ответственности за аптайм и безопасность, но не автоматическое снижение затрат.
- **Почему это важно:** Без подходов FinOps (контроля затрат) облако может быть дороже серверов. При этом DevOps — это в первую очередь культура доверия в команде, а Kubernetes — стандарт гибкости, требующий высокой квалификации.
- **Ключевой пример:** Использование Яндекс Облака (машина `zamesin-sandbox`) для командной работы.

### 4. Экономика токенов: API vs Подписка
- **Суть идеи:** Прямое использование API (например, Claude 3.5 Sonnet) в процессе обучения может стоить тысячи долларов из-за быстрого сгорания токенов в итерациях.
- **Почему это важно:** Для обучения выгоднее использовать фиксированные подписки (Pro-аккаунты), даже если они стоят $100–200.
- **Ключевой пример:** Решение Ивана использовать Claude Pro вместо API, чтобы избежать бесконтрольных трат.

## Фреймворки и инструменты
- **Claude Code**: Терминальный ИИ-агент. Позволяет писать и править код прямо через консоль.
- **Cursor IDE**: Редактор кода на базе VS Code, глубоко интегрированный с ИИ. Использовать со светлой темой и отключенными стандартными чатами для чистоты интерфейса.
- **Whisperflow**: Инструмент для перевода голоса в текст (использовался в практических задачах).
- **FinOps**: Методология управления расходами в облаке для предотвращения переплат.
- **SaaS-модель**: Подход, при котором контроль данных остается у заказчика, а ответственность за инфраструктуру — у провайдера.

## Практические упражнения
- **Настройка рабочего пространства**: Участники отключали стандартные панели «Чат» и «Агенты» в Layout Cursor, чтобы работать через Claude Code.
- **Работа с терминалом**: Отработка навигации в консоли (выбор пунктов меню цифрами на клавиатуре, а не мышкой).
- **Авторизация**: Прохождение пути «Терминал → Браузер → Терминал» для активации Claude Code.
