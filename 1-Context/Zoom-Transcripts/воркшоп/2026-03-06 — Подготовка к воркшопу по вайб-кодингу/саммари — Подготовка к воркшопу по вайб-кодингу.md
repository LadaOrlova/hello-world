## Тема воркшопа
**Подготовка к вайб-кодингу: выбор идеи и настройка окружения**
Установочная встреча перед основным днем практического программирования (вайб-кодинга).

## Структура занятия
Воркшоп был построен в формате организационной и технической подготовки:
1.  **Теоретическая часть и вводные установки:** выбор инструментов (IDE) и критерии выбора продукта.
2.  **Разбор ограничений:** что стоит и чего не стоит делать новичкам в первый день.
3.  **Практическая часть (настройка):** распределение по 7 сессионным залам к кураторам для индивидуальной установки ПО и решения технических проблем.

## Ключевые концепции

### 1. Выбор инструментария (IDE)
- **Суть идеи:** Для работы с Claude Code лучше использовать Visual Studio Code (VS Code), а не Cursor.
- **Почему это важно:** VS Code проще, стабильнее, не требует обязательной регистрации и не навязывает платные модели интерфейса, что критично для быстрой настройки «под себя».
- **Ключевой пример:** Иван отметил, что Cursor постоянно меняется и усложняет процесс интеграции с Claude Code.

### 2. Принцип «Чистого MVP»
- **Суть идеи:** Фокус на одной безупречно работающей функции вместо попытки реализовать сложную систему.
- **Почему это важно:** Это гарантирует успех в ограниченное время (5 часов) и предотвращает «расползание» проекта.
- **Ключевой пример:** «Лучше одна хорошо работающая функция, чем сто недоделанных».

### 3. Фильтрация идей по сложности
- **Суть идеи:** Для первого опыта вайб-кодинга идеально подходят веб-приложения с логикой или личным кабинетом, но не лендинги и не мобильные приложения.
- **Почему это важно:** Лендинг — слишком простая задача (10 минут), а мобильная разработка (iOS/Flutter) требует сложной настройки Xcode и долгого тестирования.
- **Ключевой пример:** Иван предложил делать калькуляторы или каталоги вместо простых одностраничников.

### 4. Минимизация «точек поломки»
- **Суть идеи:** Отказ от внешних API и тяжелых локальных вычислений (аудио/видео) в первый день.
- **Почему это важно:** Внешние интеграции — это непредсказуемые риски, которые могут затормозить работу. Локальная работа с нейросетями требует мощного железа (Apple M-серии).
- **Ключевой пример:** Попытки Гульназ и других участников взять задачи с ИИ-парсингом или обработкой видео были отклонены как слишком рискованные для старта.

### 5. Личная мотивация как драйвер
- **Суть идеи:** Выбирать нужно только ту идею, которая вызывает личный эмоциональный отклик («прёт»).
- **Почему это важно:** Высокая мотивация — ключевой фактор для преодоления технических сложностей при освоении нового навыка.
- **Ключевой пример:** Иван подчеркнул, что без интереса к продукту обучение будет неэффективным.

## Фреймворки и инструменты
- **VS Code**: Основная среда разработки (IDE). Используется как база для написания кода.
- **Claude Code**: Основное расширение/инструмент для генерации кода с помощью ИИ.
- **OpenRouter**: Сервис для подключения различных нейросетей через единый API (использовать с осторожностью из-за риска поломки ключей).
- **Apple M1–M4**: Рекомендуемое оборудование для тех, кто планирует локальную работу с тяжелыми моделями (аудио/видео).
