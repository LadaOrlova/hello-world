## Ключевые концепции

### 1. Приоритет стабильности инструментов (VS Code vs Cursor)
- **Суть идеи:** Для обучения и первого запуска продукта крайне важно использовать максимально стабильный и визуально понятный софт без лишних барьеров.
- **Почему это важно:** Инструменты вроде Cursor навязывают платные модели и требуют лишних регистраций, что отвлекает от сути. VS Code визуально проще и позволяет гибче настраивать Claude Code.
- **Ключевой пример:** Иван Замесин настоял на использовании VS Code, чтобы не тратить время на борьбу с интерфейсом и лимитами платных моделей Cursor.

### 2. Фокус на веб-технологиях (Web vs Mobile)
- **Суть идеи:** Новичкам следует создавать веб-приложения (сайты, сервисы), категорически избегая мобильной разработки (Xcode, Flutter) на старте.
- **Почему это важно:** Мобильная разработка критически усложняет процесс тестирования и деплоя. Веб-приложение проще открыть по ссылке, оно дает больше учебного опыта, чем простой лендинг.
- **Ключевой пример:** Категорический отказ студенту, который хотел делать приложение на Flutter, из-за высокого риска «сломаться» на настройке среды.

### 3. Принцип радикального упрощения (MVP)
- **Суть идеи:** «Чем проще — тем лучше». Лучше одна идеально работающая функция, чем сложный, но недоделанный проект.
- **Почему это важно:** Внешние API, парсеры и сложные интеграции — это главные точки поломки для новичков в первый день.
- **Ключевой пример:** Иван рекомендовал убирать внешние API из идей студентов и превращать простые лендинги в микро-сервисы с личным кабинетом или квизом для наработки опыта.

### 4. Мотивация как двигатель обучения
- **Суть идеи:** Выбирать нужно ту идею, которая «прет» автора эмоционально.
- **Почему это важно:** Высокая личная заинтересованность помогает легче преодолевать технические трудности и лучше усваивать навык работы с кодом через нейронки.

### 5. Технические ограничения инфраструктуры
- **Суть идеи:** Локальная работа с ИИ-моделями требует специфического железа (Mac с процессорами M-серии).
- **Почему это важно:** На других конфигурациях работа будет «больной и тяжелой» из-за нехватки мощностей. Также важно учитывать лимиты токенов в подписках (даже за $100-200 их может не хватить на сложный продукт).

## Фреймворки и инструменты
- **Visual Studio Code (VS Code)**: Основная среда разработки. Используется из-за простоты, стабильности и отсутствия принудительных платных подписок.
- **Claude Code**: Основной ИИ-инструмент для написания кода, который интегрируется в рабочую среду.
- **OpenRouter**: Сервис-агрегатор для подключения различных нейросетей через API (рекомендовано использовать с осторожностью на старте).
- **Готовые сервера для деплоя**: Инфраструктура, которую организаторы предоставляют студентам для быстрой публикации их проектов в сети.

## Практические упражнения
1. **Питчинг идей**: Студенты писали свои идеи в чат и получали фидбек от Ивана по упрощению или усложнению концепции.
2. **Работа в сессионных залах**: Распределение участников по 7 комнатам (6-7 кураторов на 40 человек) для индивидуальной помощи в настройке софта.
3. **Технический сетап**: Установка VS Code, необходимых расширений и проверка актуальных гайдов.
