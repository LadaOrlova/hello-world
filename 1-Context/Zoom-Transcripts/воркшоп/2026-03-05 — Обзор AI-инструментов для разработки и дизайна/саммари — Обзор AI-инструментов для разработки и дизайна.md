## Тема воркшопа
**Стратегия развития образовательных продуктов («Буст», «Как делать продукт») и переход на новый технологический стек (Claude Code + VS Code)**

## Структура занятия
Встреча прошла в формате стратегического синхрона команды:
1. **Технологический блок:** Смена инструментов (отказ от Cursor в пользу VS Code) и вопросы безопасности (шифрование).
2. **Методологический блок:** Изменение формата обучения и воркшопов.
3. **Операционный блок:** Статус продаж, маркетинговые задачи и подготовка кейсов.
4. **Разработка:** Обсуждение архитектуры онбординг-бота и интеграции с платформой Zis.

## Ключевые концепции

### 1. Переход от Cursor к связке VS Code + Claude Code
- **Суть идеи:** Отказ от перегруженной среды Cursor в пользу более стабильного VS Code с установленным ИИ-агентом Claude Code.
- **Почему это важно:** Cursor стал слишком нестабильным и перегруженным лишними функциями, в то время как VS Code предлагает «чистый» интерфейс.
- **Ключевой пример:** Илья Изма отметил, что интерфейс VS Code не меняется так часто, что критично для стабильной работы и обучения студентов.

### 2. Агентские пайплайны вместо жесткого кода
- **Суть идеи:** Перенос сложной логики (исследование продукта, анализ интервью) из жестко прописанного программного кода в гибкие промпты для ИИ-агентов.
- **Почему это важно:** Жесткие пайплайны негибкие; работа через агентов позволяет быстрее адаптировать логику под новые задачи.
- **Ключевой пример:** Илья Изма переносит функционал Product Research и Interview Analysis на «чистые промпты».

### 3. Образовательная среда в IDE (VS Code)
- **Суть идеи:** Обучение студентов происходит не на сторонних платформах, а прямо в среде разработки.
- **Почему это важно:** Это создает единый контекст: слева — файлы проекта, в центре — учебные материалы, справа — ИИ-помощник.
- **Ключевой пример:** Иван Замесин подготовил формат, где студент видит слайды/задания в браузере внутри VS Code и тут же работает с Claude Code.

### 4. Рост ценности фундаментальных знаний при автоматизации
- **Суть идеи:** ИИ берет на себя рутину (код, UX/UI), что делает навыки стратегии, понимания сегментов и Job-to-be-Done (JTBD) главным конкурентным преимуществом.
- **Почему это важно:** Если ИИ может «нарисовать» интерфейс или написать код, ценность специалиста смещается в сторону принятия решений и целеполагания.
- **Ключевой пример:** Появление «Vibe UX/UI дизайнеров», которые управляют видением, а не двигают пиксели вручную.

### 5. Безопасность данных: End-to-End шифрование
- **Суть идеи:** Необходимость внедрения полного шифрования, при котором даже у разработчиков нет ключей доступа к данным пользователей.
- **Почему это важно:** Обычного шифрования на бэкенде недостаточно для защиты от внутренних рисков компании; Иван настоял на E2EE для обеспечения полной приватности.
- **Ключевой пример:** Дискуссия между Ильей и Иваном о том, что наличие ключей у компании — это фундаментальная уязвимость.

## Фреймворки и инструменты
- **Claude Code**: Терминальный ИИ-агент. Используется как основной инструмент для написания кода и автоматизации задач прямо в IDE. Стоимость подписки — $100.
- **VS Code**: Основная среда разработки и обучения, заменяющая Cursor.
- **Aura (Агенты и сабагенты)**: Система разделения задач. Агенты работают в фоне, сабагенты — блокируют окно для выполнения конкретных операций.
- **Google Stitch**: Инструмент генерируемого ИИ дизайна, предвещающий трансформацию профессии UX-дизайнера.
