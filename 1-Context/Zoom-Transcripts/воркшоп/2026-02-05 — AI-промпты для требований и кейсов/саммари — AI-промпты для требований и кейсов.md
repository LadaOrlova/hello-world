# [Воркшоп] AI-промпты для требований и кейсов
**Дата**: 5 февраля 2026

---

## Тема воркшопа
Воркшоп номер четыре: "AI-промпты для требований и кейсов". Его провел Илья Измайлов. Тема была посвящена автоматизации базовых действий, а также повышению скорости и эффективности работы с помощью AI-инструментов CloD, MCP, скиллов, агентов и команд.

## Структура занятия
Занятие началось с короткого "чек-ина", где участники делились своим опытом и возникшими сложностями с использованием AI-инструментов. Затем следовала теоретическая часть в формате презентации, где Илья Измайлов представил и объяснил ключевые концепции (CloD MD, скиллы, команды, сабагенты, MCP). Далее воркшоп перешел к практической части, где участники под руководством ведущего настраивали необходимое программное обеспечение и рабочее окружение, выполняя пошаговые инструкции и решая возникающие трудности.

## Ключевые концепции

### 1. CloD MD
-   **Суть идеи:** Это специальный текстовый файл, который CloD автоматически считывает при каждом запросе, чтобы получить особые инструкции и контекст для текущего проекта или задачи. Он определяет правила и ограничения для поведения AI.
-   **Почему это важно / как применять:** CloD MD позволяет глубоко настраивать работу AI на разных уровнях, обеспечивая релевантность и согласованность ответов и действий. Существуют три уровня:
    *   **Глобальный:** Правила, расположенные в корневой папке CloD, переиспользуются между всеми проектами (например, "пиши на русском, избегай технических терминов").
    *   **Проектный:** Файл `CloD MD` в папке репозитория, задает общие правила работы для конкретного проекта или команды.
    *   **Локальный:** Файл `CloD MD local` в проекте, содержит индивидуальные настройки и правила для конкретного пользователя.
-   **Ключевой пример из воркшопа:** На глобальном уровне можно задать стиль общения AI (например, "пиши на русском"), а на проектном — требования к формату вывода или используемой терминологии.

### 2. Скиллы (Skills)
-   **Суть идеи:** Специальные сущности в CloD, которые содержат в себе набор подробных промптов и инструкций, описывающих, как AI должен выполнять определенную сложную задачу или действовать в конкретной ситуации.
-   **Почему это важно / как применять:** Скиллы позволяют AI "знать", как решать специализированные задачи. Они автоматически подгружаются в контекст, когда пользователь инициирует соответствующее действие, значительно повышая качество и эффективность работы AI за счет использования предварительно заданной "экспертизы".
-   **Ключевой пример из воркшопа:** Скилл может содержать все необходимые шаги и знания о том, "как создать красивый дизайн" или "как написать полноценный PRD" (Product Requirements Document).

### 3. Команды (Commands)
-   **Суть идеи:** Особые промпты, которые содержат в себе четкое, однозадачное действие, которое должен выполнить AI.
-   **Почему это важно / как применять:** Команды служат для быстрого инициирования стандартизированных автоматизированных операций AI. Они обычно вызываются через префикс слеша (`/`) непосредственно в интерфейсе CloD, что делает взаимодействие с AI более интуитивным и быстрым для рутинных задач.
-   **Ключевой пример из воркшопа:** Типичные команды включают `/проведи ревью кода` или `/сделай фичу`, запуская соответствующие процессы.

### 4. Сабагенты (Subagents)
-   **Суть идеи:** Независимые сущности внутри CloD, которые могут работать параллельно над разными аспектами одной большой задачи. Каждый сабагент имеет собственное, изолированное контекстное окно.
-   **Почему это важно / как применять:** За счет своего отдельного контекста, сабагенты могут очень классно фокусироваться на узкой задаче, не отвлекаясь на лишнюю информацию. Это позволяет распараллеливать сложные процессы, выполнять работу быстрее и с более высоким качеством, так как каждый агент работает с максимальной концентрацией на своей части.
-   **Ключевой пример из воркшопа:** Запуск множества сабагентов для комплексного ревью кода: один ищет "мертвый" (неиспользуемый) код, другой предлагает улучшения архитектуры, третий проверяет безопасность, и так далее.

### 5. MCP (Meta-Code Platform)
-   **Суть идеи:** Инструмент (сервер), который позволяет AI-моделям (в данном случае CloD) подключаться к внешним сервисам и приложениям для взаимодействия с ними.
-   **Почему это важно / как применять:** MCP значительно расширяет возможности AI, позволяя ему не только генерировать текст или код, но и выполнять реальные действия в других системах, получать данные из них. Это ключевой элемент для построения сквозной автоматизации и интеграции AI в существующие рабочие процессы.
-   **Ключевой пример из воркшопа:** Через MCP AI может взаимодействовать с Telegram, Google Sheets, Figma и другими практически любыми внешними приложениями.

## Фреймворки и инструменты
-   **CloD:** Основной AI-инструмент, вокруг которого построен воркшоп. Используется для взаимодействия с AI, управления промптами, скиллами, командами и сабагентами. Применяется для автоматизации задач и повышения эффективности.
-   **Cursor:** Интегрированная среда разработки (IDE), используемая участниками для работы с файлами проекта. Однако, для работы с несколькими агентами и вкладками, а также для интенсивного использования, рекомендуется использовать внешний терминал из-за возможных проблем Cursor с оперативной памятью.
-   **Warp:** Высокопроизводительный терминал (для MacOS/Linux), рекомендованный для работы с несколькими AI-агентами и большим количеством вкладок. Он обеспечивает большую стабильность и эффективность по сравнению со встроенным терминалом в Cursor, особенно при высоких нагрузках. Используется для навигации по файловой системе и выполнения команд.
-   **PowerShell:** Стандартный терминал для операционной системы Windows, используемый в качестве альтернативы Warp для пользователей Windows. Выполняет те же функции по запуску команд и навигации по файловой системе.
-   **Chrome Extension (CloD Chrome Extension):** Расширение для браузера Chrome, которое интегрирует функции CloD непосредственно в веб-браузер. Позволяет AI взаимодействовать с содержимым веб-страниц. Для установки требуется использование VPN, а активация внутри CloD производится командой `/Chrome`.
-   **GitLab:** Платформа для управления жизненным циклом DevOps. Участник воркшопа Максим привел пример своей интеграции с GitLab, где AI автоматизировал создание задач, назначение ответственных, добавление комментариев и меток в бэклоге, полностью автоматизировав процесс постановки задач для команды.

## Практические упражнения
1.  **Установка и настройка CloD Chrome Extension:** Участникам было предложено установить расширение для браузера Chrome (с использованием VPN) и активировать его в CloD с помощью команды `/Chrome`.
2.  **Скачивание проектных материалов:** Загрузка специальной папки "четыре" (Wokshop 4) из общего канала, содержащей все необходимые файлы и структуры для практической работы.
3.  **Настройка рабочего пространства в терминале:**
    *   Открытие скачанной папки в IDE Cursor.
    *   Запуск внешнего терминала (Warp или PowerShell).
    *   Переход в конкретную подпапку проекта (`Contract Creator`) внутри терминала, используя команду `CD [путь_к_папке]`.
    *   Решение проблем с путями, содержащими пробелы (использование кавычек или переименование папки).
