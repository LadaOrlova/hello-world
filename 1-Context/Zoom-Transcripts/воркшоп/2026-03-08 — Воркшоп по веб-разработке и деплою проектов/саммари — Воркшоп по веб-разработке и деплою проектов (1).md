## Ключевые концепции

### 1. Вебкодинг (Webcoding)
- **Суть идеи:** Подход к разработке, при котором 80–100% кода пишется нейросетью и практически не требует ручного ревью человеком.
- **Почему это важно:** Термин, популяризированный Андреем Карпатым, знаменует эру, где «программирование как задача во многом решена» (coding is largely solved).
- **Ключевой пример:** Спикер собрал концепт нового образовательного подхода всего за 15–20 минут во время завтрака, используя этот метод.

### 2. Контекст-инжиниринг vs Промпт-инжиниринг
- **Суть идеи:** Смещение фокуса с «вылизывания» текста запроса на предоставление нейросети максимального объема релевантных данных и файлов.
- **Почему это важно:** Для современных моделей (Opus/Sonnet 4.6) с контекстным окном в 200к токенов наличие правильных исходных файлов важнее, чем формулировка инструкции.
- **Ключевой пример:** Вместо долгого уточнения промпта, эффективнее загрузить в Claude Code все связанные файлы проекта.

### 3. Психология и физиология вебкодинга
- **Суть идеи:** Основные барьеры в освоении ИИ-разработки — психологические, а не технические; при этом успех вызывает мощный дофаминовый отклик.
- **Почему это важно:** Резкий скачок энергоэффективности может вызвать «эффект штырки» (зависимость и бессонницу на 1-2 месяца), к чему участники должны быть готовы.
- **Ключевой пример:** Сопротивление переходу на 100% автогенерацию кода сравнивается с отсутствием привычки, а не с реальной сложностью задачи.

### 4. Автономность Claude Code
- **Суть идеи:** Claude Code позиционируется как «руки и глаза» модели, обладающие новым уровнем автономности по сравнению с традиционными IDE.
- **Почему это важно:** В отличие от Cursor, который является надстройкой-посредником, Claude Code предоставляет прямой доступ к возможностям агента в терминале.
- **Ключевой пример:** Настройка режима `Edit automatically`, позволяющая агенту править файлы без ежесекундного подтверждения пользователем.

## Фреймворки и инструменты
- **Claude Code**: Основной автономный агент для написания и редактирования кода. Используется через терминал для прямого взаимодействия с файловой системой.
- **Opus 4.6**: Самая мощная и дорогая модель. Применяется для архитектурного планирования, написания PRD и решения сложных багов.
- **Sonnet 4.6**: Сбалансированная модель. Применяется для базовых тестов и оперативного редактирования кода.
- **Whisperflow**: Инструмент для голосового ввода. Позволяет общаться с кодом голосом, что значительно ускоряет процесс взаимодействия.
- **Risk Assumption Test (RAT)**: Методология проверки критических гипотез продукта перед полномасштабной разработкой.

1.  **Настройка окружения:** Участники правили `settings.json` для предоставления Claude Code прав на автоматическое выполнение команд и чтение файлов.

4.  **Маркетинг и верстка:** Генерация текста лендинга (`my_landing_page_text.md`) и последующая конвертация его в HTML-файл силами Claude Code.
5.  **Работа с сервером:** Получение VPS через Telegram-бота и подготовка к деплою.
