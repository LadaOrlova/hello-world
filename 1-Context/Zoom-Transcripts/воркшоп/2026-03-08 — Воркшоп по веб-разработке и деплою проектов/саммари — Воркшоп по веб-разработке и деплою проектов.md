## Ключевые концепции

### 1. Парадигма «Coding is largely solved»
- **Суть идеи:** Процесс написания кода перестал быть задачей для человека; современные агенты (уровня Opus 4.5+) способны автономно писать до 100% кода, который почти не требует ревью.
- **Почему это важно:** Это освобождает время разработчика/продукта для архитектуры и смыслов, превращая программирование в «вебкодинг».
- **Ключевой пример:** Спикер отмечает, что любая идея теперь реализуется за 10–20 минут.

### 2. Агентные системы против обычных LLM
- **Суть идеи:** Claude Code — это не просто чат, а агент с высокой автономностью, способный проактивно решать многошаговые задачи в файловой системе.
- **Почему это важно:** Инструмент сам выполняет действия, проверяет ошибки и доводит задачу до конца, в отличие от простых подсказчиков кода.
- **Ключевой пример:** Claude Code «рвет всех как тузик грелку», делая ненужными связки вроде Lovable + n8n.

### 3. Контекст-инжиниринг важнее промпт-инжиниринга
- **Суть идеи:** Качество результата больше зависит от объема и релевантности предоставленных файлов и данных (контекста), чем от «полировки» текста запроса.
- **Почему это важно:** Эффективнее загрузить в модель спецификации и примеры, чем пытаться составить «идеальный» промпт.
- **Ключевой пример:** Спор с участником Владимиром о необходимости «причесывания» промптов.

### 4. Дофаминовая петля вебкодинга
- **Суть идеи:** Использование ИИ для сверхбыстрого решения задач распознается мозгом как крайне энергоэффективный метод, что вызывает мощный выброс дофамина.
- **Почему это важно:** Это создает психологическую зависимость от нового метода работы и помогает преодолеть «гору в голове» (сопротивление обучению).
- **Ключевой пример:** Состояние, когда пользователя «штырит» первые 1-2 месяца от скорости реализации идей.

### 5. Смерть бизнес-модели агрегаторов (Cursor)
- **Суть идеи:** Инструменты-посредники (Cursor) проигрывают прямым интерфейсам от создателей моделей (Anthropic/OpenAI), так как вынуждены перепродавать токены дороже.
- **Почему это важно:** Для профессиональной работы выгоднее использовать нативные инструменты разработчиков моделей (Claude Code).
- **Ключевой пример:** Спикер сжег $200 в Cursor за сутки, что вынудило перейти на Claude Code Max.

## Фреймворки и инструменты
- **Claude Code**: Автономный агент для терминала. Использовать в связке с VS Code. Основной режим — `Edit automatically`.
- **Opus 4.6**: Топовая модель для сложных архитектурных задач, написания PRD и минимизации галлюцинаций.
- **Sonnet 4.6**: Модель для быстрых правок, редактирования и простых задач (экономит токены).
- **RAT (Riskiest Assumption Test)**: Методология тестирования самой рискованной гипотезы продукта перед началом разработки.
- **Whisperflow**: Инструмент для голосового ввода, используемый для оперативной доработки PRD и текстов лендингов.

## Практические упражнения
1.  **Настройка среды:** Установка расширений, настройка `settings.json` и активация режима автоматического редактирования.
2.  **Работа с моделями:** Переключение на Sonnet 4.6 для базовой проверки файлов проекта.
3.  **Проектирование:** Создание файла `about_rat.md` и проведение интервью с моделью Opus 4.6 для генерации PRD (Product Requirements Document).
4.  **Голосовое управление:** Использование Whisperflow для внесения правок в PRD «голосом».
5.  **Создание контента:** Генерация текста лендинга (`my_landing_page_text.md`) и последующее преобразование его в HTML-код.
