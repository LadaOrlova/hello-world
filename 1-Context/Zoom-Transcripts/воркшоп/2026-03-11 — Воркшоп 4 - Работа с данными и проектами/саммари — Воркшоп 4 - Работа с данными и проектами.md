## Ключевые концепции

### 1. Контекст-менеджмент важнее промпт-менеджмента
- **Суть идеи:** Эффективность ИИ теперь зависит не от качества конкретной фразы (промпта), а от качества подготовки репозитория контекста — набора файлов, расшифровок и документации, к которым у модели есть доступ.
- **Почему это важно:** Модель выдает результат на основе всей доступной базы знаний; правильная организация папок и файлов заменяет длинные инструкции.
- **Пример:** Вместо того чтобы каждый раз объяснять правила, менеджер формирует среду данных, где Claude Code сам находит нужные файлы.

### 2. Детерминированность через код (Python-скрипты)
- **Суть идеи:** Нейросети плохо справляются с прямыми вычислениями («галлюцинируют» в цифрах), поэтому для точных данных ИИ должен не считать сам, а писать и запускать Python-скрипты.
- **Почему это важно:** Это гарантирует, что цифры будут получены по чёткому алгоритму, а не предсказаны как следующее слово в предложении.
- **Пример:** Разница в цифрах конверсии у участников (80/62/87 против 80/51/87) была устранена только после того, как все перешли на расчеты через запуск скриптов.

### 3. Новая роль менеджера: "Good-enough" результат в промышленном масштабе
- **Суть идеи:** Менеджеры переходят от создания одного «идеального» документа вручную к генерации 10–50 «достаточно хороших» результатов за то же время.
- **Почему это важно:** Скорость и объем выдаваемого продукта становятся важнее перфекционизма, что радикально меняет экономику труда.
- **Пример:** Подготовка к вебинару заняла 1 минуту на раздумья, 5 минут на промпт и 5 минут на ожидание результата.

### 4. ABCD-сегментация как основа стратегии
- **Суть идеи:** Фокусировка продукта на сегментах AB (высокая маржа и удовлетворенность) и отказ от работы с сегментами CD.
- **Почему это важно:** Позволяет продукту расти за счет наиболее ценных пользователей, используя ИИ для быстрого выявления этих групп в данных.

### 5. Безопасность и анонимизация данных
- **Суть идеи:** Использование ИИ в корпоративной среде требует анонимизации данных или запуска скриптов локально.
- **Почему это важно:** Риск утечки чувствительных данных в Anthropic (Claude) остается открытым вопросом, который решается либо покупкой API, либо техническими методами локальной обработки.

## Фреймворки и инструменты
- **Visual Studio Code + расширение Claude Code**: Основной технический стек для работы с локальными файлами данных и управления AI-агентом.
- **Python**: Используется как инструмент исполнения для обеспечения точности расчетов (расчет воронок, когорт).
- **Sonnet (модель)**: Рекомендуется для написания простых скриптов.
- **Opus (модель)**: Рекомендуется для создания сложных визуализаций, HTML-отчетов и финальных аналитических документов.
- **HTML/PDF Reporting**: Формат вывода результатов анализа (отчеты могут содержать до 1000 строк кода).
