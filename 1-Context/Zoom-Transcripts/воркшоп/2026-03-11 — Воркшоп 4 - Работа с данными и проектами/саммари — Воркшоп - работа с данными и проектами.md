## Ключевые концепции

### 1. Переход от промпт-менеджмента к контекст-менеджменту
- **Суть идеи**: Эффективность работы с ИИ теперь зависит не столько от текста одного запроса, сколько от умения управлять состоянием и историей проекта.
- **Почему это важно**: Чтобы контекст не терялся при закрытии редактора, его нужно фиксировать в файлах. Рекомендуется использовать файл `index.md` в корне проекта для навигации нейросети.
- **Ключевой пример**: Сохранение истории взаимодействия в текстовые файлы для VS Code, чтобы модель всегда «помнила» архитектуру.

### 2. Детерминизм против галлюцинаций в расчетах
- **Суть идеи**: Нейронным сетям нельзя доверять математические вычисления напрямую, так как они работают недетерминировано. Для точности нужно заставлять ИИ писать код (скрипты).
- **Почему это важно**: Скрипт — это стабильный процесс, который всегда дает одинаковый результат, в то время как генеративная модель может «бабахнуть» в случайном месте.
- **Ключевой пример**: При расчете воронки участники получили разные цифры (80%, 62%, 87%), используя прямой запрос. Решением стало написание Python-скрипта, зафиксировавшего точные значения (80.4%, 51.3%, 87.4%).

### 3. Демократизация разработки («Web-coding»)
- **Суть идеи**: Роль разработчика меняется — теперь он должен проектировать систему так, чтобы любой член команды мог «кодить» через интерфейс ИИ (писать тесты, CI/CD пайплайны, логику).
- **Почему это важно**: Пользователь скоро перестанет видеть код и будет просто описывать финальный результат, принимая промежуточные итоги.
- **Ключевой пример**: Использование Claude Code для выполнения действий напрямую в файловой системе и запуска скриптов без ручного написания кода человеком.

### 4. Приоритет скорости и объема (Good-enough результат)
- **Суть идеи**: В современной экономике эффективнее выдать 10–50 «достаточно хороших» результатов быстро, чем тратить время на создание одной идеальной единицы контента/кода.
- **Почему это важно**: ИИ позволяет кратно масштабировать производительность (например, разобрать 16 продуктов за 10 минут).
- **Ключевой пример**: Спикер подготовил разбор 16 продуктов по формуле «1 минута на раздумья, 5 минут на промпт, 5 минут ожидания».

## Фреймворки и инструменты
- **Claude Code**: Инструмент для работы с файловой системой. Позволяет ИИ самостоятельно писать, редактировать и запускать скрипты.
- **VS Code + папка `Working with Data`**: Базовая среда для организации контекста проекта.
- **Модели Claude 4.6 Sonnet и 3.5 Opus**: Sonnet используется для быстрой разведки, Opus — для критического кода и сложной визуализации.
- **Режим `Allow dangerously skip permissions`**: Настройка в Claude Code для ускорения работы, позволяющая ИИ действовать без постоянных подтверждений (требует осторожности).
- **Платформа «Аура»**: Дополнительный инструмент, доступ к которому был предоставлен всем участникам.
