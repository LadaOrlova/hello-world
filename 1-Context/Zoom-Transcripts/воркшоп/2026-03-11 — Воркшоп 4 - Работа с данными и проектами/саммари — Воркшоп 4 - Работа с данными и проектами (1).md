## Тема воркшопа
**Использование ИИ и веб-кодинга для продуктовой аналитики и управления данными.**
Воркшоп посвящен переходу от классического промпт-инжиниринга к управлению контекстом и созданию детерминированных инструментов (скриптов) для анализа метрик.

Воркшоп построен как интенсивная практическая сессия:
1.  **Теоретический блок:** Концепция управления контекстом, особенности работы LLM с цифрами и методология сегментации.

5.  **Самостоятельная работа:** Исследование конкурентов.

## Ключевые концепции

### 1. Context-management вместо Prompt-management
- Суть идеи: Эффективность работы с ИИ теперь зависит не от качества разового промпта, а от умения управлять контекстом — работать в одном чате и предоставлять ИИ структуру проекта.
- Почему это важно: Это позволяет нейросети осознавать предысторию продукта и быстрее находить нужную информацию.
- Ключевой пример: Использование файла `index.md` в корне репозитория как «карты» для ИИ.

### 2. Детерминированные расчеты через Python
- Суть идеи: LLM являются генеративными моделями (предсказывают токены) и часто ошибаются в математических вычислениях. Для точности нужно заставлять ИИ писать Python-скрипт, который выполнит расчеты.
- Почему это важно: Скрипт — это приложение, которое всегда выдает одинаковый результат, исключая «плавающие» цифры нейронки.
- Ключевой пример: Разница в данных у участников (62% против корректных 51,4%) возникла из-за попытки считать «в лоб» без скрипта.

### 3. ABCD-сегментация по Илье Красинскому
- Суть идеи: Разделение пользователей на сегменты, где A и B — маржинальные и удовлетворенные клиенты, а C и D — проблемные.
- Почему это важно: Фокус на сегментах A и B является ключом к росту продукта; от сегментов C и D нужно отказываться.
- Ключевой пример: Анализ воронки подписок помог выявить проблему — отсутствие grace period, из-за чего подписка отменяется сразу после первой неудачной попытки списания.

### 4. Трансформация роли разработчика
- Суть идеи: Профессия смещается от написания кода к проектированию систем, где «веб-кодингом» могут безопасно заниматься даже не-разработчики.
- Почему это важно: Использование ИИ позволяет менеджеру выдавать в 10–50 раз больше «достаточно хорошего» (good-enough) результата.
- Ключевой пример: Генерация HTML-отчета объемом почти в 1000 строк кода силами менеджера.

## Фреймворки и инструменты
- **Claude Code (Claude 3.5 Sonnet/Opus)**: Основной ИИ-инструмент для кодинга в терминале/VS Code. Использовался для генерации аналитических скриптов.
- **Aura Framework**: Платформа для работы (участникам предоставлено 1500 кредитов на 2 месяца).
- **Visual Studio Code (VS Code)**: Редактор кода, основная рабочая среда.
- **Rainbow CSV**: Расширение для VS Code, упрощающее чтение табличных данных.
- **Режим «Allow dangerously skip permissions»**: Настройка Claude Code, позволяющая работать без постоянных подтверждений (ускоряет работу, но несет риск удаления файлов).
