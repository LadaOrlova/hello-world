## Тема воркшопа
**Вайбкодинг: Создание и деплой MVP для новичков**
Практическое занятие по быстрой разработке минимально жизнеспособного продукта с помощью инструментов ИИ (Claude Code, Cursor).

## Структура занятия
Воркшоп был построен по принципу «от философии к практике»:
1.  **Вводная часть:** Психология вайбкодинга и преодоление сопротивления.
2.  **Теория и инструменты:** Разбор отличий Claude Code от других ИИ, работа с контекстом и качеством кода.
3.  **Настройка среды:** Подготовка VS Code и Claude Code.
4.  **Продуктовая часть:** Создание PRD через интервью с ИИ и методологию JTBD.
5.  **Разработка:** Генерация лендинга и написание кода MVP через плагин FeatureDev.
6.  **Деплой:** Развертывание проекта на сервер через Telegram-бота.
7.  **Приглашенный спикер:** Кейс применения ИИ в биоинформатике.

## Ключевые концепции

### 1. Психология «Вайбкодинга» и интеграция
- **Суть идеи:** Современные инструменты позволяют реализовывать идеи за 10–20 минут, превращая «гору» психологического сопротивления в легкую задачу.
- **Почему это важно:** Традиционное программирование часто вызывает стресс; новый подход требует не только навыков, но и физической настройки организма (сон, спорт, прогулки) для переваривания высокой эффективности.
- **Ключевой пример:** Спикер подчеркивает, что после интенсивной сессии мозгу нужна «интеграция», так как нервная система не привыкла к такой скорости реализации идей.

### 2. Автономность ИИ через «обвязку»
- **Суть идеи:** Прорыв в возможностях ИИ произошел не только из-за самих моделей, но и благодаря инструментам, дающим ИИ доступ к терминалу, файловой системе и установке программ.
- **Почему это важно:** Это превращает ИИ из чат-бота в автономного агента, способного самостоятельно выполнять циклы разработки.
- **Ключевой пример:** Claude Code фундаментально отличается от ChatGPT наличием прямого доступа к компьютеру пользователя.

### 3. Управление контекстным окном
- **Суть идеи:** При работе с большими проектами важно следить за объемом контекста (лимит — 200 000 токенов).
- **Почему это важно:** При переполнении контекста модель начинает «тупеть» и ошибаться. Необходимо регулярно очищать чат или сжимать информацию.
- **Ключевой пример:** Совет использовать Sonnet 4.6 для простых задач и переключаться на Opus 3.5/4.6 только для сложной логики и PRD.

### 4. Продуктовая точность (PRD как фундамент)
- **Суть идеи:** Качество написанного кода напрямую зависит от точности продуктовых требований (PRD), а не от навыков кодинга пользователя.
- **Почему это важно:** Нейросеть не ленится прописывать детали, если её правильно направить через интервью.
- **Ключевой пример:** Использование методик AURA 2 и Jobs to be Done для формирования требований через диалог с Клодом.

## Фреймворки и инструменты
- **Claude Code**: Автономный агент в терминале. Используется для написания и запуска кода. Рекомендуется режим `Edit automatically` для скорости.
- **AURA 2 и Jobs to be Done (JTBD)**: Методологии для проведения интервью с ИИ с целью создания детального PRD.
- **WhisperFlow**: Инструмент для доработки требований голосом.
- **FeatureDev (плагин)**: Инструмент для непосредственного написания кода MVP.
- **Una (через Amazing Boost Server Bot)**: Хостинг-решение для быстрого деплоя (стоимость ~3$ в неделю).
