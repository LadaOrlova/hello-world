## Ключевые концепции

### 1. Критерии идеальной идеи для обучения
- Суть идеи: Проект для первого дня должен быть максимально простым веб-приложением, которое лично интересно автору.
- Почему это важно: Сложность — главный враг новичка. Важно дойти до работающего результата за 4–5 часов, чтобы сохранить мотивацию. Личный интерес («хочется проснуться и доделать») помогает не бросить проект при столкновении с трудностями.
- Ключевой пример: Ваня Замесин отклонил идею создания аналога Miro на технологии Canvas, так как это «технология-звездолет», слишком сложная для старта.

### 2. Управление техническими рисками (Зоны опасности)
- Суть идеи: Избегание внешних интеграций и сложных API на первом этапе.
- Почему это важно: Интеграции с Google API, AmoCRM или сложный парсинг (например, GetCourse) часто вызывают проблемы с доступами и документацией, на решение которых уходят часы.
- Ключевой пример: Предложение создать агрегатор подписок через Google API было отклонено из-за бюрократии с Privacy Policy и одобрением приложения.

### 3. Приоритет облачных решений над локальными
- Суть идеи: Использование мощных облачных моделей через API (OpenRouter) эффективнее, чем попытки поднять локальные нейронки.
- Почему это важно: Локальные модели сложнее в настройке (Docker, зависимости) и часто требуют мощного железа, что отвлекает от основной задачи — написания кода приложения.
- Ключевой пример: Участнику, предложившему локальную LLM в Docker, было рекомендовано использовать OpenRouter для стабильности и простоты.

### 4. Инфраструктурный бюджет
- Суть идеи: Качественная разработка требует вложений в инструменты (VPN и лимиты нейросетей).
- Почему это важно: Стандартной подписки Claude за $20 недостаточно для интенсивного кодинга; необходим баланс (минимум $100) для работы через API без ограничений.
- Ключевой пример: Большинство проблем с доступом связаны с некачественными бесплатными VPN.

## Фреймворки и инструменты
- **VS Code + Claude Code**: Основная среда разработки и ИИ-ассистент (плагин). Используется для написания и редактирования кода.
- **Whisperflow**: Инструмент для голосового ввода, позволяющий быстрее формулировать запросы к нейросети.
- **OpenRouter**: Единый интерфейс для доступа к разным моделям (GPT-4o, Claude 3.5 Sonnet, Flash 3.1). Позволяет менять «мозги» приложения одной настройкой.
- **Node.js**: Среда для запуска локального сервера и тестирования приложения.
- **Zarub / Подарочные карты**: Рекомендуемые способы оплаты зарубежных сервисов во избежание блокировок (в отличие от «Плати Рублем»).
