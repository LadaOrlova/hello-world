4.  **Q&A:** Обсуждение проблем с оплатой, VPN и техническими сбоями.

## Ключевые концепции

### 1. Критерии выбора первой идеи («Не-звездолет»)
- Суть идеи в том, чтобы выбрать максимально простое веб-приложение, которое можно реализовать за 4–5 часов, избегая сложных интеграций.
- Это важно, чтобы участник гарантированно дошел до финала и не «сломался» на технических сложностях (API, доступы, сложные библиотеки).
- **Ключевой пример:** Ваня Замесин отклонил идею создания аналога Miro на Canvas, так как это слишком сложно и дорого для первого дня.

### 2. Принцип «Чистой среды»
- Суть в том, чтобы полностью подготовить рабочее окружение (VS Code, Claude, Whisperflow) до начала активного кодинга.
- Это позволяет на основной неделе обучения сфокусироваться только на создании продукта, не отвлекаясь на установку Node.js или борьбу с VPN.
- **Ключевой пример:** Идеальный результат сессии — открытый файл `what-to-do.md` в настроенном редакторе.

### 3. Ресурсная база ИИ-кодинга
- Для серьезной работы с кодом через нейросети стандартных подписок недостаточно; требуются расширенные лимиты.
- Написание кода потребляет много токенов, поэтому бюджет на ИИ нужно планировать заранее.
- **Ключевой пример:** Подписки Claude за $20 не хватит для полноценного воркшопа, рекомендуется ориентироваться на лимиты уровня $100.

## Фреймворки и инструменты
- **Visual Studio Code (VS Code)**: Основной редактор кода. Использовать как базу для всех расширений.
- **Claude Code**: Платный ИИ-инструмент для генерации кода. Основной «двигатель» разработки на курсе.
- **Whisperflow**: Инструмент для голосового ввода команд. Позволяет быстрее формулировать задачи для ИИ.
- **OpenRouter**: Сервис-агрегатор для подключения различных моделей (GPT-4o, Claude 3.5 Sonnet). Применять, если нужен доступ к разным нейросетям через один интерфейс.
- **Node.js**: Программная среда. Необходима для запуска локального веб-сервера.
- **Zarub / Плати Рублем**: Сервисы для оплаты зарубежных подписок и создания виртуальных карт в условиях ограничений.

## Разногласия и ограничения
- **Запрещенные технологии:** Технология Canvas (как в Miro) — слишком дорого и сложно.
- **Рискованные идеи:** Ассистенты для AmoCRM (сложность доступа к данным), агрегаторы почты (Privacy Policy Google), локальные LLM в Docker (рекомендовано заменить на облачные решения).
