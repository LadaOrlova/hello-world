## Тема воркшопа

## Структура занятия
Воркшоп был разделен на две части:
1.  **Теоретический блок:** Определение критериев выбора идеи для проекта и обзор технологического стека.
2.  **Практический блок:** Распределение участников по сессионным залам (по 6 человек на одного куратора) для индивидуальной настройки ПО и решения технических проблем.

## Ключевые концепции

### 1. Критерий «Быстрого результата» (MVP первого дня)
- **Суть идеи:** Для первого проекта необходимо выбирать максимально простую идею, реализация которой занимает не более 4–5 часов.
- **Почему это важно:** Это позволяет гарантированно получить работающий результат в первый же день, почувствовать «мощь инструментов» и избежать фрустрации.
- **Ключевой пример:** Спикер 1 рекомендует отказаться от сложных идей вроде аналогов Miro (технология Canvas) или систем с глубокой интеграцией API (Google API) в пользу простых веб-приложений.

### 2. Личная значимость проекта
- **Суть идеи:** Продуктивность обучения напрямую коррелирует с личным интересом к выбранному продукту.
- **Почему это важно:** Высокая мотивация («желание проснуться утром и делать») помогает преодолевать технические сложности и забирать максимум пользы от интенсива.
- **Ключевой пример:** Выбор идеи, которая драйвит лично вас, важнее, чем попытка создать «правильный» или коммерчески востребованный продукт на этапе обучения.

### 3. Облачные LLM против локальных решений
- **Суть идеи:** Использование нейросетей через облачные сервисы (OpenRouter) предпочтительнее запуска локальных моделей.
- **Почему это важно:** Локальные модели в Docker слишком сложны в настройке для новичков и требуют больших ресурсов. Облачные решения — это самый дешевый и стабильный вариант для старта.
- **Ключевой пример:** Вместо настройки Докера участникам предлагается использовать OpenRouter для доступа к моделям Claude 3.5 Sonnet или GPT-4o.

### 4. Идеальное рабочее пространство (IDE Setup)
- **Суть идеи:** Эффективная разработка требует специфической организации экрана в Visual Studio Code.
- **Почему это важно:** Правильная компоновка (файлы слева, инструкция в центре, ИИ-чат справа) минимизирует переключение контекста и ускоряет работу.
- **Ключевой пример:** Использование Claude Code непосредственно внутри VS Code для бесшовного написания кода.

## Фреймворки и инструменты
- **Visual Studio Code (VS Code)**: Основная среда разработки (IDE). Используется как база для написания кода и интеграции ИИ-ассистентов.
- **Claude Code**: Расширение для VS Code. Основной ИИ-помощник для генерации кода. *Важно: подписки за $20 не хватит, требуется лимит около $100.*
- **OpenRouter**: Агрегатор LLM-моделей. Позволяет использовать Claude 3.5 Sonnet, GPT-4o или Flash 3.1 через единый интерфейс. Самый дешевый вариант для работы.
- **Whisperflow**: Инструмент для работы с голосом (преобразование голоса в текст/команды).
- **FeatureDev**: Плагин для управления процессом разработки функций.
- **Node.js**: Программная среда для запуска локального сервера, необходимая для работы веб-приложений.
