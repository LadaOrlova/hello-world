## Тема воркшопа
**Разработка функционального прототипа с подключением базы данных и деплоем через ИИ-инструменты.** 
(Воркшоп №2, развитие темы первого занятия по ускоренному созданию MVP).

## Структура занятия
1.  **Вводная часть:** Постановка усложненной задачи (БД + деплой).
2.  **Техническая настройка:** Установка базового софта (Node.js) и регистрация в сервисах (Vercel, Neon).
3.  **Практический блок (Breakout Rooms):** Работа в малых группах с кураторами (Женя, Артем, Максим) над созданием кода через Claude Code.
4.  **Лекционные вставки:** Мини-лекции по DevOps, психологии готовности и автоматизации через API/Make.
5.  **Разбор ошибок и Q&A:** Решение проблем с путями файлов, контекстом нейросети и деплоем.

## Ключевые концепции

### 1. Переход от «простого кода» к полноценному окружению
- **Суть идеи:** Делать прототип не просто в виде набора файлов, а сразу с подключением реальной базы данных и публикацией в сети.
- **Почему это важно:** Без деплоя проект остается «на компьютере разработчика» и недоступен пользователям. Наличие БД позволяет тестировать реальные пользовательские сценарии.
- **Пример:** Использование связки Vercel (хостинг) + Neon (PostgreSQL) для запуска MVP.

### 2. Психология взаимодействия с ИИ-агентами
- **Суть идеи:** Главный навык сегодня — это уверенность в том, что «всё должно работать», и настойчивость в требовании результата от ИИ.
- **Почему это важно:** Нейросети могут ошибаться или предлагать лишнее, поэтому нужно уметь «заставлять» их переделывать до победного.
- **Пример:** Если возникла ошибка, нужно сделать скриншот и написать: «Есть проблема. Должно работать вот так, почини, пожалуйста».

### 3. Культура DevOps и CI/CD в прототипировании
- **Суть идеи:** Деплой — это не разовое действие, а культура минимизации рисков через автоматизацию и тесты (Shift-left testing).
- **Почему это важно:** Ручной деплой ведет к человеческим ошибкам; Infrastructure as Code (IaC) позволяет развернуть среду за минуты.
- **Пример:** Сравнение деплоя без тестов с «прыжком с непроверенным парашютом».

### 4. Психологическая готовность к новому (аналогия со школой)
- **Суть идеи:** Успех в освоении нового (будь то школа или разработка через ИИ) зависит от мотивационной и волевой сферы больше, чем от технических навыков.
- **Почему это важно:** Важно формировать позитивный образ процесса и поддерживать самостоятельность, а не пугать сложностями.
- **Пример:** Психологическая готовность ребенка к школе как модель адаптации к любым сложным изменениям.

### 5. Автоматизация через API и Агентов
- **Суть идеи:** Использование OpenAI API и инструментов вроде Make (Integromat) для замены линейных ассистентов на 80%.
- **Почему это важно:** Это позволяет создавать автономные цепочки (например, Telegram → конспект статьи → Notion).
- **Пример:** Структурирование промпта по схеме: Role, Context, Task, Format, Constraints.

## Фреймворки и инструменты
- **Claude Code**: CLI-инструмент для написания кода. Использовать команду `/feature-dev` для проектирования архитектуры.
- **Cursor**: ИИ-редактор кода. Важно использовать плагины `FeatureDev` и `Front End Design`.
- **Vercel**: Хостинг для моментального деплоя приложений.
- **Neon**: Облачная база данных PostgreSQL.
- **Node.js**: Базовая среда, которую необходимо ставить первым шагом для работы всех утилит.
- **Make (Integromat)**: Платформа для создания автоматизированных сценариев без кода.

## Практические упражнения
- **Настройка окружения:** Участники устанавливали Node.js, регистрировались в Vercel и Neon.
- **Работа с Claude Code:** Запуск команды `/feature-dev` и «скармливание» ИИ файлов `@PRD` и `@MVP.md` для генерации логики.
- **Деплой:** Загрузка созданного проекта на Vercel и получение публичной ссылки.
- **Troubleshooting:** Решение проблем с 404 ошибками (пересборка из папки `dist`) и правами доступа (`sudo`).

## Другое (Технические нюансы и разногласия)
- **Управление контекстом:** При заполнении памяти Claude на 100% рекомендуется использовать команды `clear` или `compact`.
- **Организация файлов:** Решено создавать проект в отдельной папке, а не рядом с файлом MVP.md, чтобы не засорять контекст ИИ.
- **Баги инструментов:** Cursor лучше перезапускать через `Command+Q`, если он завис. Ссылки Vercel могут быть как `production`, так и `preview` без явной логики.
