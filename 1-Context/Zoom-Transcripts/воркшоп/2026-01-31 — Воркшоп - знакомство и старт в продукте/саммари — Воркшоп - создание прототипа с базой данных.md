**Дата**: 31 января 2026

---

Вот подробное структурированное саммари воркшопа на основе предоставленных тезисов.

---

## Тема воркшопа
**Разработка и деплой полнофункционального MVP с базой данных при помощи ИИ-инструментов.**

## Структура занятия
Воркшоп был построен по принципу интенсивной практики:
1.  **Подготовка окружения**: установка Node.js, плагинов для Claude Code и регистрация в облачных сервисах (Vercel, Neon).
2.  **Групповая работа**: сессионные залы для оперативной помощи в настройке софта.
3.  **Проектирование и разработка**: использование ИИ для создания архитектуры и написания кода на основе PRD.
4.  **Деплой и отладка**: развертывание проекта в сети и решение возникающих ошибок в реальном времени.

## Ключевые концепции

### 1. Переход от прототипа к профессиональной разработке
- **Суть идеи**: Вместо создания упрощенных макетов без сохранения данных, участники сразу создают полноценные приложения с подключенной базой данных (БД).
- **Почему это важно**: Это позволяет студентам научиться деплоить сервисы как профессиональные разработчики, создавая жизнеспособные продукты.
- **Пример**: Использование связки Neon (БД) + Vercel (хостинг) для обеспечения работы сайта даже после выключения компьютера разработчика.

### 2. ИИ как «усердный джун»
- **Суть идеи**: Нейросеть (Claude) нужно воспринимать как исполнительного, но требующего контроля младшего разработчика.
- **Почему это важно**: Это меняет подход к управлению ИИ — если результат не устраивает или стили не применились, нужно настойчиво просить переделать (иногда по 5–6 раз).
- **Пример**: Если ИИ ошибается в коде, не нужно бояться — Клод способен самостоятельно увидеть и исправить ошибку.

### 3. Эволюционная архитектура и борьба с избыточностью
- **Суть идеи**: На этапе MVP важно ограничивать сложность структуры проекта, чтобы не запутаться в ней.
- **Почему это важно**: Без ограничений ИИ может создать архитектуру на 70–150 файлов там, где достаточно нескольких, что усложняет поддержку.
- **Пример**: Использование специального файла-инструкции `MVP.md`, который заставляет Клода придерживаться простой и понятной структуры.

### 4. Визуальный фидбек и итеративность («Говно, переделай»)
- **Суть идеи**: Основной навык работы с ИИ сегодня — это предоставление визуального контекста проблемы.
- **Почему это важно**: Быстрее сделать скриншот ошибки или кривого интерфейса и отправить его ИИ с описанием желаемого результата, чем описывать проблему текстом.
- **Пример**: Использование скриншотов как главного инструмента промпт-инжиниринга в процессе доработки фронтенда.

### 5. Управление контекстом и лимитами ИИ
- **Суть идеи**: Работа с Claude Code требует понимания технических ограничений (лимиты токенов, стоимость API).
- **Почему это важно**: Знание того, что Клод умеет сжимать (compact) контекст, позволяет продолжать работу в одном чате долгое время.
- **Пример**: Лимиты Claude обновляются каждые 5 часов, а для работы через API Anthropic требуется минимальное пополнение баланса на 5 долларов.

## Фреймворки и инструменты
- **Claude Code Plugins (`FeatureDev` и `Frontend Design`)**: плагины для проектирования архитектуры по PRD и создания дизайна интерфейсов. Устанавливаются через команду `/plugins`.
- **Vercel**: сервис для хостинга и деплоя. Используется для публикации проекта в интернете. Команда деплоя — `npx vercel`.
- **Neon DB**: облачная база данных PostgreSQL. Позволяет хранить данные пользователей отдельно от кода.
- **Node.js**: программная среда, необходимая для работы всех утилит разработки. Устанавливается в первую очередь.
- **Cursor**: редактор кода, который иногда требует полной перезагрузки (Command+Q) для корректной работы плагинов.
- **MVP.md**: файл-инструкция (системный промпт), ограничивающий избыточную сложность архитектуры, создаваемой ИИ.

## Практические упражнения
- **Настройка рабочего пространства**: Установка Node.js, установка плагинов в Cursor и регистрация на Vercel и Neon.
- **Работа с окружением**: Создание файла `.env` и прописывание в нем `DATABASE_URL` для связи кода с базой данных.
- **Подготовка к деплою**: Перенос файлов проекта в отдельную папку `Project`, чтобы избежать загрузки лишнего мусора на сервер.
- **Генерация кода**: Запуск `feature-dev` и передача ИИ продуктовых требований из файла `@PRD`.
- **Локальное тестирование**: Проверка сохранения данных в БД путем перезагрузки страницы.

## Другое
- **Экономика разработки**: Использование API Клода — это дополнительные расходы поверх подписки. Аренда мощного сервера (например, на Яндекс Клауд) может стоить 3-4 тысячи рублей в месяц.
- **Технические нюансы**: При деплое через VPN могут возникать обрывы соединения (решается сменой страны в VPN). 
- **Оценка сроков**: ИИ плохо оценивает время выполнения задач (может назвать срок в 2 дня для задачи, которую сделает за 10 минут).
- **Специфика Vercel**: Ссылки после деплоя могут называться Production или Preview без явной логики сервиса.
