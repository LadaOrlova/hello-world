## Воркшоп 5: Telegram-бот для картинок
Пятый воркшоп был посвящен созданию Telegram-бота, который интегрируется с нейронной сетью OpenRouter для обработки и преобразования изображений. Участники учились создавать ботов, подключать внешние AI-модели и использовать продвинутые функции AI-помощника Cursor для разработки.

## Структура занятия
Воркшоп был построен по следующей схеме:
1.  **Теоретический блок и демонстрация:** Ведущий Илья подробно демонстрировал весь процесс создания бота, от базовой настройки до интеграции с AI, объясняя каждый шаг и его цель.
2.  **Практическая работа в комнатах:** Участники делились на группы и в отдельных комнатах под руководством кураторов самостоятельно выполняли ту же задачу, воспроизводя пройденные шаги. Кураторы оказывали помощь при возникновении сложностей.
3.  **Обсуждение и вопросы:** После основной практической части и короткого перерыва был проведен общий блок вопросов и ответов, а также демонстрация дополнительных концепций (например, маркетплейс плагинов).

## Ключевые концепции

### 1. Создание Telegram-бота с помощью AI
- Суть идеи: Использование AI-агента (Claude в Cursor) для автоматизированного создания базового Telegram-бота, способного обрабатывать входящие сообщения.
- Почему это важно / как применять: Позволяет быстро прототипировать и разворачивать функциональные боты без глубоких знаний программирования, фокусируясь на логике и интеграциях.
- Ключевой пример из воркшопа: Сначала был создан простой эхо-бот, который не имел интеграции с нейросетью, чтобы убедиться в его работоспособности, а затем функционал был расширен.

### 2. Интеграция с внешними AI-моделями через OpenRouter
- Суть идеи: OpenRouter — это унифицированный шлюз, который предоставляет доступ к множеству различных нейронных сетей от разных провайдеров (Google, Anthropic и др.) через единый API-ключ.
- Почему это важно / как применять: Упрощает процесс подключения различных AI-моделей для выполнения специализированных задач (генерация изображений, обработка речи, поиск) без необходимости регистрации на нескольких платформах. Снимает комиссию около 5%.
- Ключевой пример из воркшопа: Бот использовал модели Google Gemini 2.5 Flash Image или Nano Banana Pro для преобразования загруженных картинок согласно текстовому запросу.

### 3. Эффективное взаимодействие с AI-агентом (Prompting)
- Суть идеи: С появлением более продвинутых AI-моделей отпадает необходимость в чрезмерно детальных и замудренных промтах. Достаточно использовать простые, разговорные инструкции, а AI самостоятельно задаст уточняющие вопросы.
- Почему это важно / как применять: Оптимизирует процесс разработки, делая его более интуитивным и доступным. Разработчик может сосредоточиться на высокоуровневых задачах, передавая детализацию AI.
- Ключевой пример из воркшопа: Вместо "составления супердетальных промтов" предлагалось использовать фразы типа "Хочу сделать простенького ботика... Задай мне вопросов, чтобы детализировать задачу" или "я не знаю, сделай, как лучше".

### 4. Управление контекстом и инструкциями AI (Memory)
- Суть идеи: AI-агенты (в Cursor/Claude) могут сохранять инструкции и правила на глобальном уровне (User Memory) для использования во всех проектах, или на уровне конкретного проекта (Project Memory).
- Почему это важно / как применять: Позволяет стандартизировать рабочие процессы, автоматизировать повторяющиеся задачи и поддерживать консистентность в разработке. Можно копировать `.Claude MD` файлы с инструкциями между проектами.
- Ключевой пример из воркшопа: Была продемонстрирована команда `Memory` в терминале для просмотра текущих инструкций. Возможность попросить Клода сохранить глобальные правила, чтобы не дублировать их в каждом промте.

### 5. Использование и создание плагинов для AI-агентов
- Суть идеи: Плагины представляют собой наборы готовых команд, скиллов и агентов, которые можно устанавливать для расширения функционала AI-помощника. Существуют открытые маркетплейсы плагинов.
- Почему это важно / как применять: Позволяет быстро добавлять комплексные возможности (например, Code Review, Brainstorming, Feature Development) в среду разработки, используя уже готовые и проверенные сообществом решения. Также можно создавать собственные плагины.
- Ключевой пример из воркшопа: Демонстрация установки плагина из GitHub-репозитория через `Slash Plugin -> Manage Plugins -> Marketplaces` в Cursor, показывая его структуру (Agents, Commands, Skills) и пример работы "Brainstorm" скилла.

## Фреймворки и инструменты
-   **Cursor с плагином Claude Extension**: Что делает: интегрированная среда разработки (IDE) с мощным AI-помощником Claude. Как использовать: открывать проект, задавать задачи на естественном языке, позволять AI генерировать код, исправлять ошибки. Когда применять: для ускоренной разработки, прототипирования, исправления кода, взаимодействия с файловой системой.
-   **Botfather (Telegram)**: Что делает: официальный бот Telegram для создания и управления другими ботами. Как использовать: введите `/newbot` для создания нового бота, укажите имя и username (должен заканчиваться на `bot`), получите API-токен. Когда применять: для старта любого нового Telegram-бота.
-   **OpenRouter**: Что делает: унифицированный API-шлюз к множеству различных AI-моделей (LLM, генерация изображений, обработка аудио) от разных провайдеров. Как использовать: получить API-ключ, выбрать нужную модель (например, Google Gemini 2.5 Flash Image / Nano Banana Pro) из списка, интегрировать ключ в свой код для вызова моделей. Когда применять: для использования разнообразных AI-моделей через единую точку доступа, сравнения их производительности и стоимости.
-   **Plan Mode (в Claude)**: Что делает: режим планирования, в котором AI сначала создает детальный план действий для выполнения задачи, а затем выполняет его. Как использовать: активировать кликом или `Shift+Tab`. План сохраняется в контексте и не сжимается. Когда применять: для сложных задач, требующих структурированного подхода и последовательного выполнения, а также для контроля за действиями AI.
-   **Маркетплейс плагинов (в Cursor/Claude)**: Что делает: позволяет устанавливать готовые наборы команд, скиллов и агентов, созданные другими пользователями или Anthropic. Как использовать: ввести `/Plugin` в Cursor, выбрать `Manage Plugins`, перейти на вкладку `Marketplaces` и добавить ссылку на GitHub-репозиторий плагина. Затем активировать нужные скиллы. Когда применять: для расширения функционала AI-помощника, использования готовых решений для типовых задач (например, Code Review, Brainstorming), изучения чужих реализаций агентов/скиллов.

## Практические упражнения
1.  **Создание простого эхо-бота в Telegram:** Участники использовали AI-агента для генерации базового кода Telegram-бота, который просто отвечал на сообщения пользователя. Целью было проверить базовую работоспособность без внешних интеграций.
2.  **Получение токена бота через Botfather:** Самостоятельная регистрация нового бота в Telegram через Botfather и получение уникального API-токена.
3.  **Запуск Telegram-бота локально:** Настройка и запуск созданного бота на своей машине, тестирование его функционала.
4.  **Интеграция OpenRouter с AI-моделью для обработки изображений:** Подключение API-ключа OpenRouter и выбор конкретной AI-модели (например, Google Gemini 2.5 Flash Image) для добавления функционала преобразования изображений в бота.
5.  **Тестирование бота с AI-обработкой изображений:** Отправка изображений в бота с текстовыми промтами для их модификации (например, "добавь фон", "сделай без куртки") и проверка результата.
6.  **Модификация бота:** Участникам предлагалось минимально модифицировать бота, не усложняя логику, для закрепления полученных навыков.
7.  **Развертывание бота на сервере (опционально):** Некоторые участники успешно деплоили своего бота на виртуальный сервер.
