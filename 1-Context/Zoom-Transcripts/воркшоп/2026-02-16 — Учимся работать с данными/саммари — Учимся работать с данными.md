## Ключевые концепции

### 1. Быстрое ознакомление с данными с помощью LLM
-   **Суть идеи:** Использование больших языковых моделей (LLM) для автоматического обзора и понимания содержимого незнакомых или сложноструктурированных файлов данных (JSON, CSV, таблицы), не требуя ручного чтения или написания кода.
-   **Почему это важно / как применять:** Это позволяет быстро получить представление о структуре данных, количестве записей, полях и потенциальных выводах, которые можно сделать. Это первый шаг при работе с новым или объемным набором данных, экономящий время продакт-менеджера.
-   **Ключевой пример:** Задание 1, где LLM (Sonet 4.5) анализировала папки с данными книжного клуба (`Exit Survey`, `support_tickets_JSON`, `data_subscription_events_CSV`, `reviews_JSON`) и формировала `data_overview.md`, описывая их содержимое.

### 2. Опасность галлюцинаций LLM при точном количественном анализе
-   **Суть идеи:** При прямом запросе к LLM выполнить точные математические расчеты или статистический анализ (например, NPS, конверсии) без использования скриптов, результаты могут сильно варьироваться и быть неточными или полностью ошибочными из-за "галлюцинаций" модели.
-   **Почему это важно / как применять:** Нельзя полагаться на LLM для получения точных цифр, необходимых для принятия бизнес-решений, если не используются верифицируемые методы. Это показывает, что LLM не является заменой статистическому анализу без контроля.
-   **Ключевой пример:** Задания 2 и 3, где участники просили нейросеть посчитать NPS и воронку конверсий напрямую. Результаты у разных участников, работающих с одними и теми же данными, сильно отличались, что демонстрировало ненадежность такого подхода.

### 3. LLM как инструмент для генерации скриптов точного анализа данных
-   **Суть идеи:** Для получения точных и воспроизводимых результатов анализа данных следует просить LLM не выполнять расчеты самостоятельно, а генерировать исполняемые скрипты (например, на Python), которые затем могут быть запущены.
-   **Почему это важно / как применять:** Скрипты обеспечивают детерминированность расчетов, устраняя галлюцинации LLM. Даже если пользователь не владеет программированием, он может использовать LLM для создания, объяснения и проверки кода. Это позволяет совместить скорость LLM с точностью программирования.
-   **Ключевой пример:** Задания 4 и 5, где LLM (Opus 4.5) генерировала Python-скрипты для расчета NPS и конверсий. После выполнения этих скриптов результаты у всех участников стали одинаковыми (NPS 29.7%, конверсия 51.4%), подтверждая надежность подхода.

### 4. Качественный анализ корневых причин количественных проблем
-   **Суть идеи:** После выявления количественной проблемы в данных (например, резкое падение конверсии), LLM можно использовать для анализа большого объема неструктурированных текстовых данных (отзывы пользователей, тикеты поддержки) с целью обнаружения качественных объяснений и формулирования гипотез.
-   **Почему это важно / как применять:** Это позволяет переходить от "что произошло" к "почему произошло", находя паттерны, жалобы и боли пользователей, скрытые в текстовых данных, что критически важно для принятия продуктовых решений.
-   **Ключевой пример:** Задание 6, где, обнаружив падение конверсии на второй месяц подписки, нейросеть анализировала `reviews_JSON` и `support_ticket` для поиска упоминаний, связанных с оплатой и продлением, выявляя конкретные жалобы и формулируя гипотезу о баге в реализации списания денег.

### 5. Управление контекстным окном LLM
-   **Суть идеи:** Большие языковые модели имеют ограниченное "контекстное окно" (количество токенов, которое они могут удерживать в памяти для текущего диалога). При достижении лимита модель начинает "сжимать" предыдущую историю, что может привести к снижению качества ответов.
-   **Почему это важно / как применять:** Для поддержания высокого качества взаимодействия и избегания деградации ответов, особенно при выполнении сложных или многоэтапных задач, необходимо отслеживать заполненность контекстного окна и при необходимости начинать новый диалог/чат.
-   **Ключевой пример:** Обсуждение на воркшопе (17:40), когда спикер предложил открыть новый чат, так как контекстное окно было заполнено на 75% (150 000 из 200 000 токенов для Sonet).

## Фреймворки и инструменты
-   **Cursor / Cloud Code**:
    *   **Что делает:** Интегрированная среда разработки (IDE) с функциями, позволяющими напрямую взаимодействовать с большими языковыми моделями (LLM) для написания, рефакторинга, отладки кода и анализа данных. `Cloud Code` — это чат-интерфейс внутри Cursor для работы с LLM.
    *   **Как использовать:** Открыть проект, запустить `Cloud Code` чат, ввести запросы (промты) к LLM, используя слеш-команды для выбора модели (`/model sonet 4.5` или `/model opus`) и получения ответов или сгенерированного кода.
    *   **Когда применять:** Для быстрого ознакомления с данными, генерации скриптов для анализа, поиска информации в текстовых данных, рефакторинга кода, отладки и других задач, где требуется помощь LLM.
-   **Модели LLM (Sonet 4.5, Opus 4.5)**:
    *   **Что делает:** Это различные версии больших языковых моделей от Anthropic (через Cloud Code), предоставляющие разные возможности и производительность. Opus — более мощная и дорогая модель, Sonet — более быстрая и экономичная.
    *   **Как использовать:** Переключать модели в чате `Cloud Code` с помощью команды `/model [название модели]`, например, `/model sonet 4.5` или `/model opus`.
    *   **Когда применять:** Sonet рекомендуется для быстрых задач и общих обзоров, Opus — для более сложных задач, требующих высокой точности, понимания контекста и генерации кода.
