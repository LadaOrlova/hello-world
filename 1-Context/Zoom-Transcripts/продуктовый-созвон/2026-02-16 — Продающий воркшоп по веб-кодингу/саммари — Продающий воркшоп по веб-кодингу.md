# [Созвон] Продающий воркшоп по веб-кодингу
**Дата**: 16 февраля 2026

---

## Краткое содержание
Созвон посвящен планированию и интеграции воркшопа по веб-кодингу и использованию ИИ в курс "Как делать продукт" для большого числа студентов. Обсуждались варианты формата воркшопа (от лекции до создания MVP), проблемы с кураторами и технической подготовкой студентов, а также стратегия монетизации и финансовой мотивации для участников проекта.

## Ключевые решения
- Курс "Как делать продукт" будет переработан для интеграции современных реалий веб-кодинга и работы с нейросетями/AI-ассистентами.
- Формат предстоящего воркшопа по веб-кодингу будет зависеть от готовности студентов оплатить подписку Cloud Code (около $100).
- При количестве готовых студентов до 90 человек будет предпринята попытка провести сокращенный hands-on воркшоп по созданию MVP (как на субботних занятиях), с целью продажи второго модуля веб-кодинга.
- При очень малом количестве готовых студентов будет проведена лекция.
- Рассматривается идея создания AI-ассистента для поддержки процесса разработки MVP, который встраивался бы в workflow студентов (например, в Cloud Code/Cursor).

## Открытые вопросы
- Как обеспечить достаточное количество кураторов для эффективной работы со 100+ студентами, учитывая потенциальные сложности с установкой и настройкой инструментов (Cloud Code, Cursor)?
- Как оптимально развести контент основного курса "Как делать продукт" и дополнительные модули веб-кодинга ("бусты"), чтобы стимулировать дальнейшие покупки студентами?
- Какая конкретная дата проведения воркшопа, учитывая потенциальные сложности с Cloud Code/Cursor (упоминание 7 марта и последующие комментарии о проблемах)?

## Основные темы обсуждения
- Интеграция веб-кодинга и современных ИИ-инструментов в курс "Как делать продукт".
- Формат и содержание вводного 2.5-часового воркшопа для 100+ студентов (создание лендинга, MVP, или лекция).
- Проблемы масштабирования воркшопа: необходимость большого количества кураторов и потенциальные трудности студентов с технической подготовкой (установка, оплата Cloud Code).
- Стратегия вовлечения студентов и последующая продажа дополнительных модулей по веб-кодингу.
- Возможность разработки собственного AI-ассистента для MVP-разработки, интегрированного в рабочие процессы студентов.
- Финансовая мотивация для разработчика (Ильи) от продаж последующих модулей.
- Переработка всего курса "Как делать продукт" в соответствии с актуальными подходами.
