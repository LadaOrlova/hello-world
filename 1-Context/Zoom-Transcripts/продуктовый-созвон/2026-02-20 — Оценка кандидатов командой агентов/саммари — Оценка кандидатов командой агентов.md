**Дата**: 20 февраля 2026

---

## Контекст
Встреча посвящена планированию и разработке контента для продуктового воркшопа, сфокусированного на обучении продуктовых специалистов и предпринимателей использованию AI-агентов, в частности "Команд Агентов" (Agent Teams). Цель — создать понятные и прикладные задачи для демонстрации возможностей агентов, отойдя от задач, требующих глубоких знаний в программировании. В качестве примера продукта для воркшопа выбран "Telegram бот с AI и фоточками".

## Ключевые решения

| Решение | Обоснование | Кто принял |
|---------|-------------|------------|
| Фокусироваться на прикладных задачах для оценки работы агентов, а не на кодинге. | Пользователи (продукты, предприниматели) не понимают принципов код-ревью, что затрудняет оценку агентов на кодинговых задачах. Прикладные задачи более релевантны и понятны их аудитории. | Ivan Zamesin, Илья Изма |
| Использовать команды агентов для "разноса и доработки" продуктовых требований (PRD). | Это актуальная задача для продуктовых специалистов, которая хорошо демонстрирует возможности команд агентов в работе с противоположными мнениями и принятии бизнес-решений. | Ivan Zamesin, Илья Изма |
| В качестве продукта для воркшопа использовать "Telegram бот с АИ и фоточками". | Это конкретный, понятный пример для практических заданий, включающий чат-бот и генерацию изображений. | Илья Изма, Ivan Zamesin |
| Внедрить простую функцию расшифровки аудиофайлов (скилл/скрипт) вместо автоматической системы. | Автоматическая система расшифровки с автозапуском "геморройная штука" и сложна в реализации. Простой скилл для файла более управляем. | Ivan Zamesin |
| Добавить новый скилл в существующий плагин "Feature Dev", который будет сначала задавать вопросы для детализации задачи, а затем запускать "Team Feature" для реализации. | Это позволит агентам более полно прорабатывать задачи, собирая необходимую информацию перед переходом к реализации, улучшая текущий workflow. | Ivan Zamesin, Илья Изма |
| **Структура воркшопа:** Первая часть — работа с ботом (дополнение существующего задания). Вторая часть — использование команд агентов для улучшения PRD. | Такая структура обеспечит вариативность заданий и позволит участникам глубже изучить возможности агентов в различных сценариях, уходя от простого копи-паста. | Ivan Zamesin, Илья Изма |

## Обсуждаемые гипотезы и идеи
- **[Модель "Эксперт Арены"]**: Создание аналога "эксперт арены", где команды агентов спорят и приходят к какому-то решению. Статус: **Принята** как концептуальная основа для использования Agent Teams.
- **[Пример с наймом сотрудника]**: Использование команды агентов для оценки кандидата (поиск сильных/слабых сторон, проверка тестового задания) на основе расшифровки интервью и описания вакансии. Статус: **Обсуждалась**, но выбран вариант с PRD как более наглядный для демонстрации коммуникации агентов.
- **[Эффективность Agent Teams для ресёрча]**: Гипотеза, что Agent Teams более эффективны для задач, требующих формирования противоположных мнений и принятия финального решения на основе собранных данных, чем для чисто исследовательских задач. Статус: **Принята** (косвенно, через выбор задачи с улучшением PRD, которая подразумевает принятие решений).
- **[Удаление скилла расшифровки аудио]**: Возможность убрать функционал расшифровки аудио, если это потребуется для упрощения или сокращения объема. Статус: **Обсуждалась**, но пока остается как опция.
- **[Комбинирование картинки и текста в чат-боте]**: Добавление функционала, позволяющего ввести текст в Gemini после генерации картинки в чат-боте. Статус: **Обсуждалась**, как возможная фича для бота.
- **[Команда экспертов для брейншторма в начале воркшопа]**: Начинать воркшоп со сбора команды экспертов для первоначального брейншторма. Статус: **Отклонена (пока)**, Илья Изма решил "подумать еще, как это лучше сделать".

## Технические и продуктовые требования
- **Скилл для Agent Teams**: Студенты должны уметь создавать и использовать Agent Teams.
- **Документация/Обучение для Agent Teams**: Необходимо чёткое объяснение, когда, в каких ситуациях и для каких задач полезны Agent Teams.
- **Функционал бота (для воркшопа)**: Чат-бот "нанобанана" с AI и фото, интегрированный с Gemini, с возможностью диалога.
- **Задания воркшопа**: Должны быть вариативны и содержать открытые вопросы, дающие возможность для творчества (например, "как изменить обработку картинок нанобанане?", "как изменить логику чат-бота?", "как встроить нейронку в бота, какие сценарии?").
- **Время выполнения заданий**: Предусмотреть 15-20 минут на выполнение одного такого открытого задания.

## Приоритеты
1.  Разработка контента для воркшопа: создать задания для использования команд агентов для улучшения PRD и доработать существующее задание с ботом.
2.  Внедрение функционала "задавания вопросов" в плагин "Feature Dev" для более детальной проработки задач агентами.
