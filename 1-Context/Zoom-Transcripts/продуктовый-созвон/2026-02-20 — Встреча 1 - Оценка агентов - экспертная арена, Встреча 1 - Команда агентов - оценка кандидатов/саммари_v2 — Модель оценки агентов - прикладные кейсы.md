# [Продуктовый-созвон] Модель оценки агентов: прикладные кейсы
**Дата**: 20 февраля 2026

---

## Контекст
Встреча посвящена планированию и наполнению продуктового воркшопа/тренинга ("экспертной арены"), целью которого является обучение участников подходу к оценке работы AI-агентов и использованию команд агентов. Основная идея — дать практические задачи, понятные продукт-менеджерам и предпринимателям, чтобы они могли эффективно оценить и использовать потенциал AI-агентов.

## Ключевые решения

| Решение | Обоснование | Кто принял |
|---------|-------------|------------|
| **Отказ от оценки агентов на кодинговых задачах** | Пользователи (продукты, предприниматели) не умеют читать код, не понимают код-ревью и принципы его выполнения, что мешает им адекватно оценить работу агентов. | Ivan Zamesin, Илья Изма |
| **Фокус на понятных, прикладных задачах для оценки агентов** | Цель – научить человека подходу, дать инструмент и обеспечить понимание. Для этого нужны задачи, которые целевая аудитория (продукты, предприниматели) хорошо понимает и выполняет. | Ivan Zamesin, Илья Изма |
| **Основной практической задачей для Agent Teams станет доработка и улучшение Product Requirements Document (PRD)** | Задача актуальна для продактов и предпринимателей, позволяет продемонстрировать коммуникацию агентов и их способность принимать решения на основе собранных данных. | Ivan Zamesin, Илья Изма |
| **Отказ от реализации улучшенного PRD в рамках воркшопа** | Сфокусироваться на *улучшении* PRD, а не на его *реализации*, чтобы не усложнять и не увеличивать длительность воркшопа. | Ivan Zamesin |
| **Упрощение функциональности аудио/видео обработки для воркшопа** | Вместо сложной системы автоматической расшифровки со всеми устройства, будет реализован простой скрипт для расшифровки отдельного аудиофайла и создания его саммари. | Ivan Zamesin |
| **Интеграция функции "задавать вопросы для детализации задачи" в основной плагин/скилл** | Для повышения удобства и унификации рабочего процесса, вместо создания отдельной команды или плагина для вопросов, эта функция будет добавлена как скилл в существующий плагин реализации задач. | Ivan Zamesin, Илья Изма |
| **Структура воркшопа: 1-я половина - работа с ботом, 2-я половина - работа с PRD** | Разделение на две логические части для последовательного освоения материала. | Ivan Zamesin, Илья Изма |
| **Включение вариативности и открытых задач в воркшоп по боту** | Чтобы уйти от "копи-паста" и стимулировать активное мышление, участникам будет предложено самостоятельно модифицировать логику бота или интегрировать нейронные сети (15-20 минут). | Ivan Zamesin |

## Обсуждаемые гипотезы и идеи
- **[Оценка агентов на найме сотрудников]**: Рассматривалась идея использовать задачи по найму (анализ кандидата, вакансии, тестового задания) для демонстрации работы агентов. **Статус**: Отклонена как основная из-за сложности построения коммуникации между агентами на таких "ресёрч-задачах".
- **[Полностью автоматическая расшифровка аудио/видео на устройстве]**: Идея использовать систему, автоматически расшифровывающую весь аудио/видео контент на устройстве. **Статус**: Отклонена из-за сложности реализации ("геморройная штука", необходимость автозапуска).
- **[Отдельная команда агентов для вопросов]**: Идея создания отдельной команды, которая сначала задаёт вопросы по задаче, а потом вызывает команду для реализации. **Статус**: Отклонена в пользу интеграции функции вопросов в существующий плагин для упрощения.
- **[Добавление задачи по улучшению кода в воркшоп]**: Обсуждалась возможность включить задачи по улучшению кода в воркшоп, но было решено сосредоточиться только на PRD. **Статус**: Отклонена.
- **[Начало воркшопа с мозгового штурма экспертной команды]**: Идея начать воркшоп с брейншторма команды экспертов. **Статус**: Обсуждена, но отложена ("Не, пофиг", "подумаю ещё").

## Технические и продуктовые требования
- **Базовый проект воркшопа**: Telegram-бот "Нанобанан" с AI и фоточками.
- **Скиллы для бота (изучение/использование)**:
    - Генерация картинок.
    - Чат-бот функциональность.
    - Обработка аудио/видео (конкретно: скрипт для расшифровки аудиофайла и создания саммари).
- **Интеграции**:
    - Использование нейронов через Open Router.
    - Интеграция Gemini для работы с изображениями и текстом в чат-боте.
- **Agent Teams функциональность**:
    - Умение студентов создавать и использовать Agent Teams.
    - Четкое объяснение, когда, в каких ситуациях и для каких задач полезны Agent Teams.
    - Способность Agent Teams анализировать и дорабатывать продуктовые требования (PRD).
    - Добавление скилла "задавать вопросы по задаче для её детализации" в плагин реализации (Team Feature).
