# [Продуктовый-созвон] Разработка продающего воркшопа по веб-кодингу
**Дата**: 16 февраля 2026

---

## Контекст
Речь шла о концепции "продающего воркшопа по веб-кодингу", который является первым шагом к адаптации основного курса "Как делать продукт" под современные реалии с использованием нейросетей и веб-кодинга. Цель воркшопа — вовлечь студентов в веб-кодинг, показать его преимущества и продать им последующие модули веб-кодинга. Воркшоп рассчитан на 100+ студентов (до 140) и должен длиться 2.5 часа.

## Обсуждаемые гипотезы и идеи
- **Ужать два субботних воркшопа в один 2.5-часовой:** Суть - дать материал двух текущих воркшопов за 2.5 часа, убрав всё лишнее и оставив написание PRD по инструкции, лендинг и MVP.
    - Статус: **Требует проверки** (зависит от числа готовых студентов и проблем с кураторами/установкой).
- **Требовать установку инструментов до воркшопа:** Суть - допускать к воркшопу только тех, кто заранее установил Cloud Code.
    - Статус: **Требует проверки** (было предложено как решение проблемы с установкой, но не принято как окончательное решение).
- **Интегрировать полноценный "буст" (модуль) в основную программу курса:** Суть - вместо отдельного продающего воркшопа, сделать веб-кодинг полноценной частью основного курса.
    - Статус: **Отклонена** (Иван не уверен в релевантности для всех, считает текущий формат одной субботы крутым).
- **Разработать ассистента по MVP:** Суть - создать ассистента на базе нейросетей для помощи в создании MVP, встраиваемого в workflow (возможно, в Cloud Code/курсор). Илья уже начинал прототип.
    - Статус: **Идея на будущее** (долгосрочная продуктовая стратегия).
- **Пересобрать курс "Как делать продукт" к 60-му потоку:** Суть - адаптировать курс под современные реалии, используя нейросети (возможно, через собственного ассистента) для всех процессов. Добавить веб-кодинг как актуальное направление.
    - Статус: **Принята как долгосрочная стратегия** (будущее развитие курса).

## Технические и продуктовые требования
- **Длительность воркшопа:** 2.5 часа.
- **Целевая аудитория:** 100+ студентов (потенциально до 140 человек).
- **Основной инструмент для кодинга:** Cloud Code, подписка стоимостью $100.
- **Требуемый кураторский состав:** Для интерактивного формата (70-90 студентов) необходимо минимум 7 кураторов (соотношение 1 куратор на 10-12 студентов).
    - Текущий потенциальный состав: Артём, Максим, Женя, Миша, Илья + 2 человека, которые "подтянулись".
- **Содержание воркшопа (для интерактивного формата):** Написание PRD по инструкции, создание лендинга, написание MVP.

## Приоритеты
1.  **Сбор данных о готовности студентов (установка Cloud Code и оплата подписки):** Критический фактор для определения формата и объема воркшопа.
2.  **Решение проблемы с конфликтом Cloud Code/курсора:** Без устранения этого конфликта невозможно проведение практического воркшопа.
