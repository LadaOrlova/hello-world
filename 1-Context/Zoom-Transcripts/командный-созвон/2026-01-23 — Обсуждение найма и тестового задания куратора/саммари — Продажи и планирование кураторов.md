## Контекст
На встрече обсуждались продажи образовательных курсов/интенсивов, планирование и найм кураторов для них, а также обновления и дальнейшая разработка AI-продукта "Aura" (инструмента для продуктового анализа). Дополнительно затрагивались вопросы внутренней организации и онбординга новых сотрудников.

## Ключевые решения

| Решение | Обоснование | Кто принял |
|---|---|---|
| На первую практику (2-3 февраля) взять максимальное количество кураторов. | Большинство студентов приходит на первую практику; это обеспечит адекватную нагрузку и качественную обратную связь. | Алёна Минс (предложила), Иван Замесин (согласился) |
| Количество кураторов будет рассчитываться на основе числа команд на воркшопе и целевой нагрузки на куратора. | Обеспечение качественной обратной связи для всех команд; при необходимости число кураторов будет сокращено. | Иван Замесин |
| Отказ от скидок на раннюю запись для интенсива. | Необходимость учиться справедливо оценивать продукты, не недооценивая их. | Иван Замесин |
| Восстановить видимость скрытых кнопок в интерфейсе Aura. | Виральность и удобство использования важнее эстетики, особенно для ключевых функций (например, "Сделать публичным"). | Иван Замесин |
| Сделать обязательным указание рынка при запуске анализа в Aura. | Предотвращение неверных отчетов, так как нейронные сети базово настроены на глобальные рынки/США, а большинство клиентов из РФ. | Иван Замесин |
| Разделение ответственности в продажах между Настей, Русланом и Алиной. | Оптимизация работы с клиентами, разделение по типам запросов (оплаты, вопросы по курсу, новые клиенты). | Алёна Минс (организовала), команда продаж |
| Кураторы будут доступны на всех тарифах интенсива/курса, но с разной степенью вовлеченности. | Обеспечение поддержки всем студентам для предотвращения разочарований и получения результата; на тарифах с поддержкой — более глубокая персональная работа. | Иван Замесин |
| Ввести ежедневные "дейлики" (обновления) с контекстом для команды. | Высокая скорость изменений приводит к потере контекста командой относительно продуктов, изменений и принятых решений. | Иван Замесин |
| Срочно заняться наймом кураторов на интенсив с целевой нагрузкой не более 6 человек на куратора. | Необходимость обеспечить достаточное количество кураторов для поддержания качества обучения. | Иван Замесин |

## Технические и продуктовые требования
*   **Aura**:
    *   Унификация кнопок (однотипные, единое число, один язык).
    *   Отображение версии продукта, на которую опирается анализ.
    *   Важные кнопки (например, "Сделать публичным") должны быть видимыми, не скрытыми.
    *   Блокировка запуска анализа при отсутствии указанного рынка (для РФ клиентов).
    *   Отображение логики расчета, источников и степени уверенности для всех числовых показателей.
    *   Явное указание, что включает в себя расчет COGS.
    *   Доработка подробной оценки привлекательности: рынок, value, экономика, конкуренция (moderate/red/blue ocean), advantage (от компетенций команды и активов).
    *   Кнопка "мероприятия" должна быть поднята выше в интерфейсе.
    *   Исправление проблем с вёрсткой на продакшене.
    *   Возвращение пропавшей кнопки редактирования на продакшене.
    *   Возможность редактирования контента прямо из интерфейса для пользователей с правами, без захода в админку.
    *   Переработка чатика: раздельный диалог с нейронкой и профиль продукта.
    *   Поддержка мобильной версии.
*   **Loom**: Необходим платный доступ для Валерии для записи качественных видео с возможностью редактирования.
*   **CRM и формы**:
    *   Все ответы должны корректно попадать в CRM.
    *   Обновление форм.
    *   Устранение некорректно настроенных ограничений доступа внутри таблиц CRM.
*   **Гайды для студентов**: Отправка гайда по Figma (сегодня), гайда по Zoom (в понедельник).
*   **Материал с содержанием курса**: Добавление в личный кабинет студентам после проверки актуальности.
*   **Личный кабинет (ЛК)**: Удаление из ЛК неактуальных материалов, которые не будут записываться.
