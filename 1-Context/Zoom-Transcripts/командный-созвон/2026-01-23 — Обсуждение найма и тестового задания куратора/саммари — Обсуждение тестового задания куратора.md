## Обсуждаемые гипотезы и идеи
- **[Воронка найма]**: Идея использовать роль куратора как воронку для найма джуниор-продуктов и других специалистов, так как замотивированные джуниоры готовы тратить много времени на задачи для получения опыта. **Статус**: Принята как потенциальная выгода, но не основная цель найма кураторов.
- **[Опциональность задачи по гайду]**: Предложение сделать написание гайда необязательным тестовым заданием для снижения нагрузки на кандидатов. **Статус**: Отклонена. Задача по гайду признана обязательной для оценки мышления кандидата в обучении.
- **[Специфика тестового задания]**: Обсуждалась идея создать более специфичное тестовое задание для `code-code`, например, написать скилл для UX-теста или прожарки лендинга. **Статус**: Принята (объединена с задачей по гайду — написание гайда по созданию скилла).

## Технические и продуктовые требования
- Способность использовать AI-агентов и субагентов для выполнения комплексных задач (например, генерации вариаций лендингов).
- Опыт работы с `code-code` (Claud/Opus), предпочтительно через CLI или плагин (похожий на Cursor).
- Умение писать чёткие, понятные и структурированные гайды для пользователей, не имеющих предварительных знаний.
- Развитые коммуникативные навыки: адекватность, комфортность в общении, способность эмпатийно объяснять материал.
- Готовность тратить около 25 часов на поток (включая воркшопы и подготовку).

## Приоритеты
- **Быстрый набор 5 кураторов** на первый поток. *Обоснование*: Существует ограниченное время на проверку гипотез и формирование команды.
- **Привлечение специалистов, работающих с `code-code`**. *Обоснование*: Цель — обучение на наиболее мощном и экономичном инструменте.
