# [Воркшоп] Скилы и агенты: отличия и применение
**Дата**: 21 февраля 2026

---

## Автоматизация разработки и создания AI-агентов с Claude/Cursor

Воркшоп был посвящён углублённому изучению возможностей Claude AI, его расширения Cursor, а также использованию AI-агентов и команд агентов для автоматизации разработки, создания ботов и генерации идей. Особое внимание уделялось работе с внешними сервисами через Open Router и интеграции с Telegram.

## Ключевые концепции

### 1. Skills vs. Agents
**Skills (навыки)** – это базовые, заранее определённые функции, которые Claude может использовать для выполнения задач. Их нельзя вызвать напрямую в большом количестве или в параллельном режиме. Они вызываются через слэш (`/`). Их суть в том, что они являются неотъемлемой частью функционала Claude.
**Agents (агенты)** – это более продвинутые "субагенты" с конкретными инструкциями (промптами) и собственным контекстным окном. Их можно вызывать много параллельно для выполнения однотипных или разных задач (например, 10 агентов-ревьюеров). Агенты не вызываются через слэш. Файлы агентов, помимо `name` и `description`, обязательно содержат `tools` (инструменты для взаимодействия с системой, такие как `read`, `bash`, `write`), `model` (используемая нейросеть, например, Sonnet, Opus, Haiku) и опционально `max turns` (ограничение на количество итераций).

### 2. Плагины
**Плагины** – это наборы настроек, которые можно легко установить из открытых источников, в частности с GitHub, через "marketplace" в расширении Cursor. Плагин может содержать скилы, агенты, MCP (Module Connection Points) и хуки.

### 3. MCP (Module Connection Points)
**MCP** – это "ручки" или интерфейсы, позволяющие Claude взаимодействовать с внешними сервисами и приложениями, такими как Telegram, Slack, Figma, Notion, Jira, Chrome, Google Sheets и др. MCP настраиваются индивидуально под каждое приложение и могут быть сложны в настройке, требуя дополнительной установки Code CLI. В рамках воркшопа эта тема лишь затрагивалась, но не разбиралась детально, однако для некоторых популярных сервисов (Telegram, Google Sheets, Figma, Chrome) существуют готовые гайды.

### 4. Agent Teams (Команды агентов)
**Agent Teams** – это команды субагентов, которые могут общаться между собой, значительно повышая автономность работы AI. В отличие от просто параллельно запущенных субагентов, команда агентов работает как единый механизм для достижения общей цели. Например, агент-кодер может передать свой код агенту-ревьюеру для проверки, и они будут взаимодействовать до тех пор, пока задача не будет выполнена качественно. При этом основное окно чата (с "тимлидом") не блокируется, позволяя пользователю параллельно взаимодействовать с Claude и давать дополнительные инструкции.

### 5. Обучение через "Wipe-coding"
**Wipe-coding** – это метод обучения, при котором вы просите Claude не просто выполнить задачу, но и объяснить, как именно он это сделал, как работает сгенерированный им код или реализованная логика. Это позволяет участникам глубже понять внутренние механизмы работы AI, набраться опыта и знаний для более эффективного выполнения аналогичных задач в будущем.

## Фреймворки и инструменты
-   **Claude (через расширение Cursor)**: Основная платформа для взаимодействия с AI, создания и управления агентами и скилами.
-   **Open Router**: Сервис, позволяющий подключать практически любые нейросети (Google Gemini, OpenAI, Anthropic и др.) через единый API. Упрощает переключение между моделями, тестирование и управление ключами. Взимает 5% комиссии при пополнении баланса, но цены за запросы соответствуют ценам провайдеров.
    -   **API keys**: Доступны в разделе "Activity" -> "API keys" после регистрации.
    -   **Models**: На странице Open Router есть каталог моделей с фильтрами по входным/выходным модальностям (текст, изображение) и сортировкой по популярности. Рекомендуемые текстовые модели: Google Gemini 2.5/3.0/3 Flash Preview. Для изображений: NanoBanana.
-   **BotFather (Telegram)**: Официальный бот Telegram для создания новых ботов и получения их токенов (ключей API).
-   **Clod MD**: Специальный файл для определения правил проекта и конфигураций, которые Claude будет использовать. Например, правила для отключения проверки сертификатов или убийства предыдущих процессов для избежания конфликтов.
-   **`feature dev`**: Один из скилов, используемый для разработки новых функций.
-   **Tools для агентов**: `read` (чтение файлов), `bash` (взаимодействие с системой), `write` (запись в файлы).

## Практические задания
1.  **Создание правил проекта (`Clod MD`)**: Задание нулевое, создание файла с заранее подготовленными правилами для предотвращения распространённых ошибок.
2.  **Создание Telegram-бота поддержки**:
    *   Создание простого бота поддержки для "Золотого яблока", отвечающего на вопросы на основе предоставленной базы данных Q&A (`Gold Apple info`).
    *   Получение токена бота через BotFather.
    *   Интеграция бота с AI-моделью (Gemini 3 Flash Preview) через Open Router.
3.  **Изучение работы бота через Claude**: Запрос у Claude объяснения, как работает созданный Telegram-бот (концепция системного промпта, пошаговая логика).
4.  **Доработка бота с функцией генерации изображений**:
    *   Расширение функционала бота для классификации запросов пользователя (вопрос или генерация картинки) и выполнения соответствующего действия.
    *   Использование моделей для генерации изображений (например, NanoBanana) через Open Router.
5.  **Использование Agent Teams для генерации идей с валидацией**:
    *   Создание команды агентов, где один агент генерирует идеи, а два других спорят, взвешивают плюсы и минусы, чтобы прийти к консенсусу по поводу жизнеспособности идеи.
6.  **Индивидуальная работа с Agent Teams**: Запуск команды агентов на своём собственном продукте (PRD) для генерации идей по его улучшению.
7.  **Изучение готовых команд агентов**: Опциональное задание по изучению команд `Team Feature` и `Expert Arena`, созданных спикером, чтобы понять их функционал и протестировать работу.

## Ключевые цитаты
> "Агенты же в свою очередь это субагенты с конкретной инструкцией промтом. В этом как раз и заключается основная разница, а они получается всегда вызываются как отдельные агенты, а со своим контекстным окном. Их можно вызывать много параллельно."
>
> "MCP - это как бы ручки для Клода, чтобы он мог работать с внешними сервисами, с ними как-то взаимодействовать."
>
> "Команды агентов, Agent Teams - это, собственно, команды субагентов, которые могут общаться между собой. А, в этом основная разница между командой агентов и просто субагентов, которых мы запустили параллельно."
>
> "Если вы не знаете, что ставить, и хотите работать с текстом, ставьте [Gemini, Google Gemini 2.5 Flash, и 3.0 Flash, 3 Flash Preview]. Если не знаете, не знаете, какую модель для обработки картинок ставить, ставьте три NanoBanana."
>
> "Когда мы делаем одно и то же раз за разом, если мы обучаемся, с каждым разом мы мы делаем это быстрее и лучше за счёт того, что мы правильно ставим задачу и можем хорошо и быстро оценить результат."
>
> "Представь приёмную с двумя кабинетами. Кабинет один - специалист поддержки, кабинет два - художник, рисует картинки. Раньше все шли в кабинет один, теперь на ходе сидит администратор, который спрашивает: 'Вам на вопрос или картинку?' И направляет к нужным специалистам. А вот этот администратор тоже AI-модель, только быстрая и дешёвая." (Аналогия для классификации запросов в боте)
