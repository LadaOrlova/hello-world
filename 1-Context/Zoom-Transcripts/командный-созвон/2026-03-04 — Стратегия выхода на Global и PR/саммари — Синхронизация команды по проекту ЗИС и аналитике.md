# [Командный-созвон] Синхронизация команды по проекту ЗИС и аналитике
**Дата**: 4 марта 2026

---

Вот структурированное саммари по итогам созвона по проекту **ЗИС (Zamesin Information System)**.

## Краткое содержание
Первая синхронизация команды для выравнивания контекста по проекту ЗИС. В центре внимания — демонстрация логики прототипа (авторизация по OTP, онбординг, B2B-путь) и определение технического стека. Принято решение использовать 4-й поток курса «Boost» в качестве пилотного продукта для полной обкатки системы.

## Статусы и обновления
- **Интерфейс ЗИС (Женя):** Текущая версия является прототипом для оценки логики процессов, а не финальным дизайном.
- **Авторизация и ID (Женя):** Реализована система One-Time Password (OTP) через почту, которая является универсальным идентификатором пользователя.
- **Онбординг (Женя):** Включает выдачу LMS, создание страницы, календарь, назначение куратора и приветствие в Telegram.
- **B2B-логика (Женя):** Основана на договоре/оферте. Сотрудники компании становятся полноценными пользователями только после подтверждения оплаты.
- **Контент (Женя):** Для редактора LMS выбран формат Markdown.

## Ключевые решения
- **Пилотный запуск:** Первым продуктом для обкатки всей системы (продажи, LMS, бэкенд) станет 4-й поток курса «Boost» (Ваня).
- **Архитектурный контроль:** Максим и Виталий обладают правом решающего слова в вопросах инфраструктуры и структуры данных (Ваня).
- **Метод разработки:** Система строится под «вайпкодинг» (программирование с AI) с минимальным участием ведущих разработчиков в рутине (Ваня).
- **Ролевая модель:** Утверждено внедрение системы имперсонации (вход админа под видом пользователя) для техподдержки.
- **Управление:** Женя назначен ответственным менеджером для Максима и Виталия по задачам ЗИС.

## Блокеры и открытые вопросы
- **Техническая миграция:** Не определен механизм переноса данных из Airtable (важно для следующего этапа).
- **Интерфейс B2B:** Кто должен заполнять поля договора — владелец компании или менеджер? (Женя).
- **Шаблоны документов:** Требуется юридическое и дизайнерское согласование актов, счетов и логотипов.
- **Финансы:** Не проработана логика отображения внутренних рассрочек (платежи частями не через банк).
- **Функционал LMS:** Спор о «ревизиях» контента против полного дублирования потоков (нужно обсудить с Лерой и Аленой Т.).
- **Интеграция CRM:** Необходимы четкие требования от Руслана по составу и виду данных.
