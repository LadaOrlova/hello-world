## Обсуждаемые гипотезы и идеи
-   **[Идея] Привлечение супруги Спикера 3 для тестирования гайдов как неэксперта**: суть - использовать взгляд человека, не разбирающегося в теме, для выявления непонятных моментов в гайдах по установке ПО (Клод, VPN, оплата); статус - реализована (как личный подход Спикера 3).
-   **[Идея] Привлечение кураторов к созданию обучающих гайдов**: суть - кураторы заинтересованы и готовы создавать гайды, что может ускорить процесс и улучшить их качество; статус - требует проверки (поставлена на паузу в пользу подготовки программы).
-   **[Идея] Проведение вебинара в формате "реализация идеи участника в прямом эфире"**: суть - ведущий выбирает идею из предложенных аудиторией и за 2-2.5 часа с автором доводит ее до результата, в идеале до оплаты/регистрации; статус - принята (будет реализована Спикером 2).
-   **[Идея] Добавление "механик/плюшек" (бонусы) к продажам на вебинаре**: суть - предложить дополнительные лекции или подарки для стимулирования продаж интенсива/потока; статус - требует проверки/доработки (Спикер 2 предложил "подумать").
-   **[Идея] Создание ИИ-агента для написания статей в стиле Спикера 2 (Вани) для международных изданий**: суть - автоматизация процесса написания статей для пиара, используя стилистику Спикера 2; статус - принята (Спикер 2 будет реализовывать сам).
-   **[Идея] Переписать модуль расписаний в боте для точных уведомлений и переиспользования**: суть - создать универсальный и точный модуль для планирования и уведомлений по расписанию; статус - требует дальнейшего обсуждения/приоритезации с Спикером 3.

## Технические и продуктовые требования
-   **PRD-генератор**: Зафиксировать контекст продукта в верхнем табе. Реализовать последовательность взаимодействия: "Хочу сделать это" -> "Зачем?" -> "Накидывать идеи" -> "Выяснение деталей" -> "Критерии успеха" -> "Запуск анализа/детализация".
-   **База данных кураторов**: Занести задачи по базе данных в "Сор" до конца недели.
-   **Бот уведомлений**: Переписать логику уведомлений о мероприятиях для точной отправки за час (а не "примерно за час") путем создания нормального модуля расписаний.
-   **Задачи для Максима (Спикер 6)**: Помимо починки уведомлений, включают "расшифровку стрима самри", "скачать расшифровку", "импорт интервью в DOCX формате".
-   **Кейсы**: Обеспечить ясность и полноту кейсов, чтобы они были понятны для неспециалистов.

## Приоритеты
-   **Подготовка программы интенсива**: Высший приоритет до субботы, так как старт интенсивной программы запланирован на эти выходные.
-   **Архитектурные улучшения Aura (модуль расписаний)**: Высокий приоритет для Максима, чтобы использовать его компетенции для фундаментальных системных решений, а не мелких багов, что предотвратит будущие проблемы.
-   **Управление первыми двумя потоками "Буста" Спикером 2**: Приоритетно для сбора опыта и отработки процессов перед масштабированием и делегированием.
