## Глава 1: Главное про Jobs To Be Done

**URL**: [/book/about-jobs-to-be-done/](https://zamesin.ru/producthowto/book/about-jobs-to-be-done/)

### Кому полезна
- Слышали про JTBD и хотите разобраться
- Практикуете методологию и хотите углубиться
- Убеждены в важности "болей" клиентов и хотите узнать, что за ними стоит
- Ищете причинно-следственные связи в продуктовой работе

### Основные концепции

**Работа** = цель/задача человека. Описывается по формуле:
- **Когда** [контекст + опыт + эмоции + триггер]
- **Хочу** [результат с критериями]
- **Чтобы** [результат выше уровнем + изменить эмоции]

**Точка А** → **Точка Б**: Работа — переход из текущего состояния (задача не выполнена, негатив) в желаемое (задача выполнена, позитив).

**Consideration Set**: набор известных решений, из которых человек выбирает.

**Ключевые тезисы**:
- "Работа первична. Всё остальное производно от работ"
- "Клиенту Т-Банка не нужен счёт и приложение. Он хочет спать спокойно и переводить родителям деньги"
- Проблема — это следствие второго порядка (решение облажалось)
- Главная сегментация — по парам работа+решение
- 90% мотивации в B2B — личные работы людей, а не бизнес-работы

**Примеры работ**: Netflix, Яндекс Такси, AmoCRM, Самолёт, Грейд от Яндекс Практикума, Артдент, Биорг.Кадры.

**Контрибьюторы**: 8 человек (Краснова, Розка, Климова, Кубряк, @keyten, Лукерич, Хюнен, Голосова).

## Глава 2: Введение в Advanced Jobs To Be Done

**URL**: [/book/introduction-to-advanced-jobs-to-be-done/](https://zamesin.ru/producthowto/book/introduction-to-advanced-jobs-to-be-done/)

### Основные концепции

**Граф работ** — иерархия работ для удовлетворения потребностей. Вторая единица анализа AJTBD.

**Ценность** = более энергоэффективно выполнять работы клиентов. Научная основа: теория конструируемых эмоций Лизы Фельдман Баррет.

**Алгоритм создания ценности**:
1. Изучить графы работ фокусных сегментов
2. Генерировать гипотезы о механиках
3. Ранжировать по критичности

**50+ механик решения бизнес-задач**, включая:
- Починить проблемы
- Выполнять больше работ одним решением
- Убить работы (главная механика)
- Выйти на предыдущую/следующую работу
- Забрать работу с человека

**Стратегические примеры**:
- **Aviasales**: механика "выйти на предыдущую работу" — не продаёт билеты, а ищет самые дешёвые
- **Домклик**: решает предыдущие работы перед покупкой квартиры
- **LiFT**: "выйти на высокоуровневую работу" — всё под ключ для топ-менеджеров
- **Яндекс.Такси**: "сместиться в соседний сегмент" — от эконома к премиуму

**Правильный алгоритм решения бизнес-задач**:
1. Определиться с бизнес-задачей (ROI)
2. Экспертно предположить графы работ сегментов
3. Предположить механики с потенциалом
4. Провести исследование (кастдев, количественное)
5. Генерировать гипотезы о механиках

**AJTBD vs другие фреймворки**: CJM (граф = CJM на стероидах), ICP, WTP, AAARRR, юнит-экономика, UX.

**Границы**: не работает с аддиктивными продуктами, травматическим поведением, глубокой творческой мотивацией.

## Глава 3: Почему проблема это проблема

**URL**: [/book/what-is-a-problem/](https://zamesin.ru/producthowto/book/what-is-a-problem/)

### Кому полезна
- Создаёте ценность через "починку болей" и хотите узнать что стоит за "болью"
- Хотите лучше узнать базу создания продуктов

### Основные концепции

**Определение**: Проблема — ВСЕГДА следствие того, что на работу нанято решение, и оно облажалось.

**Что происходит при проблеме**:
1. **Эмоции как обратная связь** — негативные эмоции (злость, разочарование) сигнализируют: измени стратегию
2. **Просадка дофамина** — мозг снижает дофамин: "предсказание не подтвердилось"
3. **Налоговые работы** — вынужденные дополнительные задачи

**6 этапов в AJTBD**:
1. Есть работа → 2. Нанято решение → 3. Мозг предсказал граф → 4. Решение облажалось (негатив + просадка) → 5. Человек решает, выполнять ли "налоговые работы" → 6. Меньше вероятность повторного найма

**Проблема для бизнеса**: потеря клиента, негативные отзывы, рост CAC.

**Проблема как возможность**: сильная проблема у большого сегмента = рыночная возможность.

**Как правильно извиняться**: найти и выполнить исходную работу, а не дать бонус/промокод. Испортили кроссовки → купить такие же (не дать скидку).

**Критические последовательности работ**: чинить проблемы нужно в первую очередь в критических последовательностях — работах, которые человек не может не выполнить.

**Типичные ошибки**:
1. Не изучают сегменты
2. Не выбирают экономически привлекательные сегменты
3. Чинят проблемы неприбыльных клиентов
4. Чинят проблемы маленьких сегментов
5. Всё сводят к "починке болей"

## Глава 4: Невероятная сила привычки

**URL**: [/book/incredibly-strong-habit/](https://zamesin.ru/producthowto/book/incredibly-strong-habit/)

### Основные концепции

**Определение**: Привычка — эволюционный механизм закрепления повторяющегося поведения для экономии ресурсов мозга. Проявляется в физическом изменении структуры мозга (striatum).

**Мозг потребляет ~20% энергии** в покое. Привычки экономят: глюкозу, кислород, нейромедиаторы, когнитивные ресурсы.

**Менять привычки сложно**: по исследованию Келли Макгонигал, 74% уверены что привьют привычку, но только 51% это делают.

**Для продуктов**:
- **Положительно**: повышенная возвращаемость + снижение стоимости использования
- **Отрицательно**: привычка к конкурентам = барьер

**Главное правило**: "Переиспользуйте существующие привычки. Дайте людям то, к чему они привыкли, только лучше."

**Примеры**:
- Яндекс Музыка — импорт из Spotify (привычка к генеративным плейлистам)
- Stories в Yandex Go, Тинькофф, Сбербанк (привычка из Instagram)
- PitchMe — надстройка над ATS вместо замены

**Для редизайна**: "Не делайте одним большим обновлением". Кинопоиск (2015), Яндекс "Острова" (-5% выручки), ФНС (2023).

**Правильно**: тестировать UX → выкатывать экран за экраном → атомарно обновлять элементы.

**Для стартапов**: привитие привычки = долго, дорого, больно. Конкуренты могут переиспользовать ваши усилия. Для крупных компаний — единственный путь (Тинькофф Инвестиции, Amazon Kindle).

## Глава: OODA-цикл

**URL**: [/book/ooda-strategy-loop/](https://zamesin.ru/producthowto/book/ooda-strategy-loop/)

Четырёхэтапный процесс принятия решений: Observe → Orient → Decide → Act. Разработан полковником Джоном Бойдом.

**Observe**: построить точную картину ситуации, отсеять шум.

**Orient** (schwerpunkt): осознать свои искажения — культурные, генетические, аналитические. Метафора "снегохода" — комбинировать концепции из разных дисциплин.

**Decide**: выбрать вариант как гипотезу, быть готовым к обновлению.

**Act**: проверить решение, результаты = вход для следующего цикла.

**Принципы**: скорость, комфорт с неопределённостью (теоремы Гёделя, принцип Гейзенберга, 2-й закон термодинамики), непредсказуемость.
