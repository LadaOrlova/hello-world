# Полные правила формирования и визуализации графа работ AJTBD

Комплексный справочник для создания инструмента визуализации графов работ на основе методологии Advanced Jobs To Be Done.

---

## Часть 1: Базовая структура графа

### 1.1 Иерархия уровней работ

Граф работ имеет **фрактальную структуру** - одинаковый паттерн повторяется на всех уровнях:

```
Super Big Job (Сверхвысокоуровневая работа)
    ↓
Big Job (Высокоуровневая работа)
    ↓
Core Job (Ключевая работа)
    ↓
Small Job (Малая работа)
    ↓
Micro Job (Микроработа)
```

**Ключевые определения:**
- **Core Job** = работа, которую продукт выполняет ПОЛНОСТЬЮ, от начала до конца
- **Big Job** = работа выше уровня, в которую продукт вносит вклад, но НЕ выполняет её целиком
- **Small Job** = вспомогательная работа, которая является ЧАСТЬЮ выполнения Core Job

### 1.2 Компоненты работы (полная структура)

Каждая работа в графе содержит:

```
Контекст → Негативные эмоции → Активирующее знание → Триггер →
Ожидаемый результат (хочу + глагол) → Критерии успеха →
Позитивные эмоции → Работа выше уровня
```

| Компонент | Описание | Вопрос для изучения |
|-----------|----------|---------------------|
| **Контекст** | Особенности человека и ситуации | "В какой ситуации вы находились?" |
| **Триггер** | Событие, запустившее работу | "Что конкретно запустило действие?" |
| **Негативные эмоции** | Что беспокоит до получения результата | "Что тревожило, бесило, раздражало?" |
| **Ожидаемый результат** | Желаемое изменение (глагол неопр. формы) | "Какой результат хотели получить?" |
| **Критерии успеха** | Конкретные измеримые критерии | "По каким критериям оценивали успех?" |
| **Позитивные эмоции** | Что почувствовал после результата | "Что почувствовали, когда получилось?" |
| **Работа выше уровня** | Зачем нужен результат | "Зачем вам это нужно было?" |

### 1.3 Правила записи работ

**Правильная формулировка работы:**
- Глагол неопределённой формы: "хочу доехать", "хочу понимать"
- Смотрит из настоящего в будущее
- Каждый глагол = отдельная работа

**НЕправильно:** "Я хочу понять" (разовый результат)
**Правильно:** "Я хочу понимать" (состояние в будущем)

**Критерии успеха - конкретные:**
- НЕ: "быстро", "качественно", "надёжно" (абстракции)
- ДА: "За 15 минут", "Без ДТП", "Чисто в салоне"

## Часть 2: Типы работ для визуализации

### 2.1 Классификация по отношению к продукту

| Тип | Определение | Визуализация |
|-----|-------------|--------------|
| **Core Job** | Продукт выполняет полностью | Центральный узел, основной цвет |
| **Big Job** | Работа выше уровня (мотивация) | Узел выше, другой цвет |
| **Small Job** | Подзадача Core Job | Узел ниже, менее насыщенный |
| **Previous Job** | Работа ДО Core Job | Узел слева |
| **Next Job** | Работа ПОСЛЕ Core Job | Узел справа |

### 2.2 Классификация по повторяемости

#### Частотные работы (Frequent Jobs)
- Выполняются регулярно, циклически
- Не имеют чёткой последовательности
- Образуют "облако" работ

**Визуализация:** Группа узлов без направленных связей между собой

**Вопрос для выявления:** "Какие подзадачи вы решаете обычно, регулярно?"

#### Последовательные работы (Sequential Jobs) / Критическая цепочка
- Выполняются в определённом порядке
- Каждая следующая зависит от предыдущей
- Образуют цепочку

**Визуализация:** Узлы, соединённые направленными стрелками

**Вопрос для выявления:** "Расскажите по шагам, что делали?"

### 2.3 Специальные типы работ

#### Налоговые работы (Tax Jobs)
**Определение:** Работы, которые человек НЕ ХОЧЕТ делать, но ВЫНУЖДЕН для достижения Big Job.

**Признаки в речи:**
- "Пришлось..."
- "Вынужден был..."
- "К сожалению, надо..."
- "Бесит, что надо..."

**Визуализация:** Особый цвет (красный) или штриховка

**Источники налоговых работ:**
1. Законодательные требования
2. Корпоративные правила
3. Технические ограничения продуктов
4. Проблемы в процессах

**Примеры:**
- Распутывать наушники (AirPods убили эту работу)
- Заполнять налоговую декларацию
- Ожидать в очереди

#### Ориентационные работы (Orientation Jobs)
**Определение:** Работы для понимания "как это вообще делается", выполняемые в начале пути.

**Признаки:**
- Глаголы: понять, узнать, найти, выбрать, разобраться
- Выполняются новичками
- Точка входа в граф

**Визуализация:** Пометить как "входные точки" графа, особый цвет (жёлтый)

**Примеры:**
- "Гуглит, как переехать в другую страну"
- "Разобраться, что Figma умеет"
- "Узнать, какие документы нужны"

#### Виральные работы (Viral Jobs)
**Определение:** Работы, которые невозможно выполнить в одиночку - требуют других людей.

**Визуализация:** Узлы с пометкой о количестве участников, связи к другим акторам

**Примеры:**
- "Перевести деньги другу"
- "Согласовать документ с коллегой"
- "Провести созвон с командой"
- "Демонстрировать слайды" (виральность Figma)

## Часть 3: Критическая цепочка (Critical Chain)

### 3.1 Определение

**Критическая цепочка** — последовательность работ, которые ОБЯЗАТЕЛЬНО должны быть выполнены для достижения Big Job.

**Главное правило:** Критическая цепочка НЕ ДОЛЖНА РВАТЬСЯ. Если хотя бы одна работа в цепочке не выполнена, результат Big Job НЕ достигается.

**Термин:** Заимствован у Элияху Голдратта ("Критическая цепь").

### 3.2 Отличие от общего графа

| Характеристика | Граф работ | Критическая цепочка |
|----------------|-----------|---------------------|
| Структура | Сеть (нелинейная) | Последовательность (линейная) |
| Обязательность | Работы могут быть опциональными | Все работы обязательны |
| Пропуск работы | Возможен | Разрывает цепочку |
| Результат пропуска | Снижение эффективности | Полный провал Big Job |

### 3.3 Визуализация критической цепочки

**Рекомендации:**
1. Выделить критический путь жирной/красной линией
2. Пронумеровать шаги цепочки
3. Отметить точки потенциального разрыва
4. Показать bottleneck (узкие места)

**Пример разрыва:**
```
Переезд из Сербии в Португалию:
[Упаковать] → [Погрузить] → [Перевезти] → [Выгрузить] → [ПОДНЯТЬ НА ЭТАЖ] ← РАЗРЫВ!

Последнюю работу не выполнили → весь результат негативный
```

**Пример телемедицины:**
```
[Заполнить форму] → [Оплатить приём] → [ПОДКЛЮЧИТЬСЯ] ← РАЗРЫВ!

Подключение не работало → пациент пойдёт в обычную клинику
```

### 3.4 Запуск критической цепочки

**Формула активации:**
```
Push (толчок от проблемы) + Pull (притяжение к решению)
    >
Habit (привычка) + Anxiety (страх нового)
```

**Pressure = Push + Pull** (силы к изменению)
**Gravity = Habit + Anxiety** (силы против изменения)

Критическая цепочка запускается когда **Pressure > Gravity**

### 3.5 Правила работы с критической цепочкой

1. **Не допускать разрывов** - каждая работа должна быть выполнена
2. **Убивать циклы** - возвраты на доработку создают риск разрыва
3. **Минимизировать роли** - меньше участников = меньше точек разрыва
4. **Сокращать задержки** - время между работами = риск
5. **Снижать вариативность** - предсказуемый результат каждой работы
6. **Скорость цепи = скорость самого медленного звена**

### 3.6 Что искать в критической цепочке

| Что искать | Зачем | Визуализация |
|------------|-------|--------------|
| **Точки разрыва** | Где цепочка может порваться | Красные метки |
| **Bottleneck** | Где цепочка тормозится | Предупреждающие метки |
| **Циклы** | Возвраты на доработку | Обратные стрелки |
| **Скрытые роли** | Неизвестные участники | Дополнительные акторы |
| **Работы без ответственных** | Никто не отвечает | Пунктирные узлы |
| **Налоговые работы** | Кандидаты на убийство | Красный цвет |

## Часть 4: Правила формирования связей

### 4.1 Вопрос "Зачем?" (движение вверх по графу)

**Вопрос:** "Зачем вам ожидаемый результат этой работы?"

```
Core Job: "Записать лекцию"
    ↓ "Зачем?"
Big Job: "Обучить студентов"
    ↓ "Зачем?"
Super Big Job: "Получить признание как эксперт"
```

**"Чтобы что?" - главный инструмент восстановления графа**

### 4.2 Движение по горизонтали (Previous/Next Jobs)

**Предыдущие работы:**
"Какие задачи вы выполняли ДО этой работы для Big Job?"

**Следующие работы:**
"Какие задачи вы выполняете ПОСЛЕ этой работы для Big Job?"

**КРИТИЧЕСКИ ВАЖНО:** Вопрос обязательно задавать ЧЕРЕЗ Big Job! Если не будете задавать через Big Job - всё сломается.

**Пример:**
```
"Купить квартиру, чтобы что? → Чтобы жить в своей квартире."
"Что вы делали для этого ДО покупки? → Выбирал квартиру, копил на первый взнос."
```

### 4.3 Движение вниз (Micro Jobs)

**Для частотных работ:**
"Какие подзадачи вы решаете обычно, регулярно для достижения результата?"

**Для последовательных работ:**
"Расскажите по шагам, какие задачи выполняли для получения результата?"

### 4.4 Определение правильного уровня

**Проверочные вопросы:**
1. "Мы эту работу полностью выполняем?" → Если нет, это Big Job
2. "Чтобы что?" → Если ответ ведёт к другой работе, проверить уровень
3. Если слишком высокий уровень → спуститься ниже
4. Если слишком низкий уровень → подняться выше

**Частая ошибка:** Пропустить один или несколько уровней в графе

## Часть 5: Свойства графа для визуализации

### 5.1 Эмоции в графе

**Правило:** Чем ВЫШЕ работа по графу, тем СИЛЬНЕЕ эмоции.

**Объяснение:** Эмоции — сигнальная система мозга о том, как выполняются работы. Чем ближе к базовым потребностям (верх графа), тем интенсивнее эмоции.

**Визуализация:**
- Верхние уровни: насыщенные цвета, больший размер узлов
- Нижние уровни: менее насыщенные, меньший размер

### 5.2 Детализация и особенности продукта

**Правило:** Чем НИЖЕ по графу, тем БОЛЬШЕ деталей продукта в описании работ.

**Примеры:**
- Высокий уровень: "Записать лекцию" (продукт не упоминается)
- Низкий уровень: "Настроить источники камеры в OBS" (продукт в описании)

### 5.3 Стабильность работ

**Правило:**
- Высокоуровневые работы НЕ МЕНЯЮТСЯ десятилетиями/столетиями
- Низкоуровневые работы БЫСТРО устаревают и исчезают

**Пример:** "Хочу свой безопасный дом" — работа не менялась тысячелетиями. Но "настроить умный дом" — работа появилась недавно и изменится.

### 5.4 Фрактальность

Структура **Job → Value → Barrier** повторяется на ВСЕХ уровнях:
- Super Big Job имеет свои барьеры и ценность
- Core Job имеет свои барьеры и ценность
- Micro Job имеет свои барьеры и ценность

## Часть 6: Связи с множественными Big Jobs

### 6.1 Одна работа — много целей

**Правило:** Одна Core Job может служить НЕСКОЛЬКИМ Big Jobs.

**Визуализация:** Узел Core Job имеет несколько исходящих связей вверх

**Пример:**
```
Core Job: "Пройти курс по AJTBD"
    ├── Big Job 1: "Стать лучшим продактом"
    ├── Big Job 2: "Вырасти по карьере"
    ├── Big Job 3: "Реализовать свой проект"
    └── Big Job 4: "Помочь другу с его продуктом"

Чем больше Big Jobs, тем выше мотивация клиента
```

### 6.2 Одно решение — много работ

**Правило:** Одно решение (продукт) может выполнять разные работы:
- В одном человеке (разный контекст) → рост LTV, возвращаемости, среднего чека
- В разных людях (разные сегменты) → масштабирование продукта

**Пример Яндекс.Такси:**
- Эконом: быстро и недорого А→Б
- Комфорт: комфортно, безопасно
- Ультима: статусно, максимально комфортно

### 6.3 Появление работ в процессе

**Правило:** В процессе выполнения работы могут ПОЯВЛЯТЬСЯ дополнительные работы от других потребностей.

**Пример:**
```
Исходный граф: "Продать машину на Авито"
    ↓ [Появился покупатель с деньгами]
Появилась работа: "Обезопасить себя от мошенников"
    ↓
Добавились работы:
    - "Узнать способы обмана"
    - "Проверить покупателя"
    - "Правильно оформить сделку"
```

### 6.4 Смена работ по опыту

**Правило:** Один человек с одним продуктом выполняет РАЗНЫЕ работы в зависимости от опыта.

**Пример Figma:**

Новичок:
- Разобраться, что Figma умеет
- Сделать первый экран

Опытный:
- Рисовать макеты
- Получать фидбек
- Передавать в разработку

## Часть 7: B2B - множественные цепочки

### 7.1 Личные + бизнес работы

**Правило:** В B2B у каждого ЛПР есть ПАРА работ:
- **Бизнес-работа:** цели от руководства
- **Личная работа:** зачем это лично мне

**Визуализация:** Два параллельных слоя графа

**Типичные личные работы ЛПР:**
- Job security (сохранить работу)
- Рост по деньгам, премия
- Признание от команды
- Признание от руководителя
- Карьерный рост

**Важно:** Личные работы — это мотивация выполнять бизнес-работы!

### 7.2 Цепочка продажи B2B

**Участники:**
- Бизнес-заказчик
- ЛПР (лицо принимающее решение)
- Лица, влияющие на решение
- Конечные пользователи

**Правило:** Цепочка должна быть выполнена для ВСЕХ участников — пока не выполнены работы всех (личные + бизнес), покупки не будет.

### 7.3 B2B2C (два слоя цепочек)

**Правило "Follow the money":** При прочих равных начинайте с тех, кто платит деньги (конечный покупатель).

**Исключение:** Если спрос сильно превышает предложение — можно начать с посредников.

## Часть 8: Алгоритм построения графа

### 8.1 Быстрое прокапывание (5-10 минут)

1. **Найти Core Jobs:** "Какие задачи вы решаете с помощью [решения]?"
2. **Найти Big Jobs:** "Зачем вам результат этих Core Jobs?"
3. **Найти предыдущие работы (через Big Job!):** "Что делали ДО Core Job для Big Job?"
4. **Найти следующие работы (через Big Job!):** "Что делаете ПОСЛЕ Core Job для Big Job?"
5. **Найти работы ниже:** частотные или последовательные вопросы

### 8.2 Детальное прокапывание работы

Для каждой важной работы:
1. Контекст (внутренний и внешний)
2. Триггер (что запустило)
3. Негативные эмоции (до/во время)
4. Ожидаемый результат
5. Критерии успеха (конкретные, измеримые)
6. Позитивные эмоции (после получения результата)

### 8.3 Проверка правильности уровня

- [ ] Core Job — продукт выполняет ПОЛНОСТЬЮ?
- [ ] Big Job — работа ВЫШЕ уровня (продукт не выполняет целиком)?
- [ ] Нет пропущенных уровней?
- [ ] Вопрос "Чтобы что?" восстанавливает граф?

## Часть 9: Формы записи графа

### 9.1 Компактная форма (одна карточка)

```
Контекст → Триггер → [Core Job] → Критерии успеха
                ↑
              Big Job
                ↓
         [Previous] → [Core] → [Next]
                         ↓
                    [Small Jobs]
```

### 9.2 Форма отдельных карточек

Каждая работа — отдельная карточка с полной структурой:
- Когда (контекст)
- Что случилось (триггер)
- Что чувствовал (негативные эмоции)
- Чего хотел (ожидаемый результат)
- По каким критериям (критерии успеха)
- Что почувствовал после (позитивные эмоции)
- Зачем это (работа выше уровня)

**Правило:** Записи должны совпадать — это одно и то же, просто разные формы визуализации.

## Часть 10: Правила визуализации для инструмента

### 10.1 Обязательные элементы

- [ ] Иерархия уровней (вертикаль)
- [ ] Последовательность работ (горизонталь: Previous → Core → Next)
- [ ] Критическая цепочка (выделена)
- [ ] Типы работ (цветовая кодировка)
- [ ] Направленные связи между работами (стрелки)

### 10.2 Рекомендуемые цвета

| Элемент | Рекомендуемый цвет |
|---------|-------------------|
| Core Job | Синий/Основной |
| Big Job | Фиолетовый |
| Small Job | Голубой (менее насыщенный) |
| Налоговая работа | Красный |
| Виральная работа | Зелёный |
| Ориентационная работа | Жёлтый |
| Критическая цепочка | Жирная красная линия |
| Точка разрыва | Красная метка с предупреждением |

### 10.3 Рекомендуемая интерактивность

1. Раскрытие/сворачивание уровней
2. Фильтрация по типам работ
3. Подсветка критической цепочки
4. Отображение деталей работы при клике
5. Поиск по работам
6. Возможность редактировать работы
7. Экспорт графа в разные форматы

### 10.4 Извлечение графа из интервью (маркеры в речи)

**Высокоуровневые работы:**
- "Чтобы..."
- "Для того, чтобы..."
- "Мне нужно было..."
- "Хотел..."

**Налоговые работы:**
- "Пришлось..."
- "Вынужден был..."
- "К сожалению, надо..."
- "Бесит, что надо..."

**Триггеры:**
- "Когда..."
- "После того как..."
- "Как только..."
- "В момент когда..."

**Связь "работа → работа выше":**
- "...чтобы потом..."
- "...для того, чтобы..."

**Связь "предыдущая → следующая":**
- "До этого я..."
- "После этого я..."
- "Следующий шаг..."

**Появление новой работы:**
- "И тут я понял, что надо ещё..."
- "Оказалось, что также нужно..."

## Часть 11: Неожиданные конкуренты через граф

**Правило:** Изучая работы ВЫШЕ уровня Core Job, находим реальных конкурентов.

**Пример матрасов:**
```
Core Job: "Купить матрас"
    ↓ "Зачем?"
Big Job: "Продолжать функционировать, несмотря на недосып"

Реальные конкуренты матраса:
- Кофе
- Снотворное
- Отказ от части работы
- Делегирование
```

**Результат:** Маркетинг-кампания "Матрас по цене чашки кофе в день" (рассрочка).

**Пример складского учёта:**
```
Продукт: Приложение для складского учёта

Реальные конкуренты:
- Excel
- Тетрадка завхоза
- WhatsApp чатик
```

## Глоссарий терминов

| Термин | Определение |
|--------|-------------|
| **Job (Работа)** | Путь человека из точки А в точку Б, цель которую он хочет достичь |
| **Core Job** | Самая высокоуровневая работа, которую продукт выполняет полностью |
| **Big Job** | Работа выше уровня Core Job (мотивация клиента) |
| **Small Job / Micro Job** | Подзадача для выполнения Core Job |
| **Previous Job** | Работа, выполняемая ДО Core Job |
| **Next Job** | Работа, выполняемая ПОСЛЕ Core Job |
| **Критическая цепочка** | Последовательность обязательных работ для Big Job |
| **Налоговая работа** | Нежеланная, но обязательная работа |
| **Ориентационная работа** | Работа для понимания "как это делается" |
| **Виральная работа** | Работа, требующая других людей |
| **Триггер** | Событие, запускающее работу |
| **Контекст** | Обстоятельства, в которых возникает работа |
| **Критерии успеха** | Конкретные измеримые критерии оценки результата |
| **Активирующее знание** | Информация, приводящая к выбору решения |
| **Ага-момент** | Момент осознания ценности продукта |
| **Push** | Сила, толкающая от текущей ситуации |
| **Pull** | Сила, притягивающая к новому решению |
| **Habit** | Привычка (сила против изменения) |
| **Anxiety** | Страх нового (сила против изменения) |
| **Pressure** | Push + Pull |
| **Gravity** | Habit + Anxiety |

## Ключевые принципы (резюме)

1. **Фрактальность** — структура повторяется на всех уровнях
2. **Core Job = точка отсчёта** — от неё движемся вверх, вниз, влево, вправо
3. **Вопросы через Big Job** — предыдущие/следующие работы спрашивать только через Big Job
4. **Критическая цепочка не должна рваться** — одна пропущенная работа = провал
5. **Циклы = главная угроза** — убирать возвраты на доработку
6. **Выше = больше эмоций** — чем ближе к потребностям, тем интенсивнее
7. **Ниже = больше деталей продукта** — технические детали появляются на нижних уровнях
8. **Высокоуровневые работы стабильны** — не меняются десятилетиями
9. **Низкоуровневые работы изменчивы** — быстро устаревают
10. **Одна работа — много Big Jobs** — мотивация растёт с количеством связей
