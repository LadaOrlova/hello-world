# Правила AJTBD для AI-инструмента: Сегменты и Графы работ

Этот документ содержит ключевые тезисы из методологии Advanced Jobs To Be Done для создания инструмента, который:
1. Находит и ранжирует сегменты
2. Строит графы работ
3. Применяет стратегии и механики над графами работ

---

## Часть 1: Поиск и определение сегментов

### Что такое сегмент

- **Сегмент = группа людей с похожими работами**: Сегмент — это группа клиентов, объединённых по признаку похожих наборов работ (похожих графов работ)
- **Разные работы = разные сегменты**: Если люди хотят выполнить работу с разными критериями успеха, это разные работы и разные сегменты
- **Один человек может быть в разных сегментах**: В разных контекстах один человек относится к разным сегментам (пример: "доехать до офиса" vs "доехать до аэропорта")

### Принципы сегментации

- **Сегментация по похожести графов работ**: Есть Big Job, есть набор Core Jobs. Все люди, у которых такой набор Core Jobs, объединяются в один сегмент
- **Сегментация ≠ демография**: Нельзя сегментировать по демографии или индустрии в первую очередь. Эти параметры — корреляты, следствия. Причина — работы
- **Работа должна быть первым критерием**: Признак похожего графа работ должен стать первым критерием сегментации. Последовательность критериев сегментации крайне важна
- **Демография как дополнительный критерий**: Демография может быть дополнительным критерием сегментации вторым, но не единственным
- **Непересекаемость**: В идеальной сегментации один клиент в один момент времени относится только к одному сегменту

### Критерии сегментации применяются последовательно

- Сначала первый критерий, потом второй, потом третий
- Разный порядок применения критериев = разная итоговая сегментация
- В каждом решении — неявная последовательность критериев

## Часть 2: Ранжирование и выбор сегментов

### Четыре главных фактора выбора сегмента

1. **Возможность создать более ценное решение** — можем ли создать бóльшую добавочную ценность
2. **Возможность получать целевую прибыль на юнит** — сходится ли юнит-экономика
3. **Возможность масштабировать сегмент** — достаточно ли большой сегмент, есть ли каналы привлечения
4. **Возможность не умереть от дополнительных рисков** — нет ли критических рисков

### Четыре силы как инструмент выбора сегмента

Нам в каждый момент времени выгоднее те работы тех людей, для которых:
- **(а) Можно создать большую добавочную ценность**
- **(б) У которых сильнее проблемы с текущим решением**
- **(в) Слабее привычка** (или мы можем переиспользовать существующую привычку)
- **(г) Меньше страхи** (или мы можем эти страхи снижать)

### Формула ранжирования (скоринг сегментов)

**Скор сегмента** — безразмерная величина привлекательности сегмента. Чем больше скор — тем привлекательнее сегмент.

Компоненты скора:
- **Частотность работы** — как часто выполняется работа
- **Важность работы** — насколько критична работа для человека
- **Размер сегмента** — количество людей в сегменте
- **Неудовлетворённость** — насколько плохо текущие решения выполняют работу
- **Готовность платить** — бюджет на выполнение работы

### Карты приоритизации сегментов

**Карта 1: Частотность vs Важность**
- Оси: частотность (X) и важность (Y)
- Размер точки = размер сегмента
- Цвет = удовлетворённость (красный = неудовлетворены)
- **Приоритетная зона**: справа вверху, большие красные точки

**Карта 2: Доход vs Важность**
- Оси: потенциальный доход (X) и важность (Y)
- Помогает оценить маржинальность сегментов

### Критерии отбрасывания сегментов

- **Не за все работы всех людей нужно конкурировать**
- **Критерий отбрасывания**: Для этих людей, выполняя эти работы, мы в принципе НЕ можем получать целевую маржу на юнит
- Отбрасываем сегменты, где четыре силы складываются не в нашу пользу

### Формула ранжирования рисков (RAT)

**Приоритет риска = (Вероятность наступления риска × Последствия риска в деньгах) / Стоимость проверки риска**

Наверх всплывают предположения с:
- Высокой вероятностью наступления риска
- Дорогими последствиями рисков
- Низкой стоимостью проверки

## Часть 3: Структура и построение графа работ

### Типы работ в графе (уровни)

1. **Big Job** — работа выше уровня Core Job, которую продукт НЕ выполняет полностью. Это мотивация клиента
2. **Core Job** — самые высокоуровневые работы, которые конкретное решение выполняет полностью
3. **Small Job / Micro Job** — вспомогательные работы, которые являются частью выполнения Core Job
4. **Previous Job** — работы, которые клиент выполняет ДО Core Job
5. **Next Job** — работы, которые клиент выполняет ПОСЛЕ Core Job

### Структура описания работы

Каждая работа имеет:
- **Когда (контекст)**: В какой ситуации возникает работа
- **Триггер**: Что запускает желание выполнить работу
- **Ожидаемый результат**: Чего человек хочет достичь
- **Критерии успеха**: По каким критериям человек оценивает успех
- **Негативные эмоции**: Что беспокоит, от чего хочет избавиться
- **Позитивные эмоции**: Что хочет почувствовать в результате

### Правила построения графа

#### Core Job — точка отсчёта
- Core Job — самые высокоуровневые работы, которые конкретное решение выполняет ПОЛНОСТЬЮ
- От Core Job ищем: наверх (Big Job), вниз (Micro Job), влево (Previous Job), вправо (Next Job)

#### Big Job — работа выше уровня
- Вопрос для определения: «Зачем вам ожидаемый результат Core Job?»
- Одна Core Job может вести к нескольким Big Jobs
- Big Job объясняет мотивацию клиента

#### Previous и Next Jobs — горизонтальное движение
- **Предыдущие работы**: «Какие задачи вы выполняли для Big Job ПЕРЕД Core Jobs?»
- **Следующие работы**: «Какие задачи вы выполняете для Big Job ПОСЛЕ Core Jobs?»
- **ВАЖНО**: Вопросы обязательно задавать через Big Job!

#### Micro Jobs — работы ниже уровня
- Если частотные: «Какие подзадачи вы решаете регулярно для выполнения Core Jobs?»
- Если последовательные: «Расскажите по шагам, какие задачи вы выполняли?»

### Критические цепочки работ

- **Определение**: Последовательность работ, которые нужно выполнить, чтобы достичь Big Job
- **Разрыв цепочки**: Если одна работа в цепочке не выполняется, вся цепочка рвётся
- **Циклы = главная угроза**: Если в цепочке есть цикл (возврат на доработку), риск разрыва кратно растёт
- **Скорость цепи = скорость самого медленного звена**

### Проверочные вопросы для графа

- **«Чтобы что?»** — главный инструмент восстановления графа
- Если продукт не выполняет работу полностью → это Big Job, нужно спуститься на уровень ниже
- Если работа слишком детальная → это Micro Job, нужно подняться на уровень выше

## Часть 4: Стратегии и механики над графами работ

### Главный принцип

**Стратегии создания ценности — это движение в графе:**
- Вверх (Big Job)
- Вниз (Micro Job)
- Влево (Previous Job)
- Вправо (Next Job)
- Больше работ
- Убить работу
- Дать работу за клиента

### 16+ механик создания ценности

1. **Начать выполнять работу** — выполнять работу, которая раньше не выполнялась или выполнялась плохо
2. **Починить проблемы** — чинить проблемы текущих решений конкурентов
3. **Забрать работу с человека** — человек делал сам, теперь продукт делает за него (done-for-you)
4. **Убить работу** — работы ниже уровня исчезают как класс
5. **Выйти на предыдущую работу** — лидогенерация, выход из конкуренции
6. **Выйти на следующую работу** — удержание, рост ценности
7. **Выполнять больше работ одним решением** — производная от предыдущих механик
8. **Создавать связи с новыми Big Jobs** — с работами выше уровня, с которыми раньше связи не было
9. **Избавить от негативных эмоций** — привести к позитивным
10. **Снижение костов** — делать дешевле выполнение работы
11. **Улучшить критерий успеха** — выполнять критерий лучше
12. **Дать ясность графа** — выполнять ориентационную работу (бесплатные уроки, консультации)
13. **Чинить разрывы в цепочке** — если работа в критической цепочке не выполняется
14. **Корректировать ожидания** — снизить завышенные ожидания до реальности
15. **Уменьшать промежутки времени** — быстрее давать результат
16. **Выполнять потребности напрямую** — статус, безопасность, автономия, контроль, любовь, признание

### Мощные стратегии

#### Стратегия "Подняться на уровень выше"
- Самая мощная механика создания ценности и выхода из конкуренции
- Убивает множество работ ниже уровня
- Пример: Яндекс.Аренда vs ЦИАН — "дай ключи, мы всё сделаем" vs "разместить объявление"

#### Стратегия нишевания
- Конкурировать за все работы сегмента, но делать лучше универсальных решений
- Расти внутри сегмента по работам

#### Стратегия добавления сегментов
- Выбрать фокусный сегмент → создать ценность → создать воронку → оптимизировать → добавить соседний сегмент

#### Выход на предыдущую/следующую работу
- **Предыдущая работа**: лидогенерация, захват клиента раньше конкурентов
- **Следующая работа**: рост ценности, удержание, повторные покупки

### Алгоритм применения механик

1. **Определить бизнес-задачу**: запуск, масштабирование, выход из конкуренции, рост конверсии
2. **Построить гипотетический граф работ**
3. **Выбрать механики**: какие механики в теории могут помочь
4. **Провести ресерч**: проверить гипотезы на реальных клиентах
5. **Получить реальный граф**: без галлюцинаций, с реальными работами
6. **Применить механики**: над реальным графом применить выбранные механики
7. **Ранжировать по RICE**:
   - **R (Reach)** = Процент людей среди фокусного сегмента, на который механика может сработать
   - **I (Impact)** = Насколько субъективно каждый клиент это будет считать ценным
   - **C (Confidence)** = Насколько мы уверены
   - **E (Effort)** = Стоимость реализации
8. **Механики после ранжирования становятся стратегией**

## Часть 5: Ключевые ошибки и антипаттерны

### Типичные ошибки сегментации

- **Сегментация по демографии первой**: Демография без работы бесполезна
- **Продукт для "среднего" клиента**: Не существует среднего клиента
- **Продукт для всех = продукт ни для кого**: Без фокуса нет вау-ценности

### Типичные ошибки графа

- **Неправильный уровень Core Job**: Взяли Big Job вместо Core Job или наоборот
- **Пропуск уровней**: Пропустили один или несколько уровней в графе
- **Вопросы без Big Job**: Предыдущие/следующие работы нужно спрашивать через Big Job

### Типичные ошибки бизнеса

- **Ошибка №1**: Делать продукт для несуществующих работ несуществующих людей
- **Ошибка №2**: Маленький, сложный, убыточный сегмент (рядом есть лучше)
- **Ошибка №3**: Размывание ресурсов — фичи пилятся для других работ других людей

## Часть 6: Резюме для AI-инструмента

### Алгоритм нахождения сегментов

1. Определить рынок и наличие денег на рынке
2. Найти людей, которые ПЛАТЯТ за решение работ
3. Выявить их Core Jobs и Big Jobs
4. Сгруппировать по похожести графов работ → сегменты
5. Оценить каждый сегмент по 4 факторам выбора
6. Применить 4 силы для ранжирования
7. Рассчитать скор и отранжировать
8. Отбросить убыточные сегменты
9. Выбрать фокусный сегмент

### Алгоритм построения графа

1. Начать с Core Job (что продукт выполняет полностью)
2. Подняться наверх: «Зачем?» → Big Job
3. Определить предыдущие работы (через Big Job)
4. Определить следующие работы (через Big Job)
5. Спуститься вниз: Micro Jobs для каждой Core Job
6. Выстроить критические цепочки
7. Валидировать уровни вопросом «Чтобы что?»

### Алгоритм применения стратегий

1. Определить бизнес-задачу
2. Выбрать подходящие механики из 16+
3. Применить механики к каждой работе в графе → гипотезы
4. Отранжировать гипотезы по RICE
5. Топовые гипотезы = стратегия

### Ключевые метрики для ранжирования

- Частотность работы
- Важность работы
- Размер сегмента
- Неудовлетворённость текущими решениями
- Готовность платить (бюджет)
- Добавочная ценность (можем ли создать)
- Проблемы с текущим решением
- Сила привычки
- Уровень страхов
- Маржинальность
- Масштабируемость
- Риски
