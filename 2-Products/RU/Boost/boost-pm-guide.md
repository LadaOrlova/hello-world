# BOOST PM Guide — Полное руководство для Project Manager'а

> **Цель документа:** Полное описание роли, обязанностей и процессов Project Manager'а интенсива BOOST. Документ содержит детальные чек-листы на каждый этап работы — от межпоточной работы до завершения потока.

---

## 1. Роль и зона ответственности

PM интенсива BOOST — это **владелец продукта и операционный менеджер** в одном лице. Это full-time позиция.

**PM полностью отвечает за:**
- Качество продукта (курса) и его развитие
- Операционное проведение каждого потока
- Управление командой кураторов
- Онбординг и поддержку студентов
- Контент: воркшопы, гайды, промпты, материалы
- Инфраструктуру: серверы, бот выдачи, LMS, сайт
- Партнёрства и дистрибуцию курса
- Коммуникации со студентами (канал, чат, формы)
- Upsell выпускникам

**PM НЕ отвечает за:**заком
- Рост выручки напрямую (отвечает за качество продукта)
- Корпоративные продажи (отдельный сотрудник)
- Возврат денег студентам (финансовый менеджер)

## 3. Обзор продукта

**BOOST** — недельный интенсив по вайбкодингу (создание продуктов с AI).

- **Сайт:** https://boost-intensive.ru
- **Формат:** 7 воркшопов за неделю (установочный + 6 основных)
- **Масштаб:** ~120 студентов на поток, ~10 кураторов
- **Результат студента:** от идеи до работающего MVP на своём сервере
- **Целевая аудитория:** 95% без техбэкграунда (продакты, маркетологи, предприниматели)
- **Тарифы:** «Записи» (29 900 руб.) и «Интенсив» (39 900 руб.) + корпоративный

### Структура потока

| Воркшоп | Тема | Кто ведёт |
|---------|------|-----------|
| 1. Установочный | Установка VS Code, Claude Code, VPN, Wispr Flow. Выбор идеи продукта | **PM** |
| 2. PRD и лендинг | Продуктовые требования (PRD) и лендинг продукта | **Ваня** |
| 3. Создание и деплой MVP | Код MVP, деплой на сервер | **Ваня** |
| 4. Работа с данными | Исследование рынка, анализ данных, скрипты | **Ваня** |
| 5. Автоматизация | Субагенты, скиллы, агенты, контент-пайплайн | **Илья** |
| 6. AI-продукты | Команды агентов, Telegram-бот, генерация картинок | **Илья** |
| 7. Демо-день | Показ продуктов, подведение итогов | **Ваня** |

## 4. Ключевые метрики

| Метрика | Целевое значение | Как измерять |
|---------|------------------|--------------|
| Удовлетворённость курсом | **не ниже 9 из 10** | Выходная форма обратной связи |
| Доходимость до демо-дня | **не ниже 70%** | Количество студентов на демо vs. купивших |
| Количество проблем | Снижение от потока к потоку | Трекинг инцидентов (баны, серверы, VPN) |
| Время решения проблем | Как можно быстрее | Наблюдение в чате и на воркшопах |

## 5. Полный цикл потока — детальные чек-листы

### Фаза 1: Между потоками (после завершения предыдущего потока)

#### 1.1. Ретроспектива предыдущего потока

- [ ] Собрать и проанализировать выходные формы обратной связи от студентов
- [ ] Провести ретроспективу с кураторами: что сработало, что нет
- [ ] Собрать фидбек от ведущих воркшопов (Ваня, Илья)
- [ ] Составить список доработок на следующий поток
- [ ] Приоритизировать доработки: что внедрить обязательно, что можно отложить
- [ ] Зафиксировать список доработок в TODO.md или отдельном документе

#### 1.2. Обновление материалов

- [ ] Внедрить приоритетные доработки из ретроспективы
- [ ] Обновить файлы `what-to-do.md` в папках воркшопов (или делегировать Илье, проконтролировать)
- [ ] Обновить промпты в папках `prompts/` если нужно
- [ ] Проверить и обновить гайды на https://aura-framework.ru/guides/
- [ ] Заменить все упоминания Cursor на VS Code во всех файлах
- [ ] Обновить даты и номера потоков в материалах
- [ ] Обновить `curator-guide.md` по результатам обратной связи

#### 1.3. Мониторинг технологий

- [ ] Читать и тестировать актуальные технологии: новые нейронки, обновления Claude Code, новые MCP, новые инструменты
- [ ] Отслеживать технологические сдвиги на рынке (например: переход с Claude Code на ChatGPT, появление Codex, новые платформы)
- [ ] **Инициировать смену технологий**, если появляется более эффективный инструмент для студентов
- [ ] Предлагать адаптацию программы под новые технологии
- [ ] Тестировать изменения на себе перед внедрением в программу

#### 1.4. Записи для студентов прошлых потоков

- [ ] Убедиться, что все записи текущего потока загружены в LMS (обработка идёт после каждого воркшопа — см. Фазу 6)
- [ ] Отправить уведомление студентам тарифа «Записи» о доступности записей нового потока
- [ ] Обновить доступ к записям для студентов прошлых потоков (доступ на год)
- [ ] Проверить, что все записи доступны и воспроизводятся

#### 1.5. Развитие продукта

- [ ] **Расшифровать записи воркшопов** (стримить в Claude Code) — на основе расшифровок Claude анализирует ход воркшопа и предлагает конкретные идеи для доработки и роста ценности обучения
- [ ] Оценить, нужно ли добавить новый воркшоп или изменить существующий
- [ ] Исследовать возможности для новых форматов (отдельный курс, продвинутый уровень)

#### 1.6. Партнёрства и дистрибуция

- [ ] Поддерживать отношения с текущими партнёрами (Hello Cloud, сервис оплаты Claude)
- [ ] Искать новые партнёрства: VPN-провайдеры, платёжные сервисы, площадки для продвижения
- [ ] Менеджерить, что текущие партнёрства работают (серверы доступны, оплата проходит)
- [ ] Искать каналы дистрибуции: где продвигать курс

#### 1.7. Сообщество выпускников

- [ ] Модерировать бесплатное сообщество выпускников в Telegram
- [ ] Публиковать полезный контент, кейсы выпускников
- [ ] Менеджерить upsell: предлагать выпускникам следующий поток, новые продукты

### Фаза 2: За 2-4 недели до старта потока

#### 2.1. Кураторы

- [ ] Определить, сколько кураторов нужно (1 куратор на 10-12 студентов, при 120 студентах — минимум 10)
- [ ] Проверить, кто из текущих кураторов готов на следующий поток
- [ ] Если не хватает — начать поиск новых кураторов:
  - [ ] Разместить вакансию
  - [ ] Обратиться к выпускникам прошлых потоков
  - [ ] Провести собеседования
- [ ] Получить подтверждение от каждого куратора, что он точно будет
- [ ] Согласовать условия работы с каждым куратором (ставка единая для всех)
- [ ] Добавить новых кураторов:
  - [ ] Чат кураторов (Telegram)
  - [ ] Чат потока (студенты)
  - [ ] Канал потока (объявления)
  - [ ] Zoom (права соведущего/модератора)
- [ ] Отправить новым кураторам `curator-guide.md`
- [ ] Провести обучающий созвон со всеми кураторами (30-60 мин):
  - [ ] Объяснить формат каждого воркшопа
  - [ ] Пройтись по расписанию: когда кураторы подключаются, когда сессионные залы
  - [ ] Объяснить алгоритм помощи студентам
  - [ ] Ответить на вопросы кураторов
  - [ ] Убедиться, что каждый куратор прочитал what-to-do.md всех воркшопов

#### 2.2. Серверы (VPS)

**Основной хостер:** Hello Cloud (партнёр)

- [ ] Связаться с Hello Cloud, сообщить количество серверов (количество студентов + 10-20% запас)
- [ ] Характеристики согласованы на старте: 4 CPU, 6 GB RAM, 120GB SSD (минимум)
- [ ] Согласовать сроки доставки (получить минимум за 3 дня до старта)
- [ ] Подготовить Plan B на случай проблем (timeweb.cloud или гайд по самостоятельной регистрации на beget)

#### 2.3. Способы оплаты Claude — актуализация

- [ ] Проверить, что текущие способы оплаты Claude из РФ работают:
  - [ ] FunPay (подарочная ссылка)
  - [ ] Plati.market
  - [ ] Сервис-партнёр (если подключен)
- [ ] Найти и сделать детальные гайды для самых лучших способов оплаты зарубежных сервисов
- [ ] Обновить гайды по оплате, если что-то изменилось
- [ ] Проверить актуальность цен и комиссий
- [ ] Доработать инструкцию, как сделать дешевле: страна VPN = страна карты, США дешевле ($100 vs 112 евро)

#### 2.4. Онбординг студентов

- [ ] Подготовить вступительную форму (какой продукт хотят делать, ОС, опыт)
- [ ] Настроить флоу: студент получает доступ в чат **только после заполнения формы**
- [ ] Проверить, что автоматическое добавление в чаты работает
- [ ] Подготовить приветственное сообщение для новых студентов

### Фаза 3: За 1 неделю до старта

- [ ] Получить список серверов от хостера (IP, логин, пароль)
- [ ] Протестировать **каждый** сервер (`ssh root@IP` — проверить, что коннектится)
- [ ] Исключить нерабочие серверы из списка
- [ ] Загрузить рабочие серверы в бота выдачи
- [ ] Проверить, что бот выдачи серверов работает корректно
- [ ] Проверить все гайды на https://aura-framework.ru/guides/:

| Гайд | Что проверить |
|------|---------------|
| Установка VS Code | Ссылки актуальны, скриншоты актуальны (не Cursor!) |
| Установка Claude Code | Процесс актуален, расширение ставится |
| Оплата зарубежных сервисов | Способы работают (FunPay, Plati, партнёр) |
| Регистрация Claude | Предупреждение о банах на месте |
| VPN | Рекомендации актуальны (Red Shield, Amnezia) |

- [ ] Проверить материалы всех воркшопов (`what-to-do.md`):
  - [ ] Все ссылки рабочие
  - [ ] OpenRouter API ключ актуальный
  - [ ] Примеры промптов работают
  - [ ] Даты и номера потоков актуальные
- [ ] Провести обучающий созвон с кураторами (если не проведён ранее)
- [ ] Отправить студентам напоминание о подготовке к установочной сессии
- [ ] Проверить Zoom: ссылка рабочая, права у кураторов есть

- [ ] Подготовить PDF с планом установочной сессии

- [ ] Подготовить сессионные залы в Zoom (или план их создания)
- [ ] Проследить, чтобы Ваня отправил кругляш для знакомства в чат потока
- [ ] Мотивировать студентов знакомиться друг с другом, запустить процесс знакомства
- [ ] Напомнить кураторам о времени начала

### Фаза 5: Воркшоп 1 — Установочная сессия (ведёт PM)

#### Подготовка Zoom (заранее, до начала потока)

- [ ] Заранее создать **все Zoom-созвоны** для всех воркшопов в общем Zoom-аккаунте
- [ ] Создать календарь со всеми событиями потока (даты, время, ссылки на Zoom)
- [ ] Убедиться, что все ссылки на Zoom-созвоны готовы и отправлены студентам

#### Подготовка (за 30-60 мин до начала)

- [ ] Открыть Zoom, проверить, что ссылка работает
- [ ] Создать сессионные залы
- [ ] Проверить, что кураторы онлайн и в Zoom
- [ ] Открыть VS Code с Claude Code — подготовить экран для демонстрации
- [ ] Открыть PDF с планом сессии
- [ ] Открыть чат потока в Telegram — быть готовым отвечать

#### Проведение

1. **Приветствие и план** (5-10 мин)
   - [ ] Рассказать, что будет на установочной сессии
   - [ ] Показать экран с VS Code и Claude Code
   - [ ] Объяснить цель: «К концу сессии у вас должен быть установлен и запущен VS Code с Claude Code вот в таком виде»

2. **Выбор идеи продукта** (15-20 мин)
   - [ ] Показать файл `what-to-do.md` с требованиями к проектам
   - [ ] Объяснить, какие проекты подходят, какие — нет (без API, без Telegram webhooks на этом этапе)
   - [ ] Попросить студентов написать свои идеи в чат
   - [ ] Голосом прокомментировать каждую идею: подходит / не подходит / как упростить
   - [ ] Дать обратную связь: «Классно, берём» или «Давай упростим: вместо X сделай Y»

3. **Установка программ** (основная часть)
   - [ ] Отправить PDF с гайдами установки
   - [ ] Показать на экране процесс установки
   - [ ] Распределить студентов по сессионным залам с кураторами
   - [ ] Напомнить кураторам зайти в сессионные залы
   - [ ] Мониторить чат на проблемы
   - [ ] Возвращать студентов в сессионные залы, если их выкинуло из Zoom
   - [ ] Добавлять опоздавших студентов в сессионные залы

4. **Проверка результата**
   - [ ] Убедиться, что у студентов установлено: VPN, VS Code, Claude Code, Wispr Flow
   - [ ] Проверить, что Claude Code запускается и работает

#### После установочной сессии

- [ ] Собрать фидбек от кураторов: какие проблемы были, сколько студентов не успели
- [ ] Написать в чат потока: кто не успел — вот инструкция для самостоятельной установки
- [ ] Проверить, что у каждого студента есть идея продукта (или помочь выбрать в чате)
- [ ] Зафиксировать проблемы для улучшения следующей установочной сессии

### Фаза 6: Воркшопы 2-6 (ведёт Ваня / Илья, PM присутствует)

#### Перед каждым воркшопом (за 1-2 часа)

- [ ] Отправить напоминание в канал потока: время, ссылка на Zoom, тема
- [ ] Проверить, что ведущий воркшопа (Ваня/Илья) готов
- [ ] Проверить Zoom: ссылка работает, запись включается
- [ ] Создать сессионные залы в Zoom
- [ ] Связаться с кураторами: все на месте, знают программу воркшопа
- [ ] Открыть чат потока — быть готовым отвечать на вопросы

#### Во время воркшопа

- [ ] **Присутствовать в Zoom на протяжении всего воркшопа**
- [ ] Включить запись Zoom
- [ ] **Дать co-host права кураторам**, чтобы они могли заходить в сессионные залы
- [ ] Мониторить чат студентов в Telegram на критичные проблемы
- [ ] Распределять студентов по сессионным залам, когда ведущий объявляет практику
- [ ] Напоминать кураторам заходить в сессионные залы
- [ ] Возвращать в сессионные залы студентов, которых выкинуло из Zoom
- [ ] Добавлять опоздавших в сессионные залы
- [ ] Ходить по сессионным залам, слушать как кураторы работают
- [ ] Давать обратную связь кураторам (после, не во время работы со студентом)
- [ ] Решать сложные проблемы, которые кураторы не смогли решить
- [ ] Отвечать на вопросы студентов в чате

#### Типичные проблемы на воркшопе и действия PM

| Проблема | Кто решает | Действие PM |
|----------|-----------|-------------|
| Студента выкинуло из Zoom | PM | Вернуть в сессионный зал |
| Студент не понимает задание | Куратор | Убедиться, что студент в зале с куратором |
| Бан Claude у студента | Куратор | Если куратор не справляется — подключиться |
| Не работает VPN | Куратор | Направить в зал, дать ссылку на гайд |
| Сервер не коннектится | PM | Выдать другой сервер через бота |
| Ведущий задерживается | PM | Занять студентов, начать без ведущего если нужно |

#### После каждого воркшопа

- [ ] Остановить запись Zoom
- [ ] Провести краткий sync с кураторами: что сработало, что нет, какие проблемы
- [ ] Зафиксировать проблемы и идеи для улучшения
- [ ] **Обработать запись воркшопа:** скачать из Zoom, обрезать начало/конец, убрать технические паузы

## 6. Управление кураторами — полный процесс

### 6.1. Поиск и найм

**Норма:** 1 куратор на 10-12 студентов. При 120 студентах — минимум 10 кураторов.

**Где искать:**
- Выпускники прошлых потоков (основной источник)
- Вакансия (у нас есть стандартная вакансия)

**Процесс:**
1. Определить, сколько кураторов не хватает
2. Разместить вакансию / обратиться к выпускникам
3. Провести собеседование: понимает ли кандидат Claude Code, умеет ли объяснять
4. Согласовать условия (единая ставка для всех, оплата за поток)
5. Получить подтверждение, что куратор точно будет на всех воркшопах

### 6.2. Онбординг нового куратора

- [ ] Добавить в чат кураторов (Telegram)
- [ ] Добавить в чат потока
- [ ] Добавить в канал потока
- [ ] Дать права в Zoom (соведущий/модератор)
- [ ] Отправить `curator-guide.md`
- [ ] Объяснить, где лежат материалы (`what-to-do.md` в каждой папке воркшопа)
- [ ] На первом воркшопе поставить в пару с опытным куратором

### 6.3. Обучение перед потоком

Провести созвон (30-60 мин) со **всеми** кураторами:

- [ ] Пройтись по расписанию потока: даты, время каждого воркшопа
- [ ] Объяснить формат каждого воркшопа: когда общий зал, когда сессионные
- [ ] Разобрать `what-to-do.md` каждого воркшопа
- [ ] Убедиться, что каждый куратор сам выполнил все задания как студент
- [ ] Объяснить алгоритм помощи: короткие ответы, не делать за студента, ссылки на гайды
- [ ] Обсудить распределение по комнатам: сколько студентов на куратора
- [ ] Ответить на вопросы

### 6.4. Оценка качества работы кураторов

**Во время потока:**
- Ходить по сессионным залам, слушать как кураторы работают
- Обращать внимание: даёт ли куратор короткие ответы, не делает ли за студента, не задерживается ли

**После потока:**
- Собрать фидбек от студентов о кураторах (в выходной форме)

- Единая ставка для всех кураторов

- PM собирает данные, Алёна проводит выплату

## 10. Кризисные ситуации

### Самые частые проблемы (по убыванию частоты)

#### 1. Оплата Claude из РФ (самое сложное)

| Проблема | Решение |
|----------|---------|
| Карта отклонена | Попробовать другой сервис (FunPay / Plati / партнёр) |
| VAT добавляет 20% | Регион США, не Европа |
| Забанили сразу после оплаты | Возврат до 30 дней, но долго. Проще новый аккаунт + новая оплата |
| Страна VPN не совпадает с картой | Объяснить правило: страна VPN = страна карты |

#### 2. Бан Claude

| Проблема | Решение |
|----------|---------|
| Аккаунт заблокирован | Новый Google аккаунт + оплата через FunPay/Plati |
| Студент потерял деньги | Можно запросить возврат (до 30 дней), но проще новый аккаунт |

#### 3. Проблемы с VPN

| Проблема | Решение |
|----------|---------|
| VPN работает только в браузере | Установить VPN на уровне системы |
| VPN нестабильный | Порекомендовать Red Shield или Amnezia |
| Нет VPN | Дать гайд по установке |

#### 4. Базовые проблемы с VS Code и Claude Code

| Проблема | Решение |
|----------|---------|
| Claude Code не запускается | Проверить VPN, перезапустить VS Code |
| Корпоративный ноутбук блокирует | Использовать личный компьютер |
| Windows 32-bit | Не поддерживается, нужен другой компьютер |
| Не работает PATH на Windows | Добавить вручную (есть гайд) |

#### 5. Проблемы с серверами

| Проблема | Решение |
|----------|---------|
| Сервер не коннектится | Выдать другой сервер через бота |
| Серверов не хватает | Plan B: timeweb.cloud или гайд по самостоятельной регистрации |
| Бот не выдаёт серверы | Проверить что список загружен, username студента в базе |

### Когда PM должен срочно связаться с Ваней

- Массовый бан Claude (> 5 студентов за раз)
- Полный отказ инфраструктуры (все серверы лежат, бот не работает)
- Серьёзный конфликт со студентом или куратором
- Вопросы по партнёрству с Hello Cloud
