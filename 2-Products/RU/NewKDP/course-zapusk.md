# Курс "Запуск продукта" — Программа

> Job 1: "Запустить свой продукт, не сжигая деньги впустую"
> Формат: автономный симулятор в VS Code (0 воркшопов Вани)
> Длительность: 3 недели (18 модулей, ~15 часов контента + практика)
> Цена: 60-90K RUB
> Результат: от идеи до работающего MVP с первыми клиентами

---

## Как устроен каждый модуль

```
┌─────────────────────────────────────────────────────────────┐
│ VS Code                                                      │
├──────────────┬────────────────────────┬──────────────────────┤
│ Проект       │  Браузер (центр)       │  Claude Code         │

**Единица обучения (модуль, 30-60 минут):**
1. Видео-лекция (3-7 мин) — записана Ваней один раз
2. Тезис + иллюстрация/пример — закрепление в тексте
3. Квиз (1-2 вопроса) — проверка понимания
4. Задание — студент копирует промпт в Claude Code → получает файл в проекте
5. AI-фидбек — нейронка проверяет файл и даёт обратную связь
6. Переход к следующему модулю

## НЕДЕЛЯ 1: Понять клиента (модули 1-8)

### Модуль 1: Зачем вообще исследовать клиентов

**Видео (5 мин):** Причинно-следственная цепочка создания продукта
- Тезис из AURA: Рынок → Сегменты → Работы → Ценность → Показ ценности → Юнит-экономика → Привлечение → Конверсия → Прибыль
- Главная ошибка: "придумали продукт для несуществующих задач несуществующих людей"
- Пример: 80% стартапов не находят PMF, потому что не исследовали работы клиентов
- Пример: как Ваня за 12 лет пришёл к AJTBD через провалы

**Тезис в тексте:**
> Продукт — это не набор фичей. Продукт — это решение, которое человек "нанимает" для выполнения работы. Если вы не знаете работу — вы строите космолёт в вакууме.

**Квиз:**
- "Какова вероятность НЕ найти Product-Market Fit?" → 80%
- "Что первично при создании продукта?" → Работа клиента (не фича, не идея фаундера, не технология)

**Задание (в Claude Code):**

```
Прочитай файл prompts/module-01.md и следуй инструкциям.

Задай мне 5 вопросов по очереди (один вопрос — жди ответ — следующий вопрос):

1. Опиши свой продукт или идею продукта в 2-3 предложениях.
   Что это? Для кого?

2. Какую проблему ты хочешь решить? Опиши ситуацию клиента.

3. Почему ты думаешь, что эта проблема существует?
   (Есть ли доказательства: разговоры с людьми, свой опыт, данные?)

4. Как люди решают эту проблему сейчас? Без твоего продукта?

5. Почему именно ты можешь это решить лучше, чем существующие решения?

После всех ответов:
Сохрани результат в файл zapusk/research/my-product.md в формате:

# Мой продукт
## Описание
[ответ 1]
## Проблема клиента
[ответ 2]
## Доказательства проблемы
[ответ 3]
## Текущие решения (конкуренты)
[ответ 4]
## Моё преимущество
[ответ 5]
## ⚠️ Красные флаги
[AI анализирует ответы и отмечает: где гипотезы не подтверждены данными,
где "проблема" может быть надуманной, где конкуренты сильнее, чем кажется]
```

**AI-фидбек:** Проверяет файл `my-product.md` и добавляет секцию "Красные флаги" — где студент строит гипотезы без данных.

### Модуль 2: Есть ли деньги на рынке

**Видео (4 мин):** Как проверить, что рынок существует
- Тезис из AURA: "Рынок — это люди, которые уже тратят деньги на решение этой задачи"
- Если никто не тратит деньги → рынка нет (или ты его создаёшь, что в 100 раз сложнее)
- 3 быстрых способа проверить: поиск конкурентов, объём трат, тренды

**Квиз:**
- "Рынок — это..." → "Люди, которые уже тратят деньги на решение задачи" (не "все, кому может быть полезен продукт")

**Задание (в Claude Code):**

```
Прочитай файл zapusk/research/my-product.md.

На основе описания продукта:

1. Найди 5-10 конкурентов или альтернативных решений (включая "ничего не делать").
   Для каждого: название, что делает, примерная цена, сколько пользователей (если можно оценить).

2. Оцени размер рынка:
   - TAM: сколько людей в мире могут иметь эту проблему
   - SAM: сколько из них доступны для твоего продукта (язык, география, канал)
   - SOM: сколько реально можешь привлечь в первый год

3. Оцени: тратят ли люди деньги на решение этой задачи сейчас? Сколько?

Сохрани результат в zapusk/research/market-check.md

В конце добавь секцию:
## Вердикт
[Зелёный / Жёлтый / Красный] — есть ли деньги на рынке и стоит ли продолжать

### Модуль 3: Что такое работа (Job) и почему это важнее фич

**Видео (7 мин):** Основы AJTBD
- Тезис: "Люди не покупают продукты. Люди нанимают решения для выполнения работ"
- Модель: Точка А (текущая ситуация) → Триггер → Точка Б (желаемый результат)
- Структура работы: контекст, триггер, эмоция на входе, ожидаемый результат, критерии успеха, работа выше уровня
- Пример: "Человек покупает дрель не потому что ему нужна дрель, а потому что ему нужна дырка в стене. А дырка нужна для полки. А полка — чтобы навести порядок. А порядок — чтобы чувствовать контроль над жизнью."
- Шаблон описания работы

**Тезис в тексте:**
> **Работа** — это цель, которую хочет выполнить человек в определённом контексте, с определёнными критериями успеха, двигаясь из точки А в точку Б.
>
> Шаблон: "Когда [контекст + триггер], я хочу [действие/результат], чтобы [работа выше уровня]"

**Квиз:**
- "Что из перечисленного — работа?" → "Когда я переезжаю в новый город, я хочу быстро найти врача, которому доверяю" (не "найти врача", не "приложение для записи к врачу")
- "Что первично — работа или решение?" → Работа (решений может быть много)

**Задание (в Claude Code):**

```
Прочитай zapusk/research/my-product.md.

Помоги мне описать 3 возможные работы моих клиентов по шаблону AJTBD.

Для каждой работы заполни:

### Работа [номер]: [название]
- **Контекст:** В какой ситуации находится человек?
- **Триггер:** Что произошло, что заставило искать решение?
- **Эмоция на входе:** Что чувствует? (тревога, злость, надежда, FOMO...)
- **Ожидаемый результат:** Чего хочет достичь?
- **Критерии успеха:** Как поймёт, что получилось?
- **Работа выше уровня (Big Job):** Зачем это ему в жизни/бизнесе?

Задай мне уточняющие вопросы, если не хватает информации.

Сохрани в zapusk/research/job-hypotheses.md

В конце добавь:
## ⚠️ Это гипотезы, не факты
Эти работы нужно проверить через интервью с реальными людьми.
Самая опасная ловушка — считать что ты знаешь работы своих клиентов.
```

### Модуль 4: Как проводить AJTBD-интервью

**Видео (7 мин):** Принципы и техника интервью
- Тезис из AURA: "Вопросы всегда одинаковые. Самое важное — какие работы изучать"
- 4 принципа: (1) вопросы одинаковые, (2) важно какие работы изучать, (3) каждое интервью двигает к бизнес-задаче, (4) изучаем по факту затрат в прошлом
- Структура интервью: установление контакта → контекст → триггер → поиск решений → критерии выбора → результат → работа выше
- Что НЕ спрашивать: "Купили бы вы...?", "Сколько бы заплатили?", "Какие фичи вам нужны?"
- Демонстрация: фрагмент реального интервью из КДП (такси/Яндекс)

**Тезис в тексте:**
> Правило #1: Никогда не спрашивай про будущее ("Вы бы купили...?"). Спрашивай про прошлое ("Расскажите, когда в последний раз вы...").
>
> Правило #2: Следуй за эмоциями. Когда человек говорит "меня бесило" или "я обрадовался" — копай глубже. Там работа.

**Квиз:**
- "Какой вопрос правильный для AJTBD-интервью?" → "Расскажите, когда в последний раз вы столкнулись с этой ситуацией?" (не "Какие фичи вам нужны?")

**Задание (в Claude Code):**

```
На основе файла zapusk/research/job-hypotheses.md создай скрипт интервью.

Скрипт должен содержать:

# Скрипт AJTBD-интервью

## Вступление (2 мин)
[Текст: как представиться, объяснить цель, получить разрешение на запись]

## Разогрев (3 мин)
[2-3 вопроса про контекст: чем занимается, какая роль, какой бизнес]

## Основная часть (20-25 мин)

### Блок 1: Контекст и триггер
- Расскажите, когда в последний раз вы [ситуация из гипотезы работы]?
- Что произошло, что заставило вас начать искать решение?
- Как давно это длилось до того, как вы начали действовать?

### Блок 2: Поиск решений
- Что вы попробовали первым? Почему именно это?
- Что ещё пробовали?
- Что НЕ сработало? Почему?

### Блок 3: Критерии и результат
- Как вы выбирали между вариантами?
- Что было самым важным при выборе?
- Чем всё закончилось? Довольны ли результатом?

### Блок 4: Работа выше уровня
- А зачем вам это было нужно? Что бы изменилось в вашей жизни/бизнесе?
- Если бы эта задача решилась идеально — что бы вы почувствовали?

## Завершение (2 мин)
[Благодарность, вопрос "кого ещё можете порекомендовать для разговора?"]

## Заметки для интервьюера
- Следи за эмоциями: когда говорит "бесило/радовался/боялся" — копай глубже
- Не подсказывай ответы
- Не продавай свой продукт
- Записывай дословные цитаты

Сохрани в zapusk/research/interview-script.md
```

### Модуль 5: Тренировочное интервью с AI

**Видео (3 мин):** Как практиковаться
- "Перед реальными интервью — потренируйся на AI. AI будет играть роль твоего потенциального клиента."
- Что отслеживать: следуешь ли скрипту, копаешь ли вглубь, не продаёшь ли

**Задание (в Claude Code):**

```
Прочитай файлы:
- zapusk/research/my-product.md (мой продукт)
- zapusk/research/job-hypotheses.md (гипотезы работ)
- zapusk/research/interview-script.md (скрипт)

Ты сейчас будешь играть роль потенциального клиента моего продукта.

Правила для тебя (AI):
1. Ты — реальный человек с реальной ситуацией. Придумай себе имя, должность, компанию.
2. У тебя ЕСТЬ проблема, связанная с одной из работ из job-hypotheses.md, но ты не знаешь терминологию AJTBD.
3. Отвечай как обычный человек: иногда уходи от темы, иногда отвечай расплывчато, иногда эмоционально.
4. НЕ подсказывай интервьюеру правильные вопросы.
5. Если интервьюер задаёт плохой вопрос (про будущее, про фичи) — отвечай, но ответ будет бесполезным.

Я (студент) буду проводить интервью по своему скрипту.

После 15-20 обменов репликами — останови интервью и дай фидбек:

## Обратная связь по интервью
### ✅ Что получилось хорошо
### ⚠️ Что улучшить
### 🔍 Какие работы удалось выявить
### 💡 Какие вопросы стоило задать, но ты не задал

Сохрани весь диалог и фидбек в zapusk/research/interviews/practice-1.md
```

**Видео (5 мин):** Как строить граф работ из интервью

- Как выделить из интервью: ищи повторяющиеся паттерны
- Пример графа работ из КДП

**Тезис:**
> Граф работ — это карта того, зачем люди используют ваш продукт. Если у двух людей похожие графы — они в одном сегменте. Если разные — в разных.
