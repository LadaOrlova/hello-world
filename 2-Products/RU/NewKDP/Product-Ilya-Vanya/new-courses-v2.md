# Линейка продуктов КДП v2.1

> Дата: 2026-03-31
> Источники: jobs-v2.md (8 Jobs, 1196 студентов, потоки 55-59), карта контента AURA (36 учебных тем, 3 уровня), экосистема Замесина (КДП + BOOST + AURA), данные BOOST (83% студентов курсов НЕ из BOOST)
> Метод: проектирование продуктов на основе Jobs + доступного контента
> Изменения v2.1: реалистичный темп (3-4 модуля/нед.), вайбкодинг-онбординг, интеграция AURA

---

## Принципы проектирования

1. **Каждый продукт = конкретный Job (или 2-3 близких Job).** Не "курс для всех", а продукт для конкретного человека с конкретной проблемой.

2. **Вайбкодинг встроен в каждый курс.** Студент уходит не с "пониманием", а с работающими инструментами, которые создал сам через VS Code + Claude Code.

3. **83% студентов НЕ знакомы с VS Code.** Каждый курс начинается с вайбкодинг-онбординга (1-2 модуля), базированного на записанных BOOST-воркшопах. Каждое задание содержит пошаговую инструкцию для работы в VS Code / Claude Code.

4. **AURA интегрирована в обучение.** Студент видит два пути выполнения заданий: через Claude Code (кастомизация) и через AURA (готовый инструмент). Это создаёт воронку: курс → привычка к AURA → подписка после курса (~4000 ₽/мес).

5. **Ваня не участвует в проведении.** Контент предзаписан. Кураторы ведут. Масштабируется без Вани.

6. **Контент уже существует.** 36 учебных тем в базе AURA + записанные лекции + воркшопы. Новый контент нужен минимально.

7. **Формат: VS Code в браузере.** Студент логинится → попадает в VS Code → папка с курсом + Claude Code в терминале.

8. **Реалистичный темп: 3-4 модуля в неделю.** Один модуль = один учебный день, но студент занимается не каждый день. Длительность курсов рассчитана из этого темпа.

## Вайбкодинг-онбординг (стандартный блок)

Каждый курс начинается с 1-3 модулей онбординга. Контент уже записан в BOOST-воркшопах.

### Модуль 0a: "Что такое вайбкодинг и зачем"
*Источник: воркшоп "Веб-кодинг и новая парадигма создания продуктов", 30 янв 2026*

- Что такое вайбкодинг и как он меняет создание продуктов
- Зачем это продакту/собственнику (не для программистов)
- Экономика: вместо разработчиков — подписка $100/мес
- Психологический барьер и как его преодолеть

### Модуль 0b: "Настройка VS Code + Claude Code"
*Источник: воркшоп "Установка и настройка программ", 13 фев 2026 + "Знакомство и старт", 31 янв 2026*

- Установка Cursor/VS Code + Claude Code
- Первая команда, первый файл
- Метанавык: "Я не знаю, в чём проблема — почини!"
- Просмотр файлов, работа с папкой проекта

### Модуль 0c: "Работа с данными через Claude Code" (только продвинутые курсы)
*Источник: воркшоп "Учимся работать с данными", 16 фев 2026*
*Добавляется в: "Починка продукта", КДП Полный*

- LLM для быстрого ознакомления с данными
- Опасность галлюцинаций при количественном анализе
- LLM пишет скрипт, скрипт считает данные (детерминированность)
- Качественный анализ текстов (отзывы, тикеты)

### Пошаговые инструкции в каждом задании

Каждое задание каждого модуля содержит пошаговую инструкцию для работы в VS Code / Claude Code. Не "запусти скилл", а:
1. Откройте VS Code
2. Откройте терминал (Ctrl+`)
3. Введите команду: `claude "проанализируй файл interviews/transcript-1.md и выдели Jobs клиента"`
4. Прочитайте результат в файле `my-project/jobs.md`
5. Если что-то непонятно — напишите в терминале: `claude "я не понимаю что получилось, объясни простыми словами"`

## Отличия от new-courses.md (версия Вани)

### 1. 7 продуктов вместо 6

Добавлен "AJTBD Основы" — 3-недельный входной курс. У Вани такого не было. Это самый большой сегмент (41% аудитории), и он создаёт ступеньку перед полным КДП. Снижает порог входа (35-50K vs 90-130K).

### 2. Другой приоритет запуска

Ваня: "Запуск" первый → "Маркетинг" второй → КДП Полный третий.
Мы: "Основы" первый → "Починка" второй → КДП Полный третий.

Почему: "Основы" покрывает 41% аудитории (250+61+39 чел) и контент 100% готов. "Починка" даёт самый высокий чек. "Запуск" требует нового контента.

### 3. Сегментация по 8 Jobs вместо 5

Продукты спроектированы на основе 8 Jobs из jobs-v2.md, а не 5 Jobs из Ваниной версии. Это позволяет точнее попадать в запрос каждого сегмента.

### 4. Собственники разделены

У Вани все собственники шли в "Запуск". У нас: собственники с идеей → "Запуск", собственники с продуктом → "Починка". Это разные чеки (68K vs 91K) и разные потребности.

### 5. Экономика считается от реальных данных

У Вани чеки из старых данных (75-110K). Наши чеки из свежей выгрузки 1196 студентов (67-91K). Экономика реалистичнее.

### 6. Формат одинаковый

Оба варианта сходятся: предзаписанный формат в VS Code, без живых воркшопов Вани, с кураторами. Это подтверждает правильность направления.

### 7. Вайбкодинг-онбординг (новое в v2.1)

Каждый курс начинается с 2-3 модулей онбординга, базированных на записанных BOOST-воркшопах. 83% студентов не из BOOST и не знают VS Code / Claude Code. Без онбординга они застрянут на первом же задании.

### 8. AURA интегрирована в обучение (новое в v2.1)

Студент на каждом подходящем задании видит два пути: Claude Code (для кастомизации) и AURA (готовый инструмент). Это создаёт воронку: курс → привычка к AURA → подписка (~4000 ₽/мес) после курса. Добавляет 5.76-12.96M/год выручки от подписок.

### 9. Реалистичный темп (новое в v2.1)

Темп пересчитан с 5 модулей/неделю на 3-4 модуля/неделю. Курсы стали длиннее, но реалистичнее. Цены не изменились — студент получает больше времени на усвоение.
