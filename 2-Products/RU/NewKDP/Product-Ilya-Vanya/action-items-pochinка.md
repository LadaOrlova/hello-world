## Что делает команда Ильи

### Стадия 0: Разведка (эта неделя)

- [ ] **Допройти 59-й поток как студент** — увидеть процесс изнутри, зафиксировать что работает/ломается
- [ ] **Встреча с Владой** — что ей нужно для запуска маркетинга, каналы, сроки подготовки
- [ ] **5-10 созвонов с проектами, где обучение без живого человека** — как играют с доходимостью, удержанием, активацией в self-paced формате

### Стадия 1: Ключевые решения (нужен Ваня для части из них — см. раздел ниже)

- [ ] **Подготовить рекомендацию по формату доступа** — ограниченный (5-6 нед кураторы + бессрочные записи) vs бессрочный. Аргументы для обсуждения с Ваней
- [ ] **Подготовить варианты тарифной сетки** — стандартная (База/Макс/Макс+фидбек) или 2 плана
- [ ] **Определить модель работы кураторов** (с Лерой) — нагрузка, формат (чат/созвон), точки проверки
- [ ] **Проработать вопрос среды обучения** — cloud VS Code или гайд для своих машин. Для первого потока можно на своих
- [ ] **Определить роль AURA** (с Ильёй Измайловым) — доступ на время курса? Какие модули через AURA?

### Стадия 2: Контент

- [ ] **Нарезать записи BOOST для онбординга** (с Сашей) — 3 модуля (0a, 0b, 0c). Понять: можно нарезать из воркшопов 15-20 мин кусков, или перезаписывать
- [ ] **Инвентаризация лекций КДП** — пройтись по каждому модулю: запись есть? качество ок? длительность?
- [ ] **Написать промпты-задания** — 15 заданий для Claude Code, пошаговые, тестировать на себе
- [ ] **Тезисы/тексты к каждому модулю** — краткий конспект рядом с видео
- [ ] **Квизы** — 1-2 вопроса на модуль для проверки понимания

### Стадия 3: Сборка

- [ ] **Собрать шаблон проекта студента** — папка: онбординг → неделя 1 → 2 → 3 + README + CLAUDE.md
- [ ] **Настроить среду обучения** — гайд по установке или cloud VS Code
- [ ] **Настроить AI-скиллы** — ABCDX-анализ, interview-analyzer, граф работ
- [ ] **Подготовить кураторов** (с Лерой) — дать пройти 3-4 модуля, обучить формату
- [ ] **Система прогресса** (с Лерой) — как трекать кто на каком модуле
- [ ] **Система коммуникации** (с Лерой) — чат потока, канал объявлений, бот напоминаний

### Стадия 4: Маркетинг (параллельно со сборкой)

- [ ] **Навайбкодить лендинг** — позиционирование, программа, отзывы, CTA
- [ ] **Передать Владе материалы для прогрева** — описание продукта, аудитория, посылы, дата
- [ ] **Запустить waitlist** — лендинг с кнопкой "Записаться в список ожидания"
- [ ] **Подготовить сейла** — скрипт: кому предлагать "Починку", обработка возражений

### Стадия 5: Тестирование

- [ ] **Сам пройти весь курс как студент** — от онбординга до итогового диагноза
- [ ] **1-2 куратора проходят 3-4 модуля** — понятны ли задания, работают ли промпты
- [ ] **Бета-тест 5-10 человек** — из выпускников КДП, бесплатно/со скидкой, собрать ОС
- [ ] **Починить по обратной связи** — критичные баги, непонятные задания

### Стадия 6-7: Запуск и мониторинг

- [ ] **Набрать 50 студентов** (с Владой и сейлом)
- [ ] **Мониторинг первой недели** — кто прошёл онбординг, где застряли
- [ ] **Еженедельный трекинг доходимости** — по модулям
- [ ] **Сбор NPS и кейсов** — для второго потока и для продукта "Запуск"
