## 1. Что отличается от твоих документов

### Jobs: 8 вместо 5

| Было (Ваня, 785 студ, потоки 55-58) | Стало (Илья, 1196 студ, потоки 55-59) |
|--------------------------------------|---------------------------------------|
| 5 Jobs + 3 кластера | 8 Jobs |
| J1 "Запустить" = все вместе (25%) | **Разделил:** J1 "Запустить с нуля" (13.6%) + J2 "Собственник — вырастить" (15.7%) |
| J2 "Система вместо чуйки" (30%) | J4 "Навыки и методология" (29.6%) — по сути то же, но шире |
| J3 "Починить" (20%) | J3 "Починить" (12.4%) — подтвердилось |
| J4 "Карьера" (15%) | J5 "Карьера" (7.2%) — меньше, чем у Вани |
| J5 "Маркетинг" (10%) | **Растворился** между J2 и J3 — не отдельный Job |
| Кластер "Упаковать ценность" | **Растворился** — часть более широких Jobs |
| — | **Новый:** J6 "Стратегия и фокус" (6.4%, sat 8.8 — одна из лучших) |
| — | **Новый:** J7 "Исследовать клиентов" (4.6%) |
| Кластер "Внедрить подход" (2-5%) | **Подтверждён:** J8 (1.5%, sat 7.1 — курс не решает их проблему) |

### Ключевые отличия

- **Собственники — два типа.** С идеей (чек 69K, нет опыта) и с бизнесом (чек 80K, 8+ лет). Разные люди, разные потребности.
- **"Маркетинг" — не отдельный Job.** Запрос на упаковку реальный, но он часть "Починки" и "Собственников".
- **Чеки ниже.** Ваня: 75-110K. Данные: 67-91K. Причина: 59-й поток с тест-драйвом и базовым тарифом.
- **Удовлетворённость совпадает.** 8.5 общая — подтверждает надёжность данных.
- **Выборка больше.** 1196 vs 785. Добавлен 59-й поток (380 чел — самый большой).

## 2. Инсайты из 59-го потока

### Сдвиги

| Job | Было (55-59) | Стало (59) | Тренд |
|-----|-------------|-----------|-------|
| Запустить своё | 13.6% | **18.3%** | **Рост** |
| Починить продукт | 12.4% | **16.9%** | **Рост** |
| Карьера | 7.2% | **9.9%** | **Рост** |
| Навыки/методология | 29.6% | **25.4%** | **Падение** |

**Вывод:** Меньше "просто научиться", больше с конкретной задачей.

### Главное открытие: "вайбкодер без продукта"

- **48 человек (17%)** упоминают AI/вайбкодинг в анкетах
- ~20-25 из них — **новый архетип:** умеют строить, не знают что строить
- Цитата: *"Я могу навайбкодить что угодно, но не знаю что нужно вайбкодить"*
- **Вайбкодинг стал каналом привлечения** — многие пришли после BOOST или вебинара

### Что это значит

- AI не создал новый Job — он усилил "Запустить своё"
- Порог "построить" упал. Порог "понять, что нужно" остался прежним
- КДП для этих людей — **"вторая половина пазла"**

## 4. Proof of Concept: 2 продукта

### Почему эти два

| Критерий | "Запуск продукта" | "Починка продукта" |
|----------|-------------------|-------------------|
| Тренд в 59-м | Рост +5пп | Рост +5пп |
| Аудитория/поток | ~70 чел | ~65 чел |
| Средний чек | 69K | 91K (макс.) |
| Длительность | 4-5 нед. | 3-4 нед. |
| Цена | 60-75K | 60-80K |
| Контент готов | 90% | 100% |
| Вайбкодинг | Центральный (строим MVP) | Вспомогательный (дашборды) |
| Результат | MVP + первые клиенты | Диагноз + план роста |

### "Запуск продукта" — 16 модулей, 4-5 недель

**Обещание:** "За 5 недель — от идеи до работающего продукта с первыми клиентами."

| Нед. | Фокус | Результат |
|------|-------|-----------|
| 0 | Вайбкодинг-онбординг (из BOOST) | Умеет работать в VS Code + Claude Code |
| 1 | Понять клиента: AJTBD + интервью | Граф работ + скрипт интервью |
| 2 | Исследовать: 3 реальных интервью + сегменты | Сегменты + граф из данных |
| 3 | Стратегия: выбор сегмента + ценность + RAT | Проверенные гипотезы |
| 4 | Строим: MVP + лендинг + трекшн | Задеплоенный MVP + лендинг |

### "Починка продукта" — 15 модулей, 3-4 недели

**Обещание:** "За 4 недели — диагноз продукта и план роста. Или решение закрыть."

| Нед. | Фокус | Результат |
|------|-------|-----------|
| 0 | Вайбкодинг-онбординг + работа с данными | Умеет анализировать данные через Claude Code |
| 1 | Кто ваш клиент: ABCDX + интервью + граф | Сегментация своих клиентов |
| 2 | Что не так: ценность + юнит-экономика | Где утекают деньги, где потенциал |
| 3 | Что делать: стратегия + позиционирование | Итоговый диагноз + план |

### Что встроено в оба курса

- **Вайбкодинг-онбординг** — из записей BOOST (83% студентов не знакомы с VS Code)
- **AURA** — доступ на время курса, 4-5 модулей можно делать через AURA (воронка в подписку)
- **Кураторы** — из текущей команды КДП
- **Темп** — 3-4 модуля в неделю (реалистичный)

### Экономика (консервативно, 50 студ/поток)

| | "Запуск" | "Починка" |
|---|---------|-----------|
| Выручка за поток | 3.0-3.75M | 3.0-4.0M |
| Маржа на студента | 50-65K | 50-70K |
| 4 потока/год | 12-15M | 12-16M |

### Что нужно для запуска

- Контент: **90-100% записан** (КДП + BOOST)
- Задания (промпты): **нужно создать** (~30 промптов на оба курса)
- Cloud VS Code: **нужно поднять** (разработчик)
- Лендинги: **навайбкодить** за 1-2 дня
- Срок до запуска: **4-6 недель**

## 5. Критерии успеха POC

**Главный критерий:** Студент без опыта вайбкодинга за 4-5 недель получает конкретный результат (MVP или диагноз продукта) — без участия Вани, с удовлетворённостью 8+/10.

### Этап 1: До запуска (4-6 недель) — "Концепт рабочий"

| Критерий | Метрика |
|----------|---------|
| Шаблон проекта собран | Студент проходит первые 3 модуля без помощи |
| Онбординг работает | Новичок настраивает Claude Code и делает первое задание за 30-40 мин |
| Промпты адекватны | 8 из 10 заданий дают полезный артефакт с первой попытки |

### Этап 2: Бета-тест (5-10 чел) — "Людям полезно"

| Критерий | Метрика |
|----------|---------|
| Доходимость | **60%+** дошли до конца |
| Удовлетворённость | **8+/10** |
| Результат получен | **70%+** с конкретным артефактом (MVP или диагноз) |
| Куратор справляется | 1 куратор на 10 чел без перегрузки |
| Ваня не нужен | 0 обращений к Ване |

### Этап 3: Первый поток (50 студ) — "Бизнес работает"

| Критерий | Метрика |
|----------|---------|
| Набор | 50 студентов за 2-3 недели продаж |
| Выручка | 3M+ за поток |
| NPS | 8+/10 |
| Доходимость | 50%+ |
| Upsell | 10%+ купили второй курс или Boost Pro |
| Подписка AURA | 15%+ подписались после курса |
