# Идеи: КДП с 60-го потока — обучение через VS Code + Claude Code

Цель: студенты учатся создавать продукты в разы быстрее, используя код и AI-агентов как основной инструмент продакт-менеджера.

---

## 1. Конкурентный анализ агентами

- **Автоматический ресерч конкурентов**: настроить агента (OpenClaw / Claude Code), который каждый день обходит сайты и приложения конкурентов — прокликивает онбординги, фичи, тарифы, записывает скриншоты и изменения
- **Aura MCP для анализа сегментов конкурентов**: подключить Aura по MCP, чтобы агент автоматически определял, на какие сегменты целятся конкуренты, какие работы (jobs) они закрывают, и находил незакрытые ниши
- **Мониторинг изменений**: агент отслеживает изменения на сайтах конкурентов (новые фичи, изменения цен, новые лендинги) и присылает дайджест

## 2. Продуктовые требования и стратегия

- **PRD через локальный скилл + Aura MCP**: студенты пишут продуктовые требования (PRD) через скилл product-requirements — агенты анализируют сегменты, риски, конкурентов, а Aura MCP хранит и структурирует результаты
- **One-pager кодом**: агент генерирует одностраничный документ продукта (one-pager) из PRD — краткое описание проблемы, решения, сегментов, метрик успеха, подходит для питча стейкхолдерам и инвесторам

## 3. Исследования и интервью

- **Автотранскрипция + AJTBD-анализ**: скилл, который транскрибирует записи интервью и автоматически анализирует их по Advanced Jobs to be Done — в первую очередь выделяет сегменты
- **Выделение сегментов из транскрипции**: отдельный скилл / агент, который берёт транскрипцию и извлекает: сегменты, работы (jobs), переключающие факторы (switching triggers), барьеры, привычки, эмоциональные и социальные работы
- **Автоматическое построение графа работ**: из набора интервью агент строит граф работ (job map) — визуализирует связи между работами, выделяет кластеры и приоритетные области
- **AI-модератор для интервью**: агент, который в реальном времени подсказывает интервьюеру уточняющие вопросы на основе AJTBD-методологии (например, через наушник или второй экран)
- **Синтез инсайтов по всем интервью**: агент, который после серии интервью агрегирует данные, находит паттерны, формирует гипотезы ценности и ранжирует сегменты по привлекательности

## 4. Работа с данными (на базе воркшопа Boost)

Уже есть готовый воркшоп в `Boost-Curicculum/4-Working-with-data/`. Адаптировать для КДП:

- **LLM vs скрипт — когда что использовать**: студенты на практике видят, что LLM галлюцинирует цифры, а Python-скрипт считает детерминированно. Ключевой паттерн: LLM пишет скрипт, скрипт считает данные
- **Анализ воронки подписок через скрипт**: построение воронки, поиск аномалий, когортный анализ
- **ABCD-сегментация через скрипт**: объединение данных из нескольких источников (отзывы + подписки + exit-survey), подсчёт LTV и прибыли по сегментам
- **Выделение работ (jobs) из отзывов по AJTBD**: LLM анализирует текст отзывов, выделяет работы для каждого ABCD-сегмента
- **Исследование конкурентов через WebSearch**: автоматическое исследование конкурентов с матрицей сравнения
- **Исследование трендов рынка**: размер рынка, темпы роста, инвестиционная активность — за минуты вместо дней
- **Стратегия отстройки от конкурентов**: матрица "работы × конкуренты", фокусные работы, рекомендации по фичам
- **Визуализация и отчёты кодом**: HTML-дашборды, отчёты для руководителя в разных форматах (Slack, MD, PDF)

## 5. Прототипирование и дизайн

- **Прототипы для UX-тестов кодом**: студенты создают кликабельные HTML-прототипы через Claude Code вместо Figma — быстрее, интерактивнее, можно сразу деплоить
- **Создание прототипов по макетам из Figma**: подключить Figma MCP, чтобы агент брал макеты из Figma и генерировал рабочий HTML/CSS/JS код
- **HTML-лендинги в коде**: вместо конструкторов студенты создают лендинги кодом через Claude Code — полный контроль, кастомизация, деплой на Vercel/Netlify в один клик
- **Генерация вариантов лендингов для A/B-тестов**: агент создаёт несколько вариантов лендинга с разными ценностными предложениями для разных сегментов

## 6. UX-тестирование агентами

- **UX-тесты агентами**: AI-агент проходит прототип как пользователь из конкретного сегмента, записывает, что непонятно, где застревает, что ожидает увидеть
- **UX-тестирование кодом**: написать скрипт, который прогоняет сценарии пользователя через прототип, замеряет время на задачу, собирает метрики юзабилити
- **Мультисегментное тестирование**: агент тестирует один прототип от лица разных персон/сегментов и сравнивает результаты

## 7. QA и тестирование

- **QA-тесты агентами**: агент прокликивает MVP, проверяет все пользовательские сценарии, находит баги и UX-проблемы
- **QA-тестирование кодом**: автоматические end-to-end тесты (Playwright/Cypress), которые Claude Code пишет и запускает

## 8. MVP и деплой

- **Релиз MVP кодом**: студенты создают и деплоят рабочие MVP через Claude Code — от бэкенда до фронтенда
- **Шаблоны MVP для типовых продуктов**: готовые стартовые шаблоны (SaaS, маркетплейс, мобильное приложение, Telegram-бот), которые Claude Code кастомизирует под конкретный продукт студента

## 9. Граф работ и механики роста

- **Граф работ в коде**: агент строит интерактивный граф работ (job map) из данных интервью — визуализация связей, кластеров, приоритетов
- **Применение механик роста ценности**: по графу работ агент предлагает конкретные механики роста (cross-sell, upsell, расширение цепочки работ) и генерирует гипотезы фич для каждой механики
- **Автоматическая приоритизация бэклога**: агент берёт граф работ + данные сегментов и приоритизирует фичи по impact/effort с учётом AJTBD

## 10. Маркетинг и тексты

- **Генерация ценностных предложений по сегментам**: агент берёт данные интервью и генерирует тексты для каждого сегмента — заголовки, описания, CTA
- **Генерация текстов лендинга**: из PRD + данных интервью агент создаёт полные тексты для лендинга — hero-секция, преимущества, социальное доказательство, FAQ, CTA — отдельно для каждого сегмента
- **Генерация воронки**: агент проектирует полную воронку привлечения — от рекламного объявления через лендинг до онбординга и первой оплаты, с текстами на каждом шаге
- **Автоматическое создание рекламных объявлений**: по результатам AJTBD-анализа агент генерирует объявления для Яндекс.Директ, VK Ads, Telegram Ads — с учётом переключающих факторов каждого сегмента
- **Автоматическая генерация email-цепочек**: агент создаёт серии писем для онбординга и удержания, привязанные к конкретным работам пользователей
- **Теги и оценки ценностных предложений**: агент автоматически тегирует предложения (по сегменту, работе, этапу воронки) и скорит их по силе переключающих факторов

## 11. Аналитика и метрики

- **Дашборды кодом**: студенты создают аналитические дашборды через Claude Code (Streamlit / HTML) вместо Google Sheets — подключение к реальным данным продукта
- **Автоматический трекинг продуктовых метрик**: агент настраивает события аналитики (Amplitude, Mixpanel, PostHog) и строит воронки по ключевым сценариям
- **Автоматический анализ отзывов**: агент парсит отзывы из App Store, Google Play, соцсетей, маркетплейсов — выделяет работы, боли, сегменты

## 12. Процессы и воркфлоу

- **Aura как продуктовый хаб**: подключить Aura по MCP как центральное место для хранения сегментов, графа работ, гипотез — агенты читают и пишут туда данные
- **Автоматические weekly-отчёты**: агент собирает данные из Aura, аналитики и трекера и генерирует еженедельный продуктовый отчёт
- **CI/CD для продуктовых гипотез**: настроить пайплайн: гипотеза → прототип → тест → метрики → решение — максимально автоматизированный через агентов

## 13. Ещё идеи для автоматизации

- **Автоматический парсинг и анализ отзывов конкурентов**: агент собирает отзывы конкурентов из App Store, Google Play, отзовиков — выделяет незакрытые работы, на которые можно таргетировать свой продукт
- **Генерация пользовательских историй (user stories)**: из графа работ и PRD агент генерирует user stories для разработчиков с критериями приёмки
- **Автоматическое ТЗ для дизайнера**: агент берёт PRD + user stories и генерирует ТЗ на дизайн с описанием экранов, сценариев, edge cases
- **Скрипт для расчёта unit-экономики**: студенты задают CAC, LTV, конверсии — скрипт считает точку безубыточности, прогнозирует выручку, строит графики
- **Автоматическое A/B-тестирование текстов**: агент генерирует варианты текстов, скрипт запускает их в рекламных кабинетах через API и собирает результаты
- **Анализ поисковых запросов**: агент парсит Wordstat / Google Trends, выделяет запросы по работам (jobs) из графа работ, оценивает объём спроса
- **Генерация скриптов продаж**: по данным интервью агент создаёт скрипты для отдела продаж — с обработкой возражений, привязанных к барьерам переключения из AJTBD
- **Автоматический мониторинг NPS и CSAT**: скрипт собирает оценки, считает NPS/CSAT, алертит при падении, выделяет root cause из текстовых комментариев
- **Customer journey map кодом**: агент строит интерактивную CJM в HTML — точки контакта, эмоции, работы на каждом этапе, болевые точки
- **Автоматическая локализация**: агент переводит лендинг, интерфейс и маркетинговые материалы на другие языки с адаптацией под культурный контекст
