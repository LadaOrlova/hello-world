# Дизайн экзамена: "Как делать продукт"

## Цель экзамена

Подтвердить, что студент усвоил методологию Advanced JTBD на уровне, достаточном для самостоятельного применения. Результат — выдача сертификата об окончании тренинга.

## Целевые показатели

- Ожидаемая проходимость: 60–80%
- Количество студентов: ~30 человек
- Общее время: 2.5–3 часа

## Структура экзамена

### Часть 1: Теория (40 минут)

| Параметр | Значение |
|----------|----------|
| Количество вопросов | 25 |
| Время | 40 минут |
| Формат | Онлайн-форма на сайте |
| Проведение | Все студенты одновременно |
| Допуск к Части 2 | Автоматический (независимо от результата) |

**Типы вопросов:**

| Тип | Кол-во | Описание |
|-----|--------|----------|
| Multiple choice / multiple select | 12 | Закрытые вопросы с одним или несколькими верными ответами |
| Короткий открытый ответ | 5 | 2–3 предложения, свободный текст |
| Мини-кейс | 5 | Ситуация + выбор/описание подхода |
| "Найди ошибку" | 3 | Неправильная формулировка работы, сегментация, применение фреймворка |

**Распределение по темам:**

| Тема | Кол-во |
|------|--------|
| AJTBD-основы, формула работы, принципы | 4 |
| Граф работ: Core/Big/Small Job, критическая цепочка | 4 |
| Ценность и механики создания ценности | 3 |
| Сегментация по работам + ABCDX | 3 |
| RAT и валидация | 2 |
| Алгоритмы бизнес-задач: конверсия, ретеншн, чек, отток | 4 |
| Интервью: AJTBD, Switch, B2B | 3 |
| Коммуникация и позиционирование | 2 |

### Часть 2: Практика — AI-кейс (2 часа)

| Параметр | Значение |
|----------|----------|
| Формат | Работа с AI-ботом через чат на сайте |
| Бизнес-кейс | B2C — сеть химчисток «Васильки» (8 точек, Москва) |
| Бизнес-задача | CEO: «Починить маркетинг». Реальная задача: диагностика + репозиционирование |
| Время | 2 часа |
| Проведение | Все студенты одновременно |
| Результат | Структурированный ответ из 5 блоков |

**Что сдаёт студент:**

1. Диагностика проблемы: почему маркетинг не работает (корневая причина, разбор идей CEO)
2. ABCDX-сегментация: кто A, кто D и почему (с причинными критериями)
3. Граф работ целевого (A) сегмента: Core Job → Big Job, Small Jobs
4. Стратегия репозиционирования: фокус, one-liner, 3–5 конкретных изменений
5. RAT: 3 рискованных предположения, эксперимент для проверки первого

**Данные, доступные студенту:**
- 30 интервью с клиентами (в тексте кейса) — премиум, скидочники, B2B вперемешку
- Метрики бизнеса (чек, LTV, CAC, retention — средние по базе)
- Юнит-экономика (средний заказ)
- Текущее позиционирование и маркетинг
- Данные о конкурентах

**Данные в контексте бота (CEO):**
- Всё из текста кейса + личная перспектива CEO
- ABCDX-сегментация (предоставляется по запросу)
- Юнит-экономика по сегментам (предоставляется по запросу)
- Маркетинговые каналы с детализацией по бюджетам и CAC
- Вводящие в заблуждение идеи CEO (сменить агентство, больше скидок, B2B-продажники)

## Правила проведения

### Общие правила

1. Экзамен проводится онлайн, на единой веб-странице
2. Студент вводит своё имя перед началом
3. Переход между частями — автоматический после завершения таймера или по кнопке "Завершить часть"
4. После отправки ответов изменить их нельзя

### Часть 1: Теория

- Таймер: 40 минут, обратный отсчёт на экране
- Разрешено: конспекты, записи, книга курса
- Запрещено: помощь других людей, AI-ассистенты
- При истечении таймера — автоматическая отправка текущих ответов

### Часть 2: Практика

- Таймер: 2 часа, обратный отсчёт на экране
- AI-бот играет роль CEO компании, отвечает на вопросы о бизнесе, но НЕ даёт готовых решений
- Разрешено: конспекты, записи, книга курса
- Запрещено: помощь других людей, внешние AI-ассистенты
- Финальный ответ вводится в структурированную форму
- При истечении таймера — автоматическая отправка текущих ответов

### Пересдача

- Возможна 1 пересдача в течение 2 недель
- Формат аналогичный, вопросы частично изменены
- Проходной балл тот же

## Техническая реализация

### Стек

- Статический HTML/CSS/JS (без фреймворков)
- Tailwind CSS для стилей
- Supabase для хранения ответов и авторизации
- OpenAI/Anthropic API для чат-бота (через серверную функцию)

### Функциональность страницы

1. **Экран входа**: Ввод имени студента
2. **Часть 1**: Форма с вопросами + таймер 40 мин
3. **Часть 2**: Чат-бот + форма для ответа + таймер 2 часа
4. **Экран завершения**: Подтверждение отправки
5. **Дашборд преподавателя**: Список студентов, баллы, статусы

### Сбор данных

- Имя студента
- Время начала и окончания каждой части
- Ответы на каждый вопрос теории
- Лог чата с ботом
- Финальный структурированный ответ практики
- Автоматический расчёт баллов за теорию
