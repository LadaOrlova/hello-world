# INV-07: Сотрудник отдела продаж может самостоятельно сформировать счёт или ссылку на оплату для B2B-клиента, не привлекая финансового менеджера для каждого запроса

**Статус в прототипе:** Работает

## Параметры реализации

### Критерии приёмки

- [ ] Менеджер продаж может создать счёт для B2B-клиента через админку без участия финансового менеджера.
- [ ] При создании счёта система автоматически формирует документ (включая PDF) и сохраняет его в реестре документов компании.
- [ ] Менеджер может сформировать счёт в контексте B2B-заявки или компании на выбранный продукт, поток и тариф.
- [ ] В рамках B2B-flow менеджер может подтвердить оплату и запустить последующую автоматическую обработку (зачисления и онбординг).
- [ ] Сквозной сценарий «создать счёт -> получить документ -> подтвердить оплату -> запустить зачисление» выполняется операционно силами sales-функции.
- [ ] Все ключевые действия менеджера по счёту и оплате фиксируются в системе и доступны для последующего контроля.
- [ ] Для штатного B2B-сценария не требуется ручное подключение финансового менеджера на каждом заказе.

## Что работает в прототипе

Самостоятельное формирование документов менеджером:

1. Менеджер открывает карточку компании в админке (`/admin/companies/[id]`)
2. Менеджер переходит в раздел счетов и нажимает «Создать счёт»
3. Система автоматически генерирует PDF-счёт с реквизитами компании через фоновую задачу
4. При одобрении B2B-заявки менеджер может сразу сформировать счёт и договор
5. Менеджер подтверждает оплату — система автоматически зачисляет студентов
6. Весь процесс от формирования счёта до зачисления не требует участия финансового менеджера

### Пользовательский опыт

- Быстрое создание: формирование счёта в один клик на основе предыдущего

### Расширенные сценарии

- Лимиты и согласования: счета выше определённой суммы требуют подтверждения руководителя
- Автоматическое напоминание клиенту о неоплаченном счёте
