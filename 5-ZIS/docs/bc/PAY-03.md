# PAY-03: Менеджер видит в единой админке все транзакции пользователя вне зависимости от того, через какой продукт была совершена оплата

**Статус в прототипе:** Работает

## Параметры реализации

### Критерии приёмки

- [ ] В админке доступен единый раздел биллинга `/admin/billing` со списком заказов по всем продуктам.
- [ ] Для каждого заказа отображаются минимум: дата, пользователь, продукт, тариф, сумма, статус заказа.
- [ ] Для каждого заказа отображаются связанные платежи и их статусы.
- [ ] Менеджер может найти все транзакции конкретного пользователя через поиск или фильтрацию в биллинге.
- [ ] В карточке пользователя `/admin/users/[id]` отображается консолидированная история его заказов и платежей.
- [ ] Транзакции пользователя агрегируются межпродуктово в одном контуре админки, без необходимости переходить в отдельные системы.
- [ ] Ограничение текущего прототипа по фильтрам (по продукту и статусу оплаты) зафиксировано как follow-up и не блокирует приёмку базового сценария.

## Что работает в прототипе

Единый раздел биллинга в админке:

1. Менеджер открывает раздел «Биллинг» в админке (`/admin/billing`)
2. Менеджер видит таблицу всех заказов со всех продуктов: дата, студент, продукт, тариф, сумма, статус
3. Каждый заказ показывает привязанные платежи с их статусами
4. Менеджер может отфильтровать заказы и найти все транзакции конкретного пользователя
5. В карточке пользователя (`/admin/users/[id]`) менеджер также видит все заказы и платежи этого пользователя в одном месте
