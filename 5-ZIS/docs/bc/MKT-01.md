# MKT-01: UTM-трекинг и атрибуция — система сохраняет источник трафика (UTM-метки и referer) при создании лида или заказа для последующей маркетинговой аналитики

**Статус в прототипе:** Не реализовано

> **Источник:** Требования Лады (Владлены Орловой), маркетинг, 2026-03-13.

## Параметры реализации

### Критерии приёмки (MVP — только хранение данных)

- [ ] При создании лида (из любого источника: форма, API, TypeForm webhook, бот) система сохраняет UTM-параметры, если они были переданы: `utm_source`, `utm_medium`, `utm_campaign`, `utm_content`, `utm_term`.
- [ ] При создании лида сохраняется `referrer_url` (полный URL страницы-источника), если он доступен.
- [ ] При создании заказа (B2C checkout) UTM-параметры из сессии пользователя сохраняются в данных заказа.
- [ ] UTM-поля доступны в карточке лида и в карточке заказа в админке (отображаются как справочная информация).
- [ ] Публичный API для приёма лидов (LEAD-01) принимает UTM-параметры как часть запроса.
- [ ] Данные не теряются: если UTM были в URL — они сохраняются в БД навсегда.

## Что нужно реализовать в прототипе (MVP)

Минимальный сценарий — сохранение UTM-данных:

- [ ] Добавить поля в модель лида: `utmSource`, `utmMedium`, `utmCampaign`, `utmContent`, `utmTerm`, `referrerUrl`, `channel` (авто)

- [ ] На лендингах / формах / checkout пробрасывать UTM-параметры из URL в данные формы
- [ ] В карточке лида и заказа в админке отображать UTM-данные (read-only)
