# LMS-06: Команда может добавить в LMS новую функциональную страницу (например, страницу группы со статистикой) и опубликовать её самостоятельно — через вайб-кодинг, без необходимости ждать разработчика

**Статус в прототипе:** Не реализовано

## Параметры реализации

### Критерии приёмки

- [ ] В LMS реализован механизм добавления новой функциональной страницы в рамках существующего контура ZIS (без создания отдельного сервиса и отдельной БД).
- [ ] Команда может создать и подключить новую страницу по шаблону или гайду «вайб-кодинга» без архитектурных изменений ядра LMS.
- [ ] Новая страница может читать необходимые данные LMS через существующий API-контур (минимум: прогресс, группа или когорта, пользователь в рамках прав доступа).
- [ ] Новая страница публикуется и становится доступной целевой роли в LMS после стандартной процедуры публикации в ZIS.
- [ ] Ролевой доступ на новой странице соблюдается: студент видит только свои данные или допустимую агрегацию, куратор и менеджер — только в рамках разрешений.
- [ ] В качестве приемочного примера реализована и доступна одна рабочая страница (например, статистика группы) с реальными данными.
- [ ] Подготовлен короткий практический гайд для команды: как добавить следующую страницу по тому же процессу.

## Что нужно реализовать в прототипе

Минимальный сценарий расширяемости LMS:

- [ ] Архитектура системы позволяет быстро добавить новую страницу с помощью AI-инструментов (вайб-кодинг)
- [ ] Новая страница получает доступ к данным системы: пользователи, прогресс, группы
- [ ] Страница публикуется и становится доступна студентам без развёртывания новой версии системы
- [ ] Пример: страница группы со статистикой — средний прогресс, лидеры, общие показатели

### Что уже есть в платформе

- Модульная архитектура на Next.js — добавление новой страницы технически несложно
- API для чтения данных: прогресс, зачисления, пользователи — уже работают
- Ролевой доступ — новая страница автоматически защищена системой ролей
- tRPC роутеры — стандартизированный способ создания новых API-эндпоинтов
