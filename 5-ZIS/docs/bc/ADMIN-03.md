# ADMIN-03: Сотрудник без технической экспертизы может выполнить типовую операцию (добавить запись, изменить статус, выгрузить данные) без помощи разработчика

**Статус в прототипе:** Частично

## Параметры реализации

### Критерии приёмки

- [ ] Сотрудник без разработки может создавать базовые сущности через AdminUI (минимум: пользователь, продукт, тариф или поток, заказ, зачисление).
- [ ] Сотрудник может выполнять типовые изменения статусов через интерфейс (минимум: лиды, заказы или зачисления в пределах доступных действий).
- [ ] Сотрудник может управлять LMS-контентом через интерфейс (создание, редактирование, деактивация уроков) без участия разработчика.
- [ ] Сотрудник может выполнять типовые операции по B2B-заявкам и компаниям через интерфейс.
- [ ] Для ключевых операций система предоставляет валидируемые формы и пользовательские сообщения, достаточные для выполнения действий без техподдержки разработчика.
- [ ] По выполненным действиям сохраняются изменения в данных и они сразу отражаются в соответствующих разделах админки.
- [ ] Ограничение текущего прототипа по выгрузке CSV или Excel зафиксировано как follow-up и не блокирует приёмку базового сценария self-service операций.

## Что работает в прототипе

Типовые операции через интерфейс:

1. Менеджер создаёт пользователя: вводит имя, email, роль — через форму в админке
2. Менеджер создаёт продукт с тарифами и потоками — через формы в соответствующих разделах
3. Менеджер создаёт заказ и получает ссылку на оплату — через раздел «Биллинг»
4. Менеджер зачисляет студента на курс — через раздел «Зачисления»
5. Менеджер управляет контентом курса: создаёт, редактирует, скрывает уроки — через раздел «LMS»
6. Менеджер меняет статусы заявок — через раздел «Лиды»
7. Менеджер просматривает и модерирует заявки компаний — через раздел «Компании»
8. Все операции выполняются через понятные формы — техническая экспертиза не нужна
