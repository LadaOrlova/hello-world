# LMS-07: Онбординг студента в новый поток (добавление в чаты, предоставление доступов) происходит автоматически после подтверждения оплаты

**Статус в прототипе:** Работает

## Параметры реализации

### Критерии приёмки

- [ ] После подтверждения оплаты автоматически запускается onboarding-процесс студента для соответствующего потока.
- [ ] В onboarding автоматически выполняется выдача LMS-доступа: курс появляется в личном кабинете студента.
- [ ] Для студента создаётся и доступна onboarding-страница с актуальными шагами.
- [ ] В рамках onboarding студенту назначается куратор в контексте его потока.
- [ ] После привязки Telegram студент получает доступ к чатам потока (минимум: отображаются рабочие invite-ссылки).
- [ ] Команда получает уведомление о новом студенте и запуске onboarding.
- [ ] Ключевые шаги onboarding фиксируются в системе по статусам, чтобы команда могла проверить завершённость процесса.
- [ ] Базовый onboarding выполняется без ручного вмешательства менеджера в штатном сценарии.

## Что работает в прототипе

Полностью автоматический онбординг после оплаты:

1. Студент оплачивает курс
2. Система получает подтверждение оплаты через webhook
3. Автоматически запускается процесс онбординга из 5 шагов:
   - Предоставление доступа к LMS — курс появляется в личном кабинете
   - Создание страницы онбординга — студент видит приветственную страницу с инструкциями
   - Подписка на календарь — заглушка для будущего расписания занятий
   - Назначение куратора — студент привязывается к куратору группы
   - Приветственное сообщение — отправка через Telegram (если привязан)
4. Студент заходит на страницу онбординга и привязывает Telegram-аккаунт
5. После привязки студент видит список Telegram-чатов курса с invite-ссылками
6. Менеджер получает уведомление о новом студенте
7. Весь процесс происходит без ручного вмешательства
