# MKT-04: Воронка конверсии — маркетолог видит путь пользователя от первого контакта до покупки и может анализировать, где теряются клиенты

**Статус в прототипе:** Не реализовано

> **Источник:** Требования Лады (Владлены Орловой), маркетинг, 2026-03-13.

## Параметры реализации

### Критерии приёмки

- [ ] В системе доступна визуализация воронки: первый контакт → лид → заказ → оплата → зачисление.
- [ ] По каждому этапу видно количество пользователей и конверсия в следующий этап.
- [ ] Воронку можно фильтровать по продукту, потоку, периоду, каналу (utm_source).
- [ ] Доступны данные по среднему времени прохождения каждого этапа.

> **Решение (2026-03-17):** Не MVP. Данные для воронки уже есть в БД (leads → orders → payments → enrollments). Визуализация воронки — вайб-кодинг поверх данных из ZIS. В ZIS на v2 можно добавить timeline пользователя (хронологическая история всех событий) как расширение AUTH-03.

## Что уже есть в платформе

- Полная цепочка данных: Lead → Order → Payment → Enrollment
- События в Event Log (с фильтрами по типу, источнику, пользователю)
- UTM-данные по лидам (MKT-01)
- Карточка пользователя 360 (AUTH-03) — все данные в одном месте

## Что нужно для реализации

### Итерация 1 (v2): Timeline пользователя

- В карточке пользователя (AUTH-03) добавить хронологическую шкалу: все события по пользователю в порядке времени — регистрация, лид, заказ, оплата, зачисление, прогресс

### Итерация 2: Визуализация воронки

- Кастомный инструмент маркетолога (вайб-кодинг) — агрегация данных из ZIS по этапам воронки
- Дашборд с графиками: воронка × продукт × период
