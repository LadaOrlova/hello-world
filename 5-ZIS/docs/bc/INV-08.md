# INV-08: Менеджер может вручную запустить онбординг для B2B-студентов до поступления оплаты — по гарантийному письму или согласованной постоплате

**Статус в прототипе:** Частично

## Параметры реализации

### Критерии приёмки

- [ ] Менеджер может запустить ручное зачисление B2B-студентов до фактического поступления оплаты.
- [ ] Ручное зачисление поддерживается как для одного студента, так и в batch-режиме для группы.
- [ ] После ручного зачисления запускается стандартный onboarding (доступы LMS, бот и прочие шаги по текущему flow).
- [ ] Для зачисления без оплаты обязательно фиксируется основание (минимум: гарантийное письмо, постоплата, иное согласованное).
- [ ] Основание зачисления сохраняется в системе и доступно команде в карточке зачисления или заявки.
- [ ] Ручное досрочное зачисление не ломает последующую обработку оплаты и может быть связано с оплатой постфактум.
- [ ] Команда может отфильтровать или отследить зачисления, выполненные без подтверждённой оплаты, для операционного контроля.

## Что работает в прототипе

Ручное зачисление студентов менеджером:

1. Менеджер может зачислить студентов на курс через раздел «Зачисления» в админке
2. Менеджер может использовать массовое зачисление (batch enrollment) для нескольких студентов одновременно
3. После зачисления автоматически запускается полный онбординг: доступ к LMS, Telegram-чаты, приветствие
4. Зачисление не привязано к оплате — менеджер может зачислить студента без заказа

## Что нужно доработать в прототипе

- [ ] Добавить фиксацию основания для зачисления без оплаты (гарантийное письмо, постоплата)

## Что может быть на следующих этапах

### Пользовательский опыт

- Поле «Основание» при зачислении: выбор из списка (предоплата, гарантийное письмо, постоплата)
- Напоминание менеджеру о неоплаченных зачислениях

### Расширенные сценарии

- Добавить связь между зачислением и будущим счётом — чтобы после оплаты данные автоматически связались
- Автоматический отзыв доступа, если оплата не поступила в оговорённый срок
- Отчёт по зачислениям с отсрочкой оплаты
