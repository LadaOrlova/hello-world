# DEDUP-01: При оформлении заказа пользователь автоматически регистрируется в системе (или привязывается к существующему аккаунту по email), чтобы все его попытки оплаты, брошенные корзины и завершённые покупки были в одной карточке

**Статус в прототипе:** Работает

## Параметры реализации

### Критерии приёмки

- [ ] При оформлении заказа по email система определяет: новый это пользователь или уже существующий аккаунт.
- [ ] Для неавторизованного пользователя с уже существующим email система требует авторизацию перед продолжением чувствительных действий.
- [ ] Для нового email создаётся checkout-сценарий без дублирования аккаунтов (guest или новый пользователь в рамках текущей модели).
- [ ] Повторные попытки оплаты по одному и тому же email+потоку не создают дублирующие pending-заказы.
- [ ] После успешной оплаты заказ корректно привязывается к существующему пользователю по email или создаётся новый пользователь, если его не было.
- [ ] История попыток и финальная покупка остаются связаны с одним пользовательским контуром данных (без распада на несвязанные карточки).

## Что уже работает в прототипе

Минимальный сценарий дедупликации в checkout:

- [x] При оформлении заказа пользователь вводит email
- [x] Если аккаунт с таким email уже существует и пользователь не авторизован этим аккаунтом — система требует авторизацию (`requires_auth`)
- [x] Если email новый — создаётся guest-заказ (`userId = null`, `guestEmail = email`) и создаётся попытка оплаты
- [x] Повторные попытки оплаты для того же email+cohort переиспользуют/обновляют существующий pending-заказ (без дублей заказа)
- [x] После успешной оплаты guest-заказ автоматически привязывается к существующему пользователю по email или создаётся новый пользователь

### Что уже есть в платформе

- Единый аккаунт по email — аккаунт не дублируется при авторизации
- Дедупликация лидов в CRM — повторные заявки не создают дубль-карточки
- Checkout с защитой от дублей pending/paid заказов на один и тот же поток
- Автопривязка guest-заказа к user после `payment.succeeded`
