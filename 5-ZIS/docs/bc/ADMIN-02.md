# ADMIN-02: Каждая роль (продажи, контент, финансы, кураторы) видит только релевантный для неё срез данных и инструменты

**Статус в прототипе:** Частично

## Параметры реализации

### Критерии приёмки

- [ ] В системе применено ролевое разделение доступа минимум для ролей: `student`, `curator`, `manager`, `admin`.
- [ ] Студент имеет доступ только к своему личному кабинету и своим данным; доступ к админке и кураторскому разделу запрещён.
- [ ] Куратор имеет доступ только к своему кураторскому контуру и данным назначенных групп; доступ к админке запрещён.
- [ ] Менеджер и администратор имеют доступ к административному контуру согласно текущей матрице прав прототипа.
- [ ] API-эндпоинты защищены RBAC-проверками: запрос к данным вне роли или периметра возвращает отказ, а не данные.
- [ ] При попытке открытия неразрешённого раздела пользователь получает корректный отказ или редирект в разрешённый контур.
- [ ] Ограничение текущего прототипа по гранулярности подролей внутри админки (sales, content, finance) зафиксировано как follow-up.

## Что работает в прототипе

Базовое разделение доступа по ролям:

1. Администратор и менеджер видят все разделы админки
2. Куратор видит только свой кабинет с группами и прогрессом студентов — админка ему недоступна
3. Студент видит только свой личный кабинет — ни админка, ни кабинет куратора ему не доступны
4. Все API-вызовы защищены проверкой ролей: попытка доступа к чужим данным блокируется
5. При попытке зайти в чужой раздел пользователь перенаправляется на свою страницу

## Что нужно доработать в прототипе

- [ ] Разделить интерфейс админки по подролям: менеджер по продажам видит только лидов и биллинг, контент-менеджер — только LMS и продукты
- [ ] Скрыть из бокового меню разделы, к которым у сотрудника нет доступа

## Что может быть на следующих этапах

- Настраиваемые роли: создание кастомных ролей с гибким набором прав
- Разные стартовые страницы для разных ролей
- Аудит доступов: кто к каким данным обращался
- Временные роли: предоставить доступ к разделу на определённый срок
- Делегирование: менеджер временно передаёт свои задачи коллеге
