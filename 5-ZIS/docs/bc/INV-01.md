# INV-01: B2B-клиент может самостоятельно сформировать счёт на оплату через публичную форму: указать ИНН (реквизиты подтягиваются из ЕГРЮЛ), выбрать продукт, поток и тариф — и получить PDF-счёт

**Статус в прототипе:** Частично

## Параметры реализации

### Критерии приёмки

- [ ] B2B-клиент может инициировать оформление юрлицевого заказа из публичного контура и перейти в B2B-форму без участия менеджера.
- [ ] В B2B-форме клиент указывает реквизиты компании и выбирает продукт, поток и тариф.
- [ ] После отправки формы создаётся заявка на B2B-оформление, доступная менеджеру для обработки.
- [ ] После одобрения заявки менеджером в ZIS создаются связанные сущности: компания, счёт (и связанный договор или документы по текущему flow).
- [ ] PDF-счёт формируется автоматически и доступен в кабинете компании в разделе документов.
- [ ] Клиент может получить и использовать сформированный счёт без ручной сборки документов командой.
- [ ] Автоподтягивание реквизитов по ИНН (ЕГРЮЛ) в текущем прототипе зафиксировано как follow-up и не блокирует базовую приёмку сценария.

## Что работает в прототипе

B2B-клиент самостоятельно оформляет заявку:

1. B2B-клиент заходит на страницу `/products`, выбирает у нужного продукта "Оплатить как юрлицо" и переходит к регистрации компании
2. Клиент заполняет форму: ИНН, КПП, юридический адрес, контактное лицо
3. Клиент выбирает продукт, поток и тариф для своих сотрудников
4. Заявка отправляется на рассмотрение менеджеру
5. Менеджер одобряет заявку — система автоматически создаёт компанию, договор и счёт
6. PDF-счёт генерируется автоматически через фоновую задачу
7. Документы доступны клиенту в личном кабинете компании (`/company/documents`)

## Что может быть на следующих этапах

### Интеграции

- Интеграция с DaData или аналогичным сервисом для автозаполнения реквизитов по ИНН
- Интеграция с 1С для автоматической синхронизации документов

### Пользовательский опыт

- Мгновенное формирование счёта без ожидания одобрения менеджера (для типовых заказов)
- Статус-бар: клиент видит, на каком этапе находится его заявка

### Расширенные сценарии

- Повторный заказ: клиент формирует новый счёт с теми же реквизитами одной кнопкой
- Групповые заказы: несколько продуктов в одном счёте
