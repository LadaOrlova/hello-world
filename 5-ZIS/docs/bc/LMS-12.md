## Параметры реализации

### Критерии приёмки

- [ ] Администратор/куратор может создать квиз к уроку: набор вопросов с вариантами ответов.
- [ ] Поддерживаются типы вопросов: выбор одного ответа, выбор нескольких, свободный текст.
- [ ] Студент проходит квиз на странице урока в LMS и видит результат (правильные/неправильные ответы, балл).
- [ ] Результаты квизов сохраняются и доступны куратору в матрице прогресса.
- [ ] Куратор видит агрегированную статистику по квизу: средний балл, распределение ответов.

## Черновик модели данных

```
Quiz
├── id
├── lessonId → Lesson
├── title
├── passingScore? (минимальный балл для «зачёт»)
├── order

QuizQuestion
├── id
├── quizId → Quiz
├── type: single_choice | multiple_choice | free_text
├── text (текст вопроса)
├── options (JSON: варианты ответов с пометкой correct)
├── order

QuizAttempt (попытка прохождения студентом)
├── id
├── quizId → Quiz
├── enrollmentId → Enrollment
├── answers (JSON: ответы студента)
├── score
├── passed: boolean
├── completedAt
```
