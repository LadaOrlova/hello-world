# BOT-02: Студент может узнать информацию о своих доступах, статусе оплаты и прогрессе через единый бот, не обращаясь к куратору

**Статус в прототипе:** Частично

## Параметры реализации

### Критерии приёмки

- [ ] Студент может привязать Telegram-аккаунт к своему профилю ZIS через единый бот.
- [ ] В боте доступна команда или действие «Мои курсы», показывающее актуальные активные доступы и зачисления студента.
- [ ] В боте доступна команда или действие «Мой прогресс», показывающее прогресс по каждому доступному курсу.
- [ ] В боте доступна команда или действие просмотра статуса оплаты по релевантным заказам студента.
- [ ] Ответы бота формируются на основе данных ZIS для привязанного пользователя и не раскрывают данные других студентов.
- [ ] Если Telegram-аккаунт не привязан, бот предлагает понятный flow привязки и не выдаёт персональные данные до завершения привязки.
- [ ] Данные, получаемые через бота соответствуют данным в личном кабинете и админке для того же пользователя.

## Что работает в прототипе

Базовое взаимодействие студента с ботом:

1. Студент отправляет команду `/start` боту в Telegram
2. Бот привязывает Telegram-аккаунт студента к его аккаунту в системе
3. Бот отслеживает, в каких чатах курса состоит студент

## Что нужно доработать в прототипе

- [ ] Добавить команду «Мои курсы»: бот показывает список активных зачислений студента
