# MKT-02: Сегментация базы — маркетолог может фильтровать пользователей и лидов по продукту, статусу, источнику, активности и другим параметрам для таргетированных коммуникаций

**Статус в прототипе:** Не реализовано

> **Источник:** Требования Лады (Владлены Орловой), маркетинг, 2026-03-13.

## Параметры реализации

### Критерии приёмки

- [ ] Маркетолог может создать сегмент из условий: продукт, статус (лид/студент/выпускник), источник (utm_source, utm_campaign), период регистрации, канал коммуникации.
- [ ] Доступны готовые (предустановленные) сегменты: все лиды, активные студенты по продукту, выпускники, подписчики бота без покупки, остывшая база.
- [ ] Маркетолог может создать кастомный сегмент через конструктор условий.
- [ ] Сегменты можно использовать как целевую аудиторию для рассылок (MKT-03).
- [ ] Сегменты обновляются динамически (при изменении данных пользователя он автоматически попадает/выпадает из сегмента).

- SQL-доступ к БД для аналитики

## Готовые сегменты (для реализации)

| Сегмент | Логика |
|---------|--------|
| Все лиды (не купили) | status ≠ converted, нет оплаченных заказов |
| Студенты продукта X (активные) | enrollment.product = X, status = active |
| Выпускники продукта X | enrollment.product = X, status = completed |
| Alumni (все выпускники) | любой enrollment.status = completed |
| Подписчики бота (без покупки) | есть в боте, нет оплаченных заказов |
| Остывшая база | нет активности 30+ дней |
