# LEAD-05: Задачи по лидам — менеджер получает задачи по каждому лиду и видит свой список дел на день

**Статус в прототипе:** Не реализовано

> **Источник:** Фидбек Руслана (продажи/работа с клиентами), 2026-03-17.

## Параметры реализации

### Критерии приёмки

- [ ] В карточке лида менеджер может создать задачу: заголовок, описание, ответственный менеджер, дедлайн.
- [ ] Менеджер видит список своих задач на текущий день (отдельная страница или виджет на дашборде).
- [ ] Из задачи можно быстро перейти в карточку связанного лида.
- [ ] Для всех лидов со статусом `new` автоматически создаётся задача «Связаться с лидом», назначенная на ответственного менеджера.
- [ ] Задача имеет статус: открыта / выполнена / просрочена.
- [ ] На дашборде видна сводка: сколько задач на сегодня, сколько просрочено, по каждому менеджеру.

> **Решение (2026-03-17):** Не MVP. В MVP менеджеры могут решать ту же задачу через существующие инструменты:
>
> - **«Кого обзвонить утром»** = фильтр `/admin/leads?status=new&assignedManagerId=me` → список новых лидов для обзвона.
> - **«Кто давно не контактировал»** = сортировка/фильтр по полю `lastContactedAt` → лиды без недавнего контакта.
> - **Счётчики по статусам** (LEAD-02 MVP) показывают, сколько лидов на каждом этапе.
>
> Задачи становятся нужны, когда команда продаж вырастет и одних фильтров станет мало для координации работы между менеджерами.

```
LeadTask
├── id

├── title

├── dueDate
├── status: open | completed | overdue
├── createdAt
├── completedAt?
```

| Триггер | Задача | Назначение |
|---|---|---|
