# LMS-13: Индивидуальный доступ к материалам — менеджер может точечно открыть или закрыть доступ к конкретному модулю/уроку для отдельного студента

## Параметры реализации

### Критерии приёмки

- [ ] Менеджер может открыть доступ к конкретному модулю для конкретного зачисления (вне рамок тарифа).
- [ ] Менеджер может закрыть доступ к конкретному модулю для конкретного зачисления (даже если тариф его включает).
- [ ] Индивидуальные настройки приоритетнее тарифных: override перекрывает стандартную логику Tier → Module.
- [ ] В карточке зачисления в админке видны индивидуальные override'ы.
- [ ] Студент видит результирующий набор доступов (тариф + override'ы) в LMS.

> **Решение (2026-03-17):** Не MVP. В MVP доступ управляется через тарифы (Tier → Module). Если нужно дать точечный доступ — workaround: создать спец-тариф или зачислить студента дополнительно. Полноценный механизм override'ов — v2.

## Подход к реализации (v2)

Поле `accessOverrides` (JSON) на модели Enrollment:

```json
{
  "grants": ["moduleId_1", "moduleId_2"],
  "revokes": ["moduleId_3"]
}
```

Логика доступа: `(модули из тарифа + grants) - revokes = итоговый набор`.

Альтернатива — отдельная модель AccessGrant (enrollmentId, moduleId, type: grant|revoke, grantedBy, reason) — расширяемее, но сложнее.
