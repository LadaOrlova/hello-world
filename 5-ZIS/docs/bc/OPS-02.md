# OPS-02: Данные пользователей регулярно бэкапируются; в случае блокировки внешнего сервиса данные не теряются

**Статус в прототипе:** Частично

## Параметры реализации

### Критерии приёмки

- [ ] Для основной базы данных ZIS настроен регулярный автоматический backup по расписанию.
- [ ] Зафиксирована и задокументирована политика backup: периодичность, срок хранения, место хранения, ответственные.
- [ ] Команда имеет документированную процедуру восстановления базы из backup с пошаговой инструкцией.
- [ ] Выполнена минимум одна проверка восстановления (test restore) с подтверждением, что критичные данные поднимаются корректно.
- [ ] Недоступность внешних сервисов (например, Telegram и YooKassa) не приводит к потере уже записанных в ZIS доменных данных.
- [ ] Для backup-процесса фиксируется результат выполнения (успех и ошибка) и он доступен команде для контроля.
- [ ] Критичные сущности прототипа (минимум: пользователи, заказы, зачисления) подтверждённо восстанавливаются из backup.

## Что работает в прототипе

Архитектурные основы для защиты данных:

1. Все данные хранятся в одной базе PostgreSQL — единая точка для бэкапа
2. Критичные данные (пользователи, заказы, зачисления) не зависят от внешних сервисов
3. При недоступности YooKassa заказы сохраняются в системе — данные не теряются
4. При недоступности Telegram бота — данные о пользователях и зачислениях остаются в базе

## Что нужно доработать в прототипе

- [ ] Настроить регулярный автоматический бэкап базы данных
- [ ] Проверить и документировать процедуру восстановления из бэкапа

### Что уже есть в платформе

- PostgreSQL — стандартные инструменты бэкапа (pg_dump, WAL archiving)
- Все данные в одной базе — не нужно координировать бэкапы нескольких систем
- Prisma миграции — можно воссоздать схему с нуля

## Что может быть на следующих этапах

### Интеграции

- Автоматический бэкап в облачное хранилище (S3, GCS) по расписанию
- Мониторинг успешности бэкапов

### Расширенные сценарии

- Тестовое восстановление из бэкапа раз в месяц
- Шифрование бэкапов
- Point-in-time recovery для минимизации потерь данных
