# BOT-03: Боты потоков и продуктов не множатся пропорционально числу потоков: один бот обслуживает все потоки продукта, адаптируя ответы к контексту

**Статус в прототипе:** Работает

## Параметры реализации

### Критерии приёмки

- [ ] В системе используется единый экземпляр Telegram-бота для обслуживания нескольких продуктов и потоков.
- [ ] Для каждого подключённого чата хранится контекстная привязка к продукту и потоку.
- [ ] При добавлении или удалении участника в чатах бот корректно обновляет данные членства в ZIS.
- [ ] Подключение нового потока выполняется через настройку или привязку чата к существующему боту, без создания нового бот-приложения.
- [ ] Бот при обработке событий и команд использует контекст текущего чата (продукт и поток), а не глобальные статические настройки.
- [ ] Операционные данные по чатам и членству доступны команде в едином контуре, без раздельных конфигураций «бот на каждый поток».
- [ ] Масштабирование количества потоков не требует линейного роста числа ботов и токенов.

## Что работает в прототипе

Единый бот для всех потоков и продуктов:

1. Один Telegram-бот обслуживает все курсы и потоки системы
2. Бот привязан к чатам через таблицу связей: каждый чат ассоциирован с конкретным потоком или продуктом
3. При добавлении или удалении участника из любого чата бот автоматически обновляет информацию о членстве
4. Создание нового потока не требует создания нового бота — существующий бот добавляется в новый чат
5. Бот знает контекст: к какому продукту и потоку относится каждый чат
