# LMS-04: Куратор видит матрицу прогресса своих студентов: кто что просмотрел, что ещё не начал

**Статус в прототипе:** Работает

## Параметры реализации

### Критерии приёмки

- [ ] Куратор после входа видит список назначенных ему групп (когорт) в кабинете куратора.
- [ ] Для выбранной группы доступна страница прогресса `/curator/lms/[cohortId]`.
- [ ] На странице отображается матрица формата «студент × урок» по курсу выбранной группы.
- [ ] По каждой ячейке матрицы видно статус прохождения урока (пройден или не пройден) для конкретного студента.
- [ ] Куратор может на основе матрицы определить студентов, которые не начали или отстают по прохождению.
- [ ] Куратор видит только данные своих групп в рамках RBAC и назначений, без доступа к чужим группам.

## Что работает в прототипе

Полная матрица прогресса группы для куратора:

1. Куратор входит в систему и попадает в кабинет куратора (`/curator`)
2. Куратор видит список своих групп (потоков)
3. Куратор открывает конкретную группу (`/curator/lms/[cohortId]`)
4. Отображается таблица «студент × урок»: строки — студенты, колонки — уроки курса
5. Для каждого студента видно, какие уроки он прошёл (отмечены), а какие ещё нет
6. Куратор может быстро оценить, кто отстаёт и кому нужна помощь

### Пользовательский опыт

- Добавить сортировку студентов по проценту прогресса (отстающие вверху)

- Уведомления куратору об отстающих студентах

- Экспорт матрицы в Excel для отчётности
