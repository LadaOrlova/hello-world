# ADMIN-01: Команда работает в одной AdminUI, внутри которой видны все модули: пользователи, курсы, оплата, вакансии, LMS, маркетинг. Все существующие отдельные админки (Boost, вакансии Алёны, лендинг КДП) интегрируются в единую систему

## Параметры реализации

### Критерии приёмки

- [ ] В системе доступна единая точка входа в административный интерфейс (`/admin`) с общей авторизацией.
- [ ] В единой AdminUI доступны и рабочие базовые модули прототипа: Dashboard, Users, Companies, Products, Billing, Enrollments, LMS, Leads, Events.

- [ ] Данные модулей отображаются консистентно в едином административном контуре (единые сущности пользователей, продуктов, заказов, зачислений, лидов).

- [ ] Для модулей, не доведённых в прототипе (вакансии, Boost, marketing), зафиксирован статус интеграции как открытый follow-up в рамках этого кейса.

## Что работает в прототипе

Единая админка с основными разделами:

1. Менеджер открывает админку (`/admin`) и видит боковое меню навигации
2. Доступны следующие разделы:
   - **Дашборд** — сводка по ключевым показателям
   - **Пользователи** — список всех пользователей, карточка пользователя 360
   - **Компании** — B2B-клиенты
   - **Продукты** — курсы, тарифы, потоки
   - **Биллинг** — заказы и оплаты
   - **Зачисления** — студенты и их курсы
   - **LMS** — управление контентом курсов
   - **Лиды** — заявки из всех источников
   - **События** — журнал действий в системе
3. Все разделы работают в рамках одного приложения с единой авторизацией
