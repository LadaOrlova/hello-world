> **Источник:** Фидбек Леры (LMS/обучение), 2026-03-17.

## Параметры реализации

### Критерии приёмки (MVP)

- [ ] На дашборде менеджера отображается виджет «Неактивированные доступы»: студенты с зачислением, но без единого захода в LMS.
- [ ] На дашборде отображается виджет «Неактивные студенты»: студенты без прогресса N+ дней (настраиваемый порог, по умолчанию 7 дней).
- [ ] Виджеты кликабельны: переход в список студентов с фильтром.

## Что уже есть в платформе

- Enrollment с createdAt — можно определить, когда студент был зачислен
- LessonProgress с timestamps — можно определить дату последнего прогресса
- Матрица прогресса куратора (LMS-04) — данные по каждому студенту
- Dashboard (getManagerSummary) — уже показывает KPI, можно расширить

## Что может быть на следующих этапах

### Итерация 2: Расширенная аналитика

- Per-lesson engagement: сколько студентов просмотрели каждый урок, completion rate
- Retention analytics: воронка «зачислен → активировал → прошёл 50% → завершил»
- Время прохождения: среднее время между уроками, темп обучения
- Сравнение когорт: какой поток учится активнее

### Итерация 3: Автоматические триггеры

- Уведомление куратору, если студент неактивен 3+ дня
- Уведомление менеджеру о неактивированных доступах старше 48 часов
- Еженедельный дайджест по вовлечённости для команды
