**Статус в прототипе:** Частично (бот работает, UTM-парсинг не реализован)

> **Источник:** Требования Лады (Владлены Орловой), маркетинг, 2026-03-13.

## Параметры реализации

### MVP: UTM-парсинг из стартовой ссылки бота

#### Критерии приёмки

- [ ] При переходе по ссылке `t.me/bot?start=<payload>` бот парсит payload и извлекает UTM-параметры.
- [ ] UTM-данные из start-параметра сохраняются в привязке к пользователю / лиду.
- [ ] Формат payload определён и задокументирован (например: `utm_source__telegram__utm_campaign__march2026`).
- [ ] Если пользователь уже существует — UTM-данные обновляют атрибуцию (или сохраняются как дополнительный touchpoint).

> **Решение (2026-03-17):** MVP = парсинг UTM из `?start=` параметра Telegram-бота (расширение BOT-01). Полноценный маркетинговый бот (прогревающие цепочки, автоворонки) — не MVP, остаётся в BotHelp.

## Итерация 2+: Полноценный маркетинговый бот

- Прогревающие цепочки сообщений (drip campaigns)
- Автоворонки: серия сообщений по расписанию после подписки
- Квизы и опросы для квалификации лидов
- Интеграция с сегментацией (MKT-02) для таргетированных сообщений

## Что есть сейчас

- Telegram-бот (grammY) с командой `/start` для привязки аккаунта
- Бот работает в polling (dev) и webhook (prod) режимах
- Очередь сообщений через pg-boss
- BotHelp — сторонний сервис для маркетинговых рассылок и автоворонок (продолжает работать)
