# LMS-09: Переиспользуемые контент-блоки — изменение общего элемента (термин, дата, ссылка) автоматически обновляется во всех материалах

**Статус в прототипе:** Не реализовано

> **Источник:** Фидбек Леры (LMS/обучение), 2026-03-17.

## Параметры реализации

### Критерии приёмки

- [ ] В системе можно создать переиспользуемый контент-блок (термин, ссылка, дата, фрагмент текста) как отдельную сущность.
- [ ] Контент-блок можно вставить в материал урока по ссылке/идентификатору.
- [ ] При изменении контент-блока обновление автоматически отражается во всех уроках, где он используется.
- [ ] В админке доступен справочник контент-блоков с поиском и фильтрацией.

> **Решение (2026-03-17):** Не MVP. Базовый принцип «change once, update everywhere» уже работает для структуры курса: названия и порядок модулей/уроков хранятся в одном месте и отображаются везде одинаково (LMS-05, COURSE-03). Глоссарий и smart-даты как переиспользуемые сущности — v2.

## Примеры из фидбека Леры

| Тип блока | Пример | Эффект |
|---|---|---|
| Глоссарий | Термин «OKR» с определением | Изменил определение → обновилось во всех уроках |
| Дата | Дата дедлайна ДЗ | Изменил в одном месте → обновилось в материалах, расписании, на сайте |
| Ссылка | Zoom-ссылка на лекцию | Изменил ссылку → обновилось везде |

## Что уже работает по этому принципу

- Название модуля/урока — единый источник, отображается в LMS, ЛК, кабинете куратора ✅
- LessonRevision — версионирование контента, единый источник ✅
- Тариф → модули — привязка хранится в одном месте ✅
