# PAY-10: Перенос студента на другой поток — менеджер может перенести студента с одного потока на другой с сохранением или перепривязкой оплаты

**Статус в прототипе:** Не реализовано

> **Источник:** Фидбек Насти (2026-03-11). «Участник переносится с четвёртого на пятый поток. Мы должны ему поменять поток, и он должен перекинуться в бронь следующего потока.»

## Параметры реализации

### Критерии приёмки

- [ ] Менеджер может инициировать перенос студента с текущего потока на другой поток того же продукта.
- [ ] При переносе текущее зачисление деактивируется, создаётся новое зачисление на целевой поток.
- [ ] Существующая оплата перепривязывается к новому зачислению (основание — тот же платёж).
- [ ] Доступы студента обновляются: отзываются для старого потока, выдаются для нового (LMS, Telegram-чаты).
- [ ] В истории заказов и зачислений фиксируется факт переноса с указанием основания.
- [ ] Перенос доступен как для B2C-студентов, так и для участников B2B-компаний.
- [ ] Для B2B: при переносе обновляется информация в документах (если счёт ещё не оплачен — пересоздаётся, если оплачен — создаётся корректировочный документ).

> **Решение (2026-03-17):** В MVP перенос выполняется вручную менеджером: отчислить из текущего потока + зачислить в новый. Автоматизация переноса — v2.

## Что нужно реализовать (v2)

Минимальный сценарий автоматизированного переноса:

- [ ] Менеджер открывает зачисление студента в админке
- [ ] Менеджер нажимает «Перенести на другой поток» и выбирает целевой поток
- [ ] Система проверяет доступность места на целевом потоке
- [ ] Система деактивирует текущее зачисление и создаёт новое
- [ ] Система обновляет доступы (LMS, Telegram-чаты)
- [ ] В истории фиксируется: кто, когда, с какого на какой поток, основание

### Что уже есть в платформе

- Зачисления можно деактивировать
- Можно создать новое зачисление на любой поток
- Онбординг автоматически выдаёт доступы
