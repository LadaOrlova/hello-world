# PAY-13: Финансовая аналитика — все данные по платежам, скидкам, комиссиям и возвратам доступны для построения финансовых отчётов

**Статус в прототипе:** Не реализовано

> **Источник:** Фидбек Насти (2026-03-11). «В базу должны уходить все данные для фин аналитики, чтобы мы могли их оттуда легко достать.»

## Параметры реализации

### Критерии приёмки

- [ ] По каждому заказу в системе хранятся: полная цена тарифа, размер и тип скидки, итоговая сумма к оплате, валюта, способ оплаты, комиссия платёжной системы (если доступна из webhook).
- [ ] По каждому возврату хранятся: сумма возврата, причина, канал возврата, дата.
- [ ] Для B2B-заказов дополнительно хранятся: наценка НДС, реквизиты компании, номер счёта/договора.
- [ ] Менеджер может получить сводку по выручке: общая сумма, средний чек, количество оплат за период.
- [ ] Данные доступны для выгрузки (CSV/Excel) или SQL-запросов для внешней аналитики.
- [ ] Структура данных позволяет строить отчёты по: продуктам, потокам, тарифам, типам скидок, каналам оплаты, компаниям.

## Зависимости от других кейсов

Финансовая аналитика строится поверх данных, которые собираются другими кейсами:

| Кейс | Какие данные поставляет |
|------|------------------------|
| PAY-01 | Снапшот цены, валюта, сумма на момент покупки |
| PAY-09 | Скидки: тип, размер, основание |
| PAY-06 | Возвраты: сумма, причина, канал |
| PAY-04 | Комиссии платёжных систем из webhook-ов |
| INV-01..08 | B2B-документы: суммы, номера счетов, реквизиты |
| PAY-07 | Апгрейды: исходная и целевая сумма, доплата |

## Что может быть на следующих этапах

### Итерация 1: базовые отчёты

- Сводка по выручке за период (по продуктам, потокам)
- Выгрузка всех транзакций в CSV
- Отчёт по скидкам: сколько и каких применено

### Итерация 2: расширенная аналитика

- Дашборд с визуализацией финансовых показателей
- Сравнение потоков по выручке, среднему чеку, конверсии
- Отчёт по дебиторской задолженности (неоплаченные B2B-счета)
- Аналитика по каналам оплаты: конверсия, комиссии, средний чек
- Сверка с бухгалтерскими данными (1С)

### Расширенные сценарии

- Прогнозирование выручки на основе бронирований и воронки
- LTV клиента: совокупная выручка по пользователю за всё время
- Когортный анализ: выручка по потокам и дате покупки
