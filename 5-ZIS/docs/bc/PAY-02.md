# PAY-02: Если покупка одного продукта даёт право на доступ к другому (например, курс включает в себя членство в сообществе), система выдаёт эти доступы автоматически без ручных действий команды

**Статус в прототипе:** Не реализовано

## Параметры реализации

### Критерии приёмки

- [ ] В системе можно настроить правило кросс-доступа: покупка продукта A автоматически даёт доступ к продукту B.
- [ ] После успешной оплаты продукта A правило кросс-доступа срабатывает автоматически без ручных действий менеджера.
- [ ] Для продукта B создаётся корректное зачисление на того же пользователя, привязанное к исходной покупке продукта A.
- [ ] Автоматически созданное зачисление не требует второй оплаты и не создаёт дублирующих заказов.
- [ ] После выдачи доступа студент видит в личном кабинете оба доступа: основной продукт A и бонусный продукт B.
- [ ] Выдача кросс-доступа фиксируется в событиях или истории так, чтобы команда могла отследить основание выдачи.

## Что нужно реализовать в прототипе

Минимальный сценарий автоматической выдачи кросс-продуктовых доступов:

- [ ] Менеджер настраивает связь между продуктами: при покупке продукта A студент также получает доступ к продукту B
- [ ] Студент покупает продукт A
- [ ] Система автоматически создаёт зачисление на продукт B без дополнительной оплаты
- [ ] Студент видит оба продукта в личном кабинете

### Что уже есть в платформе

- Система зачислений работает — можно создать зачисление на любой продукт
- Онбординг-процесс автоматически выдаёт доступы после зачисления
- Продукты и тарифы управляются через админку

## Что может быть на следующих этапах

### Пользовательский опыт

- Студент видит на странице покупки, какие бонусные доступы включены в тариф
- Уведомление о получении дополнительного доступа

### Расширенные сценарии

- Гибкие правила: бонусный доступ зависит от тарифа (Премиум включает сообщество, Стандарт — нет)
- Отзыв бонусного доступа при возврате основного продукта
- Временные бонусы: пробный доступ к другому продукту на ограниченный срок
