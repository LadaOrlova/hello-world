# AUTH-02: Пользователь входит в любой продукт через единую точку авторизации — отдельная регистрация для каждого продукта не нужна

**Статус в прототипе:** Работает

## Параметры реализации

### Критерии приёмки

- [ ] Для неавторизованного пользователя доступ к защищённым разделам (`student`, `curator`, `admin`, `company`) ведёт на единую страницу входа `/auth`.
- [ ] Вход во все разделы выполняется через один и тот же flow авторизации (email + OTP), без отдельных форм входа/регистрации по продуктам.
- [ ] После успешного входа пользователь возвращается в целевой раздел в рамках своих прав доступа.
- [ ] Один email соответствует одному аккаунту, который используется во всех продуктах и разделах системы.
- [ ] Пользователь, уже авторизованный в одном разделе, может переходить в другие доступные разделы без повторной авторизации.
- [ ] Для повторного доступа к другому продукту не требуется создавать новый аккаунт или проходить отдельную регистрацию.

## Что работает в прототипе

Единая точка входа для всех разделов системы:

1. Пользователь переходит в любой раздел: личный кабинет студента, кабинет куратора, панель администратора, кабинет компании
2. Если пользователь не авторизован — система автоматически перенаправляет его на единую страницу входа (`/auth`)
3. Пользователь вводит email и получает одноразовый код — один и тот же процесс для всех продуктов
4. После входа система определяет роль пользователя и направляет его в нужный раздел
5. Не существует отдельных форм регистрации или входа для разных продуктов — вход всегда через одну и ту же страницу
6. Один email = один аккаунт. Если студент записывается на второй курс — он входит тем же способом и видит все свои курсы в одном кабинете
7. Если пользователь уже вошёл в один раздел — он может перейти в другой доступный раздел без повторного ввода кода

## Что может быть на следующих этапах

### Пользовательский опыт

- После перенаправления на страницу входа показывать, из какого раздела пользователь пришёл (например: «Войдите, чтобы открыть личный кабинет»)
- Единый профиль пользователя: имя, фото, контакты — редактируются в одном месте и видны во всех разделах
- Переключение между ролями без выхода из системы (например, куратор, который одновременно является студентом на другом курсе)

### Безопасность

- Автоматическое завершение сессии при бездействии
- Уведомление при входе с нового устройства

### Расширенные сценарии

- SSO-интеграция для корпоративных клиентов (вход через рабочий аккаунт компании)
- Вход через Telegram без email
