# AUTH-03: Команда видит полную историю пользователя в одном месте: какие продукты покупал, в каких потоках учился, когда и по какой цене

**Статус в прототипе:** Работает

## Параметры реализации

### Критерии приёмки

- [ ] В админке доступен переход в единую карточку пользователя из списка пользователей.
- [ ] В карточке пользователя отображаются ключевые блоки истории: профиль, заказы/оплаты, зачисления, прогресс обучения, последние события, связи (Telegram/компания).
- [ ] В блоке заказов и оплат для каждого заказа видны минимум: продукт, сумма, валюта, статус оплаты, дата.
- [ ] В блоке зачислений отображаются все зачисления пользователя с текущим статусом.
- [ ] В блоке прогресса отображается сводка прохождения обучения по доступным курсам.
- [ ] Если данных по блоку нет, интерфейс показывает понятное сообщение (а не пустой или сломанный блок).
- [ ] Карточка доступна ролям команды, которым нужен операционный доступ к данным пользователя (в рамках текущего RBAC прототипа).

## Что работает в прототипе

Единая карточка пользователя в админке с полной историей:

1. Менеджер открывает раздел «Пользователи» в админке
2. В таблице пользователей видны визуальные метки: «Платящий» (есть оплаченные заказы), «Представитель» (контактное лицо компании)
3. Менеджер нажимает «Карточка» напротив нужного пользователя
4. Открывается страница карточки пользователя со всей историей, разбитой на блоки:
   - **Профиль** — имя, email, роль, дата регистрации
   - **Заказы и оплаты** — список заказов с суммой, валютой, статусом оплаты, датой и продуктом
   - **Зачисления** — все курсы, в которые записан пользователь, с текущим статусом каждого зачисления
   - **Прогресс обучения** — сводка по LMS: сколько уроков пройдено, процент завершения по каждому курсу
   - **Последние события** — журнал действий пользователя в системе
   - **Связи** — привязанный Telegram-аккаунт, компания (если есть)
5. Если у пользователя нет данных в каком-то блоке — показывается понятное сообщение вместо пустоты

## Что может быть на следующих этапах

### Пользовательский опыт

- Добавить возможность перехода из карточки напрямую к конкретному заказу или зачислению (сейчас информация показывается, но без ссылок)
- Временная шкала (timeline): все события по пользователю в хронологическом порядке — регистрация, покупки, начало обучения, завершение курсов (см. также MKT-04 — данные timeline нужны для воронки конверсии)
- Быстрые действия из карточки: отправить сообщение, создать заказ, изменить роль

### Расширенные сценарии

- Сравнение пользователей: прогресс двух студентов рядом
- Экспорт истории пользователя в PDF или Excel
