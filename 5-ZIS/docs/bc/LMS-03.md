# LMS-03: Студент видит групповой прогресс (например, статистику своей учебной группы) для элементов соревновательной механики

**Статус в прототипе:** Частично

## Параметры реализации

### Критерии приёмки

- [ ] Для студента в интерфейсе LMS доступен отдельный блок или экран группового прогресса по его текущей когорте.
- [ ] В блоке группового прогресса отображается агрегированная статистика группы (например: средний процент прохождения, общий прогресс группы).
- [ ] Студент видит своё положение в группе (ранг, позиция или эквивалентная метрика соревновательности).
- [ ] Данные группового прогресса рассчитываются из актуальных данных прохождения по той же когорте, к которой относится студент.
- [ ] Персональные данные других студентов не раскрываются (анонимизация соблюдена в student-view).

## Что работает в прототипе

Групповой прогресс доступен куратору:

1. Куратор открывает страницу своей группы (`/curator/lms/[cohortId]`)
2. Куратор видит матрицу «студент × урок»: кто какой урок прошёл
3. Прогресс каждого студента отображается в таблице

## Что нужно доработать в прототипе

- [ ] Показать студенту прогресс его группы: анонимизированную статистику (средний прогресс, лидеры) без раскрытия данных конкретных одногруппников
- [ ] Добавить элемент соревновательности: место студента в рейтинге группы

### Что уже есть в платформе

- Матрица прогресса для куратора — данные собираются и хранятся
- Студент видит свой собственный прогресс — базовый экран работает
- Зачисления привязаны к потокам (когортам) — группировка студентов есть

## Что может быть на следующих этапах

### Пользовательский опыт

- Визуальный рейтинг группы с аватарами (с согласия студентов)
- Еженедельная сводка: «Вы прошли больше уроков, чем 70% группы»
- Достижения и бейджи за прогресс

### Расширенные сценарии

- Командные задания: прогресс группы как единого целого
- Сравнение прогресса между потоками одного курса
