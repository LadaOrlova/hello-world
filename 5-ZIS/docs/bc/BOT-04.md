# BOT-04: Команда может расширить сценарии бота (добавить новые команды, ответы) без переработки всей бот-инфраструктуры

**Статус в прототипе:** Частично

## Параметры реализации

### Критерии приёмки

- [ ] Архитектура бота поддерживает модульное добавление новых команд и сценариев без переписывания существующих обработчиков.
- [ ] Новая команда может быть добавлена как отдельный handler и подключена в общий роутинг бота с минимальными изменениями.
- [ ] Добавление новой команды не ломает существующие команды и базовые bot-flow (регрессия отсутствует в штатном сценарии).
- [ ] Новые handler-ы могут использовать данные ZIS (минимум: пользователь, зачисления, прогресс и оплата) через стандартный слой доступа.
- [ ] В репозитории есть минимум 2-3 референсных примера команд (простая текстовая, кнопочная, с запросом данных).
- [ ] Подготовлен практический гайд для команды: как добавить новую команду бота с использованием текущего подхода и AI-инструментов.
- [ ] Для локальной проверки нового handler предусмотрен рабочий dev-сценарий (polling/test mode) без вмешательства в прод-инфраструктуру.

## Что работает в прототипе

Расширяемая архитектура бота:

1. Бот построен на фреймворке grammY с handler-паттерном
2. Каждая команда реализована как отдельный handler — добавление нового handler не затрагивает существующие
3. Бот подключён к базе данных системы — новые команды получают доступ ко всем данным

### Что уже есть в платформе

- Рабочий бот с инфраструктурой для обработки команд
- Доступ к данным: пользователи, зачисления, оплаты, прогресс
- Polling-режим для разработки — можно тестировать новые команды локально

## Что может быть на следующих этапах

### Пользовательский опыт

- Управление командами бота через админку без кода

### Расширенные сценарии
