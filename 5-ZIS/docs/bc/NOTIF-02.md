## Что нужно реализовать в прототипе

Минимальный сценарий управления подписками:

- [ ] Студент открывает настройки уведомлений в личном кабинете
- [ ] Студент видит категории: критичные, информационные, маркетинговые
- [ ] Студент включает или выключает категории — система запоминает выбор
- [ ] Уведомления отправляются только по включённым категориям (критичные — всегда)

### Что уже есть в платформе

- Личный кабинет студента — есть где разместить настройки
- Система событий — можно категоризировать уведомления по типу
- Профиль пользователя — можно хранить предпочтения

## Что может быть на следующих этапах

### Пользовательский опыт

- Предустановленные профили: «Минимум» (только критичные), «Все» (всё включено)
- Отписка одной кнопкой из тела уведомления

### Расширенные сценарии

- Гранулярные настройки по продукту: уведомления от КДП включены, от Boost — выключены
- Соответствие GDPR и 152-ФЗ по согласиям на коммуникацию
