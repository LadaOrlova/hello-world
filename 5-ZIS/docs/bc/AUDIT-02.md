# AUDIT-02: Сотрудник может в любой момент открыть карточку пользователя и увидеть все его покупки, переписки с командой и ботами, историю доступов

**Статус в прототипе:** Частично

## Параметры реализации

### Критерии приёмки

- [ ] В админке доступна карточка пользователя с единым обзором его операционной истории.
- [ ] В карточке отображаются покупки и платежи пользователя по всем продуктам.
- [ ] В карточке отображаются зачисления и текущие статусы доступов по курсам и потокам.
- [ ] В карточке отображается прогресс обучения пользователя по его зачислениям.
- [ ] В карточке отображаются ключевые события пользователя из системного event и audit контура.
- [ ] В карточке отображаются связи пользователя (минимум: Telegram-привязка, компания при наличии).
- [ ] История переписок с ботом и командой в текущем прототипе зафиксирована как follow-up и не блокирует приёмку базового User 360.

## Что работает в прототипе

Карточка пользователя с основной историей:

1. Менеджер открывает карточку пользователя в админке (`/admin/users/[id]`)
2. В карточке видны все заказы и платежи пользователя
3. В карточке видны все зачисления с их статусами
4. Виден прогресс обучения по каждому курсу
5. Отображаются последние события пользователя в системе
6. Показаны связи: привязанный Telegram-аккаунт, компания
