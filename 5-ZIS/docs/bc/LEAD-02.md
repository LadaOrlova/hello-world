# LEAD-02: Лид, пришедший с любого лендинга, автоматически попадает в единую базу и отображается в соответствующем разделе админки

**Статус в прототипе:** Работает

## Параметры реализации

### Критерии приёмки

- [ ] Лиды из разных источников (минимум: TypeForm webhook, внутренняя форма `/lead`, публичный API) автоматически попадают в единую базу ZIS.
- [ ] В админке `/admin/leads` отображается единый список лидов без разделения по отдельным системам.
- [ ] Для каждого лида в списке доступны минимум: источник, дата создания, контактные данные, текущий статус.
- [ ] Дедупликация по email предотвращает создание дублирующих лидов при повторной заявке с тем же адресом.
- [ ] Для нового лида автоматически назначается ответственный менеджер по текущему правилу auto-assign.
- [ ] Менеджер может переводить лид по статусам воронки (новая -> в работе -> конвертирована или отклонена).
- [ ] Все изменения статуса лида сохраняются и доступны для последующего просмотра командой.

## Что работает в прототипе

Единая база заявок из всех источников:

1. Заявка приходит из TypeForm — автоматически сохраняется в системе через webhook
2. Заявка приходит с внутренней формы `/lead` — сохраняется в системе
3. Заявка приходит через публичный API — сохраняется в системе
4. Менеджер открывает раздел «Лиды» в админке (`/admin/leads`) и видит все заявки из всех источников в одной таблице
5. Для каждой заявки показан источник, дата, контактные данные и текущий статус
6. Система автоматически дедуплицирует заявки по email — повторная заявка с того же адреса не создаёт дубль
7. Менеджер автоматически назначается на новую заявку
8. Менеджер может менять статус заявки: новая → в работе → конвертирована / отклонена

## Что может быть на следующих этапах

### Пользовательский опыт

- Уведомление менеджеру о новой заявке (Telegram, email)
- Быстрые действия из списка: позвонить, отправить письмо, создать заказ

### Расширенные сценарии

- Автоматическая квалификация лидов: система оценивает «теплоту» заявки по заполненным полям
- Воронка продаж: визуализация конверсии от заявки до оплаты
- Массовые действия: изменение статуса, назначение менеджера для группы заявок
