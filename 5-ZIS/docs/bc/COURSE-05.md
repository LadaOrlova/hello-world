# COURSE-05: Команда может добавить к существующему курсу новый тариф или новый поток, не затрагивая доступы уже зачисленных студентов

**Статус в прототипе:** Работает

## Параметры реализации

### Критерии приёмки

- [ ] В существующий курс можно добавить новый тариф через админку без изменений уже созданных зачислений.
- [ ] Новый тариф становится доступным для новых покупок, при этом текущие студенты сохраняют ранее выданные доступы.
- [ ] В существующий курс можно добавить новый поток (когорту) через админку.
- [ ] Новый поток принимает новых студентов, а студенты из предыдущих потоков продолжают обучение в своих потоках без потери данных и доступов.
- [ ] Изменение цены тарифа влияет только на новые заказы.
- [ ] Исторические заказы и зачисления сохраняют исходный ценовой снапшот и условия доступа на момент покупки.
- [ ] Операция добавления тарифа или потока не требует миграции или ручной правки данных существующих студентов.

## Что работает в прототипе

Безопасное добавление тарифов и потоков без влияния на существующих студентов:

1. Менеджер открывает продукт в админке (`/admin/products`)
2. Менеджер добавляет новый тариф: указывает название, цену, лимит студентов, набор доступных модулей
3. Новый тариф становится доступен для покупки — существующие зачисления остаются без изменений
4. Менеджер создаёт новый поток (когорту) для курса: указывает даты, открывает набор
5. Новый поток принимает студентов — студенты из предыдущих потоков продолжают обучение в своём потоке с прежними доступами
6. Менеджер может изменить цену тарифа — это влияет только на новые покупки, уже оплаченные заказы сохраняют прежнюю цену
