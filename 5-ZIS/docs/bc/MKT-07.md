## Параметры реализации

### MVP: Счётчики набора на дашборде

#### Критерии приёмки

- [ ] На дашборде менеджера (`/admin/dashboard`) отображается виджет «Набор на активные потоки».
- [ ] По каждому активному потоку (cohort со статусом `enrolling` или `active`) показано: название продукта, название потока, количество зачисленных студентов, цель набора (capacity из cohort).
- [ ] Визуальный индикатор заполненности (прогресс-бар или дробь: набрано / цель).

## Итерация 2: Раздел «Маркетинг» в AdminUI

- Пункт «Маркетинг» в боковом меню AdminUI
- Внутри — ссылки на кастомные инструменты маркетолога:
  - Сегментация (MKT-02)
  - Рассылки (MKT-03)
  - Аналитика каналов (MKT-01)
  - Воронка конверсии (MKT-04)
- Принцип: ZIS = данные и API, инструменты маркетолога = вайб-кодинг поверх данных

## Что уже есть в платформе

- Дашборд менеджера (`/admin/dashboard`) с KPI-сводкой
- Данные для счётчиков: `Enrollment` (количество зачислений), `Cohort.capacity` (цель набора)
- Данные для аналитики: leads, orders, enrollments, events
