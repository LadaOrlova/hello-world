# MKT-06: A/B тестирование — маркетолог может тестировать разные варианты страниц/предложений и видеть, какой работает лучше

**Статус в прототипе:** Не реализовано

> **Источник:** Требования Лады (Владлены Орловой), маркетинг, 2026-03-13.

## Параметры реализации

### Критерии приёмки

- [ ] Маркетолог может создать A/B-тест с двумя и более вариантами (лендинг, оффер, цена, текст).
- [ ] Трафик автоматически распределяется между вариантами по заданным пропорциям.
- [ ] По каждому варианту собирается метрика конверсии (показы → лиды → оплаты).
- [ ] Результаты доступны в реальном времени с указанием статистической значимости.

> **Решение (2026-03-17):** Не MVP. Сроки реализации не определены. A/B тестирование может быть реализовано как кастомный инструмент маркетолога (вайб-кодинг) или через внешние сервисы (Google Optimize, VWO и т.д.).

## Возможные подходы

### Вариант 1: Внешний сервис
- Использовать Google Optimize, VWO или аналог для A/B-тестов на лендингах
- ZIS предоставляет данные по конверсиям через API / UTM-трекинг

### Вариант 2: Встроенный механизм (v2+)
- Параметр варианта в UTM (`utm_content=variant_a`)
- Сравнение конверсий по вариантам через аналитику ZIS
