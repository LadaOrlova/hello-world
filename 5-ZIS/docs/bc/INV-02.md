# INV-02: При формировании счёта для B2B клиент может сразу указать данные студентов (ФИО, email), чтобы после оплаты онбординг запускался автоматически, без дополнительной переписки

**Статус в прототипе:** Работает

## Параметры реализации

### Критерии приёмки

- [ ] В B2B-контуре компания может заранее указать состав студентов (минимум: ФИО и email) для зачисления по счёту.
- [ ] Данные студентов сохраняются в реестре сотрудников компании и используются при оформлении заявки на обучение.
- [ ] После подтверждения оплаты B2B-заявки система автоматически обрабатывает каждого указанного сотрудника без ручного заведения менеджером.
- [ ] Для каждого сотрудника создаются или используются корректные сущности пользователя, заказа и зачисления.
- [ ] Для каждого зачисленного сотрудника запускается стандартный onboarding-процесс (доступы LMS, бот и прочие шаги по текущему flow).
- [ ] Компания и команда видят итоговый результат обработки: кто успешно зачислен, по кому есть ошибки.
- [ ] Сценарий выполняется без дополнительной переписки с менеджером по базовому happy-path.

## Что работает в прототипе

Автоматический онбординг B2B-студентов после оплаты:

1. Компания ведёт реестр сотрудников в личном кабинете (`/company`)
2. При создании заявки на обучение компания выбирает конкретных сотрудников из реестра
3. Заявка отправляется на рассмотрение менеджеру
4. Менеджер подтверждает оплату счёта
5. Система автоматически для каждого указанного сотрудника:
   - Создаёт аккаунт пользователя (если его ещё нет)
   - Создаёт заказ и зачисление
   - Запускает полный онбординг (доступ к LMS, Telegram-чаты, приветствие)
6. Никакой дополнительной переписки и ручных действий не требуется
