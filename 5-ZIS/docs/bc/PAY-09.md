# PAY-09: Система скидок — менеджер может применить скидку при создании заказа или счёта, скидка фиксируется как отдельная сущность для финансовой аналитики

**Статус в прототипе:** Не реализовано

> **Источник:** Фидбек Насти (2026-03-11). «Размер скидки должен существовать как сущность для применения дальнейшей фин аналитики. Менеджер имеет возможность выставлять ссылки на сумму со скидкой.»

## Параметры реализации

### Критерии приёмки (MVP)

- [ ] При создании заказа или счёта менеджер может указать скидку (процент или фиксированная сумма).
- [ ] В заказе хранятся отдельно: полная цена тарифа, размер скидки, тип/основание скидки, итоговая сумма к оплате.
- [ ] Тип скидки выбирается из справочника: повторный студент, early bird (раннее бронирование), ручная (менеджерская), накопительная (для компаний), реферальная, прочее.
- [ ] При выставлении платёжной ссылки (B2C) итоговая сумма к оплате = полная цена − скидка.
- [ ] При формировании счёта (B2B) в документе отображается: полная стоимость, скидка, итого к оплате.
- [ ] Если скидка = раннее бронирование (early bird до определённой даты), то стоимость заводится как полная цена + скидка, в визуализации клиенту показывается итоговая сумма.
- [ ] Скидка фиксируется в данных заказа и доступна для финансовой аналитики.
- [ ] Менеджер в админке видит применённую скидку в карточке заказа.

## Что нужно реализовать в прототипе

### MVP (итерация 1): ручная скидка менеджером

- [ ] При создании заказа менеджер видит поле «Скидка» (процент) рядом с полной стоимостью тарифа
- [ ] Менеджер вводит размер скидки — система автоматически рассчитывает итоговую сумму
- [ ] Менеджер выбирает тип/основание скидки из выпадающего списка
- [ ] Итоговая сумма используется для формирования платёжной ссылки или счёта
- [ ] В базе данных сохраняются: `basePrice`, `discountPercent`, `discountAmount`, `discountType`, `finalPrice`
- [ ] В карточке заказа и в финансовых отчётах скидка отображается отдельной строкой

### Что уже есть в платформе

- Заказы и платежи хранятся в системе со снапшотом цен (PAY-01)
- Менеджер может создавать заказы через админку
- Определение повторного студента — частично (DEDUP-02)

## Наценка для B2B (НДС)

> **Фидбек Насти (2026-03-11):** «Стоимость для компаний имеет свою величину (выше на 5% НДС или любую другую настраиваемо, вдруг изменится НДС или ещё что-то).»

### MVP: автоматический расчёт + ручная корректировка

- [ ] Система автоматически рассчитывает B2B-цену по формуле: цена тарифа × (1 + коэффициент НДС)
- [ ] Базовый коэффициент НДС задаётся в конфигурации системы (по умолчанию 5%)
- [ ] Менеджер может заменить рассчитанное значение на любое вручную при создании B2B-заказа/счёта
- [ ] В данных заказа хранятся: базовая цена тарифа, коэффициент/сумма наценки, итоговая B2B-цена
- [ ] Наценка отображается отдельно в финансовых данных

## Что может быть на следующих этапах

### Итерация 2: автоматические скидки

- Автоматическое применение скидки для повторных студентов (DEDUP-02)
- Накопительная скидка для компаний: хранится в системе, обновляется автоматически по количеству оплат, применяется к новым заказам
- Скидка раннего бронирования: автоматически применяется до указанной даты потока
- Реферальная программа: скидки друзьям, приглашённым существующими студентами

### Расширенные сценарии

- Промокоды (ввод кода при оплате)
- Отчёт по скидкам: сколько скидок применено, на какую сумму, по каким типам
- Влияние скидок на финансовые показатели (аналитика)
- При апгрейде тарифа скидка сохраняется пропорционально (PAY-07)
