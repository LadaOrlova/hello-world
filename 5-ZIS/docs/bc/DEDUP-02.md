# DEDUP-02: Повторный студент (ранее проходивший обучение) при возврате на новый поток автоматически распознаётся системой, и ему предлагается соответствующая скидка без ручного вмешательства менеджера

**Статус в прототипе:** Частично

## Параметры реализации

### Критерии приёмки

- [ ] При оформлении новой покупки система автоматически определяет, является ли пользователь повторным студентом (по истории зачислений и покупок).
- [ ] Для повторного студента в checkout или операционном flow применяется правило скидки без ручного расчёта менеджером.
- [ ] Размер и правило применения скидки для повторных студентов задаются в конфигурации (или в явно определённой бизнес-логике прототипа).
- [ ] В заказе и оплате фиксируется факт, что цена была рассчитана как повторному студенту (с указанием основания скидки).
- [ ] Менеджер в админке видит признак «повторный студент» и применённую скидку в контексте заказа.
- [ ] Студент в пользовательском контуре видит, что ему применена скидка для повторного обучения (без ручного промокода менеджера).
- [ ] Если пользователь не является повторным студентом, скидка по этому правилу не применяется.

## Что работает в прототипе

Распознавание повторного студента:

1. Система хранит полную историю зачислений каждого пользователя
2. В таблице пользователей в админке есть метка «Платящий» — менеджер видит, кто уже покупал
3. В карточке пользователя видны все предыдущие покупки и зачисления
4. При создании нового заказа менеджер может проверить историю пользователя

## Что нужно доработать в прототипе

### MVP (итерация 1): ручное распознавание + ручная скидка

- [ ] Автоматически определять повторного студента при оформлении заказа и показывать бейдж/уведомление менеджеру
- [ ] Менеджер вручную применяет скидку для повторного студента через систему скидок (см. PAY-09)

### Итерация 2: автоматизация

- [ ] Автоматически предлагать скидку для повторных студентов (настраиваемый процент)
- [ ] Показать студенту на странице покупки: «Вы уже проходили у нас курс — для вас скидка»
- [ ] Скидка привязывается к аккаунту участника и применяется автоматически при оформлении заказа (в т.ч. курс КДП)

> **Решение (2026-03-17):** В MVP — бейдж «повторный студент» + ручная скидка менеджером. Автоприменение — v2.

### Что уже есть в платформе

- Полная история зачислений и покупок в базе данных
- Метка «Платящий» в списке пользователей — базовое распознавание
- Карточка пользователя 360 — менеджер может проверить историю

## Что может быть на следующих этапах

### Пользовательский опыт

- Персонализированный лендинг для повторных студентов
- Программа лояльности: каждая покупка увеличивает скидку на следующую

### Расширенные сценарии

- Промокоды для выпускников: автоматическая рассылка после завершения курса
- Реферальная программа: студент приглашает друга — оба получают бонус
- Аналитика retention: сколько студентов возвращаются за вторым курсом
