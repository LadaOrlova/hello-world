## Этап 1 — Ядро (месяц 1-2): биллинг + LMS + TG-онбординг

**Цель**: Запустить следующий поток КДП на новой системе. Студенты покупают, автоматически получают доступ к LMS и чатам.

1. **Auth (better-auth + Email OTP)** — авторизация по email + OTP-код (без пароля), 4 роли (admin, manager, curator, student)
2. **Users & Companies** — единый профиль (email = идентификатор), привязка TG-аккаунта, карточка компании (B2B)
3. **Products, Tiers, Cohorts** — справочник курсов, тарифов (с telegram_chats[] и upgrade_to_tier_id), потоков
4. **LMS: Modules & Lessons** — контент-админка: модули, уроки (text/video/text_video, Kinescope embed, markdown), drag-and-drop для порядка
5. **LMS: LessonRevision (SoK)** — версионирование контента уроков: каждое редактирование → новая ревизия, история изменений, Lesson.current_revision_id как указатель на SoK ([docs/11-content-versioning.md](docs/11-content-versioning.md))
6. **LMS: Progress** — прогресс студента по урокам, чекбокс "Изучено", дэшборд куратора (таблица студент × урок)
7. **Billing: Orders & Payments** — Order (покупка) + Payment (транзакция), интеграция ЮKassa (создание, webhook), Т-Банк рассрочка
8. **Billing: Invoices** — счета для юрлиц (PDF-генерация), ручное подтверждение оплаты
9. **Enrollments** — зачисление студента, привязка к Order, статусы, price_snapshot
10. **TG: Chats & Memberships** — TelegramChat, TelegramAccount, ChatMembership (should_be_member), онбординг-страница (веб)
11. **TG Bot (grammY)** — привязка аккаунта (/start), отслеживание вступлений в чаты (chat_member events), базовые уведомления
12. **Leads** — приём заявок (TypeForm webhook, лендинг), воронка, назначение менеджера, дедупликация
13. **Job Queue** — pg-boss, один worker, Process с JSON-шагами
14. **Onboarding Process** — полный поток: LMS access + онбординг-страница + Calendar + Curator + Welcome
15. **Event Logging** — таблица Event, запись при каждом действии
16. **Dashboard** — сводка для менеджера: лиды, оплаты, студенты, прогресс онбординга, dead jobs
17. **AuditLog + WebhookLog** — журнал действий + журнал входящих webhook'ов
18. **AirTable Import** — однократный импорт данных

**Результат этапа 1:**

- Студенты покупают и автоматически получают доступ к LMS и чатам
- Кураторы видят прогресс студентов по урокам
- Команде не нужно вручную добавлять в чаты (онбординг-страница)
- Менеджеры работают с лидами и оплатами в едином интерфейсе
- Все события записываются для будущей аналитики
- Контент уроков версионируется: история изменений, diff-лог (SoK)

## Этап 2 — Автоматизация (месяц 2-3): ДЗ + CRM + расписание + боты

**Цель**: B2B-расширение, CRM pipeline, полный учебный цикл с ДЗ, TG-ботами, follow-up.

19. **Homework & Submissions** — домашние задания, сдача, рецензирование куратором, уведомления
20. **Schedule & Attendance** — расписание воркшопов, интеграция Zoom + Google Calendar, посещаемость
21. **CuratorGroups** — назначение кураторов, группы студентов, breakout rooms
22. **CRM: Lead Pipeline (B2B)** — расширенные статусы Lead (negotiation → proposal → contract → invoice), Kanban-доска
23. **LMS: CohortContent (SoK fork + sync)** — fork контента при создании потока, auto-sync, ручной sync/откат, индикаторы рассинхронизации в дэшборде ([docs/11-content-versioning.md](docs/11-content-versioning.md))
24. **Billing: Upgrades** — апгрейд тарифа (доплата = разница, перенос в чаты нового тарифа)
25. **PromoCode** — промокоды, реферальная программа (бонус рефереру — подписка Maker)
26. **Billing: Refunds** — возвраты: запрос в ЮKassa + откат доступов (LMS, TG-чаты)
27. **TG: Access Control** — ежедневная проверка ChatMembership ↔ Enrollment, предупреждения, удаление
28. **TG: Webinar Reminders** — напоминания о вебинарах (за 1 день, 1 час, 15 мин)
29. **TG: Student Bot** — уведомления: новый урок, дедлайн ДЗ, результат проверки, кросс-продажи
30. **Notifications** — системные уведомления (TG + email)
31. **Follow-up** — автоматические напоминания менеджеру ("Лид X не обработан 24ч"), цепочка follow-up студенту
    32\*. **Заливщик контента (полуавтомат)** — webhook из Kinescope → предзаполнение урока в LMS (видео, название), менеджер подтверждает и привязывает к модулю/потоку. Закрывает боль ручной заливки лекций

**Результат этапа 2:**

- Полный учебный цикл: уроки + ДЗ + проверка + посещаемость
- Sales-менеджер работает с Kanban-доской сделок
- Реферальная программа работает
- Автоматический контроль доступа к TG-чатам
- Напоминания о вебинарах и ДЗ
- Fork контента для потоков: каждый поток видит свой снимок, auto-sync с SoK

## Этап 3 — Аналитика и завершение (месяц 3-4)

**Цель**: Сертификаты, подписки, глубокая аналитика, B2B-документы.

32. **Feedback** — кампании сбора обратной связи, AI-анализ, дэшборд NPS
33. **Certificates** — генерация PDF (проверка критериев: уроки + ДЗ + посещаемость), QR-верификация
34. **Acts** — акты выполненных работ (PDF, автогенерация для B2B)
35. **Subscriptions (AURA/Maker)** — подписочная модель, рекуррентные платежи, контроль доступа к чатам
36. **Cross-product Bonuses** — автоматическая подписка Maker при покупке КДП/BOOST
37. **Offboarding** — завершение потока: сертификаты + alumni + Makers + чаты выпускников + финальная ОС
38. **Analytics Dashboards** — воронка, конверсии, LTV, retention, операционные метрики
39. **Reports** — еженедельный автоматический отчёт (email/Telegram)

**Результат этапа 3:**

- Студенты получают сертификаты автоматически
- Подписочная модель AURA/Maker работает
- Полная воронка видна на дашборде
- Еженедельные отчёты автоматически

## Будущие фазы (месяц 4+)

40. **TG: Warmup Bot** — бот прогрева (замена BotHelp): воронки, сбор данных, автоцепочки
41. **B2B Self-Service** — кабинет компании: "накликать себе счёт", список сотрудников, акты
42. **MCP-серверы** — API для AI-агентов (users, billing, lms, telegram, analytics, content)
43. **CLI (zamesin)** — командная строка для скриптов и автоматизаций
44. **API Tokens** — управление токенами для внешних интеграций
45. **Stripe** — валютные платежи (USD)
46. **1С-интеграция** — синхронизация с бухгалтерией
47. **BOOST + "Жопа" на LMS** — перенос остальных продуктов на новую LMS
48. **Sandbox-режим** — тестовая среда для разработки ботов
