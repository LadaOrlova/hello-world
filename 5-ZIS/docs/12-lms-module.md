## Обзор

LMS (Learning Management System) — модуль управления учебным контентом и прогрессом обучения. Обеспечивает:

- **Контент-админку для менеджеров** — создание и редактирование модулей, уроков, домашних заданий
- **Учебный интерфейс для студентов** — просмотр уроков, отметка прогресса, сдача ДЗ
- **Дэшборд прогресса для кураторов** — таблица "студент x урок" по своей группе
- **Версионирование контента (SoK)** — история изменений, fork для потоков, auto-sync

## Сущности

### Иерархия контента

```
Product (КДП, BOOST, ...)
└── Module (Модуль курса)
    └── Lesson (Урок — метаданные)
        ├── LessonRevision (Ревизия — контент)    [1]
        └── Homework (Домашнее задание)            [2]
```

### Module (Модуль курса) [1]

Группирует уроки в логические блоки. Порядок модулей настраивается через drag-and-drop.

```
Module
├── id, product_id, title
├── position (порядок — для drag-and-drop)
├── type: main | workshop | interview | demo | case
└── is_active
```

**Типы модулей:**

- `main` — основной учебный блок (теория + практика)
- `workshop` — воркшопы и практические занятия
- `interview` — интервью с экспертами
- `demo` — демо-занятия / пробные уроки
- `case` — кейсы и разборы

### Lesson (Урок — метаданные) [1]

Урок хранит метаданные: позицию, тип, доступность по тарифам. Контент (текст, видео, длительность) хранится в LessonRevision.

```
Lesson
├── id, module_id, title
├── type: text | video | text_video
├── current_revision_id (FK → LessonRevision — указатель на SoK)
├── position (порядок внутри модуля)
├── deadline_at (опциональный дедлайн)
├── is_new (метка "новый контент")
├── available_for_tiers[] (JSON — для каких тарифов доступно)
└── is_active
```

**Типы уроков:**

- `text` — текстовый урок (markdown)
- `video` — видеоурок (Kinescope embed)
- `text_video` — комбинированный (текст + видео)

**Доступность по тарифам:** `available_for_tiers[]` — если не пусто, урок виден только студентам с указанными тарифами. Если пусто — доступен всем.

### LessonRevision (Ревизия — контент) [1]

Каждое редактирование контента создаёт новую ревизию. Ревизии не удаляются (append-only).

```
LessonRevision
├── id, lesson_id (FK → Lesson)
├── revision_number (auto-increment per lesson)
├── content (markdown)
├── video_url (Kinescope embed)
├── duration_minutes
├── change_summary (что изменилось)
├── created_by (FK → User)
├── created_at
└── UNIQUE(lesson_id, revision_number)
```

### CohortContent (Снимок контента потока) [2]

Определяет, какую ревизию урока видят студенты конкретного потока.

```
CohortContent
├── id, cohort_id (FK → Cohort), lesson_id (FK → Lesson)
├── pinned_revision_id (FK → LessonRevision)
├── is_synced (boolean)
├── synced_at
└── UNIQUE(cohort_id, lesson_id)
```

Подробнее: [11-content-versioning.md](11-content-versioning.md)

### Progress (Прогресс студента) [1]

Отслеживает прохождение каждого урока каждым студентом.

```
Progress
├── id, enrollment_id (FK → Enrollment), lesson_id (FK → Lesson)
├── status: not_started | in_progress | completed
├── completed_at
├── time_spent_seconds
└── UNIQUE(enrollment_id, lesson_id)
```

Связь со студентом — через `enrollment_id` (→ Enrollment.user_id). Привязка к зачислению, а не к User, т.к. студент может проходить курс повторно в другом потоке.

### Homework (Домашнее задание) [2]

```
Homework
├── id, lesson_id (FK → Lesson), title, description
├── deadline_days (дней после открытия доступа)
└── is_active
```

### HomeworkSubmission (Сдача ДЗ) [2]

```
HomeworkSubmission
├── id, homework_id (FK → Homework), enrollment_id (FK → Enrollment)
├── content (текст ответа)
├── status: submitted | reviewed | revision_needed | accepted
├── reviewer_id (FK → User, куратор)
├── feedback (текст обратной связи)
├── submitted_at, reviewed_at
└── UNIQUE(homework_id, enrollment_id)
```

## Роли и доступ

| Роль         | Возможности в LMS                                                                                                            |
| ------------ | ---------------------------------------------------------------------------------------------------------------------------- |
| **Менеджер** | CRUD модулей/уроков, drag-and-drop порядка, редактирование контента → ревизии, fork/sync контента потоков, история изменений |
| **Куратор**  | Таблица прогресса своей группы (read-only), проверка ДЗ, обратная связь студенту                                             |
| **Студент**  | Просмотр уроков (в рамках тарифа), отметка "Изучено", сдача ДЗ, просмотр записей                                             |

## Процессы

### Выдача доступа к LMS (онбординг) [1]

```
Event: payment.success → Process (type: "onboarding")
    │
    └──→ [Job] grant_lms_access (LMS.1):
         ├── Определить уроки Product → Module → Lesson
         ├── Отфильтровать по available_for_tiers (тариф студента)
         ├── Создать Progress записи (status: not_started)
         ├── Enrollment.lms_access_granted = true
         └── Event: lms_access_granted
```

### Редактирование контента → ревизия [1]

```
[Менеджер сохраняет изменения в уроке]
    │
    ├── [DB.33] Создать LessonRevision
    │   ├── revision_number = max(existing) + 1
    │   ├── content, video_url, duration_minutes из формы
    │   ├── change_summary из формы
    │   └── created_by = текущий менеджер
    │
    ├── Обновить Lesson.current_revision_id → новая ревизия
    │
    ├── [Этап 2] Пометить CohortContent.is_synced = false
    │   для всех CohortContent этого урока
    │
    ├── Event: lesson_content_updated
    └── AuditLog: lesson_updated
```

### Fork контента при создании потока [2]

```
[Менеджер создаёт Cohort]
    │
    └──→ [Job] fork_cohort_content (DB.34):
         Для каждого активного Lesson продукта:
         └── CohortContent(
               cohort_id, lesson_id,
               pinned_revision_id = Lesson.current_revision_id,
               is_synced = true, synced_at = now
             )
```

### Auto-sync контента [2]

```
[pg-boss cron: "content.auto_sync" — ежедневно]
    │
    ├── CohortContent WHERE is_synced = false
    │
    ├── Для каждого → проверить eligible:
    │   ├── Cohort.status IN (enrolling, active)
    │   ├── Есть active Enrollment с valid support_expires_at
    │   └── Auto-sync не заблокирован для потока
    │
    ├── Eligible → sync (DB.35):
    │   pinned → current, is_synced → true
    │
    └── Не eligible → остаётся is_synced = false
```

### Отзыв доступа к LMS (возврат/рефанд) [2]

```
[Менеджер оформляет возврат]
    │
    └──→ [Job] revoke_lms_access (LMS.2):
         ├── Enrollment.lms_access_granted = false
         ├── Progress записи остаются (для истории)
         └── Студент не видит уроки в LMS
```

## Этапы реализации

### Этап 1 — Ядро

- Module, Lesson (метаданные)
- LessonRevision (контент с версионированием)
- Progress (прогресс студента)
- grant_lms_access (выдача доступа при онбординге)
- Контент-админка: CRUD модулей/уроков, drag-and-drop, редактор markdown
- Студенческий LMS: навигация по курсу, просмотр уроков, отметка прогресса
- Дэшборд куратора: таблица прогресса группы

### Этап 2 — Автоматизация

- CohortContent (fork + sync контента потоков)
- Auto-sync через pg-boss cron
- Homework, HomeworkSubmission (ДЗ и проверка)
- revoke_lms_access (отзыв при возврате)
- Загрузка записей (Kinescope → ревизия)
- Дэшборд: индикаторы рассинхронизации контента

### Этап 3 — Аналитика

- check_completion_criteria (проверка критериев для сертификата)
- Аналитика: прохождение, время на уроки, конверсия по шагам
