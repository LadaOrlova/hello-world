# New Article Thesis

## Overview

Этот скилл реализует фазу расхождения (Double Diamond): расширяет тезисное поле, добавляет контраргументы, повышает глубину и актуальность.

Ключевой принцип: качество тезисов важнее количества.

## Рабочие пути

Перед началом вычисли корень репозитория:

```bash
ROOT="$(git rev-parse --show-toplevel 2>/dev/null || pwd)"
```

Используй пути:

- База знаний: `$ROOT/1-Context/AURA-Theses/`
- Концепт книги: `$ROOT/3-Marketing/2-Writing-Book/bookconcept.md`
- Текущие тезисы: `$ROOT/3-Marketing/2-Writing-Book/2-Theses/`
- Временные файлы: `$ROOT/.codex/temp/thesis-{slug}/`

Создай temp-папку сразу:

```bash
mkdir -p "$ROOT/.codex/temp/thesis-{slug}"
```

## Входные данные

Режимы запуска:

1. CREATION:
- Тема/название статьи
- Сырые заметки (raw thoughts)

2. ITERATION:
- Путь к существующему `Theses-*.md`
- Фидбэк: что удалить, углубить, добавить, проверить

## Этап 0: Определи режим

Правила:

- Если есть путь к `Theses-*.md` или явный запрос "доработай/углуби/поправь" -> `ITERATION`
- Иначе -> `CREATION`

Для `ITERATION` проверь файл:

- Есть ли заполненная секция `## 🔄 Фидбэк автора для итерации`
- Есть ли `**Версия:**`

## Общие правила качества

1. Сохраняй контекст после каждого крупного этапа:
- `_spring-context.md`
- `01-raw-theses.md`
- `research-knowledge.md`
- `research-expert.md`
- `research-contrarian.md`
- `coverage-round-{N}.md`
- `04-synthesis.md`

2. Перед каждым новым этапом перечитывай `_spring-context.md` и релевантные сохраненные файлы.

3. Каждый источник должен быть проверяем:
- Веб: реальный URL
- KB: точный путь файла внутри `1-Context/AURA-Theses/`

4. Не выдумывай ссылки и не оставляй "голые" упоминания экспертов без URL.

5. Для внешних данных приоритет свежести:
- 2025-2026 -> высокий
- 2024 -> средний
- Старше -> только если это фундаментальный источник

## CREATION workflow

### Этап 1: Извлеки тезисы из raw thoughts

Извлеки:

- Явные тезисы
- Неявные гипотезы
- Примеры/кейсы/метрики
- TODO и незавершенные мысли
- Ссылки/цитаты из исходного текста

Для каждого тезиса фиксируй:

- Тип: утверждение / гипотеза / пример / цитата / TODO
- Суть: 3-10 предложений
- Ключевые цитаты из исходника

Сохрани в `01-raw-theses.md`.

### Этап 2: Проведи 3 исследовательских трека

Если есть возможность, выполняй треки параллельно; иначе последовательно.

Трек A `knowledge-researcher`:

- Изучи `_index.md` и релевантные файлы в `1-Context/AURA-Theses/`
- Обязательно прочитай:
  - `AJTBD/fundamentals.md`
  - `AJTBD/value-creation.md`
  - `AURA/overview.md`
- Углубляй тезисы автора материалом KB, не тащи в статью отдельные темы, не связанные с текущим фокусом.

Трек B `expert-hunter`:

- Найди сильные внешние мнения по теме
- Ищи свежие статьи, треды и исследования
- Выдели потенциальные новые тезисы (`🆕 НОВЫЙ`), только если они реально усиливают тему статьи

Трек C `contrarian`:

- Найди сильные контраргументы к ключевым тезисам
- Проверь, что могло устареть
- Отметь, где AI/LLM меняет правила
- Превращай критичные контраргументы в самостоятельные новые тезисы

Сохрани результаты:

- `research-knowledge.md`
- `research-expert.md`
- `research-contrarian.md`

### Этап 3: Quality Control (до 2 раундов)

Считай `Coverage Score` по 8 критериям:

- Экспертное покрытие
- Свежесть источников
- Глубина тезисов
- Контраргументы
- Связь с Knowledge Base
- Actionability
- Наличие ссылок
- Качество новых тезисов (релевантность, не количество)

Если `Coverage Score < 85%`, сделай точечный дополнительный ресерч по самым слабым критериям.

Ограничение: максимум 2 раунда.

После каждого раунда сохраняй `coverage-round-{N}.md`.

### Этап 4: Синтез

Перед синтезом перечитай:

- `_spring-context.md`
- `01-raw-theses.md`
- `research-knowledge.md`
- `research-expert.md`
- `research-contrarian.md`
- последний `coverage-round-{N}.md`

Собери 2 блока:

1. `📝 Тезисы автора (обогащенные)`:
- пометка `📝 Авторский`
- глубина: `Поверхностный / Средний / Глубокий`
- актуальность: `Вечнозеленый / Актуальный 2025+ / Устаревающий`
- подтверждения, контраргументы, практическая применимость

2. `🆕 Новые тезисы от команды`:
- только релевантные теме статьи
- для каждого: источник, связь с авторскими тезисами, спорность
- не более 10-12 новых тезисов

Сохрани в `04-synthesis.md`.

### Этап 5: Запиши финальный файл

Создай файл:

`$ROOT/3-Marketing/2-Writing-Book/2-Theses/Theses-{Название-статьи}.md`

Минимальная структура:

```markdown
---
## 📝 Инструкция для draft-этапа

> *[Поле для автора]*

---

## 🔄 Фидбэк автора для итерации

> *[Заполни это поле и запусти $new-article-thesis снова]*

---

# {Название статьи}

> **Статус:** Расширенные тезисы (фаза расхождения)
> **Версия:** 1
> **Дата:** {YYYY-MM-DD HH:MM}
> **Тезисов автора:** {N}
> **Новых тезисов:** {M}
> **Coverage Score:** {X}%

## Главная тема
[1-2 предложения]

## 📝 Тезисы автора (обогащенные)
[список]

## 🆕 Новые тезисы от команды
[список]

## Все ссылки и источники
[URL и пути к KB]

## Исходный поток мыслей
>[оригинал]
```

## ITERATION workflow

### И1: Разбери фидбэк

Классифицируй каждый пункт:

- Удалить
- Переформулировать
- Объединить
- Разделить
- Переставить
- Углубить
- Добавить
- Проверить актуальность

### И2: Выбери стратегию

- `Light`: только простые правки, без ресерча
- `Targeted-Deep`: есть задачи "углубить/добавить/проверить", делай прицельный ресерч только под конкретные тезисы

### И3: Выполни

Для `Targeted-Deep`:

- Делай узкий ресерч (3-5 качественных источников на конкретный запрос)
- Не перезапускай полный цикл CREATION
- Сохраняй результаты в temp-файлы `iteration-*.md`

### И4: Обнови исходный файл in-place

1. Примени изменения в том же `Theses-*.md`
2. Инкрементируй `**Версия:**`
3. Обнови `**Дата:**`
4. Добавь строку `**Последняя итерация:** ...`
5. Очисти секцию `## 🔄 Фидбэк автора для итерации` до placeholder

## Чек-лист перед завершением

- [ ] Режим определен корректно (`CREATION` или `ITERATION`)
- [ ] Все тезисы автора сохранены без потери смысла
- [ ] Источники проверяемы (URL/путь), без выдумок
- [ ] У новых тезисов есть связь с темой текущей статьи
- [ ] В итерации изменения внесены in-place и версия повышена
- [ ] Сохранены промежуточные файлы в `.codex/temp/thesis-{slug}/`
