# План запуска: КДП Полный (предзаписанный формат)

> Дата: 2026-03-31
> Продукт: КДП Полный (Job 2 — "Получить систему вместо чуйки")
> Формат: предзаписанные лекции + задания в VS Code (в браузере) + AI-фидбек + кураторы
> Целевая аудитория: действующие продакты (Middle/Senior/Head), 30% всех студентов
> Цена: 90-130K RUB, 6 недель
> Статус: планирование

---

## Суть проекта

Превратить текущий КДП из курса с живыми воркшопами Вани в **предзаписанный продукт с кураторами**. Студент заходит через браузер в личный кабинет, попадает в VS Code, где уже лежат все материалы курса. Учится в своём темпе. Claude Code в терминале — основной инструмент для выполнения заданий. Кураторы поддерживают.

**Зачем:** масштабирование без участия Вани. Ваня записал контент один раз — дальше продукт работает сам.

## Команда и роли

### Илья — Продюсер

**Что делает:**
- Координирует всех участников, ставит задачи, контролирует сроки
- Принимает финальные решения по продукту (что включаем, что нет)
- Проводит инвентаризацию контента (какие лекции есть)
- Тестирует продукт перед запуском
- Общается с Ваней по утверждению программы
- Набирает и готовит кураторов
- Запускает продажи

**Не делает:** не пишет код, не записывает лекции

### Разработчик — Инфраструктура

**Что делает:**
- Поднимает cloud VS Code (code-server / Gitpod / аналог)
- Делает систему авторизации (студент логинится → попадает в своё окружение)
- Настраивает автоматическое развёртывание шаблона проекта при первом входе
- Настраивает деплой и обслуживание серверов
- Помогает с интеграцией Claude Code в окружение

**Ключевые решения, которые нужно принять:**
- Какую технологию использовать (code-server, Gitpod, Coder, GitHub Codespaces)
- Где хостить (свой сервер, облако)
- Сколько ресурсов на одного студента (RAM, CPU)
- Как масштабировать при росте числа студентов

### Методист — Структура обучения

**Что делает:**
- Берёт записанные лекции и выстраивает в логичный порядок
- Определяет, какие лекции нужно перезаписать или дополнить
- Пишет описание к каждому модулю: что студент узнает, что сделает, как проверим
- Определяет точки контроля: после каких модулей куратор проверяет результат
- Пишет задания к модулям совместно со скилл-мейкером
- Создаёт регламент для кураторов

**Кто это может быть:**
- Действующий методист из команды Вани (если есть)
- Методист на фрилансе с опытом в онлайн-образовании
- Илья сам (если готов взять эту роль, но это удлинит сроки)

### Скилл-мейкер — AI-задания

**Что делает:**
- Пишет скиллы для Claude Code — промпты, которые превращают задания в интерактивный опыт
- Настраивает AI-фидбек: студент сделал задание → Claude Code проверяет и даёт обратную связь
- Адаптирует существующие AI-инструменты (AURA, ABCDX) под формат курса
- Пишет инструкцию по Claude Code для студентов

**Какие скиллы нужны (предварительно):**
1. **Аналитик интервью** — студент загружает транскрипцию, скилл вытаскивает Jobs, сегменты, триггеры, барьеры
2. **Граф работ** — по результатам интервью строит интерактивный граф связей между работами
3. **ABCDX-сегментация** — автоматическая сегментация клиентов по данным
4. **Калькулятор юнит-экономики** — интерактивный HTML-дашборд
5. **Приоритизация бэклога** — ранжирование фич по данным из графа работ и сегментов
6. **AI-фидбек по заданиям** — проверяет каждое задание и даёт обратную связь

**Кто это может быть:**
- Человек, который умеет писать промпты + понимает AJTBD
- AI может помочь с написанием скиллов (я готов участвовать)

### Кураторы — Живая поддержка

**Что делают:**
- Отвечают на вопросы студентов (в чате или на созвонах)
- Проверяют ключевые задания (интервью, граф работ, сегментация)
- Проводят групповые разборы (1 раз в неделю, 1-1.5 часа)
- Мотивируют отстающих студентов

**Формат работы (нужно утвердить):**
- Сколько студентов на куратора? (ориентир: 15-25 человек)
- Канал связи: Telegram-чат группы? Чат внутри VS Code? Slack?
- Частота проверок: после каждого модуля или в ключевых точках?
- Живые созвоны: еженедельно или по запросу?

### Ваня — Экспертиза и валидация

**Что делает:**
- Утверждает программу курса (финальное слово по содержанию)
- Записывает недостающий контент (если выявится при инвентаризации)
- Консультирует по методологии AJTBD при создании заданий

**Не делает:** не участвует в проведении курса, не ведёт воркшопы

**Кто:** Илья

**Зависимости:** нет, можно начинать сразу

4. Результат оформить в таблицу

**Результат:** Таблица контента — полная картина "что есть / чего нет"

**Шаблон таблицы:**

#### 1.2 Составить программу курса

**Кто:** Методист + Илья
**Срок:** 3-5 дней
**Зависимости:** после 1.1 (нужно знать, какие лекции есть)

**Что сделать:**
1. Взять текущую структуру КДП (6 недель) как основу
2. Разбить на модули. Каждый модуль = 1 учебная единица:
   - Записанная лекция (видео)
   - Тезисы в тексте (закрепление)
   - Квиз (1-2 вопроса, проверка понимания)
   - Задание (промпт для Claude Code)
   - AI-фидбек (автоматическая проверка)
3. Определить точки контроля куратора: после каких модулей куратор смотрит результат
4. Определить последовательность: какие модули обязательные, какие опциональные
5. Описать, какой результат (файл/артефакт) студент получает после каждого модуля

**Результат:** Документ "Программа курса" — список всех модулей с описанием

**Структура модуля (пример):**

```
## Модуль X: [Название]

**Неделя:** N
**Лекция:** [название записи] (NN мин)
**Тезис:** [ключевая мысль в 1-2 предложениях]
**Квиз:** [1-2 вопроса]
**Задание:** [описание + промпт для Claude Code]
**Результат:** Файл [path/filename.md] в проекте студента
**AI-фидбек:** [что проверяет нейронка]
**Проверка куратором:** Да / Нет
```

**Результат:** Утверждённая программа + список лекций для перезаписи (если нужно)

#### 1.4 Определить формат работы кураторов

**Кто:** Илья (+ методист, если уже есть)
**Срок:** 1-2 дня
**Зависимости:** можно делать параллельно с 1.1-1.2

**Что сделать:**

#### 1.5 ТЗ для разработчика

**Кто:** Илья + Разработчик

5. Определить, как подключается Claude Code: предустановлен или студент ставит сам
