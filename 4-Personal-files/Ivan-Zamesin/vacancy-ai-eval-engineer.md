# AI Evaluation & Pipeline Engineer

## Зачем нам этот человек

Мы строим AI-продукт: pipeline агентов, который генерирует сегменты клиентов, графы работ (AJTBD), критические цепочки и бизнес-модели. Главная проблема — мы не умеем измерять качество выхода каждого этапа pipeline. Без метрик невозможно ни улучшать промпты, ни прикрутить автоматическую оптимизацию (autoresearch-подход Карпати).

Нам нужен человек, который:
1. Построит систему оценки (evals) для каждого этапа pipeline
2. Настроит autoresearch-цикл: агент модифицирует промпты → прогоняет на тестах → keep/discard
3. Будет итеративно улучшать качество генерации

## Роль: AI Evaluation & Pipeline Engineer

### Core задачи (что делает каждый день)

- **Проектирует evals:** рубрики для LLM-as-Judge, структурные проверки, композитные метрики для каждого этапа pipeline
- **Создает и поддерживает gold standard:** вместе с Иваном (доменный эксперт AJTBD) создаёт эталонные выходы для 5-10 тестовых продуктов
- **Строит eval pipeline:** автоматический прогон промптов по тест-кейсам, скоринг, логирование, дашборд результатов
- **Оптимизирует промпты:** ручной и автоматический (autoresearch) prompt engineering
- **Строит agent orchestration:** multi-step pipeline, передача контекста между этапами, обработка ошибок
- **Мониторит качество:** observability, логирование, алерты при деградации метрик

### Что этот человек решал в прошлом

- Строил LLM-based pipelines (RAG, multi-agent, chain-of-thought) и доводил их до production quality
- Писал evals для LLM-выходов (не просто "работает/не работает", а рубрики с весами)
- Использовал LLM-as-Judge для автоматической оценки качества
- Оптимизировал промпты систематически (не vibe-based, а metric-driven)
- Работал с structured output: JSON schemas, validation, parsing
- Интегрировал несколько LLM-вызовов в pipeline с обработкой ошибок и retry

### Требования (must have)

- **Опыт 2+ года** работы с LLM в production (не pet projects)
- **Python** на уровне построения data/ML pipelines
- **Понимание eval-методологий:** LLM-as-Judge, rubric-based scoring, inter-annotator agreement, A/B testing промптов
- **Prompt engineering:** умение систематически улучшать промпты, а не гадать
- **Structured output:** работа с JSON schema, Pydantic, validation
- **Английский B2+** (документация, промпты, LLM-коммуникация)
- **Самостоятельность:** мы не будем ставить таски каждый день, нужен человек, который сам видит что улучшать

### Nice to have

- Опыт с autoresearch / automated prompt optimization (DSPy, ADAS, TextGrad)
- Знание фреймворков: LangChain, LangSmith, Weave, Braintrust, Promptfoo
- Опыт с multi-agent systems (CrewAI, AutoGen, Claude Agent SDK)
- Знание продуктовых методологий (JTBD, Lean, Design Thinking) — не обязательно, но поможет понять домен
- Опыт работы с observability (трейсинг LLM-вызовов, стоимость, латентность)

### Что точно НЕ подходит

- Человек, который "немного баловался с ChatGPT" — нам нужен инженер, не энтузиаст
- Data Scientist без опыта LLM pipelines — другая специализация
- Backend-разработчик без опыта работы с LLM — нужен именно AI engineering
- Человек, который ждёт детальных ТЗ — у нас исследовательская работа, нужна инициатива

### Тестовое задание

Дать кандидату реальную задачу:

> Вот промпт, который генерирует core jobs для продукта. Вот 3 примера выходов.
> 1. Напиши рубрику оценки качества (5-7 критериев с весами)
> 2. Реализуй LLM-as-Judge, который скорит выходы по этой рубрике

Это задание проверяет ровно те навыки, которые нужны: eval design + prompt optimization + measurement.

### AI Evaluation & Pipeline Engineer — оптимизация качества LLM pipeline

Мы в Zamesin Academy строим AI-продукт, который моделирует бизнес-стратегии: генерирует сегменты клиентов, графы задач (jobs), критические цепочки и каналы привлечения. Под капотом — pipeline из нескольких LLM-агентов, каждый решает свой этап анализа.

**Что будешь делать:**
- Проектировать систему оценки качества (evals) для каждого этапа pipeline
- Строить LLM-as-Judge с рубриками и весами
- Создавать gold standard вместе с доменным экспертом
- Оптимизировать промпты метрик-driven подходом
- Настраивать автоматический цикл оптимизации
- Мониторить качество и стоимость pipeline

**Что нужно:**
- 2+ года с LLM в production
- Python (data/ML pipelines)
- Опыт с eval-методологиями (LLM-as-Judge, rubric scoring)
- Системный prompt engineering (не vibe-based)
- Самостоятельность и инициатива

**Условия:** удалённо, асинхронно, part-time → full-time. Пересечение с UTC+3.
