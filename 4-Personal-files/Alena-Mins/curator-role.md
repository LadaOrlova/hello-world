## Часть 1. Должностная инструкция куратора

### 1.1. Роль и смысл

Куратор — это **ментор по содержанию**, а не FAQ-бот и не техподдержка.

Студент должен понимать: куратор помогает разобраться с методологией, продвинуться в задании, преодолеть затык — а не объяснять, как войти в платформу.

Ключевая функция: **помочь студенту перейти от понимания к практике** — особенно в месте, где пересекаются методология AJTBD и работа в VS Code + Claude Code.

---

### 1.2. Что делает куратор

| Зона | Что конкретно |
|------|--------------|
| **Контакт** | Выходит на связь с каждым студентом в первые 24 часа после старта потока |
| **Сопровождение** | Отслеживает прогресс каждого студента минимум 1 раз в неделю |
| **Диагностика** | Выясняет реальный статус студента: на каком модуле, есть ли затыки, какое настроение |
| **Менторинг** | Задаёт наводящие вопросы — не даёт ответ сразу, помогает студенту прийти к нему самому |
| **Техническая помощь** | Помогает разобраться с VS Code, Claude Code и заданиями курса |
| **Эскалация** | Сигнализирует ответственному, если студент в красной зоне (молчит 5+ дней, собирается уходить) |
| **Фиксация** | Заполняет карточку студента по стандартному шаблону после каждого контакта |

---

### 1.3. Что куратор НЕ делает

- **Не делает задания за студента.** Даже если просят — только наводящие вопросы.
- **Не отвечает за операционные вопросы** (оплата, перенос доступа, технические баги платформы) — это другой процесс.
- **Не игнорирует молчание.** «Не донимаю» — не стратегия. Молчит 3 дня — пишешь сам.
- **Не ставит диагноз «всё в порядке»** без подтверждения: студент написал что-то содержательное или рассказал о своём прогрессе.

---

### 1.4. Обязательные компетенции

Для новых курсов («Запуск», «Починка») куратор должен уверенно владеть **двумя** областями:

#### А. Методология AJTBD / AURA
- Понимает суть AJTBD: люди покупают прогресс, а не продукт
- Умеет объяснить разницу между Jobs, Pain, Gain — своими словами
- Знаком с ABCDX-сегментацией и Unit Economics на базовом уровне
- Прошёл КДП (поток 50+) или эквивалентную подготовку

#### Б. Вайбкодинг (VS Code + Claude Code)
- Умеет работать в VS Code: открывать файлы, запускать терминал, читать результаты
- Понимает базовый флоу Claude Code: промпт → файл → результат
- Прошёл BOOST или эквивалент (или имеет практику работы с AI-кодингом)

#### В. Менторские навыки
- Задаёт вопросы, а не читает лекции
- Не осуждает за медленный темп или незнание
- Умеет удерживать студента в процессе без давления

---

### 1.5. Стандартная карточка студента

После каждого контакта куратор заполняет карточку. Формат основан на практике Тани (лучшей по качеству диагностики из потоков КДП):

```
Студент: [имя]
Дата контакта: [дд.мм]
Контакт: развёрнутый / по делу / краткий / не ответил
Инициатива: сам написал / ответил на мой запрос / нет
Текущий модуль: [номер] / не знает
Темп: в потоке / немного отстаёт / сильно отстаёт / не начинал
Трудности: [конкретно — что именно непонятно] / не обозначал
Настроение: спокойное / воодушевлён / тревожится / демотивирован
Действие куратора: [что сделал — вопрос, объяснение, направил к ресурсу]
Красная зона: да / нет
```

**Красная зона** = комбинация из трёх и более признаков:
- Не отвечает 5+ дней
- Не смотрит лекции
- Не сдавал задания 2+ недели
- Написал что хочет уйти или "не понимаю зачем это"

При красной зоне → эскалация ответственному в течение 24 часов.

---

### 1.6. Протокол работы с неотвечающими студентами

1. **День 3** после отсутствия контакта: написать в личный чат, простой вопрос ("как дела с курсом?")
2. **День 5**: написать повторно, другой угол ("видел ты прошёл модуль 2 — как ощущения?")
3. **День 7**: написать в третий раз, предложить помощь ("если что-то застряло — давай разберём вместе")
4. **День 10**: эскалация ответственному + отметить в карточке

Молчание — это не норма, а сигнал. Студент либо тихо учится сам (хорошо), либо потерян (плохо). Не узнаешь — не поможешь.

---

### 1.7. Скрипт первого сообщения студенту

```
Привет, [имя]! Я [имя куратора], твой личный куратор на курсе.

Моя задача — помочь тебе не застрять. Если что-то непонятно в материале, 
задание кажется размытым или VS Code ведёт себя странно — пиши мне напрямую.

Пара вопросов чтобы познакомиться:
— Над каким продуктом работаешь (или будешь работать на курсе)?
— Был ли у тебя опыт с VS Code или Claude Code раньше?

Первый модуль уже открыт — как ощущения?
```

---

### 1.8. Скрипт содержательных вопросов

Вместо "как дела?" — конкретные вопросы по содержанию:

- «На каком модуле сейчас? Что было самым сложным?»
- «Начал исследование? Уже говорил с клиентами?»
- «Что получилось из последнего задания — покажи, разберём вместе?»
- «В прошлый раз ты говорил, что не понимаешь джобы. Стало понятнее?»
- «Есть ли у тебя реальный продукт для практики — или работаешь с учебным кейсом?»

## Часть 2. Регламент подбора и обучения кураторов

### 2.1. Принципы найма

**Нанимаем под конкретный курс, а не "вообще куратора".** Запуск и Починка требуют двойной экспертизы (AJTBD + вайбкодинг) — это нетиповой профиль.

**Критерии (must have):**
1. Прошёл КДП (поток 50 или позже) — понимает методологию изнутри как студент
2. Работал с VS Code + Claude Code — не теория, а практика
3. Умеет задавать вопросы, не даёт готовые ответы — проверяется на собеседовании

**Критерии (nice to have):**
- Опыт менторства, преподавания, коучинга
- Опыт работы продактом или основателем
- Прошёл BOOST

**Красные флаги (не нанимать):**
- Говорит "я объясню студентам как надо" вместо "я помогу им разобраться"
- Не может объяснить AJTBD своими словами
- Нет практики с Claude Code — только слышал

---

### 2.2. Процесс подбора

**Шаг 1. Скрининг (15 минут, асинхронно)**

Кандидат отвечает письменно на 3 вопроса:
1. Объясни AJTBD своими словами — как бы ты объяснил это студенту, который первый раз слышит?
2. Студент пишет: "Я не понимаю как строить граф работ, это слишком абстрактно". Что ты ответишь?
3. Расскажи о своём опыте работы с Claude Code. Что делал, что получилось?

**Критерий отсева:** если в ответе на вопрос 2 кандидат начинает объяснять теорию — это FAQ-бот, не ментор.

---

**Шаг 2. Тестовое задание (1-2 часа)**

Кандидат проходит 1-2 модуля курса в роли студента:
- Открывает VS Code
- Выполняет задание по промпту курса
- Записывает голосовое / текстовое: "что было непонятно, где застрял, что помогло"

Цель: кандидат должен понять боль студента изнутри.

---

**Шаг 3. Собеседование (30 минут)**

Структура:
1. Разбор тестового задания — что было сложно, что понравилось (10 мин)
2. Кейс: дан реальный кейс студента (из анализа кураторов КДП) — как бы отреагировал? (10 мин)
3. Вопросы про мотивацию: зачем это тебе, сколько времени готов вкладывать (10 мин)

**Ключевые вопросы на собеседовании:**
- "Студент уже 10 дней молчит. Что делаешь?"
- "Студент говорит: хочу уйти, не мой курс. Как реагируешь?"
- "Студент просит: сделай задание за меня, у меня нет времени. Что отвечаешь?"

---

**Шаг 4. Решение**

Решение принимается в течение 2 рабочих дней после собеседования.

Ответственный за найм кураторов: Илья Рублевский (задача 5.4 апреля 2026).

---

### 2.3. Обучение перед стартом потока

Обучение занимает **3-5 рабочих дней** до старта первой когорты.

#### День 1-2: Погружение в курс

- Куратор проходит все модули курса самостоятельно (полностью, как студент)
- Выполняет все задания через VS Code + Claude Code
- Фиксирует: где застревал, что было непонятно, какие вопросы возникли

**Результат:** куратор знает курс изнутри, может предсказать боли студентов.

---

#### День 3: Стандарты работы

Совместная сессия (2-3 часа) с ответственным:

1. **Разбор должностной инструкции** — обсуждение кейсов, вопросы
2. **Практика карточки студента** — заполнить 5 карточек по реальным кейсам из анализа КДП
3. **Скрипты** — разобрать первое сообщение, содержательные вопросы, протокол молчунов
4. **Красные флаги и эскалация** — кто, куда, в какие сроки

---

#### День 4: Калибровка

Разбор 3-4 реальных кейсов из кураторов КДП — как бы поступил, что ответил?

Примеры кейсов для разбора:
- Студент: "Я не понимаю джобы, это слишком абстрактно, у меня нет своего продукта"
- Студент молчит 8 дней, на последнее сообщение не ответил
- Студент: "Зачем вы мне пишете, я сам разберусь"
- Студент: "Я не согласен с методологией, это всё теория"
- Студент: "Объясните мне граф работ, у меня воркшоп завтра"

Цель калибровки: убедиться, что куратор задаёт вопросы, а не читает лекцию.

---

#### День 5: Пробный чек-ин

Куратор проводит пробный чек-ин с реальным студентом из прошлого потока КДП (или с коллегой в роли студента):

- 15-минутный созвон или переписка
- Ответственный наблюдает и даёт фидбек после

**Критерий готовности:** куратор задал 3+ содержательных вопроса, не объяснял теорию без запроса, заполнил карточку после разговора.

---

### 2.4. Сопровождение кураторов в процессе потока

**Еженедельно:**
- Куратор сдаёт карточки по всем студентам (каждую пятницу)
- Ответственный просматривает карточки, отмечает красные зоны
- Короткий синк 30 мин: разбор сложных кейсов

**Ежемесячно:**
- Калибровочная сессия: разбор 2-3 кейсов всеми кураторами вместе
- Сравнение подходов, выравнивание стандарта

**После потока:**
- Ретроспектива: что сработало, что нет, что менять в следующей когорте

---

### 2.5. Нагрузка и оплата

| Параметр | Значение |
|----------|---------|
| Студентов на куратора | 20-30 |
| Касаний в неделю | 1-2 на студента (текст или звонок) |
| Среднее время в неделю | 5-8 часов |
| Карточки студентов | Обновляются еженедельно, сдача по пятницам |

> Конкретные ставки фиксируются отдельно (задача 5.4 апреля 2026).
