# Архитектура Telegram-бота: от хаоса к системе

## Проблема

Один бот на ~15K подписчиков, в который свалено всё:
- Регистрация на вебинары и цепочки прогрева
- Контент-план (посты про кейсы, методологию)
- Меню с продуктами (КДП, BOOST, AURA)
- Кейсы по категориям
- Квиз "Подходит ли мне?"
- Функционал конференции (25-27 марта)
- Пост-вебинарные продающие цепочки

Результат: все 15K получают всё подряд, люди отписываются, конверсия падает.

---

## 1. Система тегов

Каждый подписчик получает теги из трёх категорий. Теги присваиваются автоматически по действиям пользователя.

### Источник (откуда пришёл)

| Тег | Когда ставится |
|-----|---------------|
| `src:webinar_120326` | Регистрация на вебинар 12 марта |
| `src:webinar_ДДММГГ` | Регистрация на любой будущий вебинар |
| `src:conf_250327` | Регистрация на конференцию 25-27 марта |
| `src:organic` | Подписался сам (через поиск, ссылку в канале) |
| `src:youtube` | Пришёл с YouTube |
| `src:linkedin` | Пришёл с LinkedIn |
| `src:referral` | Пришёл по реферальной ссылке |

### Интерес (к какому продукту)

| Тег | Когда ставится |
|-----|---------------|
| `int:kdp` | Нажал на описание КДП / прошёл квиз с результатом КДП |
| `int:boost` | Нажал на описание BOOST / прошёл квиз с результатом BOOST |
| `int:aura` | Нажал на описание AURA |
| `int:consultation` | Нажал "Консультация с Русланом" |
| `int:free_content` | Только читает контент, не кликал на продукты |

### Стадия (уровень прогрева)

| Тег | Когда ставится |
|-----|---------------|
| `stage:new` | Только подписался (автоматически) |
| `stage:engaged` | Прочитал 3+ постов / прошёл квиз / смотрел кейсы |
| `stage:hot_lead` | Кликнул на продукт 2+ раза / запросил консультацию |
| `stage:customer` | Купил любой продукт |
| `stage:alumni` | Завершил обучение |

> **Важно:** теги не удаляются, а накапливаются. Человек может быть одновременно `src:webinar_120326` + `int:kdp` + `stage:hot_lead`.

## 2. Постоянное меню бота

Меню доступно всегда через кнопку Menu в Telegram. Это НЕ рассылка — пользователь сам выбирает, что смотреть.

```
BOOST – продуктовый вайбкодинг. За неделю от идеи до MVP без опыта кода → описание + ссылка на лендинг
Makers → описание + ссылка
AURA – AI-платформа для ускоренного создания продуктов на базе AJTBD → описание + ссылка на лендинг
Кейсы и материалы → кейсы по категориям + бесплатные ресурсы
📅 Ближайшие события → динамический блок (ближайший вебинар или конференция)
```

### Реализация меню

- В BotHelp / SendPulse это делается через **persistent menu + inline-кнопки**.
- Каждый клик по разделу меню присваивает соответствующий тег интереса.
- Блок "Ближайшие события" обновляется вручную раз в 1-2 недели (Аника меняет текст и ссылку).

## 3. Правила рассылок: кто что получает

### Матрица рассылок

| Тип контента | Кому отправлять | Частота |
|-------------|----------------|---------|
| Контентные посты (кейсы, советы, методология) | Всем подписчикам | 2-3 раза в неделю |
| Приглашение на вебинар | Всем, КРОМЕ `stage:customer` + `stage:alumni` | За неделю до вебинара |
| Цепочка после вебинара (прогрев) | Только `src:webinar_ДДММГГ` конкретного вебинара | 3-5 сообщений за 5 дней |
| Сообщения конференции | Только `src:conf_250327` | По расписанию конференции |
| Продающая цепочка КДП | `int:kdp` + (`stage:engaged` ИЛИ `stage:hot_lead`) | Разово, после триггера |
| Продающая цепочка BOOST | `int:boost` + (`stage:engaged` ИЛИ `stage:hot_lead`) | Разово, после триггера |
| Пост-вебинар (укороченный КДП + BOOST) | `src:webinar_ДДММГГ` + НЕ `stage:customer` | 3-4 сообщения за неделю |
| Upsell для выпускников | `stage:alumni` | Не чаще 1 раз в месяц |

### Правило "не больше 4 касаний в неделю"

Независимо от тегов, один человек не должен получать больше 4 сообщений в неделю (включая контентные посты). Если человек попадает в несколько цепочек одновременно — приоритет:

1. Продающая цепочка (если запущена) — заменяет контентный пост
2. Событийные сообщения (вебинар/конференция)
3. Контентные посты

> В BotHelp это решается через задержку отправки и проверку "последнее сообщение было менее 24 часов назад". В SendPulse — через условия в автоматизациях.

## 4. Онбординг нового подписчика

Универсальный флоу, независимо от точки входа.

```
Подписался на бота
       │
       ▼
[Сообщение 1 — сразу]
"Привет! Я Ваня Замесин. Создал методологию Advanced JTBD, обучил 12 000+ человек создавать продукты, которые покупают. Сейчас активно работаю на стыке продуктового мышления и AI.

В этом боте я делюсь кейсами, как вырастить выручку x3–x13, приглашаю на бесплатные вебинары и конференции. Анонсирую запуски новых интенсивов раньше, чем где-либо.

Здесь ты научишься делать продукт, за который платят, и проверять идеи за пару дней с помощью вайбкодинга, без зависимости от разработчиков.

Что тебе сейчас актуальнее?"

  [Хочу разобраться в продуктах]    →  ставит int:kdp или int:boost (по дальнейшему выбору)
  [Просто хочу читать контент]      →  ставит int:free_content
  [Записаться на ближайшее событие]  →  показывает ближайший вебинар/конференцию
       │
       ▼
[Сообщение 2 — через 1 день]
Лучший кейс из библиотеки (по категории, если выбрал; универсальный, если нет)
       │
       ▼
[Сообщение 3 — через 3 дня]
"Кстати, у нас есть короткий квиз — поможет понять,
какой формат обучения подойдёт именно тебе."
  [Пройти квиз →]  →  ведёт в квиз, по результату ставит int:kdp / int:boost / int:aura
  [Пропустить]     →  ничего не делаем
       │
       ▼
Далее — обычный контентный поток по правилам из раздела 3
```

**Если человек пришёл через регистрацию на вебинар:**
- Сообщение 1 заменяется на "Ты зарегистрирован на вебинар [название]. Дата: ... Ссылка придёт за час."
- Сообщение 2 и 3 — те же, но отправляются до вебинара
- После вебинара запускается цепочка прогрева (только для этого вебинара)

**Если человек пришёл через конференцию:**
- Сообщение 1 заменяется на приветствие конференции + расписание
- После конференции — переход в общий контентный поток

## 6. Чеклист для Аники: что сделать

### Неделя 1: Подготовка
- [ ] Выбрать платформу (BotHelp или SendPulse) — если ещё не выбрана
- [ ] Создать систему тегов в платформе (скопировать таблицы из раздела 1)
- [ ] Настроить постоянное меню с кнопками из раздела 2
- [ ] Написать тексты для каждого пункта меню (описания продуктов, списки кейсов)

### Неделя 2: Онбординг и контент
- [ ] Собрать онбординг-цепочку из 3 сообщений (раздел 4)
- [ ] Настроить автоматическую простановку тегов по кликам
- [ ] Создать квиз "Подходит ли мне?" (4-5 вопросов → рекомендация)
- [ ] Подготовить шаблон контентного поста

### Неделя 3: Рассылки и цепочки
- [ ] Настроить сегменты для рассылок по матрице из раздела 3
- [ ] Создать шаблон вебинарной цепочки (регистрация → напоминание → follow-up)
- [ ] Создать шаблон продающей цепочки (3-4 сообщения)
- [ ] Протестировать на себе: подписаться как новый пользователь, пройти все сценарии

### Перед конференцией (до 25 марта)
- [ ] Настроить отдельный тег `src:conf_250327`
- [ ] Подготовить цепочку конференции (приветствие, расписание, напоминания)
- [ ] Убедиться, что участники конференции НЕ получают параллельно другие цепочки

## 7. Частые ошибки, которых стоит избежать

1. **Не делать отдельного бота для конференции.** Тот же бот, но контент по тегу. Иначе — потеря подписчиков после события.

2. **Не отправлять продающие цепочки всем.** Только тем, кто показал интерес (клик по продукту, прохождение квиза).

3. **Не забыть про стоп-лист.** Если человек купил КДП — убирать его из продающей цепочки КДП (тег `stage:customer` + проверка).

4. **Не усложнять.** Начать с 3-5 тегов, а не с 20. Добавлять по мере необходимости.

5. **Контентные посты — не продающие.** Пост про кейс должен быть полезным сам по себе. Продажа — только в продающих цепочках.
