## 1. Суть

Студент, который начал обучение на КДП или BOOST, получает в боте персональную реферальную ссылку. Если по этой ссылке кто-то **покупает** любой наш продукт — студент получает бонус, а покупатель получает скидку.

## 2. Флоу: пошагово

### 2.1. Когда студент получает реферальную ссылку

```
Студент купил КДП/BOOST
       │
       ▼
  Прошёл онбординг в боте (чаты, ЛК, форма)
       │
       ▼
  Начал обучение (посетил 1+ воркшоп / открыл 1+ урок)
       │
       ▼
  Бот отправляет сообщение:
  ┌─────────────────────────────────────────────────┐
  │  "Нравится обучение? Поделись с друзьями!       │
  │                                                 │
  │  Твоя персональная ссылка:                      │
  │  t.me/[botname]?start=ref_[ID]                  │
  │                                                 │
  │  Что получит друг:                              │
  │  → Скидка [X]% на любой наш курс               │
  │                                                 │
  │  Что получишь ты за каждого купившего:           │
  │  → 1 месяц в Мейкерах бесплатно                │
  │  → Скидка [Y]% на следующий курс               │
  │                                                 │
  │  [Скопировать ссылку]  [Мой прогресс]"          │
  └─────────────────────────────────────────────────┘
```

**Почему не сразу после покупки:** если человек оформит возврат до начала обучения, реферальная ссылка уже не действует. Привязка к началу обучения решает проблему возвратов (вопрос Леры).

**Триггер:** определяется по действию в боте (например, нажатие «Я на воркшопе» или автоматически через N дней после старта потока). Конкретный триггер — на усмотрение Леры, главное чтобы человек уже начал учиться.

## 3. Ответы на вопросы Леры

### «Для кого действует скидка — только для новых или и для бывших студентов?»

**Для всех.** Если бывший студент пришёл по реферальной ссылке и купил новый продукт — считается. Реферер получает бонус. Это упрощает логику и не наказывает за привлечение знакомых.

### «Сколько раз ссылка может быть использована?»

**Безлимитно.** Одна ссылка, сколько угодно друзей. Чем больше продадут — тем лучше нам.

### «Где применяется ссылка — на сайте при оплате?»

**Нет, ссылка ведёт в бота.** Флоу такой:
1. Друг переходит по ссылке → попадает в бота
2. В боте видит скидку и кнопку «Посмотреть курсы»
3. Кнопка ведёт на лендинг (zamesin.ru) с промокодом или UTM-меткой
4. Покупает на сайте

**Связка «кто привёл — кто купил»** фиксируется так:
- В боте у друга записан `referred_by = ID реферера`
- При покупке менеджер (или автоматика) проверяет, есть ли у покупателя реферер в боте
- Если есть — начисляет бонус

> **MVP-вариант (если автоматику делать долго):** менеджер вручную проверяет в боте, пришёл ли покупатель по рефералке, и вручную активирует бонус рефереру.

### «Что делать с возвратами?»

Реферальная ссылка выдаётся **только после начала обучения** (см. п. 2.1). Это значит:
- Рефереру не нужно ничего «забирать» — он уже учится
- Если **друг** купил и сделал возврат — бонус рефереру не начисляется (или отзывается). Это проще отследить вручную на MVP

## 5. Техническая реализация (для Леры)

### Что нужно в боте:

| # | Элемент | Описание |
|---|---------|----------|
| 1 | **Кастомное поле `referred_by`** | Текстовое поле. Заполняется автоматически при переходе по ссылке `?start=ref_[ID]`. Хранит ID реферера |
| 2 | **Кастомное поле `referral_count`** | Числовое поле у реферера. Сколько друзей купили по его ссылке |
| 3 | **Тег `src:referral`** | Ставится автоматически другу при входе по реферальной ссылке |
| 4 | **Тег `ref:active`** | Ставится студенту, когда он активировал (получил) свою реферальную ссылку |
| 5 | **Поток «Выдача реферальной ссылки»** | Триггер: начало обучения. Генерирует ссылку `t.me/[bot]?start=ref_[USERID]`, отправляет сообщение |
| 6 | **Поток «Реферальный онбординг»** | Триггер: старт бота с параметром `ref_*`. Записывает referred_by, ставит тег, показывает скидку |
| 7 | **Кнопка «Мой прогресс»** | Показывает рефереру: сколько друзей подписалось, сколько купили, какие бонусы получил |
| 8 | **Кнопка «Скопировать ссылку»** | Повторно показывает реферальную ссылку |

### Про deep link `?start=`

В BotHelp прямая ссылка `t.me/bot?start=ref_123` **запускает бота**, но **не передаёт UTM и кастомные параметры автоматически**.

**Два варианта решения:**

**Вариант A — через `r.bothelp.io` (рекомендуется):**
```
https://r.bothelp.io/tg?domain=[botname]&start=[FLOW_ID]&referred_by=[USERID]
```
Этот формат автоматически записывает значение в кастомное поле `referred_by`.

**Вариант B — через парсинг start-параметра:**
Если бот умеет читать start-параметр (`/start ref_123`), то в первом шаге потока разобрать строку и записать ID в поле. Зависит от возможностей конкретной версии BotHelp.

> **Лера, проверь:** какой из вариантов работает в твоей версии BotHelp. Документация: https://help.bothelp.io/en/kak-peredat-utm-metki-i-parametry-v-pryamyh-ssylkah-na-botov-telegram/

### Про подсчёт покупок (referral_count)

BotHelp **не может** из потока одного подписчика изменить поле у другого подписчика. Поэтому:

**MVP (ручной):**
- Менеджер видит покупку
- Проверяет в боте: есть ли у покупателя поле `referred_by`
- Если да — находит реферера, вручную увеличивает `referral_count`, отправляет сообщение

**Автоматический (через Make.com / Google Apps Script):**
- При покупке webhook отправляет данные наружу
- Скрипт проверяет `referred_by` у покупателя
- Через BotHelp API обновляет `referral_count` у реферера
- Через API отправляет уведомление рефереру

> **Рекомендация:** начать с MVP (ручной), автоматизировать позже.

## 6. Что НЕ входит в эту задачу

- Интеграция с платёжной системой (пока ручная проверка)
- Автоматическая генерация промокодов на скидку (пока один общий промокод для всех рефералов)
- Личный кабинет на сайте с реферальной статистикой (всё в боте)
- Интеграция с ЗИС (пока рано, Женя ещё проектирует)

## 7. Где в боте это живёт

```
Меню бота (постоянное)
  │
  ├── BOOST — описание + лендинг
  ├── КДП — описание + лендинг
  ├── Makers — описание + лендинг
  ├── Кейсы и материалы
  ├── Ближайшие события
  │
  └── 🎁 Пригласи друга          ← НОВЫЙ ПУНКТ
        │
        ├── [если ref:active] → "Твоя ссылка: ... | Прогресс: N друзей"
        └── [если НЕ ref:active] → "Программа доступна после начала обучения"
```

Также ссылка отправляется **проактивно** — бот сам пишет студенту после начала обучения (п. 2.1).
