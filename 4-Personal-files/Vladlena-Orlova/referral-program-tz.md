> Автор: Владлена Орлова | Дата: 2026-03-31

---

## 1. Суть

Выпускник КДП или BOOST получает **персональный промокод на скидку 10% на BOOST**. Передаёт его друзьям. Если друг покупает по промокоду — выпускник получает бонус.

---

## 2. Что получает друг

| Бонус | Размер |
|-------|--------|
| Скидка на BOOST | **10%** по промокоду |

---

## 3. Что получает выпускник (когда друг купил)

| Бонус | Размер |
|-------|--------|
| Дополнительный месяц в Мейкерах | +1 месяц за каждого купившего друга |
| Скидка на будущий продукт экосистемы | **10%** |

---

## 4. Промокод

**Формат:** `ZAMESIN-XXXX` (4 случайных символа, буквы + цифры)
Примеры: `ZAMESIN-7KM2`, `ZAMESIN-R4NW`, `ZAMESIN-3FHL`

- Генерируется автоматически в боте
- Уникальный для каждого выпускника
- Количество использований не ограничено
- Сейчас применяется при оплате BOOST, в будущем — на любой продукт экосистемы

---

## 5. Кто получает промокод

Два сценария:

### 5.1. Текущие студенты (КДП и BOOST)

Получают промокод **в боте после начала обучения** (посетил 1+ воркшоп / открыл 1+ урок).

Бот отправляет сообщение:

```
У тебя есть персональный промокод для друзей.

Если кто-то купит BOOST по твоему промокоду —
ты получишь +1 месяц в Мейкерах и скидку 10%
на будущие продукты.

Твой промокод: ZAMESIN-XXXX

Друг получит скидку 10% на BOOST.

[Скопировать промокод]  [Мой прогресс]
```

### 5.2. Старые выпускники (все прошлые потоки КДП и BOOST)

Основная аудитория — **выпускники КДП прошлых потоков**, которых нет в боте.

**Механика:**

1. **Email-рассылка** по базе выпускников
2. **Посты в старые чаты потоков** (у кого есть доступ)

Рассылка ведёт в бота, где выпускник получает свой промокод.

---

## 6. Email-рассылка для старых выпускников

### Задача рассылки

Не просто «вот тебе промокод», а:
1. Напомнить о себе — мы живы и развиваемся
2. Показать, что экосистема растёт — запускаются новые продукты
3. Дать причину оставаться на связи
4. Выдать промокод как бонус за лояльность

### Текст рассылки

**Тема:** У нас новое — и для тебя кое-что есть

```
Привет!

Ты проходил(а) курс «Как делать продукт» — и с тех пор
у нас многое изменилось.

Мы запустили BOOST — недельный интенсив по вайбкодингу,
где продакты и предприниматели без техбэкграунда собирают
работающий MVP за неделю. Мы развиваем Мейкеры — сообщество
для тех, кто делает продукты. И планируем новые продукты
в экосистеме.

Мы хотим оставаться на связи с выпускниками — и делаем
для этого реферальную программу.

Как это работает:
→ Ты получаешь персональный промокод на скидку 10% на BOOST
→ Передаёшь его другу или коллеге
→ Если он покупает — ты получаешь месяц в Мейкерах бесплатно
  и скидку 10% на наши будущие продукты

Чтобы получить промокод — зайди в нашего бота:
[Получить промокод →]

До связи!
Команда Замесина
```

### Текст для старых чатов потоков

```
Привет, выпускники!

У нас запустилась реферальная программа — для вас
персональный промокод на скидку 10% на BOOST для друзей.
Если друг купит — вы получите месяц Мейкеров и скидку
на будущие продукты.

Промокод — в боте: [ссылка на бота]
```

## 7. Техническая реализация (для Леры)

### Что нужно в боте

| # | Элемент | Описание |
|---|---------|----------|
| 1 | Поле `promo_code` | Текстовое. Хранит персональный промокод выпускника |
| 2 | Поле `referral_count` | Числовое. Сколько друзей купили по его промокоду |
| 3 | Тег `ref:active` | Ставится выпускнику, когда он получил промокод |
| 4 | Поток «Выдача промокода» | Триггер: начало обучения. Генерирует `ZAMESIN-XXXX`, сохраняет в поле, отправляет сообщение |
| 5 | Поток «Выдача промокода (alumni)» | Триггер: вход в бота из email-рассылки. Генерирует промокод для старого выпускника |
| 6 | Кнопка «Мой прогресс» | Показывает: сколько друзей купили, какие бонусы начислены |
| 7 | Кнопка «Скопировать промокод» | Повторно показывает промокод |

### Подсчёт покупок по промокоду

**MVP (ручной):**
- Менеджер видит покупку с промокодом `ZAMESIN-XXXX`
- Находит в боте выпускника с этим промокодом
- Увеличивает `referral_count`, отправляет уведомление

**Позже (автоматический):**
- Через Make.com / Google Apps Script: вебхук от платёжки → обновление полей через BotHelp API

## 8. Где в боте это живёт

```
Меню бота
  │
  ├── ...существующие пункты...
  │
  └── 🎁 Промокод для друзей
        │
        ├── [если ref:active] → промокод + прогресс
        └── [если НЕ ref:active] → «Доступно после начала обучения»
```

## 9. Тексты уведомлений

### Друг купил — уведомление выпускнику

```
Отличная новость!

Кто-то купил BOOST по твоему промокоду.

Тебе начислено:
→ +1 месяц в Мейкерах
→ Скидка 10% на будущие продукты

Всего друзей привёл: [N]

[Пригласить ещё]
```

### Кнопка «Мой прогресс»

```
Твоя статистика:

Купили по промокоду: [N]

Бонусы:
→ Мейкеры: +[N] мес. начислено
→ Скидка 10% на будущие продукты: да/нет

Твой промокод: ZAMESIN-XXXX
[Скопировать промокод]
```

## 10. Что НЕ входит

- Интеграция с платёжной системой (пока ручная проверка промокодов)
- Интеграция с ЗИС
