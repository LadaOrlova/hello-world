## Доменная структура

| Домен | Что на нём | Статус |
|---|---|---|
| `zamesin.ru` | **Основной сайт Академии** — главная, курсы, кейсы, знания, блог, инструменты | Цель: всё обучение здесь |
| `zamesin.ru/boost` | **BOOST** — вайб-кодинг интенсив (сейчас на `boost-intensive.ru`, переносим) | Переезжает под основной домен |
| `cpo-aura-framework.ru` | **AURA** — отдельный AI-продукт (AI CPO), подписка, свой домен | Остаётся отдельным |

### Почему AURA отдельно:
- AURA — это **платный AI-продукт** (подписка, токены), а не обучение
- Своя бизнес-модель, свой рынок (в т.ч. международный, интерфейс на EN)
- 2000+ в waitlist, ~300 в бете
- Часть экосистемы Замесина, но другой тип продукта
- На сайте zamesin.ru — упоминание как часть методологии, ссылка на `cpo-aura-framework.ru`

### Почему BOOST под zamesin.ru:
- BOOST — это **обучение** (интенсив), часть линейки курсов Академии
- Та же аудитория, те же кураторы, тот же Замесин
- Логично жить как `zamesin.ru/boost`

## Главная страница — блоки сверху вниз

| # | Блок | Для кого работает |
|---|---|---|
| 1 | **Hero** — "Создавайте продукты, которые покупают" + proof-метрики (11K+, 9.2/10, 59 потоков, #1) | Все |
| 2 | **Методология** — "Почему это работает" (4 карточки AURA) + теги индустрий и ролей | Холодные → доверие, широкий охват |
| 3 | **Автор** — Иван Замесин, credentials, bio, цифры | Холодные → доверие к эксперту |
| 4 | **Кейсы** — 6 карточек с реальными именами и цифрами + ссылка на полную страницу | Все → social proof |
| 5 | **Инструменты** — лид-магниты (лекция, мини-тест, диагностика) | Холодные → захват контакта |

### Логика порядка блоков:
1. **Обещание** → что вы получите (Hero)
2. **Доказательство метода** → почему это работает (AURA + индустрии/роли)
3. **Доверие к автору** → кто за этим стоит (Замесин)
4. **Social proof** → реальные результаты (Кейсы)
5. **Для нерешительных** → инструменты и лид-магниты

## Страница кейса — структура проваливания

Весь сайт — **взаимосвязанная система**. Человек нигде не упирается в тупик.

### Список кейсов (`/cases`)
- Карточки с фильтрами по джобу, роли, индустрии
- Клик по карточке → подробная страница кейса

### Страница отдельного кейса (`/cases/:slug`)

```
┌─────────────────────────────────────────────┐
│  Имя, роль, компания, индустрия             │
│                                             │
│  ТОЧКА А: боль на входе (цитата)            │
│  ЧТО СДЕЛАЛ: действия на обучении           │
│  ТОЧКА Б: результат (цитата + цифры)        │
│                                             │
│  ─────────────────────────────────────────  │
│                                             │
│  CTA: "Хотите такой же результат?"          │
│  Варианты (тестируем):                      │
│  • Получи разбор своего продукта → Руслан   │
│  • Диагностика: насколько сработает для     │
│    вашего продукта                          │
│  • Записаться на обучение                   │
│                                             │
│  ─────────────────────────────────────────  │
│                                             │
│  📎 Похожие кейсы (по джобу / индустрии)    │
```

**Принцип:** страница кейса не заканчивается кейсом. Всегда есть куда провалиться дальше — в другие кейсы, в статьи, в продукт.

### Распределение по страницам

| Где | Лид-магнит | За что отдаём контакт |
|---|---|---|
| **Главная** | Вводная лекция | Доступ к записи за контакт |
| **Главная** | Мини-тест "Проверь себя" | Результат теста по AJTBD/AURA |
| **Кейс (каждый)** | "Получи разбор своего продукта" → Руслан | Запись на диагностику |
| **Страница курса** | Форма "Узнать о запуске" | Контакт привязан к конкретному продукту (видим в CRM) |
| **Знания / статья** | Связанный гайд или шаблон | Скачать материал |
| **Календарь** | Напоминание о событии | Email/Telegram подписка |

### Лид-магниты на страницах курсов

На каждой странице продукта:
- Форма "Узнать о запуске первым" (Имя + Email + TG)
- **Важно:** сохраняем, **с какой именно страницы продукта** пришёл контакт → видно в CRM
- Менеджеры сразу понимают интерес (Job 1-5, BOOST и т.д.)

**Принцип:** каждая страница сайта = точка входа в воронку. Нигде человек не уходит с пустыми руками. Лид-магниты — только реальные, упакованные.
