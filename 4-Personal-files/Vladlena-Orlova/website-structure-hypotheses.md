# Гипотезы по структуре сайта Вани Замесина

Дата: 2026-03-06

---

## Часть 1: Общий сайт (zamesin.ru) — структура экосистемы

### Проблемы текущей ситуации
- Продукты живут на разных доменах (zamesin.ru, boost-intensive.ru) — нет единой точки входа
- SEO-трафик размазан между доменами, нет внутренней перелинковки
- Нет контентного хаба — весь экспертный контент (12 000+ выпускников, 40+ кейсов, методология AJTBD) не работает на привлечение
- Пользователь, пришедший за одним продуктом, не узнаёт о других

---

### Гипотеза 1: Hub-and-Spoke архитектура сайта

**Структура:**

```
zamesin.ru/                          — Главная (хаб экосистемы)
├── /producthowto                    — Лендинг "Как делать продукт"
├── /boost                           — Лендинг BOOST интенсива
├── /exam                            — Экзамен по AJTBD
├── /aura                            — AI-платформа AURA (LaunchPilot)
├── /book                            — Книга (когда выйдет)
├── /blog                            — Контентный хаб (SEO-ядро)
│   ├── /blog/ajtbd                  — Статьи про AJTBD
│   ├── /blog/cases                  — Кейсы выпускников
│   ├── /blog/vibe-coding            — Статьи про вайбкодинг
│   └── /blog/guides                 — Практические гайды
├── /about                           — Об авторе + миссия
├── /for-teams                       — Корпоративное обучение
├── /webinars                        — Расписание и записи вебинаров
└── /community                       — Сообщество выпускников
```

**Зачем:** Весь домен zamesin.ru работает как единый SEO-организм. Контент на /blog привлекает трафик, внутренние ссылки ведут к продуктам. Домен набирает авторитет целиком.

**Метрика:** Органический трафик на zamesin.ru растёт на 30%+ за 3 месяца.

### Гипотеза 2: Главная страница как навигатор по ситуациям (не по продуктам)

**Идея:** Вместо классического перечисления продуктов — интерактивный навигатор: "Выбери свою ситуацию → получи рекомендацию продукта."

**Блоки главной:**

1. **Hero:** "Ваня Замесин. Помогаю создавать продукты, которые покупают" + цифры (12 000 выпускников, 40+ кейсов)
2. **Навигатор по ситуациям:**
   - "Хочу научиться создавать продукты системно" → Как делать продукт
   - "Хочу создать свой продукт за неделю без программистов" → BOOST
   - "Хочу AI-ассистента для продуктовых решений" → AURA
   - "Хочу обучить команду" → Корпоративное обучение
   - "Хочу сертификат по AJTBD" → Экзамен
3. **Социальное доказательство:** Лого компаний + ротация кейсов
4. **Свежий контент:** Последние статьи и вебинары (тянет SEO)
5. **Подписка на канал/рассылку**

**Зачем:** Посетитель за 5 секунд находит "свой" продукт. Конверсия с главной в лендинг продукта растёт.

**Метрика:** CTR с главной на продуктовые страницы > 40%.

### Гипотеза 3: SEO-контентный хаб (/blog) как основной генератор трафика

**Идея:** Создать 50-100 SEO-статей на основе уже существующего контента (тезисы AURA, транскрипции лекций, кейсы).

**Кластеры:**

| Кластер | Примеры статей | Целевые запросы |
|---------|----------------|-----------------|
| JTBD-методология | "Что такое Jobs To Be Done", "JTBD vs персоны", "Как проводить JTBD-интервью" | jtbd, jobs to be done, jtbd интервью |
| Продуктовые фреймворки | "Что такое Product-Market Fit", "Unit Economics для стартапов", "ABCDX-сегментация" | product market fit, unit economics, сегментация клиентов |
| Кейсы | "Как X увеличил конверсию в 3 раза с AJTBD" (40+ готовых кейсов!) | кейс продукт, рост конверсии, как увеличить продажи |
| Вайбкодинг и AI | "Что такое вайбкодинг", "Как создать MVP без программистов", "AI для продакт-менеджера" | вайбкодинг, mvp без программирования, ai продакт |
| Карьера в продукте | "Как стать продакт-менеджером", "Навыки CPO", "Сертификация PM" | стать продакт менеджером, навыки продакта |

**Каждая статья содержит:** CTA на релевантный продукт + подписка на канал.

**Зачем:** У Вани огромная база контента (транскрипции, тезисы, кейсы), но она не индексируется. Переупаковка в SEO-статьи = бесплатный трафик.

**Метрика:** 5 000+ органических визитов/мес через 6 месяцев.

### Гипотеза 4: Публичный календарь вебинаров + записи

**Структура:**
```
/webinars
├── Расписание предстоящих вебинаров (с регистрацией)
├── Архив записей (с гейтом: email для доступа)
└── Тематические подборки вебинаров
```

**Зачем:** Вебинары — основной инструмент привлечения. Страница с расписанием + архивом:
- Индексируется поисковиками (SEO)
- Собирает лиды через email-гейт на записи
- Показывает экспертизу (социальное доказательство)

**Метрика:** 500+ email-подписчиков/мес с архива вебинаров.

### Гипотеза 5: Страница /for-teams для корпоративных продаж

**Блоки:**
1. Проблема: "Команда делает фичи, а не решает задачи клиентов"
2. Решение: корпоративный тренинг AJTBD + BOOST для команд
3. Кейсы B2B-клиентов (лого, результаты)
4. Формат: адаптивный (онлайн/офлайн, 1 день / 1 неделя)
5. Форма заявки (не цена, а "обсудить формат")

**Зачем:** B2B-продажи = высокий средний чек. Сейчас нет отдельной точки входа для HR/L&D менеджеров. Они ищут "корпоративное обучение продакт-менеджмент" — и не находят Замесина.

**Метрика:** 10+ B2B-лидов/мес с этой страницы.

### Гипотеза 6: Единая страница кейсов /cases

**Идея:** Собрать все 40+ кейсов на одну страницу с фильтрами:
- По отрасли (fintech, edtech, e-commerce, B2B SaaS...)
- По задаче (рост конверсии, запуск продукта, снижение оттока...)
- По роли (продакт, фаундер, маркетолог...)

**Каждый кейс:** Компания → Проблема → Что сделали → Результат → CTA на продукт.

**Зачем:** Кейсы — самый сильный конверсионный контент. 40+ кейсов = уникальное преимущество, которого нет ни у одного конкурента. Сейчас они разбросаны и не работают.

**Метрика:** Конверсия посетителей /cases в заявку > 5%.

## Часть 2: Лендинг "Как делать продукт" (/producthowto)

### Текущая ситуация
- 12 000+ выпускников, 9.2/10 рейтинг, 40+ кейсов — сильнейший продукт на русскоязычном рынке
- Конкуренты: Product University (Ильяхов), Нетология, Skillbox, Product Star

---

### Гипотеза 7: Структура лендинга по модели "Проблема → Трансформация"

**Порядок блоков:**

```
1. HERO
   "Научись создавать продукты, которые покупают"
   Подзаголовок: "Тренинг по Advanced JTBD от Вани Замесина. 12 000+ выпускников."
   CTA: "Записаться на тренинг" + "Посмотреть бесплатную лекцию"

2. ПРОБЛЕМА (идентификация)
   "Узнай себя" — 4-5 ситуаций:
   - "Делаю фичи, но продукт не растёт"
   - "Не понимаю, что хочет клиент"
   - "Конкуренты давят ценой, не знаю как выделиться"
   - "Запустил продукт, а продаж нет"
   - "Принимаю решения на интуиции, а не на данных"

3. РЕШЕНИЕ (методология)
   "AJTBD — система, а не набор инструментов"
   - Визуальная схема: Сегментация → Работы → Ценность → Коммуникация
   - "Не очередной курс, а операционная система для продуктовых решений"

4. ПРОГРАММА (8 модулей)
   Раскрываемые блоки с кратким описанием каждого модуля
   + Что вы научитесь делать (в формате навыков, не тем)

5. РЕЗУЛЬТАТЫ (кейсы) — КЛЮЧЕВОЙ БЛОК
   Карусель из 6-8 сильнейших кейсов:
   - "Конверсия x13 за 3 месяца" — название компании, лого
   - "Выручка x3 за полгода" — цитата выпускника
   Кнопка: "Смотреть все 40+ кейсов"

6. ДЛЯ КОГО
   Три колонки: PM / Фаундер / Маркетолог
   Под каждой: что получит, какие задачи решит

7. АВТОР
   Ваня Замесин: био, цифры, ссылки на публикации
   "12 000+ выпускников, 80+ тренингов, автор Advanced JTBD"

8. ТАРИФЫ
   3 тарифа в сравнительной таблице
   Рекомендуемый выделен ("Практика" — самый популярный)
   Рассрочка: "от X ₽/мес"

9. ОТЗЫВЫ
   Видео-отзывы + текстовые (с фото, должностью, компанией)
   Средняя оценка 9.2/10 с количеством

10. FAQ
    Типовые вопросы + возражения

11. ФИНАЛЬНЫЙ CTA
    "Записаться" + таймер ближайшего потока + "Остаталось X мест"
```

**Зачем:** Текущий лендинг может не следовать логике "проблема → решение → доказательства". Перестройка под эту модель увеличивает конверсию.

**Метрика:** Конверсия лендинга в заявку растёт на 20%+.

### Гипотеза 8: Бесплатная лекция как лид-магнит (top of funnel)

**Идея:** Сделать 1-ю лекцию тренинга публичной на YouTube + встроить на лендинг.

**Зачем:**
- SEO: YouTube-видео индексируется, люди ищут "jtbd обучение", "как создать продукт"
- Квалификация лида: человек посмотрел 40 мин → уже "подогрет"
- Снижает барьер: "а вдруг не подойдёт" → посмотри бесплатно
- Уже есть контент: транскрипции лекций и материалы для создания

**Реализация:**
- На лендинге: блок "Посмотри бесплатную лекцию" между Hero и Программой
- Email-гейт на полную версию (30 сек превью без гейта)

**Метрика:** 30%+ посетителей лендинга смотрят лекцию. Конверсия "посмотрел лекцию → заявка" > 8%.

---

### Гипотеза 9: Калькулятор ROI обучения

**Идея:** Интерактивный блок на лендинге: "Посчитай, сколько ты заработаешь после тренинга."

Поля ввода:
- Текущая конверсия (%)
- Количество лидов/мес
- Средний чек (₽)

Результат: "Если конверсия вырастет хотя бы на 30% (средний результат выпускников), ваша дополнительная выручка составит X ₽/мес. Окупаемость тренинга — Y дней."

**Зачем:** Переводит стоимость тренинга из "расхода" в "инвестицию". Работает на сегмент "хочу вырастить конверсию/выручку".

**Метрика:** 15%+ посетителей используют калькулятор. Конверсия после калькулятора в 2x выше средней.

### Гипотеза 10: Отдельные SEO-лендинги под сегменты

**Идея:** Создать суб-страницы под ключевые сегменты:

```
/producthowto/for-product-managers   — "AJTBD для продакт-менеджеров"
/producthowto/for-founders           — "AJTBD для фаундеров и предпринимателей"
/producthowto/for-marketers          — "AJTBD для маркетологов"
/producthowto/for-teams              — "Корпоративное обучение AJTBD"
/producthowto/for-career-changers    — "Как стать продакт-менеджером"
```

**Каждая страница:** Те же блоки, но с адаптированными заголовками, кейсами и CTA под конкретный сегмент.

**Зачем:**
- SEO: ловит запросы "обучение продакт-менеджеров", "курсы для маркетологов jtbd"
- Конверсия: человек видит "это ДЛЯ МЕНЯ", а не общий лендинг
- Рекламные кампании: можно лить трафик на сегментную страницу

**Метрика:** Каждая сегментная страница конвертирует на 30%+ лучше общего лендинга.

### Гипотеза 11: Блок "Выпускники работают в" (лого компаний)

**Идея:** Лента логотипов компаний, где работают/работали выпускники: Яндекс, Сбер, Ozon, VK, Тинькофф, МТС и т.д.

**Зачем:** Мгновенное социальное доказательство. Работает на сегмент "хочу карьерный рост" и на корпоративных покупателей.

**Метрика:** Уменьшение bounce rate на 10%+ в блоке после лого.

---

### Гипотеза 12: Интеграция BOOST как upsell / cross-sell

**На лендинге КДП:** Блок внизу: "Уже прошёл тренинг? Создай свой продукт за неделю → BOOST"
**На лендинге BOOST:** Блок: "Хочешь сначала разобраться в методологии? → Как делать продукт"

**Зачем:** Кросс-продажи между продуктами. Человек, пришедший за одним продуктом, узнаёт о другом. LTV клиента растёт.

**Метрика:** 5%+ покупателей одного продукта покупают второй в течение 3 месяцев.

---

### Гипотеза 13: Sticky CTA + Exit Intent

**Идея:**
- Sticky bar внизу экрана: "Ближайший поток: X марта | Осталось Y мест | Записаться"
- Exit intent popup: "Подожди! Посмотри бесплатную лекцию перед уходом" (email-гейт)

**Зачем:** Sticky bar напоминает о действии. Exit intent ловит уходящих — даже если не купят сейчас, попадут в воронку email-маркетинга.

**Метрика:** Exit intent собирает 3-5% уходящих посетителей.

## Часть 3: Глобальный англоязычный сайт

### Гипотеза 14: Отдельный домен для глобального рынка

**Варианты:**
- launchpilot.app (если платформа — основной продукт)
- zamesin.com (если бренд автора — основной)
- aura-framework.com (если методология — основной)

**Структура:**
```
domain.com/
├── /                    — Hero: "Build products people actually buy"
├── /course              — From Zero to First Product
├── /platform            — LaunchPilot AI Platform
├── /methodology         — AURA Framework (SEO-контент)
├── /blog                — English content hub
├── /cases               — Case studies
└── /about               — About Ivan Zamesin
```

**Зачем:** Русскоязычный и англоязычный контент на одном домене путает поисковики (hreflang-проблемы). Отдельный домен для EN — чище для SEO.

**Метрика:** EN-сайт выходит в топ-20 по "jobs to be done course" за 6 месяцев.
