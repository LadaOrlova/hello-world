# Правила структуры кейс-поста

## Заголовок

Результат + цифра. Не описание процесса.
«Принёс сделки на $500 000 при ROMI 2000%», а не «как мы внедрили AJTBD».

## Подзаголовок

Одно предложение-саммари всего кейса: что сделали → что получили. Даёт контекст до чтения.

## Структура текста

### 1. Результаты в цифрах — первый блок

Буллетами, без пояснений. Только цифры и факты.

### 2. Контекст — коротко

3–4 предложения. Кто, масштаб, проблема. Без подзаголовков внутри.

### 3. Действия — только ключевые

- Без нумерации шагов (не «Шаг 1, Шаг 2»)
- Убирать детали, которые не нужны для понимания логики «проблема → действие → результат»
- Техстек — одним предложением, не отдельным блоком
- Перечисления сегментов/подпунктов — оставлять принцип, убирать полный список

### 4. Выводы — максимум 3 пункта

Формат: **принцип** — пояснение через тире. Не больше трёх.

### 5. Цитата в конце

Одна, как финальный акцент.

## Общий принцип

Резать детали. Оставлять только то, что нужно для понимания логики «проблема → действие → результат».
