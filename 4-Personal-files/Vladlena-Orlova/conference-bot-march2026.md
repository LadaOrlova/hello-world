## Реферальная механика — детали

### Как работает

1. Каждый подписчик бота получает уникальную реферальную ссылку (формат: `t.me/[bot_name]?start=ref_[user_id]`)
2. Когда новый человек переходит по ссылке и нажимает /start — бот засчитывает реферал
3. Пригласивший получает уведомление: "Ты пригласил(а) ещё одного друга! Всего: N"

### Что показывать

- Счётчик приглашённых — в реферальном сообщении при старте
- Обновление счётчика — через уведомление при каждом новом реферале

### Награды (опционально, решить до запуска)

Варианты:
- **Просто**: без наград, только соцмотивация ("пригласи тех, кому полезно")
- **С наградой**: за 3 приглашённых — бонусный материал (запись закрытого выступления, PDF, доп. скидка на OTO)

**Рекомендация**: начать без наград. Если будет слабая виральность — добавить награду на ходу.

### Кнопка "Поделиться"

Кнопка открывает стандартный Telegram-шеринг с преднаписанным текстом:

```
25-27 марта — бесплатная онлайн-конференция
[НАЗВАНИЕ]

Спикеры: Илья Красинский, Ваня Замесин
и ещё [N] экспертов.

Регистрация: [РЕФЕРАЛЬНАЯ ССЫЛКА]
```

## Технические заметки для Аники

1. **Стек**: любой удобный (aiogram, python-telegram-bot — на выбор Аники)
2. **Хранение данных**: минимум — user_id, referrer_id, referral_count, дата подписки
3. **Расписание сообщений**: cron или scheduler. Все сообщения отправляются ВСЕМ подписчикам (без сегментации)
4. **Кружочки**: хранить file_id после первой загрузки, чтобы не загружать повторно
5. **Реферальная ссылка**: deep link через start-параметр (`?start=ref_123456`)
6. **Шеринг**: использовать `inline_keyboard` с `switch_inline_query` или url-кнопку на `t.me/share/url?...`
7. **Нет ветвлений**: бот не обрабатывает никакие команды кроме /start. Все остальные сообщения от пользователя — игнорировать (или отвечать "Расписание конференции: [повторить]")
